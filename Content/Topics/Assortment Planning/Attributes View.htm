Attributes view

The Attributes view is used to view and manage product attributes for placeholders and collections.

For more information on Attribute Families, Values, and how their eligibility is set for items within the product hierarchy, see Attribute Management and Assignment Overview.

The Attributes view is displayed or hidden in Placeholder view using the View Selection dropdown menu.

Viewing Attribute values that are longer than 256 characters

NVARCHAR(MAX) is a non-editable Attribute type enabling users to view attribute values longer than 256 characters. NVARCHAR(MAX) is only available in the Collection / Placeholder Attributes supporting view.

A related functionality allows the Notes field in the main planning view to display greater than 256 characters.

Data population

The view populates with all attribute families that any of the selected placeholders or collections is eligible for. Attribute values available for selection include all values that any of the selected items is eligible for. Attribute family and value eligibility can be set via Maintenance > Attribute Management.

Applying changes across multiple items

Whether multiple lines are selected in the main planning view, changes are being applied in Collection view, or both the system automatically resolves any conflicts that exist between the applied values and the attribute value eligibility of the items the change is being applied to.

Single-select attribute value changes are applied to all placeholders and collections that are eligible for them. Ineligible placeholders and collections remain unchanged.

Multi-select attribute value changes are applied additively to placeholders and collections that are eligible for them. If only some of the applied values are eligible, then those are applied. Ineligible placeholders and collections remain unchanged.

Dual-directional attribute flow

Depending on where attributes are assigned – to placeholders in Assortment Planning or to variants in Variant Manager – attributes are transferred when a variant is assigned to a placeholder

System policy Multi Value Attribute Inheritance governs the inheritance of multi-value attribute families when variants are assigned. When set to FALSE, multi-value families are applied additively to existing values when variants are assigned.

A placeholder has the multi-value family Embellishments and the values Flocked and Sequinned currently set. If a user assigns a variant with the Embellishments values Flocked and Embroidered to that placeholder, then the resulting values for the Embellishments family are: Flocked, Sequinned, and Embroidered.

Assigning a variant

Product attributes attached to a variant are transferred to the placeholder. Any attributes that are already assigned to the placeholder that don’t conflict with the variant attributes (i.e. single-select families that the variant already had assigned) remain on the placeholder and also are transferred to the variant.

When set to TRUE, multi value families overwrite the existing values when variants are assigned.

Using the same case as the example above, the resulting values for Embellishments would be: Flocked and Embroidered only.

Attribute changes in Assortment Planning

System policy Update Variant Attributes In Assortment Planning (Module: Assortment) governs whether changes are allowed to attribute families attached to variants. When set to FALSE, all attribute families attached to the assigned variant appear grayed out and are locked for editing.

Creating attribute families that only have the entity type Variant Placeholder assigned allows additional attribute families that cannot be attached to variants and will only be used inside of Assortment Planning for placeholders.

When set to TRUE, attribute families can be locked or unlocked individually within the Attribute Management solution. Families assigned to the variant entity type and having the AP Updates Variants setting checked are editable in Assortment Planning even when variants are assigned. Changes made to these families in Assortment Planning are transferred to the assigned variant for the product hierarchy assigned to the active assortment plan but are not transferred to other placeholders that variant is assigned to.

For more information on the AP Updates Variants setting, see Creating and maintaining attribute families.

Attribute changes in Variant Manager

Changes made to variant attributes in Variant Manager are transferred to all placeholders that variant is currently assigned to across all existing assortment plans that have the selected Variant Hierarchy assigned.

Field descriptions

Field Description
Family Name The name of the attribute family.
Family Description The description for the attribute family.
Value The attribute value of the attribute family for the selected placeholder or collection. Supported attribute types include boolean, numeric, dates, and strings.

Buttons

Save: Saves all changes made to attribute values for the selected placeholder/collection.

Cancel: Cancels all changes made to attribute values for the selected placeholder/collection and reverts them to previously saved values.
