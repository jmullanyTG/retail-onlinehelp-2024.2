Cluster Sets view

The Cluster Sets view contains all the essential metrics for building sales plans and the functionality to make Cluster Set or Cluster level changes to the single placeholder or collection selected in the main planning view.

The Cluster Sets view is displayed or hidden in Placeholder view using the View Selection dropdown menu.

Creating preferred layouts within the Cluster Sets view

Clicking on the Fields button displays a screen similar to the standard Fields Customization screen. The only difference is that the filter, row, and column sections are absent. This is because only the data items can be amended.

Users can move fields between the Data Area and None Area to show or hide fields. Both sections are multi-select enabled to allow for bulk editing.

When a desired layout has been achieved, users can right-click in the Cluster Sets grid and select Save Layout. The original or another layout can be selected using the appropriate right-click menu option.

Sub clustering by attribute for placeholders and collections

Sub clustering by attribute allows for clusters to be created which support clustering by grade and sub clustering by climate. For example, swimwear might have an earlier phase-in date for Warm stores than for Cold stores. To avoid having to update the phase-in date for A-Warm, B-Warm, C-Warm, etc. one at a time the Sub Cluster field can be used at an aggregated level (for example, update the phase-in date for all Warm stores at once).

The Sub Cluster field is best placed in the Column Area of the grid.

The ability to bulk edit across sub clusters supports user stories such as defining phase dates by sub cluster attribute at an aggregate level, rather than for each cluster-sub cluster at a time. Values are apportioned and pushed down according to existing rules within the system. Editability is as per the existing rules.

The value that displays in the Sub Cluster field is the value selected in the Sub Clustering Attribute field in the Add Cluster Set or Update Cluster Set screens within the Site Clustering module. This field displays N/A if the cluster does not have a sub-cluster.

For more information, see Configuring sub-clusters. refer to the Site Clustering User Guide.

The Cluster Sets view responds to the selection of multiple placeholders or placeholder collections in the main planning view. To facilitate this functionality, three fields can be used to identify the placeholders selected:

  • Placeholder ID

  • PH Code

  • PH Desc

In the Collection view these fields are the corresponding Placeholder Collection identifiers.

These fields are best used in the Row Area and are most useful when showing multiple placeholders at once. For example, users can view values at a more aggregated level across multiple placeholders / clusters.

Visibility to actual ROS and how aligned sales plans are with actual sales

The ROS metric reflects a combination of actual and planned sales in a single ROS metric. The following metrics split actual and planned sales out. This provides better visibility to the actual ROS for active sites and how aligned sales plans are with actual sales.

  • Reg Ftr Site Count, Reg Ftr ROS ($), Reg Ftr ROS (U), Reg Ftr Site ($), Reg Ftr Site (U), Reg Ftr AUR ($), Act ROS ($), Act ROS (U)
  • SPV ROS ($), SPV ROS (U), SPV Sales ($), SPV Sales (U)

Changes that cause Total Sales Units to change cascade down to the Regular Future figures as well.

Abbreviation Description
ROS Rate of Sale
Reg Ftr Regular Future, as in the ROS for the portion of the sales plan that is pre-markdown (e.g. Regular) and is non-actualized.
Act Actualized, as in the ROS for the portion of the sales plan that has already happened.
SPV Sales Plan View, as in the ROS for the sales plan inclusive of overrides made in the Sales Plan or Top Down Core Plan (TDCP) views.

Metric notes

For plans with:

  • wholesale lines - Sales, Sls and AUR renamed to Bookings, BK and ASP.

  • mixed wholesale and retail - Sales, Sls and AUT renamed to Sales/Bookings, Sls/Bk and AUR/ASP.

Editability

  • Discount %, At Once $/U, Cancel %, Cancelled $/U are all editable when the line is a wholesale line or have an underlying Cluster Set that is wholesale.

  • All the wholesale fields are non-editable if the line does not have any Wholesale Cluster Sets.

  • Cancel %, Cancelled $/U are non-editable after actualization begins.

The Wholesale check box is purely informational and indicates a line is wholesale.

Calculations

Amending:

  • At Once $ recalculates At Once U and vice versa.

  • Cancel % recalculates Cancelled $ and Cancelled U

  • Cancelled $ recalculates Cancelled U and Cancelled %

  • Cancelled U recalculates Cancelled $ and Cancelled %

  • Blended AUT recalculates Cancelled $

Any amendment that affects Sales Units and/or Total Bookings recalculates Cancelled U and Cancelled %.

Metrics

  • Actual First Sale Period: The period where the first sale actualized.

  • AUC: The average unit cost price for the cluster set/cluster. Displays the weighted average of AUC values for all periods, both actualized and future, beneath.

    AUC is not directly editable once a placeholder has begun to actualize (current date is greater than Phase In date).

  • AUR: The average unit retail price for the cluster set/cluster.

    Because of the potential for multiple currencies, VAT (value added tax) rates, and promotional participation to vary by site within a single cluster, AUR is not directly editable and is only changed as a result of changes to Home AUT, Phase In, Phase Out, and promotion applications. Details for this calculation are available in a supplemental MS Excel model.

    • Calculation: (Ttl Sls $ / Ttl Sls U)

  • AUR ex VAT: Average retail (selling) price excluding VAT per unit for the placeholder item.

    • Calculation: AUR / (1 + VAT %)
  • Home AUT: The average unit ticket price for the cluster set/cluster in home currency. Displays the weighted average of Home AUT values for all future periods beneath. Home AUT is assumed to be inclusive of the Home Site’s VAT and is treated as such for all dependent calculations.

    Because of the potential for multiple currencies, VAT (value added tax) rates, and promotional participation to vary by site within a single cluster, changes to Home AUT require saving to properly calculate the effects on the rest of the plan for a given placeholder/collection. Details for this calculation are available in a supplemental MS Excel model.

    Period level changes to Home AUT can be made in Sales Plan view.

    • Blended AUT (Blended Average Unit Ticket) and AUR (average unit retail) being an outcome of the calculation mentioned above is not directly editable.

    If Import Config setting AUT for planned periods is set to 2 (AUT is only managed through imports), then Home AUT is read-only for Placeholders with a status of Assigned, Committed, Replen, and Allocate. For placeholders with a status of Open and Created, Home AUT is editable.

  • Blended AUT (Blended Average Unit Ticket): The average unit ticket price for the cluster set/cluster, accounting for all price conversions across different currencies and countries within the cluster. Displays the weighted average of Blended AUT values for all periods, both actualized and future, beneath. Blended AUT is derived from Home AUT in future periods, but it comes from provided sales history for actualized periods. Blended AUT is assumed to be inclusive of VATs for all sites within the cluster and is treated as such for all dependent calculations.

    Blended AUT incorporates each of the following, all of which are individually controlled via system policy:

    • VAT Rate differences between the Home Site and sites within each cluster
    • Price Exception factors
    • Price Matrix conversions
    • Calculation: (Ttl Sls / Ttl Sls U).

  • Curve Mode: The mode of the selected Seasonality Curve.

    • Static: The selected curve is applied exactly as it exists at the time of application and the cluster set/cluster’s curve will not change if changes are made to the curve, whether manually in the Curves module or through dynamic recalculation.
    • Dynamic: Changes made to the curve push down to the cluster set/cluster via automated system processes and the dependent sales plans recalculate when those changes are pushed.

  • Gross Margin %: An editable value that adjusts the Receipt AUC metric to allow this margin to be achieved. Coloring rules follow the Receipt AUC pattern. The gross margin percentage for the cluster set/cluster

    • Calculation: (AUR ex VAT – AUC) / AUR ex VAT

  • Markdown Curve: The curve applied to the cluster set/cluster. Used to break down Ttl Sls $ and Ttl Sls U by period in Sales Plan view. Curves must be generated with the Enhanced Filter type to be eligible for use within Assortment Planning and selectable here. For more information, see Sales Plan View.

    Click the ellipsis () within the cell to launch the Curve Selection screen. For more information, see Curve Selection Screen.

  • Markdown Date: The markdown date for the cluster set/cluster. Selections are limited to periods from Phase In through Phase Out. A markdown date defines which period to stop respecting the various constraints utilized in the Receipt Plan supporting view including Min Pres.

    Min Pres drops to zero in the period specified by Markdown Date when viewing the by-period plan for the cluster set/cluster in Sales Plan or Receipt Plan view.

    The markdown date can be pushed by changing Phase In or Phase Out.

  • Min Pres : The desired minimum presentation stock per site for the cluster set/cluster. This represents additional non-sales units to be purchased for the cluster set/cluster to maintain adequate stock for presentation within each store.

    • Calculation: SUM (cluster site count * cluster min pres – for all clusters within a set) / cluster set Site Count (yields a weighted average) or (cluster site count * cluster min pres) for single clusters

  • Phase In Date: The phase in date for the cluster set/cluster. If set, selections limit to periods prior to Phase Out.

    Defaults to the earliest non-actualized period within the Window of Time in the data selection parameters.

  • Phase Out Date: The phase out date for the cluster set/cluster. If set, the selections limit to periods after Phase In. For placeholders/collections of Replen Status, Phase Out dates are derived from the maximum Phase Out date set in the Forecasting module for all relevant inventories.

    Defaults to the last period within the Window of Time in the data selection parameters.

  • Planned Phase In: Editable with the date chosen by clicking in the field and selecting a date from a calendar.
  • Receipt ($): The sum of Receipt Cost from Receipt Plan view for the placeholder/collection at cluster level. This is included as a comparison of the actual receipt plan to the estimated Ttl Buy $ in Cluster Set View.
  • Receipt (U): The sum of Receipt Units from Receipt Plan view for the placeholder/collection at cluster level. This is included as a comparison of the actual receipt plan to the estimated Ttl Buy U in Cluster Set View.
  • Reg S/T %: Read-only. The calculation is Reg SPV Sales U / Ttl Orders U.

  • Retrend: (Visible when View Style Rate of Sale is selected) The fields for Retrend metrics populate only after the selection of the Retrend option in the Process Queue section of the main planning view toolbar. The following display the results and status of the Retrend operation:

    • Retrend ROS ($)

    • Retrend ROS (U)

    • Retrend Ftr ROS ($)

    • Retrend Ftr ROS (U)

    • Retrend Status (showing a status of either Clean or Dirty)

    An additional metric, Last Retrend Date, displays the date the Retrend operation was last performed.

    There are Notifications for Retrend submission status in the Inbox/process queue.

  • Reviewed?: If checked, then all underlying Cluster Set/Clusters have been reviewed. Unchecked indicates that none have been reviewed and multi-select indicates a mix of reviewed and non-reviewed at the Cluster Set level.

    This field is visible when system policy Require Placeholder Review before Commitment Generation is set to True.

  • ROS $: The rate of sale (average sales per site per period) in revenue for the cluster set/cluster.

    • Calculation: (Ttl Sls $ / No. Periods / Site Count)

  • ROS U: The rate of sale (average sales per site per period) in units for the cluster set/cluster.

    • Calculation: (Ttl Sls U / No. Periods / Site Count)

  • S/T %: Sets the target sell-through percentage for the cluster set/cluster. Changes to this metric alter Ttl Sls U in relation to Ttl Buy U to achieve the entered sell-through percentage.

    • Calculation: (Ttl Sls U / Ttl Buy U)
  • S/T % Delta: Compares the targeted sell-through rate against the actual. The calculation is (Target S/T % - S/T %) / (S/T %).
  • S/T %: Read-only. Compares the lifetime sales for the item against the lifetime orders. Calculation: (SPV Sls U / Ttl Orders U)

  • Site Count (Active): The average total site count and the average active site count (in parenthesis) of the cluster set/cluster between Phase In and Phase Out. Typically, this metric is a static whole number that equates to the sum of the sites in the cluster set or cluster. However, when multiple cluster set snapshots are present during a placeholder or collection’s life, it is possible for site count by cluster to vary over time which necessitates the need for an average count.

  • SPV Diff ($), SPV Diff ($) %, SPV Diff (U), SPV Diff (U) %: The difference between SPV Sales and SPV Diff ($) Sales.

  • SPV Sales ($): The sum of all sales for the placeholder or collection in Sales Plan view. Provided for easy comparison to the plan at the placeholder or collection level.

  • SPV Sales (U): The sum of all sales Units for the placeholder or collection in Sales Plan view. Provided for easy comparison to the plan at the placeholder or collection level.

  • Target S/T %: Editable, has no calculation effect or relationship. User can enter a target sell-through value to compare against the calculated sell-through percentage.

  • Ttl Buy $: Editable. The total buy in revenue for the cluster set/cluster. Changes to this metric directly affect the cluster set/cluster sales plan.

    • Calculation: (Ttl Sls U + (Min Pres * Site Count) + Additional Buy) * AUC

  • Ttl Buy U: The total buy in revenue for the cluster set/cluster. Changes to this metric directly affect the cluster set/cluster sales plan.

    • Calculation: (Ttl Sls U + (Min Pres * Site Count) + Additional Buy)

  • Ttl Sls $: The total sales revenue for the cluster set/cluster from Phase In through Phase Out.

    • Calculation: (ROS $ * No. Periods * Site Count)

  • Ttl Sls U: The total sales units for the cluster set/cluster from Phase In through Phase Out.

    • Calculation: (ROS U * No. Periods * Site Count)

  • Units Per Location: The average units per location for a cluster set/cluster based on Ttl Buy U and Site Count.

    • Calculation: (Ttl Buy U / Site Count)
