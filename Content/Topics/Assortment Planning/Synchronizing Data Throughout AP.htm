Synchronizing data throughout Assortment Planning

Throughout AP, the system may need to queue processing tasks to keep data synchronized. The following is a summary of the various asynchronous data processing tasks that may occur while using AP.

The order in which the tasks start and stop does not matter.

Name Description Executes when...
Actualization Reaggregate detailed historical sales, stock on hand, and receipt data in order to better inform future plans.

  • Variant assignment changes in main planning view.

  • A new cluster is added to the assortment in main planning view > Cluster Details.
Allocation Push Push AP rate of sale, cluster assortment, minimum presentation, and phase dates to matched allocation strategies. The button Process Queue > Push Allocations is clicked in the main planning view.
Core Data Plan Ingest On Order data into future periods of the AP stock positioning plan. If configured, ingest AUT and/or Receipt AUC values from Variant-Site price and cost data.

  • Variant assignment changes in main planning view.

  • A new cluster is added to the assortment in main planning view > Cluster Details.

  • Placeholder phase dates are extended in main planning view, Collection View, or the Cluster Sets side pane.
External Import If configured, ingest any AP data from external systems into various AP metrics.

  • Variant assignment changes in main planning view.

  • A new cluster is added to the assortment in main planning view > Cluster Details.

  • Placeholder phase dates are extended in main planning view, Collection View, or the Cluster Sets side pane.

  • The button Process Queue > Recalculate External is clicked in main planning view.
Receipt Plan Recalculate suggested receipts based on the sales plans and receipt constraints.

  • A new placeholder is created through the New, Hindsight, Duplicate, Split, or Smart Start processes.

  • Receipt flow dates are changed through the Manage > Receipt Periods screen.
Replen Import Ingest replenishment data into AP sales, receipt. and stock positioning plans.

  • Data Source configurations are changed for Replen-type items.

  • Placeholder status is changed to Replen.

  • A new cluster is added to the assortment in main planning view > Cluster Details.

  • The button Process Queue > Recalculate Replenishment is clicked in main planning view.
Retrend Calculate a suggested Rate of Sale based on actualized sales history. The button Process Queue > Retrend is clicked in main planning view.
