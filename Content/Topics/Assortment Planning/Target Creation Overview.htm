Target creation

Target Creation is the establishing of plan targets for specific areas of a business and the determining of the number of items required to meet those plan targets. Assortment Planning has the ability to segment hierarchy levels by a wide array of product attributes, both qualitative and quantitative, and then further subdivide and define plan targets and desired new choice counts all the way down to a cluster level.

Traditional assortment planning tools are typically rigid in that planning is by product and site hierarchy only. This has led to many retailers attempting to gain detail by further extending their hierarchies, but this approach still does not offer true flexibility to adjust strategy as often as desired based on the changing needs of the business.

Target Creation can:

  • create scenarios in as simple or complex as manner as desired.
  • link scenarios to existing merchandise financial plans or base plan targets on specified historical period.
  • define business segments by product hierarchy or based on a wide array of variables defined through product filters.
  • model new areas of business, adjust percent contributions and plan targets at aggregate or more granular levels through the Pivot Grid functionality, and calculate recommended new choice counts based on targets through Rationalization (a marginal return curve calculation that suggests optimal assortment breadth and depth based on cleansed historical data).

Scenarios can be used to create a foundation for assortment plans through a process called Smart Start. For more information, see Smart Start .

Parameters

Use the following parameters to filter the displayed Targets:

Parameter Description
Product Hierarchy The product hierarchy levels a Target Creation scenario must contain to be shown in the Target Creation Scenarios section. Selectable levels are based on the intersection of the product hierarchy assigned to the current assortment plan and the product hierarchy permissions of the active user.
Site Hierarchy The site hierarchy levels a Target Creation scenario must contain to be shown in the Target Creation Scenarios section. Selectable levels are based on the intersection of the site hierarchy assigned to the current assortment plan and the site hierarchy permissions of the active user.
Window of Time The window of time a scenario must intersect with to be shown in the Target Creation Scenarios section. Selectable periods are based on those configured in the system down to the Period set for the current assortment plan.

For more information, see Creating floor set updates for varying time frames.
Display Currency The currency to display for all revenue based fields in the view. Defaults to the Home Currency for the active Assortment Plan.

Toolbar buttons

  • New: Launches the Target Creation (New) window to create a new scenario.

  • Copy: Launches the Target Creation window populating the Target Data Selection, Target Segment Definitions, and Target Worksheet (segment cluster details) section using the selected scenario in the Target Creation Scenarios section.

    • All fields in the Target Data Selection and Target Segment Definitions sections are editable in copied scenarios.

      To unlock target segment definitions and cluster details for editing, copied scenarios must first be saved. To do this without changing the scenario’s core data structure, change the scenario name to a unique name then click Save.

  • Edit: Launches the Target Creation window, populated with all data from the selected scenario in the Target Creation Scenarios section.

    • All parameters are editable in scenarios of status Created, Calculated, or Rationalized.
    • For scenarios of status Smart Started, fields in the Target Segment Definitions and Target Worksheet sections can be edited, but doing so requires recalculation, re-rationalization, and re-Smart Starting for changes to take effect in the active assortment plan.
  • Delete: Deletes the selected Target Creation scenario.

  • Compare: Launches the Target Comparison window in which users can compare two scenarios.

    • Standard Pivot Grid functionality, including ability to graph selected data elements and Export or Open analysis in a variety of popular formats, including Microsoft Excel and Adobe PDF.
    • Only prerequisite for scenario comparison is that scenarios be in Calculated, Rationalized, or Smart Started status.

    • Site count is a SUM across cluster sets and clusters, but an AVERAGE across segments and scenarios.

    • Comparisons of scenarios representing disparate parts of the product/site hierarchies and/or windows of time are possible - they are just of limited usefulness.
  • Smart Start: Initiates the Smart Start process for the selected scenario, generating the number of placeholder items required, and seeding with beginning sales plan data elements.
  • Cluster Config: Launches the Target Configuration screen allowing users to determine which clusters should always have more choices than others. For more information, see Cluster Configuration.

  • Fields: Launches the customization dialog.
