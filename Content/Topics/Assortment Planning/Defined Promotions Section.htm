Defined Promotions section

When there are active applications for a given promotion, all fields are locked for editing except Name, Description, Privacy, and Sharing Settings. Promotions with active applications are conditionally formatted with bold text with locked fields also being italicized.

Toolbar buttons

Button Description
New Adds a new line in which the user can define a new promotion.
Delete Removes the selected promotion from the list of defined promotions.

Users are not asked to confirm deletion of promotions, so be certain the promotion should be deleted.
Fields Opens the Customization dialog in which users can drag fields to or from the tab’s header area and this dialog to customize the view. Custom layouts can be saved or restored by right-clicking in the grid and selecting Save Layout or Restore Layout, respectively.
Save Saves all newly created promotions and changes or deletions of existing promotions made since the last save.
Cancel Removes all newly created promotions and undoes deletions and change of existing promotions made since the last save.

Grid fields

  • Name: The name for the promotion
  • Description: The description for the promotion

  • Privacy: The user access settings for the promotion

    • Public: everyone with access to Promotions can edit this promotion
    • Private: only the user who created this promotion can edit this promotion
    • Shared: only specified users with access to Promotions can edit this promotion
  • Sharing Settings: The specific users to grant permission to edit this promotion. Only valid when Privacy is set to Shared.
  • Type: The type for the promotion. Promotion types are defined during product implementation. In this version of Promotions, types are informational only and do not affect the promotion definition or the way it is applied.

  • AUR Effects: Only one AUR effect from the below list can be applied to a given promotion. On setting one of the following, the other fields are grayed out and made inactive but can be made active again by clearing the value from the field set.

    Option Description
    Replacement AUR An explicit new AUR to be assigned to all placeholder/clusters the promotion is applied to.
    Promotion Discount ($) The amount by which the AUR of placeholder/clusters the promotion is applied to is decreased.
    Promotion Discount (%): Sets a percentage by which the AUR of the placeholder/clusters the promotion is applied to is decreased.

  • Global: Whether the promotion is a global promotion or not. Global promotions gain the added behavior option of Supersede.

    Aside from the above, the global promotion designation is purely informational. While global promotions are typically applied to all products or large section of the product hierarchy, this is not required.

  • Exclusion Filter: A product filter of placeholders to exclude from any applications of the promotion. Clicking in this field displays the Items dialog in which the product filter to be used is set. Once an exclusion filter is set the field cannot be blanked out. To avoid using an exclusion filter after one is set, define a filter with no criteria so it returns no items and assign that (nothing is excluded from the promotion).

    • :

    Any items that belong to an exclusion filter or their sales plans do not fall within the eligibility period if one is set are not modifiable within the Promotion Detail View tab.

    Option Description
    Launches the Maintain Filters dialog where users can create new product filters, edit, delete, or export existing product filters, and import product filter files.
    Cancel Closes the product filter selection dialog without setting or changing the product filter selection.
    Ok Sets or changes the product filter selection.

  • Behavior: Sets the stacking behavior for the promotion.

    Option Description
    Stacking All stacking promotions that are active in a given period combine their effects. Promotional stacking is always applied in order depending on the type of AUR effect assigned to each: New AUR, % Off, $ Off.
    Non-Stacking: Non-stacking promotions operate on best-deal logic meaning the resulting AURs of applied promotions are compared and the one with the lowest AUR is applied.
    Supersede The supersede behavior is only valid for Global promotions. When set, supersede promotions are applied in place of any others applied during a given period regardless of whether their resulting AURs are lower.

  • Final Index: The promotional index for the promotion. Changing the Final Index at this level only affects newly created Promotion Applications. Existing applications are not changed.

  • Is Always Eligible?: Whether the promotion is a standing promotion with no limits on when it can be applied: Checking this clears and locks Eligibility Start Date and Eligibility End Date for the promotion.

  • Eligibility Start Date: The start date (as a period) for the time frame during which the promotion can be applied. Start dates at the Promotion Application and Application Details levels cannot be set prior to this date.

  • Eligibility End Date: The end date (as a period) for the time frame during which the promotion can be applied. End dates at the Promotion Application and Application Details levels cannot be set after this date.

  • Created By: The user name of the creator of the promotion.
  • Create Date: The date the promotion was created.
  • Apply to Markdown Periods: When checked, prevents promotions from being applied to any markdown periods.
