Data imports including Sales ex VAT

As periods begin to actualize, so VAT amounts are actualized for those periods. If an organization deals with VAT, it is important that the data import includes Sales ex VAT, as the system uses the difference between Sales and Sales ex VAT to calculate historic VAT percentages for actualized periods.

If any records imported for a given variant/cluster/period lack Sales ex VAT, the system nulls the VAT percentage for the entire variant/cluster/period, as an indicator that there is missing data in the import. This may cause anomalies in AUR ex VAT in the main planning view and with metrics in the Price Exceptions view.

A way to check for this is to look at the VAT percentages in the Sales Plan view for periods that have actualized VAT percentages of zero though VAT percentages were in effect for the product.

System policy Enable VAT Conversion in AUT Calculation determines whether to use VAT rate conversion steps when deriving currency-specific AUT (Blended AUT) values from the given Home AUT value.
