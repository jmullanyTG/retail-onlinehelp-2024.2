Assortment Rollup data parameters

The Assortment Rollup is controlled by the following parameters:

  • Product: A product filter to define the items to be included in the aggregate rollup at the selected product hierarchy level.
    • Launches the Filter Conditions dialog allowing users to edit the criteria for the selected filter prior to executing it. The option also exists to create a new filter from the adjusted criteria.
    • Launches the Maintain Filter dialog allowing users to create or import a new filter, or update, delete, or export an existing filter.

  • Product Hierarchy Level: The level of the product hierarchy at which to view and analyze the metrics contained in the view.

    This selection drives the level of the product hierarchy in which the assortment plan data is aggregated to. 

    For example, if a sub-department is selected as the product hierarchy level parameter selection, the assortment roll up displays all aggregated data in this view at the sub-department level, but no lower than that level from a product hierarchical perspective. The data displayed in the view can be aggregated to levels above sub-department dynamically.

  • Site Hierarchy: The specific levels of the current plan’s site hierarchy to include in the current view.
  • Cluster Sets: The cluster sets to include in the current view. The cluster sets listed include all default and additionally available cluster sets for the product level selected in the Product parameter.

    While cluster sets are not explicitly part of the site hierarchy they are directly related. Cluster sets in a given assortment plan must be contained within a single lowest level of that hierarchy. Because of this, the Cluster Sets dropdown populates with the cluster sets contained beneath the selected site hierarchy levels.

  • Site Hierarchy Level: Selects the level of the site hierarchy at which to view and analyze the metrics contained in the view.

    This selection drives the level of the site hierarchy in which the assortment plan data aggregates to.

    For example, if a cluster set is selected as the site hierarchy level parameter selection, the assortment rollup displays all aggregated data in the view at the cluster set level, but no lower than that level from a site hierarchical perspective. The data displayed in the view can be aggregated to levels above cluster set dynamically.

  • Window of Time: The time frame to display in the current view. The selected time frame must be a contiguous range of periods. If it is not, the system still uses the time frame defined by the first and last selected periods.
  • Comparison Snapshot: A snapshot for the current plan from which to populate comparison metrics in the view. 

    For detailed information on taking snapshots to create comparisons against the assortment plan, see Viewing snapshots and snapshot templates.

  • View Mode: The view mode:
    • Total Company: Plan revenue and unit metrics as total company values (meaning the total volume for all sites contained in whatever level the pivot grid is aggregated to).
    • Average Store: Plan revenue and unit metrics as average store values (meaning the per site volume for the number of sites contained in whatever level the pivot grid is aggregated to).
  • Display Currency: The currency to display for all revenue based fields in the view. Defaults to the Home Currency for the active Assortment Plan.

    Due to the amount of data that can potentially be returned in the view via these selection parameters, there are levels at which the performance of the pivot grid will become sub-optimal. A message displays when the data to be returned exceeds the threshold. Users can revise their parameters.
