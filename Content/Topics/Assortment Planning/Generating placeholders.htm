Generating placeholders

Placeholders are added to collections to help plan items before the details of these items are officially known, such as color and style. With placeholders, the financial target can be planned and the sales projection built.

To create a new collection, see Creating a new collection.

To add placeholders to collections

  1. Navigate to the Placeholder view or Variants supporting view in the Collection view.

  2. Click New to display the Add Placeholders window.

  3. Enter the number of new placeholders in the # of Placeholders field.

  4. Configure the Mirror Options as required.

  5. Ensure Current Collection is checked.

    Uncheck Current Collection to select different or create new collections.

  6. Click Generate.

To manually select an existing collection when creating a placeholder

  1. In the Add Placeholders window, ensure that Current Collection is unchecked.
  2. Click in the Coll Code column to display a collection selector.

  3. From the dropdown list, double click a collection to select it, or enter the collection name in the Coll Desc column.

For information, see Creating new collections.

Parameter descriptions

  • # of Placeholders: The number of new placeholder lines to add when Generate is clicked.

  • Mirror Options: When checked, copies the selections made for the topmost placeholder line to any beneath it. It is not necessary to uncheck this option to deviate from the selections made in the topmost placeholder line. Editing unchecks the box automatically.

  • Current Collection: When checked, automatically sets Coll Code and Coll Desc to the values for the collection of the placeholder that was selected when the Add Placeholders dialog was launched.

Viewing product codes and/or product descriptions in a product hierarchy dropdown

The product selection dropdown in Placeholder and Collection views can be configured to display:

  • product description

  • product code

  • product code and description

Which fields to display is set using system policy Product Level Display (Module: Assortment).

Metric notes

For plans with:

  • wholesale lines - Sales, Sls and AUR renamed to Bookings, BK and ASP.

  • mixed wholesale and retail - Sales, Sls and AUT renamed to Sales/Bookings, Sls/Bk and AUR/ASP.

Editability

  • Discount %, At Once $/U, Cancel %, Cancelled $/U are all editable when the line is a wholesale line or have an underlying Cluster Set that is wholesale.

  • All the wholesale fields are non-editable if the line does not have any Wholesale Cluster Sets.

  • Cancel %, Cancelled $/U are non-editable after actualization begins.

The Wholesale check box is purely informational and indicates a line is wholesale.

Calculations

Amending:

  • At Once $ recalculates At Once U and vice versa.

  • Cancel % recalculates Cancelled $ and Cancelled U

  • Cancelled $ recalculates Cancelled U and Cancelled %

  • Cancelled U recalculates Cancelled $ and Cancelled %

  • AUT recalculates Cancelled $

Any amendment that affects Sales Units and/or Total Bookings recalculates Cancelled U and Cancelled %.

Grid columns

  • Coll Code: The collection code to which the placeholder line is assigned.
  • Coll Desc: The corresponding description for Coll Code for the placeholder line.
  • PH Code: A unique system generated code for the placeholder line.
  • PH Desc: The description for the placeholder line.
  • Product Hierarchy: The product hierarchy level to which the placeholder line will be assigned. This defaults to the lowest common product hierarchy level present in the Product filter in the data selection parameters for the Placeholder view.
  • Cluster Sets: The cluster sets to which the placeholder line will be assigned. Defaults to any cluster sets assigned to the Product Hierarchy level in Assortment Plan Defaults.

    New placeholders are assorted to all clusters within the assigned cluster sets by default.

  • Phase In: The phase in date for the placeholder. Selections are limited to periods prior to Phase Out, if set. Defaults to the earliest non-actualized period within the Window of Time in the data selection parameters.

  • Markdown: The markdown date for the placeholder/collection.

    For information regarding extending markdown dates, see Extending the markdown date in Placeholders.

  • Phase Out: The phase out date for the placeholder.

    Defaults to the last period within the Window of Time in the data selection parameters.

  • Mkdn No Periods: The time frame for the markdown date to phase out date.
  • Markdown Offset Locked: Links the markdown and phase out dates to each other, so they work dependently on each other.

  • Add to Promos?: If checked, new placeholders are automatically included in all applicable existing promotions.

    Whether the checkbox is checked by default is determined by the system policy New Placeholders Added to Promotions Default.

  • Home AUT: The home currency average unit ticket (AUT) for the placeholder line.
  • Map From Variant: (Optional) Another placeholder item to use as a Like Style from which the system can seed a sales plan. The pop-up only displays placeholders that do not have a variant assigned. Populates with placeholder items that exist within the selected Product Hierarchy for the line. Type in the relevant field to filter the results.

    If set, this copies the original placeholder's Size Option, Curve Mode, and Seasonality Curve. Home AUT (unless specified in Home AUT) can calculate a sales plan based on sales present for the selected Like PH between the Comp Start and End dates specified

    The read-only Variant checkbox indicates which placeholders have a variant assigned.

  • Attrib Copy: Only valid if Map From is set. Defines whether any product attributes assigned to the Like PH are copied to the new placeholder line or if any defined attribute defaults for the Product Hierarchy level are assigned instead. For more information, see Assortment Planning Overview.For more information, refer to the Attribute Management User Guide.
  • Mod Factor: Defaults to 1.0. Only valid if Map From is set. Defines a factor to multiply the sales history of the Map From by.

  • Comp Start: Only valid if Map From is set. Defines the start date for the desired window of time from which to capture sales for the Like PH.

    • Selections are limited to periods prior to Comp End, if set
    • Defaults to the Phase In of the Map From

  • Comp End: Only valid if Map From is set. Defines the end date for the desired window of time from which to capture sales for the Like PH.

    • Selections are limited to periods after Comp Start, if set
    • Defaults to the Phase Out of the Map From.

Toolbar functionality

  • New: Adds a single placeholder line.
  • Delete: Deletes the selected placeholder line.
