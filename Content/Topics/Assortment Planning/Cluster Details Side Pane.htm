Cluster Details side pane

The Cluster Details side pane allows users to manage the assortment of placeholders and collections to clusters and sub-clusters.

For more information on working with Sub-Clusters, see Using Sub-clusters to adjust Phase Periods based on site attributes

There are queue processing tasks that may be triggered when a new cluster is added to an Assortment. For more information, see Synchronizing data throughout Assortment Planning.

While in the Placeholder or Collection views, the Cluster Details side pane is opened by:

  • selecting View Options > Side Pane > Cluster Details, or
  • right-clicking the grouping section of the main planning view and selecting Show Cluster Details.

Editing cluster assignments

The side pane contains a column group for each cluster set selected using the Cluster Sets parameter. Under each group is a column for every cluster in that cluster set.

Because not every cluster shown in the Cluster Details side pane is available to each placeholder or collection, the fields in each row are color coded to indicate whether a cluster is available to that placeholder or collection.

Color Cluster Set Description
  Available
  Not available to this product level
  Not available because sites in this cluster overlap with one or more already-selected clusters

To edit cluster assignments

  1. Select the placeholder or collection row.

  2. Click in a cluster column to assign or unassign that cluster.

    Use the All column under each cluster set group to toggle assigning and unassigning all clusters in the cluster set.

  3. Click Save when finished.

Notes

  • Placeholders generated by Smart Start are assorted to clusters per the logic described in Smart start .
  • Placeholders and collections generated using the Add Placeholders dialog are assorted to all cluster sets they were assigned to when added.

  • Assorting a placeholder or collection to a cluster it was not previously assorted to will not automatically create a sales plan for that cluster.

    A sales plan can be created through manual entry of metric values or use the Rate of Sale Calculator.

    For more information, see:

    • Rate of Sale Calculator view Supporting Views
    • Rate of Sale Calculator overview
    • The Rate of Sale Calculator User Guide.

Clusters that are unassorted from placeholders or collections still retain their sales plans in the database. They are just zeroed out in the user interface for calculation purposes and they are reinstated if the cluster is reassorted to the placeholder or collection. This enables users to quickly try different assortment strategies without losing work done previously.
