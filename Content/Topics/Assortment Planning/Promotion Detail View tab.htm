Promotion Detail View tab

The Promotion Detail View tab provides flexibility and visibility to promotions applied to placeholders across time and geography. This lets users effectively manage all active placeholder promotions at one time in one view. This view is modifiable and cluster set clusters can be opted in to participate in a particular promotion application.

Filter in the data area can be dragged into the Pivot Grid.

Parameters

In addition to the parameters found in the Assortment Promotion tab, the Promotion Detail View tab includes the following:

Parameter Description
Promotion Applications Allows for the selection of all Active and Inactive Promotion Applications as defined in the Promotions Definitions tab.

For more information, see Promotions Screen Parameters.

Pivot grid fields

Field Description
Included? Whether cells are included or not. When checked for the first time, users need to assign a new promotion index before proceeding. If a user unchecks a checked field, then the promotion index is removed.

Checking or unchecking a promotion from each time period and cluster acts as an override to previous assignments.

A square fill in indicates that some periods are skipped in the life of the promotion.
Promo Index Displays the index is for the plan.

Adjusting index values for each placeholder by time period and cluster acts as an override to previous assignments.

For example, 1.4 means there is a 40% lift in sales plan units while the promotion is in effect.

Grid formatting scheme

The cells in the grid follow the formatting scheme indicated below:

Color Description
Lighter gray Past periods
Darker gray Non-editable periods

See below for why metadata might be non-editable.
Blue Editable regular periods
Green Editable promotion periods
Yellow Editable markdown periods
Bold Active promotions
Italicized Inactive promotions

Why promotion metadata may be non-editable

Promotion metadata may be non-editable if:

  • It is in the past.
  • It is a replenishment item.
  • The placeholder is part of the promotion header’s exclusion list.
  • The engine status is queued or processing.
  • The item is not assorted to the given cluster set / cluster.
  • The item is not in phase for the selected time periods.
  • The item is in markdown and the Apply to Markdowns setting is off.
