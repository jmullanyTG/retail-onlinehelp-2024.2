Placeholder and collection view toolbar functionality

The Placeholder and Collection views are where the majority of work in Assortment Planning is done. Expandable side and bottom pane views provide access to core functionality that can be quickly configured on one or more screens and then saved as work spaces for quick recall later. This allows visibility to the core of a plan to be maintained, including product images. The effect of changes can be seen at different levels of the plan as they are made.

Placeholders help plan the items before the details of these items are officially known, such as color and style. Use placeholders to plan the financial target and build the sales projection.

The following are available on the toolbar:

New

Launches the Add Placeholders dialog (The Add Placeholder dialog is available in the Collection view via the Variants supporting view). For more information, see Add Placeholders dialog.

Use the Style Bank view to link variants to placeholders. For more information, see Assigning variants

Hindsight

Launches the Hindsight Placeholder Management screen in Placeholder view and the Hindsight Collection Management screen in Collection view. For more information on the Hindsight functionality, see Hindsight management.

Copy

(not available in Collection view) Launches the Copy Placeholder dialog. Any placeholders not in a locked or Replen status can be copied. Copied placeholders are created in Open status.

If the placeholder has a Health Error (something is wrong with the data), then the Copy button is disabled.

Field Description
Number of Sets The number of duplicate placeholders to be created.
Add to Promos? If checked, the new duplicate evaluates all promotions it could be a part of and adds them.

Exceptions to exact duplication

  • If the original item is Assigned or above, the Variant assignment is not copied into the duplicate. All data associated with a variant is not duplicated:
    • Actualized sales, receipts, SOH
    • Planned orders from Order Manager or imported supplier order data
    • If the status of the original is Replen or Allocate, the Replenishment or Allocation links are not duplicated. For Replen items, the sales total will duplicate, but the sales plan (e.g. how that sales total is distributed across weeks) will not necessarily duplicate. This is recalculated according to the seasonality curve.
  • Receipt data is not copied. This must be regenerated by the engine.
    • Receipt AUC is a weighted average by receipt units. Since receipt units are generated by the engine, the weightings may differ and thus AUC might be slightly different between original and duplicate.
  • Promotions are not duplicated. If Add to Promos? is selected, the new duplicate evaluates all promotions it could be a part of and adds them. This would include any Promotions created since the original placeholder was created. Therefore, the copy’s promotion membership may or may not match original’s promotion membership.

Split

(not available in Collection view) Launches the Split Placeholders dialog. For more information, see Split Placeholders dialog.

Assign

Variants can be automatically imported and assigned from MS Excel spreadsheets or from the Clipboard. This functionality significantly reduces the time and repetitive work required to make Variant information (descriptions, color, vendor style code, costs, etc) from Vendors available in AP.

To import, click the Assign button on the toolbar and select one of the options on the dropdown.

The menu option Unassign Selected unassigns variants attached to the selected placeholder rows.

  • Not available at header level in Collection view. Available in Placeholder view or Variant view of Collection view.
  • If any product attributes were attached to the variants being unassigned, the system reverts the overwritten attribute values back to their pre-variant assignment states.
  • Variants cannot be unassigned variants from placeholders in Committed status.

The import functionality functions in the same way as the Import Excel Data menu option in Order Manager.

Requirements

  • The Variants and PLM Codes to be assigned must exist in the Style Bank.
  • The Placeholders being assigned must be eligible for Variant and PLM Code assignment (i.e. Active and not Locked, not in Status Allocation, Committed, or Replen and not actualized).

It is possible to reassign Placeholders from one PLM code to another as long as the placeholder is not committed or actualized.

Delete

Deletes all selected placeholder or collection rows. To undo, use Undo > Deletions rather than Cancel. Deletions are permanent after clicking Save. Committed placeholders and collections cannot be deleted.

Commitments

Allows users to create Commitments in bulk. There are two options:

  • Selected: Attempts to create Commitments for all highlighted rows (Placeholder Collections / Placeholders).
  • All: Attempts to create Commitments for all rows.

Grouping commitments

Undo

The Undo sub-menu has the following options:

  • Selected Row Changes: Reverts unsaved changes for the selected rows only. Use this when there are changes to multiple rows but only certain rows should be undone. Using Cancel undoes all changes.
  • Deletions: Undoes all deletions of placeholder or collection rows. Use this method instead of clicking Cancel to undo row deletions.

Locations

Launches the Manage Locations window (not available in Collection view). Use this dialog to exclude specific sites from clusters to which a placeholder is assorted and adjust their sales plans accordingly. For more information, see Manage Locations dialog.

View Options

The following options can be selected:

Option Description
Column Chooser Opens the Customization dialog. Drag fields to or from the main planning area and this dialog to customize the view. Custom layouts can be saved and restored by right-clicking in the Snapshot Templates area and selecting Save Layout or Restore Layout, respectively.
Side Pane Choose which side pane to view at the right of the screen. (Both cannot be open at the same time.)
  • Cluster Details (See, Cluster Details side pane)
  • Lifespan (See, Lifespan side pane)
  • None (Hides the side pane)
Image Tooltip Toggles whether image tool tips appear when hovering the mouse over a thumbnail image in the main view (not available in Collection view).
Active Style Bank If selected, the contents of the Style Bank are evaluated based on the currently selected row in the main planning view.

For more information, see Style Bank view.

Selecting this option causes the system to cycle through the full style bank data set and select all Variants that are in the same Hierarchy as the selected placeholder. Then each selected Variant is evaluated to determine if it is already assigned to a placeholder on the screen. If the Variant is already assigned, then the following is recorded:

  • which Clusters Sets they are assigned to
  • any overlapping Cluster Sets

If the placeholder selected in the main planning view is assigned to any of those recorded Cluster Sets, then the Variant cannot be assigned and is not added to the Style Bank view. The grid in the Style Bank view is then populated with the results.

When Active Style Bank is selected, the PLM Code field acts as a dropdown and shows the same Variants as the Style Bank view. Alternatively, double-click a selection within the Style Bank view.
Attribute Synch If checked, disables the automatic synchronization of changes in the Attributes view. By default, changes in the Attributes view are automatically synchronized to all affected placeholders in the main planning view. Occasionally, quick bulk changes such as might occur to pre-season assortments, causes a system slowdown while the UI automatically synchronizes the changes to all affected placeholders.

If Attribute Synch is unchecked then the main planning view grid will not automatically reload while users edit and maintain Attributes via the Attributes view. This alleviates any related UI slowdowns. However, once finished working with the Attributes users need to click Execute to reload Attributes into the primary main planning view grid.

Process Queue

There are queue processing tasks that may be triggered when an option in the Process Queue menu is selected. For more information, see Synchronizing data throughout Assortment Planning.

The Process Queue sub-menu has the following options:

  • Receipts section

    • Recalc Req. %: Displays the percentage of placeholders or collections in the current view that require receipt plan recalculation based on changes that have been made. This value is directly tied to the Receipt Status field in the main view. Placeholders and collections with a Receipt Status of Dirty require recalculation.

    • Recalculate Receipts: Recalculates receipt plans for all placeholders and collections with a Dirty status in the Receipt Status field.

      Receipt plan default constraints are set via Manage Assortment Plans > Defaults. System policies affecting the Assortment Engine output are set via Maintenance > Policies > Engine: Commitments.

    Receipt recalculations are sent to the Workload Queue module for scheduling and processing because they require complex calculations. Usually receipts are recalculated immediately. However, they may be deferred for later calculation based on the number of placeholders requiring recalculation and other jobs currently in the queue. Placeholders awaiting recalculation are locked and conditionally formatted to indicate this.

    Receipts for placeholders or collections in Replen status are calculated in the Forecasts module and read in, thus they never appear Dirty and are not recalculated when this button is clicked.

    All changes to plans, including changes made to constraints in the Receipt Plan supporting view, must be saved prior to recalculating receipts or they will not be taken into account by the Receipt Engine.

  • Replen section

    Recalculate Replenishment: Recalculates sales plans for placeholders or collections of status Replen with a Dirty status in the Replen Status field. The date placeholders or collections of status Replen were last recalculated is shown in the Last Replen field.

    As Replenishment recalculations require complex calculations, they are sent to the Workload Queue module for scheduling and processing. Though replenishment placeholders are frequently recalculated immediately, they may be deferred for later calculation based on the number of placeholders requiring recalculation and other jobs currently in the queue. Placeholders awaiting recalculation are locked and conditionally formatted to indicate this.

  • Alloc section

    Push Allocations: Initiates a push of variant data for the selected placeholders in Allocation status to the related allocation strategy in the Allocations module. The data pushed from Assortment Planning to allocation strategies is determined by the allocation strategy.

    The Last Alloc Push field updates on successful completion of the push to Allocations.

  • Sales Plan section

    Retrend: Runs Sales Plan retrend logic (specifically Monte Carlo) against the selected Placeholders or PlaceholderCollections. For more information on the Monte Carlo method, see Statistical explanation of the Monte Carlo method. For more information, refer to Statistical explanation of the Monte Carlo method.

    The re-trending of ROS is a valuable tool to consider for products as they transition from a pre-season plan to in-season. As products begin to sell, their sales patterns inevitably trend away from the initial plan (sometimes in the positive, sometimes in the negative). The included re-trend mechanisms in the application account for the trends in sales and form an amended go-forward sales plan against which continued allocations may be structured.

    Reviewing the results of running the Retrend logic

    The grid in the main planning view has several Retrend metrics showing the results and status of the Retrend operation.

    The Cluster Sets view has a View Style labeled ROS which displays the following ROS fields as well as Actualized Sales (if applicable).

    • Retrend ROS ($)

    • Retrend ROS (U)

    The Sales Plan view has Retrend Sales and Retrend Sales Units metrics.

    Configuring Retrend in Assortment Planning

    The system policies for Retrend in AP are:

    • Retrend: Evaluation Window Size
    • Retrend: Max Multiplicative Factor
    • Retrend: Min Multiplicative Factor
    • Retrend: Selling Days

  • External section

    Recalculate External: Queues placeholders with a status of External for processing. The Placeholders are locked until the process is completed.

Health %

Indicates what percentage of an assortment has issues that need to be addressed. More details are provided in the Health Error and Health Error Details columns in the main planning view.

For information on the Health Error codes, see Health Error Codes.

Workspaces

Within Assortment Planning, users can create and save workspaces. Workspaces are configurations of visible views which enable particular tasks such as setting (and saving a workspace with) Placeholder, Style Bank, and Cluster Sets views docked and visible to support Cluster Assortment planning. Use the Workspaces field to manage workspaces. Multi-screen workspace configurations can be saved as well as single screen ones.

  • Reset: reverts the workspace configuration to the system default for the view

  • Save:  launches the Manage Workspaces dialog allowing users to save the current workspace configuration for later recall. Requires a workspace name to be entered in the Workspaces dropdown.

    Field or option Description
    Name The name for the workspace.
    Description An optional description for the workspace.
    Family A workspace family in which to save the workspace.
    Protection Type The permissions for the workspace: Public (all users can access) or Private (only the user who created the workspace can access).
    Default Select to designate as saved layout as a default workspace. If set, the workspace is automatically applied when either the Collection or Placeholder views are opened. This functionality is specific to individual users only. Therefore, only private workspaces can be made default.
    Cancel Closes the Manage Workspaces dialog without saving the workspace.
    Save Saves the current workspace with the specified parameters.

  • Edit:  Launches the Manage Workspaces dialog in which some fields are editable. Name is read-only.

  • Delete: Deletes the current workspace.

    The system does not ask the user to confirm deletion of workspaces.

Execute

Refreshes everything in the current view, including all supporting views, based on the selected Product, Site Hierarchy, Cluster Sets, and Window of Time.
