Health Error Codes

Error Meaning Action
Actualization Has Not Run AP has a process called Actualization that is meant to run weekly after new core data is imported into the system (sales history, stock on hand, etc). This process incorporates actualized data into sales plans and makes any necessary adjustments. It is important that actual data is up-to-date in AP before making any subsequent changes to sales plans. Run actualization from the Job Manager. Contact Support if there are errors.
Actualized Sales Greater than Total The Total Sales metric is less than the Actualized Sales metric. Since Total Sales represents the lifetime of the item, adding Actualized Sales and Future Sales together, Total Sales should always be the greater number. This circumstance would only occur if there were some unexpected data condition (like extremely negative sales values in actualized data); there are many defenses in the application to prevent this from happening. Modify Total Sales to be greater than Actualized Sales, and Save. The Actualized Sales and Total Sales values can exactly match (and these values should match after the item has phased out), but there can be conflicts if entering in the exact Actualized Sales value into Total Sales and there are unrounded decimals not visible in the screen, so the value should be increased by 1.
Cluster Zero Site Count There are periods in the sales plan where there are no sites for the cluster set / cluster. This occurred because the cluster set snapshot active for the period contains no sites, or all of the sites in the cluster have been marked as closed. A sales, receipt, and inventory plan cannot exist where there are no sites. There are multiple actions to take to resolve this error. Reevaluate the cluster set snapshots over time, ensuring that all clusters have the right membership of sites. Reevaluate site status information, ensuring at least one site is open for each period of time during the life of the placeholder. If the cluster is truly meant to have no sites for some time (because, for example, a cluster has been removed or added), then change the AP plan to exclude the time where the cluster doesn't exist. This can be done by changing phase dates of the placeholder to not include the periods where the cluster is empty, or un-assorting from the cluster altogether using the Cluster Details pane.
Data Expiration Has Not Run AP has a process called Data Expiration that is meant to run weekly after Actualization. This process prunes old data out of data tables in order to reclaim space in the database. How far out the data is pruned is governed by a system policy (which can also disable this feature altogether). It is important that expiration run weekly so that the expected amount of data is present in the sales plan. Run data expiration from the Job Manager. Contact Support if there are errors.
Markdown Period Longer Than Expiration Period This error would occur if a very long markdown period was configured. In particular, there is an error if there is a Markdown set on an item with no Phase Out date, or if the Markdown period is longer than the Data Expiration period (a system policy). A Markdown phase this long is unexpected in AP data. Remove the Markdown date, or set the Markdown date closer to phase out. Note that Markdown Dates should not be set for placeholders with no phase out date; in other words, the placeholder cannot be on markdown indefinitely.
Missing Receipt AUC A receipt period with planned receipt units does not have a corresponding Receipt AUC value. This can impact cost calculations throughout the sales plan. Add a Receipt AUC value. This can be done for all periods by modifying the value in main planning view, or by period in the Receipt Plan View.
Negative Sales Plan There are negative values in at least one period of the sales plan for this placeholder. Change sales values to a zero-valued or positive-valued number either at the aggregate level in Main Planning View or by period in Sales Plan View.
No Selling Periods The sales plan for this placeholder has no active selling periods. This would occur if the user manually toggled Selling Period to "off" for all periods, or the system set Selling Period to "off" for all periods if there are no sites in the cluster. Depending on the desired outcome for the user, several actions could be taken. If all periods were opted out in error, then the user should opt some periods back in as selling. If the periods are marked as non-selling due to having no sites in the cluster, then actions should be taken to correct cluster definitions (see the Cluster Zero Site Count error). If the item is meant to not be in phase yet, set the phase dates accordingly. If the placeholder should never be sold for the cluster, rather than opting out each period individually, the assortment should be removed using the Cluster Details pane.
Period Horizon Past Calendar In order for AP to function properly, a calendar extending into the future must be defined. This error will be surfaced if the defined calendar does not extend far enough into the future for placeholders with no phase out date. Placeholders with no phase out date have sales plans extended to the Period Horizon configured for the plan. If the calendar does not have periods defined at least that far out, then there is a conflict. The best thing to do is to run maintenance or imports to add more future periods into the calendar. In the short term, the placeholders with no phase out can have phase out set temporarily to the last defined period in the calendar until the calendar is extended.
Insufficient Sales Plan There are values in the Sales Units plan that are too small. The threshold for this limit is defined by the system policy Lower Limit for Unit-Based Metrics. If the policy is set, any Sales Units value between 0 and this threshold (e.g. 0.0001) is meant to be zeroed out. We cannot allow very-near-zero numbers in Sales Units, because Sales Units is often a divisor, e.g. AUR = Sales / Sales Units. If a very small number is used as a divisor, the resulting quotient will be extremely large, breaking the numeric limits of the application. Verify the Total Sales Units metrics against the Actualized Sales Units metric. For example, if the Actualized Sales Units is 1000 but Total Sales Units is 1001, then the system will attempt to spread the remaining 1 unit across the planned future weeks, potentially splitting into very small fractions.

Ensure that the Total Sales Units value moderately exceeds Actual Sales Units (if this is already the case, enter in a slightly different Total Sales Units value to ensure the system reprocesses the sales plan and zeroes out too-small units values). If it is desired to have no sales in future sales plans, then either:

  • set Total Sales Units to exactly match Actual Sales Units

  • phase out the item, or

  • set periods to Non-Selling in Sales Plan view.
Overstated Ticket Price There are values in Home AUT or Blended AUT that are too large. The threshold for this limit is defined by the system policy Upper Limit for Value-Based Metrics. If the policy is set, any AUT value exceeding the threshold (e.g. $1,000,000,000) is meant to be cleared. We cannot allow extremely large numbers in the system, as this will break the numeric limits of the application. Reset any too-large Home AUT values through main planning view, Cluster Sets view, or Sales Plan view.
Too May Sites Excluded The user has excluded all sites from selling through the Location Exceptions screen. The screen will not allow a user to deselect all sites from a cluster (if there is a desire to not sell to any sites in a cluster, the entire cluster should be de-selected in the Cluster Details screen). However, if the cluster definitions changed over time to move the non-excluded sites out of a cluster, and the cluster then has all sites excluded, this error will be shown. Re-include selling sites in the cluster, or close selling in the cluster, either by setting Phase Out Date or by un-assorting to the cluster altogether in Cluster Details.
Stock On Hand Negative In a future period of the sales plan, the EOP Units value has become negative. This means that the receipt plan is not sufficient to support the sales plan. Increase receipts or move receipts forward in time, or decrease the sales plan.
Missing Data In Price Matrix If a Price Matrix is set for a placeholder, then the system will attempt to use the Price Matrix data to convert AUT values. If any of the following components are missing, the error displays:

  • an entry in the matrix for the home site's country / currency

  • an entry in the matrix for the selling sites' country / currencies

  • a row in the matrix for the starting Home AUT value

  • currency conversion data for converting non-Home currency values back to the home currency

 Assign a different price matrix, or ensure the price matrix is defined for all countries, currencies, and AUT values.
