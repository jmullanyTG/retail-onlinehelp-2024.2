Assortment Out-of-Tier Report view (not available in Collection view)

For a selected placeholder with sales or stock at a non-assorted site, the Assortment Out-of-Tier Report view can be used to drill down further into the data. The Assortment Out-of-Tier Report view can still display data even when the OOT Sales and OOT Stock fields are not marked. This is because the threshold for sales or stock was not met or exceeded.

Thresholds for sales and stock at non-assorted sites are set in the Assortment Plan Defaults tab.

The report displays the placeholder’s sales and stock data that correspond to the selected Window of Time, how much is in which cluster, and what the percentage is for non-assorted sites.

Metrics are actualized values (for example: using ActualSalesUnits versus TotalSalesUnits) so they may not match totals in the main planning view.

The Assortment Out-of-Tier Report view is displayed or hidden in Placeholder view using the View Selection dropdown menu. The report is only visible for one placeholder at a time. If multiple placeholders are selected in the main planning view, the report resolves only for the placeholder that has focus.

View layout

The Assortment Out-of-Tier Report view has four sections.

Assortment Inventory Report section

The Assortment Inventory Report section uses the cluster snapshot date selected in the Cluster Snapshot Date parameter.

Out of Tier Sites section

This section returns sales units for the period of actualization. Multiple sites can be selected to show sales history data and cluster candidates.

Sales History section

The Sales History section can be modified to show sales units at week level by right-clicking within the section and selecting Setup.

Cluster Candidates section

The Cluster Candidates section uses the cluster snapshot date selected in the Cluster Snapshot Date parameter. The grid lists the cluster sets and cluster set clusters the listed sites could belong to.

Cluster candidates- Example, let's say SiteID 123 belongs to Cluster Set 1 / Cluster 1, and also could be part of Cluster Set 12 / Cluster 2, and also could be part of Cluster Set 876 / Cluster 3 The totals SOH is from ALL the OOT sites that belong to 1 / 1, and the total that belong to 12 / 2, and the total that belong to 876 / 3 those cluster set / clusters may or may not contain the same numbers of sites, some overlap, some not
