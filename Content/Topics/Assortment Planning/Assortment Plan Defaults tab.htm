Assortment Plan Defaults tab

Accessing the Assortment Plan Default tabs

  • From the Assortment Planning module, click Manage and select Defaults on the Manage Assortment Plans screen.

Updating multiple rows simultaneously

  1. Select a row and press Shift.
  2. Click on all the rows to be updated or select a row further down the list to highlight a range of rows.
  3. Right-click in the field to be updated and choose Multiple Changes from the menu.
  4. Choose the appropriate value.
  5. Click Ok.
  6. Click Save.

Accounting for new store openings through the life of items

Two metrics account for new store openings through the life of items. The metrics are:

  • New Site Min Pres Multiplier
  • New Site Multiplier Number of Periods

This functionality increases the safety stock (Minimum Presentation or Minimum Display) at particular points during the life of the item by front loading or ordering more stock earlier to account for new store openings. The increase in safety stock occurs at each flow date, instead of at the beginning of the life of the item. Additionally, instead of applying a multiplier for all sites, a multiplier is only applied for the count of new sites.

To function as designed and apply the multiplier for the count of new sites only, it is crucial that the Date Opened and Date Closed fields be accurate within Site Manager.

Field descriptions

Path Code: The code for the corresponding level of the product hierarchy assigned to the current plan.

Path Description: The full description of the product hierarchy path corresponding to the Path Code of the product hierarchy assigned to the current plan.

Cluster Sets: The default cluster sets assigned to new placeholders generated within the specified level of the product hierarchy. Also, sets (along with Additional Allowed Cluster Sets) the cluster sets that will be available in the Cluster Sets parameter when the associated product level is chosen. Available choices are only cluster sets which have been snapshotted and passed validation for use within the current assortment plan.

The following message displays upon clicking Save when users attempt to assign multiple cluster sets containing overlapping sites to a single level of the product hierarchy:

Additional Allowed Cluster Sets: Sets (along with Cluster Sets) the cluster sets that will be available in the Cluster Sets parameter when the associated product level is chosen. Cluster sets selected here are not automatically assigned to new placeholders. Available choices are only cluster sets which have been snapshotted and passed validation for use within the current assortment plan.

If users attempt to assign multiple cluster sets containing overlapping sites to a single level of the product hierarchy, an error message appears.

Minimum Presentation: Sets the minimum presentation stock in units per site for a given period within the current assortment plan.

This setting persists across time for any periods for which Minimum Presentation has not been set in the Assortment Plan Defaults By Time tab, with values entered there acting as overrides to this value.

Seasonality Curve: Sets the default seasonality curve applied to placeholder instances generated within the specified level of the product hierarchy.

Curve Mode: Sets the mode of the seasonality curve being applied to a placeholder:

  • Static: The selected curve is applied exactly as it exists at the time of application and the placeholder’s curve will not change if changes are made to the curve, whether manually in the Curves module or through dynamic recalculation. Think of this as making a copy of the applied curve
  • Dynamic: Changes made to the curve are pushed down to the placeholder and the dependent sales plans recalculates when those changes are pushed.

Lead Time Days: The number of days prior to a given receipt date that an order must be placed to arrive on time.

Used to calculate Next Order Date in the main planning and Receipt Plan views and Due Date in the Assortment Upcoming Commitments report for placeholders belonging to the corresponding product hierarchy level for which commitment orders have not been created and which either don’t have variants assigned or their assigned variants lack supplier association.

Number of Markdown Periods: The time frame for the markdown date to phase out date.

Margin Read-only. The gross margin percentage for the placeholder. The calculation is: ((AUR ex VAT / (1 + VAT %)) – AUC) / (AUR ex VAT / 1 + VAT %).

ROS $: Read-only. The current ROS $ for the placeholder. Blank for new placeholders.

ROS U: Read-only. The current ROS U for the placeholder. Blank for new placeholders.

Ttl Sls $: Read-only. The current total sales revenue planned for the placeholder. Blank for new placeholders.

Ttl Sls U: Read-only. The current total sales units planned for the placeholder. Blank for new placeholders.

S/T %: Read-only. The target sell-through percentage for the placeholder.

Calculation: (Ttl Sls U / Ttl Buy U)

Home AUT: Editable currency ($10.01) The home currency AUT (average unit ticket) for the placeholder. If a Map From is specified, then AUT is set to that of the selected variant.

AUT is assumed to be inclusive of the Home Site VAT rate for the purpose of all calculations within Assortment Planning. For companies in which VAT is not in play for the home currency assigned to the active assortment plan, no special actions are required as long as the associated VAT % is set to zero. Alternatively, the system policy that enables the use of VAT in AUT calculations can be disabled. For more information, see Currency and VAT Management Overview.For more information, refer to the Currency and VAT Management User Guide.

AUR: Read-only. The average unit retail price for the placeholder.

Markdown Sales Plan Factor: The ratio between pre-markdown and post-markdown sales. This is used to calculate the post-markdown sales.

Markdown Offset Lock: Links the markdown and phase out dates to each other so that they work dependently on each other.

Periods of Supply: The periods of supply (period type as defined by the current assortment plan) for placeholders as a minimum stock constraint for receipt engine outputs within the Receipt Plan supporting view on the main planning view.

Actual by period constraint is defined as the sales planned for this many periods forward from a given period.

This setting interacts with the system policy Assortment Engine DC-to-Cluster Lead Time (Maintain > Policies > Engine: Commitments).

Assortment Engine DC-to-Cluster Lead Time is set to 7 days. Periods of Supply is set to 2 with a user planning a receipt to arrive in Wk 1.

For this constraint to be considered fulfilled for Wk 1, the period must end with at least 60 units of inventory on hand to cover weeks 2 and 3. The lead time of 7 days stipulates that stock being received in given week cannot be used to cover the sales planned in that week. Thus, Periods of Supply begins calculating need at Wk 2 rather than Wk 1.

Var Order Multiple: The number by which receipt quantities generated by the Recalculate Receipts functions in the main planning view and Receipt Plan views will be evenly divisible by. This persists in orders generated from assortment plans within Order Manager.

For more information, see:

  • Order Document details – break by size
  • Order Document details – Prepack tab

For more information, refer to the Break by Size and Prepack sections of the Order Manager User Guide.

This constraint can also be set in the Placeholder Constraints area of Receipt Plan view.

The system always rounds up to meet Order Multiple constraints. This ensures that the base sales plan on which the receipt plan is built will always be satisfied.

The following is a common scenario where very small parts are sold individually but it is not practical to ship them that way. For a sales plan of 241 units for weeks 1-4, and an Order Multiple of 50 set, the receipt quantity is 250 units or the lowest multiple of 50 that satisfies the entire sales plan.

Size Curve: The size curve assigned to placeholders generated within the specified level of the product hierarchy for the current plan. Works in conjunction with Rec Size Min in the Receipt Plan view and passed to Order Manager for use in the Break by Size stage of the commitment process.

All available size curves populate for selection in this field.

Periods Before Phase In: The default number of periods prior to Phase In date that initial receipt quantities for placeholder instances should be scheduled to be received. Used in the Receipt Plan supporting view on the main planning view. For example, a placeholder with a Phase In date of wk 4 and a Periods Before Phase In of 2 has a default calculated initial receipt for the placeholder of wk 2.

Flow Cadence: The default number of periods between receipt dates calculated by the receipt planning engine. Used in the Receipt Plan supporting view on the main planning view.

A placeholder with a Phase In date of wk 3, a Periods Before Phase In of 2, and a Flow Cadence of 4. 3 receipts will occur in wk 1, wk 7, and wk 11.

This is because the purpose of Periods Before Phase In is to ensure product can be in stores by the Phase In date. Thus, the first flow date is considered to be the Phase In date for the purposes of receipt engine calculation.

Min Stk/Sales: (Minimum Stock to Sales) Editable to 2 decimal places. The minimum stock to sales ratio for placeholder instances as a minimum stock constraint for receipt engine outputs within the Receipt Plan supporting view on the main planning view. Actual by period constraint is defined as the sales planned for a given period multiplied by the constraint value (period sales of 10 * Min Stk/Sls of 4 = EOP stock requirement of 40).

Shrink % of Sales: Sets the anticipated shrink units per period as a percentage of planned period sales units for placeholder instances as a minimum stock constraint for receipt engine outputs within the Receipt Plan supporting view on the main planning view.

For example, a period with planned sales of 500 units and a Shrink % of Sales of 2.00% must have an EOP units of at least 10 for this constraint to be satisfied for the period.

Maximum Pack % Total: Defines the maximum % of units in a commitment order of the units within prepacks can represent as a constraint for the prepack optimization engine within Order Manager. When set to less than 100%, units not placed in prepacks are fulfilled as single units.

For two placeholder orders for wk 4 for 500 units each, prepacked together they have a Maximum Pack % Total of 60.00%, the maximum number of units existing in prepacks for this order is 600. However, the prepack output could be less than 600 if the prepack configurations being used don’t evenly roll up to 600 units based on the need calculation by site.

Pack Tolerance: The number of units over the total order quantity the engine may suggest in an attempt to fulfill the Maximum Pack % Total, expressed as a percentage of the total order quantity. Used as a constraint for the prepack optimization engine within Order Manager

For a total order quantity of 1,000 units to be prepacked, a Maximum Pack % Total of 100%, and a Pack Tolerance of 1.00%, the engine may suggest an adjustment of the total order quantity of up to 1010 units to allow for 100% of the original order quantity to be fulfilled by units in prepacks.

When calculating prepack optimization against a Pack Tolerance, the engine will prioritize stores with greater need first

Maximum Packs Per Location: The maximum number of prepacks of any configuration that can be allocated to a single site within the prepack optimization calculation to fulfill the anticipated sales for that site. Used as a constraint for the prepack optimization engine within Order Manager

The prepack optimization engine determines that a site requires 50 units from an order being prepacked, Maximum Packs Per Location is set to 5, and the selected prepack configurations all contain 9 units - ignoring site need by size for this example, the engine can only use 5 prepacks, containing 45 units total, as the prepack component of order fulfillment for the site and must assume the remaining 5 units will be fulfilled as single units.

Underage Weight: Sets the relative preference for underages when calculating optimal unit allocation by site. This is used as a constraint for the prepack optimization engine within Order Manager.

Used in conjunction with Overage Weight and Pack Weighting and calculated as a % to total of the 3 parameters (i.e. all 3 are summed then divided by the number of constraints that were not NULL. Thus, if only 2 of the 3 are set, the sum of their values is divided by 2 rather than 3 to determine the % to total).

Overage Weight: Sets the relative preference for overages when calculating optimal unit allocation by site. Used as a constraint for the prepack optimization engine within Order Manager.

Used in conjunction with Underage Weight and Pack Weighting and calculated as a % to total of the 3 parameters (i.e. all 3 are summed then divided by the number of constraints that were not NULL. Thus, if only 2 of the 3 are set, the sum of their values is divided by 2 rather than 3 to determine the % to total).

Maximum Overage: The number of units per site the prepack optimization engine may over allocate during calculation to reach an acceptable solution. This is used as a constraint for the prepack optimization engine within Order Manager.

Minimum Units Per Pack: The minimum number of units the prepack optimization engine may suggest in single prepack configurations. Used as a constraint for the prepack optimization engine within Order Manager.

Only valid when Optimize Prepack is checked for a given prepack group, otherwise user selected prepack configurations are used.

Only valid when Optimize Prepack is checked for a given prepack group, otherwise user selected prepack configurations are used.

Num Pack Configs: Sets the maximum number of configurations the prepack optimization engine may suggest when determining optimal fulfillment of need within an order. This is used as a constraint for the prepack optimization engine within Order Manager.

Pack Type Pick Cost: Used in the Pack Instruction Parameters section of the Packing Instructions view and as part of the calculation of Picking Cost Ratio.

Pack Type Max Weeks Cover: The number of Weeks cover that the Pack should be below.

Pack Type Max Cost Ratio: Used in the Pack Instruction Selector section of the Packing Instructions view. The Picking Cost Ratio field is formatted red if the Picking Cost Ratio goes above this value.

Pack Type Max Error Ratio: Used in the Pack Instruction Selector section. The Absolute Error Ratio field is formatted red if the Absolute Error goes above this value.

Pack Type Smallest Size Quantity: On packing instruction calculation the smallest size ratio within the curve receives this amount. This acts as the starting point of the pack and the rest of the size quantities are extrapolated from this.

Calculation

The system retrieves all Sites associated with the selected placeholder and a size curve for each Site is calculated. The individual Site size curves are rolled up to the placeholder level and re-normalized to give an overall placeholder size curve.

The system assigns the smallest % a value that corresponds to the hierarchy default Pack Type Smallest Size Quantity. The rest of the size quantities are extrapolated from this.

The system multiplies the initial Pack Quantity by 2, 2.5, 3, and 3.5 to give a range of Pack options to be used. The system suggests a Pack that is below the Max Weeks of Cover, below the Max Cost Ratio, and has the lowest error rate. If no Pack fits these requirements, then the Pack with the Multiplier that corresponds to Pack Type Default Multiplier.

For example, for S M L where S = 20% M = 30% L = 50%. When calculating the pack, size S receives the Smallest Size Quantity and the other 2 sizes are calculated from that.

Pack Type Default Multiplier: This is used as a tie break, if the system cannot select a Pack then the pack with the Multiplier corresponding to this value is selected. Valid values are 1, 2, 2.5, 3, 3.5.

Pack Type Default ROS: A default ROS can be set against the hierarchy and users can choose to use this value instead of the placeholder ROS in the calculation. To use this, Apply Hierarchy ROS must be checked in the Pack Instruction Parameters section.

Lower Stock to Sales Threshold: Sets the lower threshold for Stock to Sales. If the STS ratio falls below the lower threshold, then the text is formatted so users can identify that it is out of threshold bounds.

Upper Stock to Sales Threshold: Sets the higher threshold for Stock to Sales. If the STS ratio falls above the upper threshold, then the text is formatted so users can identify that it is out of threshold bounds.

Min Presentation Multiplier: Multiplies the minimum presentation units quantity passed into the receipt engine by the value entered in this field for the number of weeks specified in Min Pres Multiplier Number of Periods.

Min Presentation Multiplier and Min Pres Multiplier Number of Periods together have an effect on the receipt plan by weighting stock received toward the beginning of the receipt plan according to the values set.

Min Presentation Multiplier is set at 2.5 and Min Pres Multiplier Number of Periods is set to 5. If the minimum presentation is 15, then the minimum presentation is raised to 37.5 for the first 5 weeks.

Min Pres Multiplier Number of Periods: The number of periods that the multiplier is in effect from the phase-in date. A value of zero means effectively there is no multiplier. Once the number of periods has expired the minimum presentation values return to the original value.

As the item starts actualizing, the number of periods declines until it is zero. Therefore, if the multiplier is to be used over the first 3 periods of a receipt plan, then:

  • Before phase in, wk1, wk2, and wk3 all use the multiplier.
  • When wk1 is actualized and the current week is wk2, then wk2 and wk3 will use the multiplier.
  • When wk2 is actualized and the current week is wk3, only wk3 will use the multiplier.
  • When it is wk4 or later, the multiplier is no longer used, even though 3 periods continues to be the Min Pres Multiplier Number of Periods value set.

New Site Min Pres Multiplier: This value is applied against the minimum presentation for only the site count with new store openings.

New Site Min Pres Multiplier Number of Periods: Applies only to the site count of new store openings.

Use Cube for Flow Period Frequency?: Specifies which levels use item cube logic.

Use the Assortment Cube Matrix view to specify flow period frequency based on Item Cube (ROS & Cube size). The Assortment Cube Matrix view is located in the Views module. For more information, see Setting flow period frequency based on Item Cube (ROS & Cube size).

Out-of-Tier Sales Units Threshold: If set, a threshold at the product hierarchy level for the number of units sold outside an assorted cluster. Setting a default at a higher product level applies that default to the lower product levels beneath it. Once the threshold is met or exceeded the checkbox in the OOT Sales field in the Placeholder, Collection, and Hindsight (both Current and Historical) views is checked.

When evaluating sales for an item, lifetime sales (both sales that occurred at regular price as well as at markdown) contribute to reaching the threshold. Filters may be applied to view only regular or markdown sales as desired.

If not set or is set to zero, then there is no threshold and no action is taken regardless of how many sales occurred at sites that were not assorted to.

Metric Definition
Sales Sales units for the entire life of the item - planned (with overrides included) + actuals.

Out-of-Tier Stock Units Threshold: If set, a threshold at the product hierarchy level for the number of units in stock at sites that were not originally assorted to. Setting a default at a higher product level applies that default to the lower product levels beneath it. Once the threshold is met or exceeded the checkbox in the OOT Stock field in the Placeholder, Collection, and Hindsight (both Current and Historical) views is checked.

If current stock meets or exceeds the threshold, even if currently at markdown, then the OOT Stock checkbox is marked. However, filters may be applied to view only regular or markdown stock as desired.

If not set or is set to zero, then there is no threshold and no action is taken regardless of how many units are in stock at sites that were not assorted to.

To ensure a transfer or reallocation is worthwhile, the variant stock level must be greater than X for the threshold to apply.

Metric Definition
Stock on Hand Triggered by EOP Units for the last actualized period (cluster-specific EOP combined with its portion of DC-split EOP)
