Hindsight management

The Hindsight button on the toolbar launches the Hindsight Placeholder Management screen in Placeholder view and the Hindsight Collection Management screen in Collection view. This screen can be used to create new placeholders and manage Like Style mappings for new and existing placeholders and collections within plans.

Customizing the columns included in the Current and Historical views

Right-click in either grid header and select Column Chooser to customize the columns that appear in the Current and Historical views.

Editing multiple rows simultaneously

To edit multiple rows simultaneously:

  1. Select multiple rows.
  2. Right-click in a field that is blue in color.
  3. Choose Multiple Changes from the context menu.
  4. Update the fields as required in the Multi Data Change Form.
  5. Click Save.

Dragging and dropping

Regardless of the view configuration (with Current on the left and Historical on the right or vice versa) users can drag and drop items from both sections to the other to set Map From and Map To (see below for details on these fields).

Filtering results

To filter results, hover the mouse in the top right of each column header and click the icon to display a filter dialog.

In the Coll Code and Coll Desc column headers, click the icon and select a value or type in the text field to filter the results returned. % is a wildcard to do “contains” type filtering instead.

To create a new collection, enter a description for the collection in the space provided and click New.

Toolbar functionality

  • # Placeholders, # Collections: The number of new placeholder or collection lines to add when New is clicked.

  • New:

    In Placeholder view: Adds the number of new placeholder lines specified in # Placeholders to the bottom of the Current grid.

    In Collection view: Adds the number of new collections specified in # Collections to the bottom of the Current grid. The number of placeholders to be added is specified in the PH Count field.

  • Copy: Lets users bulk select and copy placeholders or collections in the Historical grid. All details, attributes, and the new adjusted ROS, comparison stock and end date are copied. For collections, copies all existing placeholders from the original collection, but drops variant mapping, so the new collections consist of the same number of placeholders as LY.
  • Clear: Removes mapped Like Styles for selected placeholders or collections.
  • Delete: Deletes selected new placeholders or collections. Existing placeholders are deleted in the main planning view.

  • Mirror Options: When highlighted, copies the selections made for the topmost new placeholder or collection line to any beneath it. It is not necessary to un-highlight this option to deviate from the selections made in the topmost placeholder line. Edit as normal and the button un-highlights automatically.

    The default behavior is to save the state of the button (whether selected or not) when the view is exited. For example, if the Mirror Options button is selected when the view is exited, then the button is selected (highlighted) the next time the view displays.

  • Current Collection: (Placeholder view only) When highlighted, automatically sets the Coll Code and Coll Desc columns for newly created placeholders to the values for the collection of the placeholder that was selected when the Hindsight Placeholder Management screen was launched.

  • Historical: Flips the Current and Historical sections in the view.

    The dynamic of assigning historical variants to placeholders is essentially the same regardless of the layout. Be aware that while a single historical variant can be assigned to many current placeholders, only one variant can be mapped to a current placeholder when assigning in reverse. (If a blend of multiple variants is desired to seed rate of sale for a placeholder, the Rate of Sale Calculator can be used.)

    Placeholders that are Inactive or in Committed status will not be shown in the Current section of the view.

    For more information, see Rate of Sale Calculator View.

  • Parameters: Shows or hides the data selection parameters for the view.

Parameter options are the same as in the main planning view defaulting to the same product, site, and cluster set selections as in the main view. Window of Time defaults to the LY equivalent of the time selected in the main view.

Data selection parameters only determine the variants/placeholders that populate in the Historical section of the view. Current placeholders are populated based on those shown in the main planning view.

Current section

Attribute columns within the Hindsight Placeholder Management screen are controlled in the same way as Attribute columns for Variant Manager i.e. within Attribute Management.

  • Active: If checked, then the new placeholder or collection is active and can be added to existing promotions. If unchecked (inactive), then the Add to Promos? field is deselected and not editable. For more information, see Main planning view .

    Sales plans created from mapped variants should persist.

  • Add to Promo?: If checked, new placeholders are automatically included in all applicable existing promotions.

    Whether the checkbox is checked by default is determined by the system policy New Placeholders Added to Promotions Default.

  • Adj ROS ($): Blank unless Map From is set. Displays the placeholder level ROS $ calculated from the Map From based on the Comp Start and Comp End Dates and any Mod Factor specified that will be assigned to the placeholder.

    As rates of sale are calculated and assigned at the cluster level, actual rates of sale assigned to individual clusters may vary from the value shown here, just as differences can exist between the ROS $ shown in Placeholder view versus those in Cluster Set view.

  • Adj ROS (U): Blank unless Map From is set. Displays the placeholder level ROS U calculated from the Map From based on the Comp Start and Comp End Dates and any Mod Factor specified that will be assigned to the placeholder.

    As rates of sale are calculated and assigned at the cluster level, actual rates of sale assigned to individual clusters may vary from the value shown here, just as differences can exist between the ROS U shown in Placeholder view versus those in Cluster Set view.

  • Attrib Copy: Only valid if Map From is set. Defines whether any product attributes assigned to the Map From are copied to the new placeholder line or if any defined attribute defaults for the Product Hierarchy level is assigned instead.

    For more information, see Assortment Planning Overview. For more information, refer to the Attribute Management User Guide.

  • AUR: The average unit retail (AUR) price for the placeholder.

  • Home AUT: Sets the home currency average unit ticket (AUT). Home AUT is set to that of the selected variant if a Map From is specified.

    AUT is assumed to be inclusive of the Home Site’s VAT rate for the purpose of all calculations within Assortment Planning. For companies in which VAT is not in effect for the home currency assigned to the active assortment plan, no special actions are required as long as the associated VAT percent is set to zero. Alternatively, the system policy that enables the use of VAT rates in AUT calculations can be disabled.

    For more information, see Currency and VAT Management Overview. For more information, refer to the Currency and VAT Management User Guide.

  • Cluster Copy: Only valid if Map From is set. Defines whether the placeholder copies the cluster assignments of the selected Map From or uses those set by the user on the placeholder line.

    Existing cluster assignments are overwritten when a Map From is assigned and its cluster assignments are copied for placeholders that existed prior to launching the Hindsight Placeholder Management screen.

    When a Map From is assigned but its cluster assignments are not copied for placeholders that existed prior to the launching of the Hindsight dialog, it is possible that some of the pre-assigned clusters will not have rates of sale if there were no sales in those clusters between the Comp Start and Comp End dates for the Map From selected.

  • Cluster Set Message: Displays system messages regarding the copying of cluster assignments from a Map From for the specified placeholder.

    An example of a scenario that would generate a message:

    Scenario: Placeholders X and Y are both assigned to Variant A.  Placeholder X is assorted to Cluster Set 1 and Placeholder Y is assorted to Cluster Set 2. Placeholder X in the Hindsight view is to be changed.

    If a Map From that is assorted to Cluster Set 2 is selected, then cluster sets cannot be copied because it would cause both Placeholders X and Y to be assorted to Cluster Set 2 for the same variant.

    Cluster set messages

    • The Like Style is assorted to more cluster sets than are visible: (<list of cluster sets if the length of list is less than 512 characters OR the text "x cluster sets">), If applied with Cluster Assortment Copy checked, these assortments will also be created for this placeholder
    • The Placeholder to be altered is assorted to more cluster sets than are visible: (<list of cluster sets if the length of list is less than 512 characters OR the text "x cluster sets">). If applied with Cluster Assortment Copy checked, these assortments will also be removed from this placeholder
    • The Placeholder to be altered is assorted to cluster sets to which the logged-in user does not have access: Assortments to this placeholder cannot be altered using this method
    • Changing assortments for this variant would cause overlapping cluster sets to exist over the same time ranges: This is when the placeholder to be altered is assorted to a variant, that variant is also assigned to other placeholders in the plan, and those other placeholders' combined cluster set / cluster / date assignments would be in conflict if the change was made.
  • Cluster Sets: The cluster sets to which the placeholder is assigned. The cluster sets listed include all default and additionally available cluster sets for the product level selected in the Product parameter.

  • Coll Code: (Editable for new placeholders in Placeholder view) Double-click in the dropdown list to select a collection code to which the placeholder is assigned.

  • Coll Desc: The corresponding description for the Placeholder or Collection. Editable for new Collections.
  • Color Code: The color code for the placeholders / variants shown. The field is filterable and sortable. Editable for new rows.
  • Color Desc: A description of the color.
  • Comp End: Defaults to the Phase Out of the Map From. Only valid if Map From is set. Defines the end date for the desired window of time from which to capture sales for the Map From. If set, selections are limited to periods prior to Comp Start.
  • Comp Start: Defaults to the Phase In of the Map From. Only valid if Map From is set. Defines the start date for the desired window of time from which to capture sales for the Map From.
  • Image: The primary image associated with the placeholder. If no primary image exists, then No image data displays
  • Locked: For full description, see Main planning view .

  • Map From: (Optional) Selects a placeholder item to use as a Like Style from which the system can seed a sales plan and other information.

    Like Styles can also be dragged from the Historical section onto the target placeholders in the Current section. Dragging a Like Style onto a placeholder that already has a Like Style assigned will replace the previous assigned style with the new one.

    Populates with placeholders and/or variants shown in the Historical section that exist within the selected Product Hierarchy for the line.

    If set, this copies the selected item’s Size Option, Curve Mode, Seasonality Curve, Home AUT (unless specified in Home Ticket Price), Min Pres, Location Exceptions, and receipt plan constraints (Receipt Variant Min, Receipt Size Min, Variant Order Multiple, Periods of Supply, Periods Before Phase In, Flow Period Frequency, Min Stock to Sales, Shrink % of Sales, Recalc Buy, and Hold Receipt Units) and calculates a sales plan based on sales present for the selected Like PH between the Comp Start and End dates specified.

  • Margin %: Gross margin percentage for the placeholder.
    • Calculation: ((AUR ex VAT / (1 + VAT %)) – AUC) / (AUR ex VAT / 1 + VAT %)
  • Markdown: The markdown date for the placeholder.

  • Mkdn No. Periods: The time frame for the markdown date to phase out date.

  • Mkdn Offset Locked: Links the markdown and phase out dates to each other so they work dependently on each other.

  • Mod Factor: Defaults to 1.0. Only valid if Map From is set. Defines a factor to multiply the sales history of the Map From.

  • OOT Sales: Whether the item meets or exceeds the product hierarchy level default threshold for sales units outside an assorted cluster. A mark indicates the threshold has been met or exceeded.

    The OOT Sales field indicates that there have been noteworthy sales of units at non-assorted sites. This helps provide a total picture of sales including those sales outside of the assorted cluster for a placeholder or variant. Users may decide to include the site in the cluster set moving forward.

  • OOT Stock: Whether the item meets or exceeds the threshold at the product hierarchy level for the number of units in stock at sites that were not originally assorted to. A mark indicates the threshold has been met or exceeded.

  • PH Code: Unique system generated code for the placeholder

  • PH Count: (not present in Placeholder view). The number of the placeholders assigned to a collection. Editable for new Collections.

  • PH Desc: The description for the placeholder

  • Phase In: The phase in date for the placeholder. If set, selections limit to periods prior to Phase Out.

    Defaults to the earliest non-actualized period within the time frame selected.

  • Phase Out: The phase out date for the placeholder. If set, selections limit to periods after Phase In.

    Defaults to the last period within the Window of Time in the data selection parameters.

  • Product Level: The product hierarchy level to which the placeholder is assigned.

    This defaults to the lowest common product hierarchy level present in the Product filter in the data selection parameters for Placeholder view.

  • Promo?: Whether the placeholder has any active promotions applied to it.
  • Receipt AUC: Average unit cost price of receipts coming from the sales plan view.
  • ROS ($): The current ROS $ for the placeholder. Blank for new placeholders.
  • ROS (U): The current ROS U for the placeholder. Blank for new placeholders.
  • S/T %: The target sell-through percentage for the placeholder.
    • Calculation: (Ttl Sls U / Ttl Buy U)
  • Ttl Sls ($): The current total sales revenue planned for the placeholder. Blank for new placeholders.
  • Ttl Sls U: The current total sales units planned for the placeholder. Blank for new placeholders.

Historical section

This section displays the placeholders and variants contained within the Product, Site Hierarchy, Cluster Set, and Window of Time parameters. Rows in this section can be dragged onto new placeholders or collections in the Current section.

The label Historical is a generic term that is based on the selected parameter. Placeholders and variants from any time frame can be displayed here.

All metrics in this section are essentially the same as their Current section counterparts except that none are editable when the standard view configuration is engaged (meaning the Current section on the left and the Historical section is on the right).

When the Historical Section is on the left

This is achieved by clicking the Historical button on the toolbar. The Attrib Copy, Mod Factor, Comp Start, Comp End, Adj ROS ($), and Adj ROS (U) fields become part of the Historical section.

Map to: Selects a placeholder from those in the Current section with compatible product levels to map the item to as a Like Style.

This field functions very similarly to the Map From field except that each placeholder in the Current section can only be selected as a Map to once in the Historical section, as this view is not capable of mapping multiple items to a single placeholder. To blend multiple items together for the purpose of seeding rates of sale, users must use the Rate of Sale Calculator supporting view. For more information, see Rate of Sale Calculator View.

Users can also drag and drop placeholders from the Current section to set the Map to. If the placeholder being dropped is already selected as the Map To for another line in the Historical section, it is removed from that line and set as the Map To for the new line.

Buttons

  • Cancel: Closes the screen without committing any changes.
  • Save: Adds all new placeholders or collections and commits all Map From and Map To changes. Also, this initiates the recalculation of placeholder sales plans based on changes made.
