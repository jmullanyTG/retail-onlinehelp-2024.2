Managing locations in clusters to which a placeholder is assorted

The Manage Locations view is used to include or exclude specific sites from clusters to which a placeholder is assorted and adjust their sales plans accordingly.

Click the Locations button in the main planning view to launch this view.

Sites excluded by default when moved to a new cluster

By default, sites that are specifically excluded within Assortment Planning remain so. This is based on the system policy described in the following table. If the system policy is enabled, then when a site that has been excluded moves from one cluster to another cluster it remains excluded. Having an exclusion hold by default when a site moves from one cluster to another reduces the risk of assorting and later allocating items that were not intended to be sent out to that store. If a site should receive the item after changing cluster it can easily be included again.

Data is not destroyed on a system policy change, so if a switch is done in error there are no consequences, unless a manual data migration was done and data loss occurred. If no manual data migration is performed then all data is preserved but not actioned or displayed depending on how the system policy is defined.

System Policy Location
Allow Strict Site Exclusion Maintenance > Policies > Module: Assortment

Users can make exceptions for more than one placeholder at once by selecting multiple placeholders in the main planning view prior to clicking the Locations button. Items selected are stacked in both the top and bottom sections of the Manage Locations window.

The Cluster % Cont column lets users see the impact of including or excluding a site in a cluster based on a specific period of time.

Adding or removing rows and columns from the grid

  1. Right-click on a column header in the bottom section to display a menu.
  2. Select Show Field List.
  3. Use the PivotGrid Field List window to add or remove rows and columns.
  4. Close the PivotGrid Field List window when finished.

Cell formatting

Formatting is applied to cells to provide additional information.

A cell with a gray checkbox and a gray background indicates that there is nothing underlying that data i.e. the location has no E-Com stores or that all of the underlying stores are locked.

A white checkbox with a gray background means the stores are locked. To see the reason, bring in the Lock Reason field into the Data Area. In this case, one suggestion would be to filter the Locked entries out so as to ensure that those values do not interfere with setting at a higher level. Either by placing in the Filter Area or by placing in the Row Area.

Top section fields

Fields Description
Code The system codes for all selected placeholders.
Description The description the placeholder.
Cluster Set The cluster sets the placeholder is assorted to.
Cluster The clusters within each cluster set the placeholder is assorted to.
Location Count Count of the sites currently included in the corresponding cluster’s assortment of the placeholder, based on exclusions made in the bottom section of the dialog.

This field reflects the actual number of assorted sites within a cluster for the placeholder, but the Site Count metric in the main view will not update based on exclusions made here. This is because when saving exclusions the cluster level rates of sale are recalculated. Based on the % contribution of included sites and decreasing Site Count would cause double-decrementing of the related sales plan.
Total Sales ($) The total sales revenue for the placeholder.
Total Sales (U) The total sales units for the placeholder.
Adjusted Sales ($), Adjusted Sales (U) The % contribution changes made to the cluster in this view.

Users can restore and go back to sales numbers before any exclusion was made.

Bottom section fields

Fields Description
Code The system codes for all selected placeholders.
Description The description the placeholder.
Cluster Set The cluster sets the placeholder is assorted to.
Cluster The clusters within each cluster set the placeholder is assorted to.
Loc ID Unique system identifiers for the sites contained in each cluster.
Loc Desc The corresponding site descriptions for the sites contained in each cluster.
Cluster % Cont Percentage based on assigned Product Hierarchy level of the placeholder.

The level of product hierarchy data used to calculate the location % contribution tables is set by the Level Contribution parameter for the current assortment plan. The sales included are determined by the Sales Types parameter for the current plan. See Manage Assortment Plans above for more information.

When calculating location % contribution for location exceptions, sales defined by the above mentioned parameters and that correspond to the LY periods of the lifecycle (periods from Phase in to Phase Out) of each placeholder/cluster are used.
Include? (Default is checked) Whether a site is planned to receive the placeholder or not.

The sites displayed by cluster in this dialog are based on all effective cluster set snapshots for the placeholder over its life (or through the plan horizon, for placeholders without Phase Out dates). For more information, see Cluster manager overview.For more information, refer to the Cluster Manager User Guide.

Buttons

Button Description
Cancel Closes the Manage Locations dialog without making any changes to the selected placeholders.
Save Saves the changes made to the selected placeholders and updates the main planning view ROS $ and ROS U fields with the Adj ROS $ and Adj ROS U values shown in the top section of the Manage Locations dialog.
