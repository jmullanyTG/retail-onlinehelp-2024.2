Multi-Currency Price View

The Multi-Currency Price View view is used to both manage price exceptions by placeholder for all of the currencies used in the clusters to which that placeholder is assorted and view the specific usage of a price matrix. The two read-only Price Matrix columns, Price Matrix Local AUT and Price Matrix Home AUT, let users see how, for each period, the Home AUT value converts into each specific currency's value.

System policy Line Visualization Allow Price Exceptions sets whether this view is read-only or editable. If set to False, the view displays but is read-only. To affect a change in this policy on an already open plan, the plan must be reopened through the Manage Assortment Plans dialog.

Conditional formatting

Orange shaded cells indicates a change in AUT from the prior period for the currency indicated by the column section header.

Resetting default factors

Default translation factors for all non-actualized periods for the product level of the selected placeholder (as defined in the Price Translation Management, and Price Translation Hierarchy Assignment views) can be restored by selecting Reset Default Factors on the toolbar. Restoring default factors causes the Local AUT and Home AUT Equivalent fields to recalculate accordingly.

Price Matrix conversion method enabled

If Price Matrix conversions are enabled via system policy Price Conversion Method and a Price Matrix has been set on a given placeholder, then the configured Price Matrix is used to convert the provided Home AUT value to Blended AUT.

Price Matrix functionality example

First, a Price Matrix is created. Each row in the matrix indicates a series of prices that are equivalent to each other across countries and currencies.

In the example, this indicates that USD $67.99 is equivalent to PLN zł270. The matrix is meant to encapsulate both currency conversions and elegant rounding, or the standard price endings based on currency and locale, like ending prices in 99 cents in USD.

The Price Matrix should be specified as a hierarchy default so that newly created placeholders can automatically inherit this property. Then, when new placeholders are created with a given Home AUT, a Blended AUT value can automatically be created.

It is not necessary to set a Price Matrix Rounding rule. If a Price Matrix is specified, but no Rounding rule is specified, then Nearest is used.

The Multi-Currency Price View also indicates whether any value was missing from the indicated Price Matrix value, for example, if a site in the cluster had currency GBP but this currency was not included in the Price Matrix.

After the AUT values are converted from Home AUT values to currency-specific values using the Price Matrix, all AUT values are then converted back to Home currency using currency conversion rates, and a weighted average is used to calculate the cluster's combined Blended AUT value.

Each period and cluster's Home AUT values can be modified in the Sales Plan view, and Blended AUT for each period and cluster can be viewed there as well.
