Assortment Choice Count Report

The Assortment Choice Count report provides the ability to view and evaluate planned number of choices and productivity of the planned choices of an assortment in aggregate over a defined window of time. The report contains key measures at an aggregate level of the product hierarchy or product filter related to the choices planned for an assortment separating the choice counts by sales type. 

The report displays how many choices for a given period are regular price, on promotion, or designated as markdown. Additionally, choice counts can be measured against the same period last year to determine the balance of assortment as it relates to sales by type (regular price, promotion, and markdown).  Other key measures that are contained within this report are related to Total Sales, Total Cost of Sales, and Total Margin.

This report is used to manipulate the assortment by reviewing choice counts for a defined time period in aggregate and can be utilized to pass key information to users without access to the system. For example, merchandise planners can export this report as a PDF or spreadsheet to share with merchants or buyers or visual merchandisers. 

Additionally, this report provides insight to how an assortment is planned and how balanced an assortment looks in comparing regular price, promotional and markdown choices into the assortment. For example, a denim planner can illustrate the balance of fit types planned in aggregate for a period and how many of each fit type are planned to be regular price, on promotion or on markdown. 

Similar to the Assortment Choice Count By Period report, this report makes it easy to draw comparisons to last year in total sales, COGS, margin, or productivity, though in aggregate rather than across time. The primary benefit of this report is that it can subdivide a business at any level by segments that live beneath and see the percent to total of each.

A shirt planner wants to know the contribution to total sales different types of graphics represented to graphic shirts as a whole. These insights may identify what the trends of the business are and if the trends are covered in reviewing future period plans.

Parameters

The Assortment Choice Count report is controlled by seven parameters:

  • Product: Selects a product filter for the current view.

    •   Launches the Filter Conditions dialog in which the criteria for the selected filter can be edited prior to executing it. The option also exists to create a new filter from the adjusted criteria dynamically.
    •   Launches the Maintain Filter dialog to create or import a new filter, or update, delete, or export an existing filter.
  • Site Hierarchy: The levels of the current Plan’s Site Hierarchy to include in the current view.

  • Cluster Sets: The cluster sets to include in the current view. The cluster sets listed include all default and additionally available cluster sets for the product level selected in the Product parameter.

    While cluster sets are not explicitly part of the Site Hierarchy, they are directly related in that cluster sets in a given assortment plan must be contained within a single lowest level of that hierarchy. Therefore, the Cluster Set dropdown populates with the cluster sets contained beneath the selected Site Hierarchy levels.

  • Window of Time: Selects the window of time to display in the current view.

    The selected time frame must be a contiguous range of periods. If it is not, the window of time defined by the first and last selected period is still used.

  • Attribute Family: Each attribute family contains attributes that pertain to that family and can be imported from host systems if available.

    Attribute Family = Fabric Type, and a value belonging to this family could be cotton.  This selection defines the values within a family or group of families that will be available for selection and viewing in the sixth parameter.

  • Attribute Values: This selection returns the values associated with the attribute families selected in the fifth parameter.

    It is not necessary to select Attribute Family or Attribute Values in the parameters selections as it is possible to review choice counts and productivity through the utilization of a product filter, Site Hierarchy, Cluster Sets for a defined window of time.

  • Display Currency: Sets the currency to display for all revenue based fields in the view. Defaults to the Home Currency for the active Assortment Plan.

Deriving Choice Counts:

  • Historical Periods - choice counts for each type (Regular, Promo, Mkdn) are derived from the presence of sales of the corresponding type for a given variant.

  • Planned (future) Periods

    • Regular Choices - sales are planned for the period but no promotion is in effect, and the Mkdn Date, if set, has not yet been reached
    • Promo Choices - sales are planned for the period, a promotion IS in effect, and the Mkdn Date, if set, has not yet been reached
    • Mkdn Choices – sales are planned for the period and the period is the same as or past the period set as the Mkdn Date
