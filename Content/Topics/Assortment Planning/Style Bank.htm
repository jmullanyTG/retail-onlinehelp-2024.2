Style Bank view

Where a product lifecycle management (PLM) or similar system is integrated with Assortment Planning, use the Style Bank view to quickly locate and assign variants to placeholders within a plan. The Style Bank view can only be used to assign variants to single placeholders at a time. The Style Bank view is available in both Placeholder and Collection modes.

Add fields by right-clicking any column header, selecting Column Chooser, and dragging the fields to the column header area.

Depending on the structure of the assigned product hierarchy, product hierarchy fields are available to add to the Style Bank view to aid in identifying the desired styles.

Product attribute fields can be added to the Style Bank view to assist in qualifying selections of items to assign to placeholders in the main planning view. Fields are available for all attribute families configured in the Attribute Management tab of the Attribute Management module.

Attribute family fields are available for inclusion regardless of whether actual values are assigned to the variants displayed. Thus, it is possible to add attribute family columns that are partially or entirely blank.

The Style Bank view includes all Attribute data types for the Variants, in both the main planning view and the docked view in the Collection view.

The Style Bank view populates all variants assigned to any product hierarchy levels that exist in the Product filter selection in the view parameters.

Filtering the list

To filter the Style Bank by any column, type in the space provided below the header. Use % at the beginning to do a Contains type filter instead of a Starts With filter. More complex filters are applied by clicking the icon that appears when hovering the cursor over a column header and then clicking (Custom).

Displaying only relevant variants applicable to a selected placeholder

By default, the Style Bank view displays all variants regardless if they are within the selected product hierarchy. This can lead to a very large list to look through to find a desired style.

If selected, a View Option labeled Active Style Bank evaluates the contents of the Style Bank based on the currently selected row in the main planning view. This option provides a much smaller relevant list of styles that would make sense to be applied to the selected placeholder. Double-click on a variant to assign it to the selected Placeholder.

Assigning variants

To assign a variant, drag it from the Style Bank and drop it on the desired placeholder line. The full hierarchy path of the variant to be assigned is verified against the path of the placeholder. The cursor changes to a circle with a diagonal line through it if there is a mismatch. In this case, the variant cannot be assigned to the placeholder.

Variants that have already been assigned to placeholders in the current plan are conditionally formatted for easy identification (see the above screenshot for an example of this).

The same variant cannot be assigned to two placeholders within a collection. However, it is possible to assign the same variant to two placeholders in the plan as long as they are assorted to different cluster sets. This is to allow for plan seeding via multiple Target Creation scenarios for different areas of the site hierarchy that will share the same variants.

A variant cannot be assigned over another variant of a placeholder in Committed status.

Assigning a variant may overwrite or add to any attributes already assigned to the target placeholder. For more information, see Dual-directional attribute flow.
