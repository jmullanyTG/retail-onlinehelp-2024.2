AUT view

Deprecated 2021.4

The AUT view is used to make period level changes to an average unit ticket price (AUT) for the purpose of planning mid-season permanent price adjustments. This view can be used to make such changes even after a placeholder or collection has begun to actualize.

If system policy Editability of AUT is set to 2 (AUT is only managed through imports), then AUT is read-only for Placeholders with a status of Assigned, Committed, Replen, and Allocate. For placeholders with a status of Open and Created, AUT is editable.

Conditional formatting

The fields in the grid automatically format. Once a markdown date is applied black text refreshes to red text.

Toolbar options

The following toolbar options are available:

Option Description
Export Exports and saves the Pivot Grid in many common formats including Microsoft Excel and Adobe PDF.
Open Opens the Pivot Grid in many common formats including Microsoft Excel and Adobe PDF – does not require save prior to viewing in the desired format.
Graph Graphs the selected data in the view.
Fields Allows the user to customize the field layout for the view.

Custom layouts can be saved and restored by right-clicking in the Pivot Grid and selecting Save Layout or Restore Layout, respectively.
Save Saves all AUT changes made since the last Apply and initiates recalculation of blended AUR values for the selected placeholder.

Placeholder/collection level AUT calculates as a weighted average based on sales units of the periods in the phase window for each cluster.
Cancel Cancels all changes and reverts them to previously saved values.

Data fields

  • Period: The period being changed. This aligns with the period type for the current assortment plan.
  • Cluster Set: The cluster set containing the cluster to be changed.
  • Cluster: The cluster to be changed.
