Assortment Promotion Approximation window

The Assortment Promotions Approximation window displays by clicking in the Approximation field in the Promotion Application section in the Assortment Promotion Definitions tab in the Promotions view.

Grid columns

Column Description
Promotion Application ID The unique system identifier for the promotion application.
Seasonal Index The aggregate seasonal index for all placeholder/clusters within the period based on their assigned seasonality curves.
ROS The average sales revenue per site prior to the promotion application for the period for all placeholder/clusters included in the promotion application.
ROS Units The average sales units per site prior to the promotion application for the period for all placeholder/clusters included in the promotion application.
Sales Total sales revenue prior to the promotion application for the period for all placeholder/clusters included in the promotion application.
Sales Units The total sales units prior to the promotion application for the period for all placeholder/clusters included in the promotion application.
Promo Index The promotional index for each period for the promotion application being approximated.
Effective Index The index applied to period rate of sale to calculate period sales. Derived by multiplying the seasonal and promotion indices for each period then dividing by the average result of all periods from the minimum Phase In date to the maximum Phase Out date for all placeholder/clusters included in the application.
Home AUT The average unit ticket price for the period for all placeholder/clusters included in the promotion application in home currency.
Adjusted Sales The total sales revenue after the promotion application for the period for all placeholder/clusters included in the promotion application.
Adjusted Sales Units The total sales units after the promotion application for the period for all placeholder/clusters included in the promotion application.
AUR The average unit ticket price after the promotion application for the period for all placeholder/clusters included in the promotion application.
Sales Difference The difference between the sales revenue for the period before and after the promotion is applied. The calculation is Adjusted Sales – Sales.
Sales Difference Units The difference between the sales units for the period before and after the promotion is applied.

The number of periods shown before and after the periods the promotion is to be active is controlled by the Promotion Planning Time Periods system policy (Module: Assortment).

This system policy will not cause the approximation to include periods before the minimum Phase In Date or after the maximum Phase Out Date of all placeholders included in the promotion application, or of the single placeholder being approximated, when used in the Promotion Application Detail section.

The calculation is Adjusted Sales Units – Sales Units.
Applied By The user name of the person who created the promotion application.
Apply Date The date the promotion application was created.

Toolbar buttons

Button Description
Export Exports and saves the pivot grid in many common formats including Microsoft Excel and Adobe PDF.
Open Opens the grid in many common formats including Microsoft Excel and Adobe PDF. Does not require saving prior to viewing in the desired format.
Graph Graphs the selected data in the view.
Fields Lets users customize the field layout for the view. Custom layouts can be saved and restored by right-clicking in the grid and selecting Save Layout or Restore Layout, respectively
Save Saves changes made to the Include? field by site.
Cancel Undoes changes made to the Include? field by site.
