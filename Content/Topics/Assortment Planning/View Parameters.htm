Placeholder and collection view parameters

The following parameters are available in the Placeholder and Collection views.

Product

Selects a product filter for the current view.

  •   Launches the Filter Conditions dialog allowing users to edit the criteria for the selected filter prior to executing it. The option also exists to create a new filter from the adjusted criteria.
  •   Launches the Maintain Filter dialog allowing users to create or import a new filter, or update, delete, or export an existing filter.

Placeholder metadata and cluster metadata must be distinguished when creating or editing product filters:

  • Placeholder metadata fields are prefixed with Placeholder- and apply to the total placeholder.

    For example, filtering by Placeholder - Total Sales (Rev) compares the filter criteria against the total combined sales revenue of all clusters that the placeholder is assorted to.

  • Cluster metadata fields are prefixed with Cluster- and apply at the individual cluster level. When filtering by these fields, the filter criteria is compared to each assorted cluster individually. If any cluster meets the filter criteria, the placeholder is included in the filter. For example, filtering by Cluster- Total Sales (Rev) compares the filter criteria against the total sales revenue of each individual cluster the placeholder is assorted to. If any cluster meets this criteria, the placeholder is included in the filter.

Given sufficient permissions, it is possible for users to change the Product Level assignment or other properties of a placeholder causing it to no longer meet the criteria specified by the selected product filter. This causes the placeholder to disappear from the current view after clicking Execute. To view the placeholder, change the filter selection or adjust the filter’s criteria to include it again.

Site Hierarchy

Selects the levels of the current Plan’s Site Hierarchy to include in the current view

Cluster Sets

Selects the cluster sets to include in the current view. The cluster sets listed include all default and additionally available cluster sets for the product level selected in the Product parameter.

To assist users within organizations that have a large number of cluster sets in use in a given assortment plan, an additional filterable field labeled Product, has been included in this dropdown. This column displays the product levels to which each cluster set is available to as either a default or additionally available cluster set.

While cluster sets are not explicitly part of the Site Hierarchy they are directly related. Cluster sets in a given assortment plan must be contained within a single lowest level of that hierarchy resulting in the Cluster Set dropdown populating with the cluster sets contained beneath the selected Site Hierarchy levels.

Regardless of whether any of the placeholders or collections returned by Product, Site Hierarchy, and Window of Time selections are currently assorted to the cluster sets selected, the cluster sets will still show in the Cluster Details view, thus allowing users to assort to them.

For placeholders or collections to display in the main planning view, they must be currently assorted to at least one of the cluster sets selected in this dropdown. Therefore, to expand a placeholder to a cluster set it is not currently assorted to, select both a currently assorted cluster set and the one to expand to so the placeholder is visible. Then show the cluster details to assort it to the new cluster set. Once assorted, remove the originally assorted cluster set from the view parameters if needed.

An error message displays if a user tries to un-assort the last assorted cluster from a placeholder.

Window of Time

Selects the window of time to display in the current view.

Any placeholder or collection whose life cycle intersects with any part of the selected Window of Time displays in the view. This lets users work with placeholders and collections holistically.  Displaying subdivisions of a life cycle might alter users decisions if they have forgotten or are unaware of the full length of time a placeholder or collection is for sale.

Display Currency

Sets the currency to display for all revenue based fields in the view. This defaults to the Home Currency for the active Assortment Plan.

View Selection

Selects the supporting views to be enabled within Assortment Planning. Deselecting views that will not be used helps improve system performance by eliminating the need to store them in memory.

This also reduces the number of active system handles, if there are errors regarding the number of handles currently in use or red X marks across supporting views when they launch.

Displaying or hiding view options in Placeholder and Collection modes

System policy Hidden Items on the View Selection dropdown (Module: Assortment) defines which view options display in the View Selection parameter dropdown.

For more information on individual supporting views, see Supporting Views.

This setting is currently global for the plan and not specific to each user, thus deselecting views affects other users’ ability to see them. Only super users or system administrators should be given permissions to modify View Selection to help avoid issues.

Cluster Snapshot Date

The list of snapshots displayed in the Cluster Snapshot Date parameter is normally restricted by the dates selected in the Window of Time parameter. For example, selecting a small window of time restricts the list by the start and end dates selected in the Window of Time parameter.

To see all available snapshots for the plan including those outside the selected Window of Time, system policy Return all Cluster Set Snapshot Dates should be set to True.
