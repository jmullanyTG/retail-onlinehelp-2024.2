Assortment planning related system policies

The following table describes the available Assortment Planning related system policies.

Target Creation

For system policies specifically related to the Target Creation functionality, see Target creation related system policies.

Module: Assortment

System Policy Description
Algorithm To Match Lifecycle Curve to Sales Plan This policy determines what mechanism is used to match a lifecyle curve to a sales plan.

The two options are:

Period Match - this will search for the phase in week within the lifecycle curve (on any year, LY, LLY, a future year, etc, depending on the configurations of the time periods of the curve and sales plan). Then it will match up subsequent periods one by one, looping the curve around as many times as necessary as to cover the whole sales plan.

Scale To Fit will align the first period of the curve with the first period of the plan. If the sizes of the curve and plan are different, the curve will either be compressed or elongated to match the size of the sales plan.
Algorithm To Match Markdown Curve to Sales Plan This policy determines what mechanism is used to match a markdown curve to a sales plan. The two options are: Period Match - this will search for the markdown week within the markdown curve (on any year, LY, LLY, a future year, etc, depending on the configurations of the time periods of the curve and sales plan). Then it will match up subsequent periods one by one, looping the curve around as many times as necessary as to cover the whole markdown plan. A value of Scale To Fit will align the first period of the curve with the first period of the markdown plan. If the sizes of the curve and plan are different, the curve will either be compressed or elongated to match the size of the sales plan.
Algorithm To Match Seasonality Curve to Sales Plan This policy determines what mechanism is used to match a seasonality curve to a sales plan.

The two options are:

Period Match - this will search for the phase in week within the seasonality curve (on any year, LY, LLY, a future year, etc, depending on the configurations of the time periods of the curve and sales plan). Then it will match up subsequent periods one by one, looping the curve around as many times as necessary as to cover the whole sales plan.

Scale To Fit will align the first period of the curve with the first period of the plan. If the sizes of the curve and plan are different, the curve will either be compressed or elongated to match the size of the sales plan.
Algorithm To Match Static Curve to Placeholder This policy determines what mechanism is used to copy a placeholder from like style where the style has a static curve assigned. The two options are: False - this will keep the same existing static curve that is already copied and assigned to the like style, True - Re-copy the original dynamic curve as a new static curve.
Allow custom grouping of Placeholder Commitment Lines If True this allows the user to specify a grouping before Commitments are generated.
Allow DCs as Home Site Determines whether sites of type DC are allowed to be selected as the Home Site (used in determining Home VAT %) in Assortment Planning. Recommended setting: FALSE.
Allow Strict Site Exclusion Site Exclusion is controlled at either Placeholder [True] or PlaceholderInstance [False]. Recommended setting: FALSE.
Allow supplemental commitments Allow for supplemental commitment orders for a period which already contains a confirmed commitment order.
  • 0 = Do Not Allow (recommended setting)
  • 1 = Allow Supplemental Commits and Enforce Constraints on Supplemental Commit.
  • 2 = Allow Supplemental Commits and Do Not Enforce Constraints on Supplemental Commits
Assortment days history to actualize The number of days back to go when actualizing sales, receipts and stock on hand for assortment planning. Recommended setting: 60.
Assortment days history to maintain The number of days back maintain in a live assortment plan. Detail data older than this date will be summarized and then deleted. Entering the value "0" will keep all history. Recommended setting: 730.
Assortment Engine DC-to-Cluster Lead Time Default Lead Time Days from the DC to the Cluster that will be used to calculate Next Order Date information. This will be rounded to the nearest period.
Assortment Engine Supplier-to-DC Lead Time Default Lead Time Days from the Supplier to the DC that is used to determine eligible receipt dates. This will be rounded to the nearest period. Default value is 7.
Assortment sales types to actualize The sales types, as defined in tbl_SalesType, to use when computed actualized sales numbers. Recommended setting: 1, 2, 3.
Attribute Display Determines which fields to show in the attributes columns. 0 - Description, 1 - Code, 2 - Code and Description
Attribute to reference when applying item cube The name of the attribute to reference when applying flow period frequency logic based on item cube.
AttributeID to determine which default values to apply The AttributeID to reference when determining which default values in assortment.Attribute_Default to apply. Value of 0 indicates functionality is not used.
Automatically Mark Periods As Selling Default is False. If True, then when site counts change from zero to non-zero (e.g. when a cluster snapshot changes to re-populate a cluster), the period will automatically be marked as a selling period. If False, then the period will remain non-selling (but eligible for manual changes).
Batch size used in Assortment Actualization. The number of PlaceholderIDs processed in a single batch during Actualization. Recommended setting: 50.
Batch size used in Assortment Price / Cost Imports The number of PlaceholderInstanceIDs processed in a single batch during imports. The default is 500.

The process batches through placeholder / cluster set / clusters (instances). Upon install, it is recommended that a technical resource watches the import batches, and increases or decreases the batch size and monitors how that affects batch completion time. Changes to the system policy will be incorporated in the next batch; no stopping and starting is required.
Cost Type for Supplier Variant Cost. Cost Type associated with the Supplier Variant Cost within the Receipt Plan View.
Days To Keep Deleted Data The maximum number of days that deleted data will be held (hidden) in the database. Set a lower number for more aggressive deleting and more space conservation. Recommended setting: 0 (zero).
DC Stock Splits future number of periods Number of periods forward from the current period to determine % DC stock splits by cluster set / cluster. Default is 8.
Default Commitment Merge Default Merge OrderLines when commitments are created:
  • 0 = None
  • 1 = Merge by Variant (recommended setting)
Description for Generated Placeholder Collections from Seed This is the description used for newly generated placeholder collections from the seed process, followed by the Style description.
Description for Generated Placeholders from Seed This is the description used for newly generated placeholders from the seed process, followed by the Variant description.
Display products on Cluster Set parameter If enabled, display the field Products in the Cluster Sets dropdown. This will show the default product hierarchy levels associated with each cluster set. If there are too many levels to display, the column will show a count of product levels instead.
Editable Elements on Line Visualization when Replen Editable Fields / Objects when Placeholder Status has been set to Replen.
Enable Supersessions Determines whether item supersession information should be used to drive stock on hand and sales units inheritance between variants. If this is enabled, a new item's initial BOP value will be the supersession item's EOP, and actualized sales will show for the new item.
Enable VAT Conversion In AUT Calculation Determines whether to use VAT rate conversion steps when deriving currency-specific AUT values (Blended AUT) from the given Home AUT value. The default is True.

This system policy is referenced in the following processes:

  • when a placeholder has the AUT/AUR values recalculated from any numerical metric change (e.g. change a phase date, change total sales, change Home AUT)
  • when a placeholder is created from a like style
  • when a placeholder is created from Smart Start
  • when the Home Site property is changed on an assortment plan
  • when daily maintenance runs, it only considers changed VAT rates if the feature is enabled
  • if a VAT rate is changed live in the UI, it only applies changes if VAT is enabled
Enforce Supplier matching before Variant Assignment If True a Placeholder must have the same Supplier as a Variant before being assigned
Generate Receipts On New Item Creation Controls whether to generate receipt plans automatically on item creation. A value of 0 will not generate receipts automatically, but create new items in Dirty status. A value of 1 will queue receipts to be generated for items created through Smart Start. A value of 2 will queue receipts to be generated for items created through the Main Planning View (i.e. through New and Hindsight). A value of 3 will queue receipts to be generated for new items created through all means.
Hidden Items on the View Selection dropdown Views that will be hidden on the View Selection dropdown separated by a ~. Options are StyleBank, SeasonalityCurve, RateOfSaleCalculator, Variants, Promotions, SalesPlan, ReceiptPlan, ImageView, ClusterSet, Attributes, AUT, PriceExceptions
Image View Maximum number SQL Data Rows The maximum number of SQL data rows that can be queried in a view that returns images. Recommended setting: 20000.
Line Visualization Allow Price Exceptions Allow View / Edit of Price Exceptions in Line Visualization. Recommended setting: TRUE.
Line Visualization Maximum number SQL Data Rows The maximum number of SQL data rows that can be queried in a Line Visualization view. Recommended setting: 5000.

An error displays if the data selection parameters exceed this maximum.
Lower Limit for Unit-Based Metrics Default value is 0.0001. The lower limit for unit-based metrics. This is to prevent infinitesimally small numbers from entering sales plans. The value will be used as an absolute value. A value of 0 disables this feature. For information on setting an upper limit, see Upper Limit for Unit-Based Metrics.

Currently, this system policy will only be applied to Sales Units. When a value between -0.0001 and 0.0001 is detected in future periods, it will be set to 0.
Markdown Curve Default Behavior This policy controls what will happen to the markdown sales plan if no markdown curve is selected. The default selection of "True" will indicate that a line trending to zero will be used to decrease the sales plan to zero at phase out. A selection of "False" will not slope the sales plan to zero, but leave the markdown sales plan unchanged.
Multi Value Attribute Inheritance Defines the inheritance of multi-value attribute families attached to variants by the placeholders to which those variants are assigned within Assortment Planning. Value of 1 indicates placeholder values will be entirely overwritten by the variant attributes on assignment of the variant. Value of 0 indicates unique values attached to the variant will be added to the existing placeholder values on assignment of the variant. Recommended setting: TRUE.
New Placeholders Added to Promotions Default Determines the default selection for whether the option for adding new placeholders to promotions on creation is set to true or false.
New Placeholders Added to Promotions Filter Limit The number of product filters that will be refreshed when new placeholders are added and their promos are to be refreshed. Increasing this value will have performance implications when creating new placeholders.
Prepack Max Overage Percent Resolution Aggregation Function to resolve default collisions. Recommended setting: MIN.
Prepack Max Overage Resolution Aggregation Function to resolve default collisions. Recommended setting: MIN.
Prepack Max Pack Configs Resolution Aggregation Function to resolve default collisions. Recommended setting: MAX.
Prepack Max Pack Per Location Resolution Aggregation Function to resolve default collisions. Recommended setting: MIN.
Prepack Max Pack Percent Total Resolution Aggregation Function to resolve default collisions.
Prepack Max Units Per Pack Resolution Aggregation Function to resolve default collisions. Recommended setting: MAX.
Prepack Min Pack Configs Resolution Aggregation Function to resolve default collisions. Recommended setting: MIN.
Prepack Min Units Per Pack Resolution Aggregation Function to resolve default collisions. Recommended setting: MIN.
Prepack Overage Weight Resolution Aggregation Function to resolve default collisions. Recommended setting: MIN.
Prepack Pack Tolerance Resolution Aggregation Function to resolve default collisions. Recommended setting: MIN.
Prepack Pack Weighting Resolution Aggregation Function to resolve default collisions. Recommended setting: MAX.
Prepack Underage Weight Resolution Aggregation Function to resolve default collisions. Recommended setting: MAX
Price Conversion Method Determines which Price Conversion Method to use when deriving currency-specific AUT values (Blended AUT) from the given Home AUT value. 1 - Will use the original Price Exception method only, 2 - Will use the Price Matrix method only, 3 - Will use neither method. Default value: 1.
Product Level Display Determines which fields to show in the Product Columns. 0 - Description, 1 - Code, 2 - Code and Description
Promotion Planning Time Periods Promotions - Approximation. Integer representing the number of time periods before and after a promotion's application period that will be shown in Promotion Planning. Recommended setting: 2.
Promotions Applied to Markdown Default Determines default property of whether promotions can be applied to a markdown period for an item. This can be overwritten at a promotion and promotion application level. If True, the default is that promotions can be applied to markdown periods. Recommended setting: TRUE.
Receipt Cluster Merge All When commitments are generated, whether to group receipts for variants together across clusters by default, or whether to keep them separate by default. Recommended setting: TRUE.
Remove User Override Receipt Units on Commit If True remove User Override Receipt Units from Sales Plan / Receipt Plan views after creating Commitments
Require Display Primary Image Require usage of Primary Image Classification in Assortment Planning. Recommended setting: TRUE.
Require Placeholder Review before Commitment Generation If True this requires Placeholders to be Reviewed before a Commitment is allowed to be generated.
Retrend: Evaluation Window Size The evaluation window size used during the retrend calculation.
Retrend: Max Multiplicative Factor The maximum multiplicative factor that the system is allowed to use during the retrend calculation.
Retrend: Min Multiplicative Factor The minimum multiplicative factor that the system is allowed to use during the retrend calculation.
Retrend: Selling Days The number days (which will be converted into periods) in which there must be positive sales before the item is included in retrending.
Return all Cluster Set Snapshot Dates If set to True the system will return all Cluster Set Snapshot dates, ignoring the selected Window of Time
Show Effective Forecast Units in Receipt Plan View Show Effective Forecast Units in Receipt Plan View
SOH Actualization Number Of Periods Per Loop This policy should be set by a technical resource doing performance tuning of the actualization process. This will determine how many periods' worth of stock on hand (SOH) data is actualized at once. Depending on data size, hardware considerations, SQL version, etc, one period may need to be processed at a time (so the value set should be 1), or many periods could be processed in batch.
Style Bank Line Visualization Maximum number SQL Data Rows The maximum number of SQL data rows that can be queried in the Style Bank in the Line Visualization view. Recommended setting: 5000.
Toggle on/off snapshot aware seasonal index calculation Allows the assortment sales plan calculation to update based on future cluster site snapshots through manipulation of the curve indices. If Enabled, site counts by period will be used to scale seasonality indexes. If Disabled, seasonality indexes will be used as generated by the Curves module.
Update Variant Attributes in Assortment Planning Defines the ability to make attribute changes on assigned placeholders.
  • 1 = Indicates that placeholder attribute changes are allowed on assigned placeholders.
  • 0 = Indicates attribute changes cannot be made to assigned placeholders.

Recommended setting: TRUE.
Upper Limit for Value-Based Metrics Default value is 100,000,000. The upper limit for value-based metrics. This is to prevent infinitely large numbers from entering sales plans. The value will be used as an absolute value. A value of 0 disables this feature. For information on setting a lower limit, see Lower Limit for Unit-Based Metrics.

Currently, this system policy will only be applied to Home AUT, Blended AUT, and AUR. When a value greater than 100,000,000 or more negative than -100,000,000 is detected in future periods, it will be set to NULL.

Use Size Curve By Site If enabled, then the size curve used for the placeholder will be by-site, meaning, the site-specific breakdowns of size distributions will be used. If false, then the aggregate size curve across all sites will be used.
Variant Level Duty Percentage If True the Duty Percentage will be pulled from the Variant Level, if False from the Supplier record
Variant Primary Image Classification Update Allow Update of Primary Image Classification if Variant Present - Assortment Planning / Variant Manager. Recommended setting: TRUE.
VAT Rounding Digits The number of digits to round to when calculating historical VAT. A number 4 here means that the VAT value will be rounded to 0.xxxx, or xx.xx%. Recommended setting: 4.

System Policy Description
Flow Sequence From First Flow Date When this is enabled, flow dates are generated using the Flow Period Frequency starting from the first pre-phase-in flow. If disabled, then flow dates are started from phase-in, effectively treating all pre-phase-in flows as having arrived at phase in.
