Reconciliation Report parameters

The Reconciliation Report is controlled by eight parameters:

Parameters

Parameter Description
Product Filter based product selection.

Care must be taken to use filters judiciously while comparing to MFP to get an apples-to-apples comparison.

For example, if a hierarchy in MFP is configured as Company > Dept > Cat, and a filter is selected that contains the hierarchy selection of Company > Mens > Shirts, then placeholders for Mens Shirts and the MFP plan data for Mens Shirts are returned and are comparable.

However, if a filter is selected that is configured as Color = Blue, and there are blue items in Mens Shirts, Mens Pants, Womens Shirts, and Womens Pants, then the AP lines show totals for all blue items across those categories, but MFP data is unable to constrain on only blue items, so MFP data shows total: Mens Shirts + Mens Pants + Womens Shirts + Womens Pants.

This must be kept in mind when choosing a filter for use when running the Reconciliation report against MFP.

Site Hierarchy The level of the common site hierarchy between Comparison Plan 1 and Comparison Plan 2 at which to compare the reconciliation metrics.
Cluster Sets The cluster sets to include in the current view.

While cluster sets are not explicitly part of the Site Hierarchy, they are directly related, in that cluster sets in a given assortment plan must be contained within a single lowest level of that hierarchy. Therefore, the Cluster Set dropdown populates with the cluster sets contained beneath the selected Site Hierarchy levels.
Window of Time The window of time to compare for Comparison Plan 1 and Comparison Plan 2.

The selected time frame must be a contiguous range of periods. If not, the window of time used is defined by the first and last selected periods
Comparison Type 1 The plan or snapshot type of Comparison Plan 1 – either Assortment Plan or Merchandise Financial Plan. Plans and snapshots are split out as separate lines.

Merchandise Financial plans display in the same tree grid as the Financial Plan Maintenance view.

Merchandise Financial plan snapshots show in a hierarchy view, with the parent plans shown. The Master and Branch plans are prepended with Plan and the actual snapshots underneath listed alphabetically and prepended with Snapshot.

Sorting snapshots by name allows organizations to provide user direction on the naming convention to use to achieve desired sorting results. For example:

  • YYYY-MM-DD (Product Hierarchy) Description

  • (Product Hierarchy) YYYY-MM-DD Description

  • Plan Year (Product Hierarchy)

Two of the same AP Comparison Type can be selected at once. While the utility of this is limited when comparing the one available AP plan to itself, it is more useful when comparing two AP snapshots. There is a limitation that disallows users from selecting 2 MFP plans or snapshots to compare to each other.

This selection determines how the variance measures are calculated in the report.  For example, if the assortment plan is selected as Comparison type 1 and the financial plan is selected as Comparison type 2, the assortment plan is the first variable in the calculation that determines the variance.
Comparison Plan 1 The first plan or snapshot to compare in the view.
Comparison Type 2 The plan or snapshot type of Comparison Plan 2 – either Assortment Plan or Merchandise Financial Plan. Plans and snapshots are split out as separate lines.

This selection determines how the variance measures are calculated in the report.  For example, if the assortment plan is selected as Comparison type 2 and the financial plan is selected as Comparison type 1, the assortment plan is the second variable in the calculation that determines the variance.
Comparison Plan 2 The first plan or snapshot to compare in the view.
Display Currency (defaults to the Home Currency for the active Assortment Plan) The currency to display for all revenue based fields in the view.
