Seasonality Curve view

Using the Seasonality Curve view, select and apply seasonality curves to the entire selected placeholder or collection or only to selected cluster sets/clusters the placeholder or collection is assorted to.

The Seasonality Curve view in Assortment Planning is a scaled down version of the full Curves module (Execute > Curves). Creation of new curves is still done within the full Curves module.

Seasonality Curve screen fields

  • Filter Tags: Selects tags to be used in filtering the curves available for selection in Saved Curves. Tags can be attached to curves within the Curves module. The list of tags can be filtered by typing in the blank field above the list.

  • Saved Curves: A curve generated within the Curves module to apply to the selected placeholder or collection. Only curves of the Manual or Specified Conditions Using Enhanced Filtering types are available for selection within Assortment Planning.
  • Curve Mode: The mode of the selected curve:
    • Static: The selected curve is applied exactly as it exists at the time of application and the placeholder or collection’s curve will not change if changes are made to the curve, whether manually in the Curves module or through dynamic recalculation. Think of this as making a copy of the applied curve.
    • Dynamic: Changes made to the curve are pushed down to the placeholder or collection via automated processes and the dependent sales plans recalculate when those changes are pushed.
  • Apply To Cluster: The cluster sets/clusters to apply the selected curve to from those assorted to the selected placeholder/collection
  • Apply Lowest Level Curve: The level of curve to be applied:
    • Unchecked: Applies the curve exactly as shown to all selected cluster sets/clusters.
    • Checked: Looks for related lower level curves to apply at the cluster set/cluster level and applies them instead of applying the higher level curve as shown.
      • During implementation, a process is configured to automatically generate curves at levels of the product/site hierarchies on a specified schedule. If desired, this process can be configured to generate these curves at multiple levels.
      • “Related lower level curves” refers to these automatically generated curves – created using the same criteria

An automatically generated channel level curve is selected from the Saved Curves dropdown and the Apply Lowest Level Curve option is checked. The selected placeholder has 2 cluster sets, each with 3 clusters assorted to it, and curves were also automatically generated using the same criteria for each of those 6 clusters. The lower level curves do not have to be applied individually as the cluster-level curves are automatically applied to each cluster in a single action.

Curve Details section

  • Generation Type: The generation type for the selected curve. Only curves of the Manual or Specified Conditions Using Enhanced Filtering types are available for selection within Assortment Planning.
  • Bucket Type: The bucket type for the selected curve. For the purposes of Assortment Planning, bucket type is the same as period type.
  • Time Range: The time range used to generate the selected curve.
  • Forecasted Demand: If configured, generates curves based on projected future demand from the Forecasts module.
  • Curve Source: The curve source for the selected curve. Historical Demand By Sales Type Using Sales History generates curves based on historical sales by type
  • Generate By: The generation type for the selected curve. This is essentially the metric used to calculate the seasonal indices.
    • Volume Contribution (Units) – sales units
    • Cost of Sales Contribution – COGS (cost of goods sold)
    • Revenue Contribution – sales $
    • Profit Contribution – gross margin $ (sales $ - COGS)
  • Sales Types: The sales types used in generation of the selected curve. Not applicable for Forecasted Demand Curve Source.
  • Includes Returns: Whether returns (negative sales) were included in the calculation of the selected curve.
  • Inactive Items: Whether inactive items were considered in the calculation of the selected curve.

Product Filtering section

The fields shown depends on the options used to generate the selected curve.

  • Hierarchy: The product hierarchy.
  • Hierarchy Items: The product hierarchy levels.
  • Filter: The product filter.

Site Filtering section

The fields shown depends on the options used to generate the selected curve.

  • Hierarchy: The site hierarchy.
  • Hierarchy Site: The product hierarchy levels.
  • Filter: The site filter.
  • Cluster Sets:The cluster sets.
  • Cluster: The clusters.
