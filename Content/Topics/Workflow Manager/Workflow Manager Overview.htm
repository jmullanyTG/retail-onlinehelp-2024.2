Workflow manager Overview

Workflows define processes related to specific objects. At each step in a workflow, which users or roles have access to what actions and what actions the system performs can be controlled. Workflows can be used to:

  • set up approval processes for quality control.
  • promote safe cross-team collaboration.
  • ensure required steps and operations are always executed.
  • automatically enforce corporate standards and policies.

Workflows can be created for the following:

  • Events
  • Promotions
  • Price Actions
  • Order Headers

Accessing Workflow manager

Users require the Access to Workflow User Right found under the Workflow Manager right group in the Users module to create, edit, and delete workflows.

Workflows

Workflows have two main components: statuses and transitions. A status is a specific step in the workflow. A transition moves the object from its current status to its next status. Statuses and transitions are illustrated in the following workflow example:

This workflow has 4 statuses: Planning, Ready for Approval, Changes Needed, and Approved / Confirmed for Export. The transition arrows that connect the statuses indicate what status can come next. For example, the Ready for Approval status can transition to either Changes Needed or Approved / Confirmed for Export.

When designing a workflow, define the conditions in which a status can move from one status to the next. In some cases, a specific person or role may need to decide (for example, a reviewer must determine whether an object in the Ready for Approval status moves to Approved or Changes Needed). In other cases, the transition can occur automatically on a specific date.
