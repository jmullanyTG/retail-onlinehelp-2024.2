Status state permission blacklist

An object (event, promotion, price action, order header) can pass through several states in the context of a workflow status. For example, events are marked In Progress while they are not ready to progress to the next workflow status.

Status permission blacklists are used to enforce different sets of permissions based on the state of an object while it is in a specific workflow status. For example, an event workflow may begin with a Planning status. For events in the Planning status, all users are granted the right to edit events that are In Progress, but edit rights are restricted to managers only when an event is marked Completed. As events progress through the workflow, permissions are granted to smaller sets of users in each progressive workflow status.

Denying permissions for specific users

It is possible to grant a permission to a Role while denying that same permission to a specific user who is associated to that Role. In this case, the specific user is blacklisted and unable to perform the action even though the Role the user is associated to is not blacklisted.

Where a Role is blacklisted but not the user, then the user is blacklisted because of their association to the Role.

User blacklist rights used for the nightly workflow heartbeat job

The system takes into account the User Blacklist Rights that is used for the nightly Workflow Heartbeat job to determine if a Price Action, Promotion, or Event must be automatically advanced to the next Workflow Status based on the appropriate solution object's current Status State.

The system evaluates the user's user right to modify status based on the current status state of the object:

  • If the Job User does not have user rights to Modify Status based on the current Status State nor permission to the Transition, the system will not change to the next Workflow Status.

  • If the Job User has user rights to Modify Status based on the current Status State and permission to the Transition, the system will change to the next Workflow Status.

Setting permissions

Select a workflow status in the Design pane to see its Status Permission Blacklist in the Status Information pane. The blacklist contains a list of user rights and a set of columns representing potential object states.

A black user icon in a state column indicates that all roles and users are granted that user right when an object is in that state. A blue user icon indicates that at least one role or user has been denied that user right for objects in that state.

Click on a user icon to view or edit its user permissions. For more information, see Editing a status permission blacklist.

Object states and workflow types

The object states in a blacklist differ according to the workflow type.

Event states

Icon Description
Work due for this status is in progress.
Work due for this status has been completed.

Some user rights in an event status permission blacklist apply at the event level and others apply at the slot set level. For more information, see Event and Slot Set User Rights.

Order header states

Icon Description
Applies to the entire duration of the status.

Price action and promotion states

Price action and promotion status states are based on the results of validations run while the price action or promotion is in the selected status.

Icon Description
All validations succeeded or had the Information severity type
At least one validation with the Warning severity level failed.
At least one validation with the Error severity level failed.
