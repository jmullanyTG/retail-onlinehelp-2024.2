Adding a status to a workflow

  1. From the Maintenance tab, click Workflow.

    The Workflows list displays.

  2. In the Workflows list, select the workflow to be modified.

  3. Click New.

    The Workflow window displays.

    TIP: If an existing status box is selected in the workflow diagram before clicking New, a transition is automatically created from the selected status box to the new status box.

  4. Edit the status information.

  5. Click Save.

After a status is created, configure a permission blacklist and add a transition:

Status information

Field Description
Name The status name appears in the status box in the Design pane.
Completion Days Out The number of days before or after the workflow object's goal date that the status is due. See Trigger Method. (Optional)

A positive number indicates that the status is due before the goal date. A negative number indicates that the status is due after the goal date.

A workflow's goal date is the object's start date (promotion goal date = the sale/promotion start date; event goal date = the ad start date; price action goal date = the price action start date, etc.). If the status is due 30 days before the start date, enter the value 30. If the status is due 14 days after the start date, enter the value -14.

Prefix Method The method that must execute before this status is applied. (Optional)

A method is an advanced process applied at the database level (such as the execution of a SQL script). This list is empty, unless methods are configured to appear in it.
Post Fix Method The method that must execute after this status is applied. (Optional)
Trigger Method The method that must execute when the completion day specified in the Completion Days Out field is due. (Optional)

This list includes the method Auto Advance Node Status by Completion Days Out for Workflow Triggers, which automatically advances the workflow to its next status when the completion day is reached. If there is more than one possible next status (for example, Approve or Change Needed), the workflow advances to the status with the highest transition weight.

Completion Days Out are tracked using the Workflow - Heartbeat task, which is in the daily job executed by Job Manager. This task stops tracking days out in a workflow on the furthest negative day out specified in the workflow. If the heartbeat task is not executed on that specific day, the trigger method is never automatically executed. For more information, see Standardized jobs.For more information, refer to the Workload Queue Manager User Guide.
Status Permission Blacklist The state-based permission sets to enforce while the workflow is in this status. The list includes a row for each user right controlled by the workflow and a column for each possible object state. A black icon indicates that the user right is granted to all users when an object governed by the workflow is in that state. A blue icon indicates that the user right is denied to one or more users when an object governed by the workflow is in that state.
