Adding a transition to a workflow

To add a transition to a workflow:

  1. From the Maintenance tab, click Workflow.
  2. Select the workflow to be modified in the Workflows list.

  3. Click Draw to toggle draw mode in the Design pane.

  4. While in draw mode, click on the earlier status (orange border) and drag the cursor to the later status (green border).

    For example, click on the Approved status and then drag the cursor to the Published status. The transition arrow appears as the cursor is dragged from one connection point to another.

  5. After the transition is created, edit the transition properties in the Transition Information pane.

  6. Click Save.

Transition information

Field Description
Name The name of the transition. The default value is [Start Status Name] -> [End Status Name].
Start Status The name of the status where the transition starts.
End Status The name of the status where the transition ends.
Weight When a start status has a trigger method that automatically advances the workflow to the next status, the weight value determines which status is transitioned to in case there are more than one possible next status.

A Start status has two possible next statuses: Approved and Changes Needed. Validation criteria in the Start status determines which status is transitioned to, based on the weight values of the Approved and Changed statuses.
Weight values are required for order header workflows and optional for all others.
Transition Access The roles or users permitted to move the workflow from the start status to the end status. Click Edit to view or change this list.
