Roles

Roles may be thought of as a preset group of rights which may apply to more than one user. Selecting the Roles button in the Navigation Menu displays the current roles list. Role names and descriptions are listed here.

Accessing detailed information about Roles

Much of the detailed information related to the roles is found in the Role Information window which is accessed by either double clicking on an existing role or highlighting a role and clicking Create, Modify, or Clone Role.

The Role Information window

The Memberships tab displays included and excluded users. User membership can be maintained here or in the individual user accounts.

The Rights tab lists rights specifically included or excluded from roles. Any included rights are granted to all users listed in the memberships section.

There is no need to grant those users the same rights in their individual user rights sections if the necessary rights are assigned to the role.

Field descriptions

  • Role Name – The name of the role.

    It is considered best practice to keep the names generalized when creating new roles.

  • Description – A high level description of the role.

Roles tab within the users and rights sections

The Roles tab within the Users and Rights sections of the Users module allow administrators to add roles to user accounts.
