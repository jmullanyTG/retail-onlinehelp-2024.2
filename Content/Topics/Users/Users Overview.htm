Users Overview

The Users module is an administrative tool used to set up user rights, roles, and groups for a particular module and sub-sections of modules. Roles can be configured with required rights, and users belonging to that role will inherit those rights. Further specific right configuration per user can be applied from that point.

Users tab

Administrators manage individual user accounts within the Users tab of the navigation menu. Tasks such as account creation, information manipulation, account deletion, and assignment of rights and roles are all accomplished in the Users tab.

The main screen displays the current active users list and high level information that differentiates the individual user accounts.

Accessing detailed user account information

Most tasks which can be accomplished in the Users tab are completed using the User Information window which has detailed information related to user accounts. To access the User Information window:

  • double click on an existing user account or highlight a user account and click Edit in the navigation pane
  • create a new user by clicking Create in the navigation pane, or
  • highlight a specific user account and click Clone User in the navigation pane.

The Assign Roles and Assign Rights functions also launch the User Information window.

Column descriptions

  • User Name – The name for the user account. If it is a domain user, the name is preceded by the domain (domain\user).
  • User Description – A description of the user account. If it is a cloned user, the description reads as Cloned from _____ until it is changed (the blank is filled by the user account from which it was cloned).
  • User Access Group – Defines the access group to which the user belongs.
  • Active - (default is hidden) Whether the user is currently active.
  • Role Names - (default is hidden) A list of the roles to which the user belongs. This lets System Administrators ensure that role memberships are aligned with business responsibilities.
