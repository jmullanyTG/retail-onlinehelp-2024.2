Price Manager

There are two sets of permissions within Price Planning. One set not affected by a Price Action Status (i.e. global), the other affected by status. Below is a list of global permissions and purpose. Permissions can be configured using the Maintenance > Users menu action.

Right Description
Access to Price Planning View the Price Planning module, via Influence > Price and Markdown menu action, and all Pricing Plans and Actions configured.
Access to Price Planning Configuration View the Price Planning configuration screen: Rounding Rules, Variables, Calculations, Validations, Strategies and Status.
Allow Override of Validation Failures Mark the override checkbox, for a failed Price Validation (i.e. Information, Warning, Error), in the Price Validation Log. Overriding a Price Validation error replaces the Validation failure severity icon with a Reviewed icon to indicate that a failure exists but has already been 'approved'.
Create/Edit a Pricing Plan Create and edit start date, add or remove products and sites, etc. associated to a pricing plan.
Delete a Pricing Plan Delete pricing plans. Plans cannot be deleted if there are associated price actions.
Create a Pricing Action Create price actions which set pricing for site products.
Emergency Pricing Action Modify a pricing plan start date and price action start date to occur tomorrow. By default, the plan start date and associated price action start dates are restricted by the number of days, from today, configured in system policy Plan Start Date Number Of Days Out (Module:Price Manager).
