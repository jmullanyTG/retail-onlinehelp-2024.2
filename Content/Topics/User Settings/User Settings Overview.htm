User Settings

The User Settings tab does not contain modules as they are represented throughout other parts of the application. It does contain settings specific to the Shell or user interface layer (UIL). Any changes that must be made to a specific user interface is done through User Settings.

Default settings

The following settings are available by default:

Auto Login

If the Auto Login button is selected, logs a user in after launching the shell stub bypassing the need to enter a password at launch. It is not recommended to be used on a shared workplace environment. The Auto Login button is highlighted when selected.

Language

The Language settings button launches the Language Settings window where users choose a language from the available translations.

Change Password

Users can change their password after logging into the user interface. The Change User Password window populates the user name of the logged in user. Users enter their current password, then the new password, and finally confirm the new password.

Notification Preferences

The Notifications Preferences screen is used to customize how notifications are received about specific messages. Options are Hard, Soft, Silent, and Ignore. For more information, see Notifications overview. refer to the Notifications User Guide.

Skin

A dropdown list allows users to change the appearance of the user interface.

About

The About window displays the version information for the User Interface and pertinent connection information.

Advanced settings

The following settings are not installed by default. They must be added manually.

Auto Update

If selected, the Auto Update button automatically updates the module files within the user interface from the company server. Any available updates are downloaded immediately after the user logs in. When selected, the Auto Update button is highlighted.

Plugin Folders

The Plugin Folders Configuration window lets users specify the location of a particular folder which contains the desired version of the plugins.

Entries may be added or deleted using the navigation buttons at the bottom of the window.

To reload plugins, users must log out of ToolsGroup JustEnough and then log back in. If the plugins are not loading, make sure to check that there are valid plugin files contained within the file path.

Refresh Plugins

Selecting Refresh Plugins prompts a reload all of the plugin files from the specified plugin folders (or the default folder if none are specified). This causes the user interface to reset. Users are not logged out. Work which is open and not saved or committed must be saved before the user interface is reloaded.

Error Explorer

The Error Explorer window launches with a detailed explanation of any and all errors from the plugins.

A detailed report of the XML exception which accompanies the error may be viewed to facilitate proper debugging. The exact error is viewed by clicking the View Error button.

The Send Error Report button sends the full stack trace of the error to members of the Support team.
