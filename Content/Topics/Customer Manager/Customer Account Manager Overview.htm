Customer Manager

The Customer entity is managed within Customer Manager.

Sites can be set up as site type Account that represents a collection of customer accounts in a many customers to one Account site relationship. Customers are grouped 1 to 20 like other entities that can be used to show a hierarchy. Users maintain the Customer entity and do the linking to the Account site entity in Customer Manager.

Accounts are not directly replenished. All forecasted account demand is replenished at DCs where the stock is held until anticipated wholesale account purchases. The DC responsible for holding stock is defined in the Site Account Source Site Contribution widget in Site Manager.

Two new fields Account Name and Location Count located under the Customer Account group can be added to the top grid in the Customer Manager Document. Both fields are editable and have the Multiple Changes right-click menu option. The value in the Location Count field can be used within Assortment Planning.

The usual rights (seen in variant manager, variant customer manager, etc) need to be associated to the module so that it displays.
