Files Utility

The Files utility is a system operations utility and is the main import mechanism for dynamic link library (.dll) files and plugins which the application needs to function. It is an important utility which may be used to check file versions and update the application.

The Files utility is most commonly used by consultants, support engineers, developers, or customer IT and power users. The files listing is a data grid. Users can type in the top bar to filter the entries for ease of navigation.

The only screen within this utility lists the currently loaded files and their pertinent information. The files listed in the files utility are typically .dll and program database (.pdb) files. Other file types (.txt, .chm, etc.) may be included as well.

  • FileName – The full name of the file including the file extension.
  • FileDescription – An optional description of the file.
  • FilePath – Most file paths will represent the root folder of an appdata folder for ToolsGroup JustEnough. This is represented by \ (backslash). File action types will determine the actual root location for the file path.
    • \ToolsGroup JustEnough.UIL.SHELL\2022.1.0.0 – Any file action types of Copy File To Local User App Data Path exist in this root. This changes with the version number.
    • \ToolsGroup JustEnough.UIL.SHELL.STUB\2022.1.0.0 – Any file action types of Copy File to Executable Path exist in this root. This changes with the version number.

  • FileVersion – The version number for the specific file. All .dll files in the namespace follow the naming convention for ToolsGroup JustEnough releases. Other namespaced files follow their respective release version numbers as drawn from their file metadata properties.

    Files without version metadata are given a version number of 1.

  • FileCheckSum – This is an informational field displaying the check sum on the file version. This ensures that the correct file is the one that has been loaded.
  • FileActionType – There are three file action types. They are determined by the type of file and their respective location in the file structure. These are important as setting the wrong file action type may cause the application to crash.
    • Copy File To Local User App Data Path – This is the file action type for plugins. When adding a file, to determine if a file is a plugin, check the filepath as any plugins will exist in the Plugins folder. Files of this action type can be reloaded without closing and reopening the ToolsGroup JustEnough shell.
    • Copy File To Executable Path – This is the file action type for all other .dll files. All files which exist within the root “SHELL.STUB\version” folder will need to be of this action type. Any files which exist in both the plugins folder and the root folder (such as ToolsGroup JustEnough.Common.dll) will need to be of this file action type.
    • No Action – Marking a file as no action results in the deletion of the file. It is intended to be a mechanism to recall deprecated files.
  • FileDateTime – The date of upload to the application.
  • FileAuditName – The name of the file as it exists in the file audit path (original source). This is for audit purposes.
  • FileAuditSourcePath – The original source from which the file was uploaded. This will ensure that the file may be traced to a certain point of origin. This is for audit purposes.

Fully understanding the distinction between plugin files, which can be updated without fully exiting the shell, and executable path .dll files is the most critical element when dealing with the files located in the Files module. To maintain the integrity of files, do not move the files from one location (plugin folders) to the other.
