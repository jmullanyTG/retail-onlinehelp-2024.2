Associating suppliers with Attributes

An import associates suppliers with Attributes. The import fields are:

  • ConstructionCode

  • SupplierCode

  • Ranking

  • ActiveFlag

A construction code can be associated with multiple suppliers. For example, if the construction code is SHIRT then there can be multiple suppliers that can manufacture construction code SHIRT. While the construction code is very specific, the actual supplier constraint is an Attribute which can be any Attribute.

Ranking is the list of suppliers that sets the order in which suppliers are used before consuming the next supplier’s capacity. It is possible to have different rankings per construction type. For example, for shirts, supplier A is ranked 1 and supplier B is ranked 2. Whereas, for ties supplier B is ranked 1 and supplier B is ranked 2.

Each supplier can have multiple construction types and multiple construction groups associated with them. For example,

  • Supplier 1: (V-necks, T-shirts, and Tank Tops)

  • Supplier 2: (T-shirts, Dress-shirts, and Collar Shirts)

Construction type can be based on any finite capacity such as material, machinery, time required to complete manufacture of required amount of items, and so on. Said another way, construction type is something associated with the constraint that a supplier has in terms of the material, fabric or other factor related to the ability to actually run the production of the product and what limitations the supplier has.

System policy Supplier Constraint: Construction Type (Module: Order Manager) defines which Attribute to use for supplier constraint.

Should a product not have an Attribute associated with it then macro, Apply Supplier Production Limitation, available for Supplier orders in Order Manager will not be applied and supply will not be constrained. The macro requires a product’s Attribute to be associated with a construction type in order to constrain by that construction type. Without that association, there is no link to the particular macro and supply cannot be constrained.

The Attribute needs to be set prior to actually creating purchase orders.
