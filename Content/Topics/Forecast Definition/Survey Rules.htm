Method surveys

Method surveys are yes or no questions that are applied to a forecast to determine its viability. These method surveys are a very simple way to fail a forecasting method that would likely yield an impractical result. The result of a series rule will be true or false.

An example of a method survey would be to check if there are enough data points to fill in the specified number of periods for a moving average to be run. If users ask for three data points and there are only two, then the method survey will fail the method.

There are several surveys that may be added to a forecasting method. Within each survey, the survey stage (either Pre-Survey or Post-Survey) must be set. Also, each individual rule will have a set of default parameters and several additional parameters that may be included to further customize the survey, and by extension the forecast.

An optional one-off survey labeled 8 weeks is possible. This survey fails the method if there are eight weeks or less of data. The expression HistoryMax < 9 presents all that there is to this survey rule.

Trend survey

The Trend Survey returns a false result (thus failing the method) if the method is looking for a trend and the data does not show a big enough trend. Default included parameters for this survey are HistoryAvg and HistorySlope.

Seasonal trend survey

The Seasonal Trend Survey returns a true or false after determining if the data shows a big enough seasonal trend. It uses the default included parameter of HistorySeasonalPercent. If it shows more than a 20% change, it is determined to be big enough.

Seasonal survey

The Seasonal Survey rule only passes a seasonal forecasting method if there are sufficient data points to justify using it. It requires that HistoryNumPoints minus HistoryZerosBetweenSales minus HistoryTrailingZeros is greater than Seasonality.

Intermittent survey

The Intermittent Survey rule fails an intermittent method if there are not enough zero points. It’s measured by adding the HistoryZerosBetweenSales to the HistoryTrailingZeros and seeing if it is greater than 0.6 x Seasonality. This will easily fail most intermittent methods for non-intermittent data.

Linear regression survey

The Linear Regression Survey rule is used to fail the linear regression method. If the HistoryStd divided by the HistoryAvg is less than 0.5, then the method fails.

Too intermittent survey

The Too Intermittent Survey rule fails several methods if the data is too intermittent. This is measured using the parameters HistoryZerosBetweenSales, HistoryNumPoints, HistoryTrailingZeros, and HistoryLeadingZeros.

Trend points

Trend Points is another survey used to fail any method looking for a trend. Unlike Trend Survey, this survey rule measures the number of data points instead of the slope of the data. The default included parameters are HistoryNumPoints, HistoryLeadingZeros, and Seasonality.

Reasonable survey

The Reasonable Survey is a very important survey rule. This survey fails a method if the results are determined to be beyond the realm of realistic possibilities.

Reasonable Survey should almost always be included in the post-survey for all forecasting methods. If it is not included (or the parameters are changed in the expression) it should only be done if it is intended to see what the forecasting engine can do when it doesn’t need to return a reasonable result. The reasonable survey looks at the history to gather ForecastSeasonalPercent.
