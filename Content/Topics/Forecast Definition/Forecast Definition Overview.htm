Forecast Overview

The Forecast module (System Utilities) holds the rules by which the forecasting tool operates.

Users can create new forecast method sets, which can be selected in the Forecast Task pop-up window in the Inventory Forecast and Group Forecast modules. Selecting one of the method sets populates the methods and their parameters.

There are five major components which define a specific forecasting method:

  • the base
  • any processors
  • any surveys
  • series rules
  • step rules

Some or all of these can be manipulated to define a forecasting method.

The forecasting engine follows a structured order of operations which is essential to understand to properly manipulate the engine and return a practical result.

As manipulating the forecast definitions can cause a significant disruption if done improperly, it is highly recommended to restrict access to this module to consultants and end users with both a strong background in statistical forecasting and administrative level rights.

The forecasting engine runs a tournament to determine the best result and will display that result along with any other methods which did not fail.

Forecast Level Optimization is a tool designed to help users optimize the accuracy of their forecasts. This is valuable for demand planners who have highly intermittent, short life cycle sales patterns. The purpose of this tool is to generate and automatically test a variety of leveling schemes, and to select the one that produces the most accurate inventory-level forecasts. Forecasting such series manually can be very inaccurate and difficult.

To combat this problem, level optimization groups inventories into levels based on a hierarchy. These levels ideally have consistent sale behavior and therefore, can be forecasted accurately. Level forecasts are then split down to inventory.

There are certain requirements needed in order to successfully run the forecast level optimization. Users should work with their ToolsGroup representative to properly configure their database instances, and ensure the correct inputs are set up for accurate results.
