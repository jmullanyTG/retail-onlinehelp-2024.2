Applying ROS scenarios using the ROS module

Users apply the results of a Rate of Sale scenario in the Apply Rate of Sale section.  Users select a product filter to identify potential placeholders/variants to apply the results to. Users have the ability to apply the ROS results to multiple placeholders/variants and collections.

The application of ROS in this view is integrated with the Assortment Planning module, so once the user has applied the results, the demand plan for the items selected adjusts to reflect the ROS values that have been applied.

Filters and parameters

Which filters and parameters display depends on whether a placeholder, item master, or variant.

Applying a ROS scenario

In the Apply Rate of Sale section:

  1. In the Filter field, select a filter containing the placeholders, item masters, or variants that are eligible for ROS application.
  2. For placeholders, select:
    1. the time range during which the placeholders are in phase in the assortment plan set for the scenario.
    2. the level at which to apply the ROS: placeholder collection or placeholder.
  3. Click Execute.
  4. In the Scenario Results section, check the Selected checkbox of the ROS to apply.
  5. Iin the Apply Rate of Sale section, check the Selected checkbox to select rows which should be updated with the ROS value chosen in the Scenario Results view.
  6. Click Copy Rate of Sale in the Apply Rate of Sale section to populate the selected ROS value.
  7. Optionally, selected the checkbox in the Locked field to prevent the ROS values from being overridden.
  8. Click Save.

Before clicking Save, users can clear any copied ROS values by clicking on the Undo Icon in the top of the Apply Rate of Sale section.
