First stage

The calculation first removes data points that violate the parameters set in the scenario. For product 1 in week 2, the price per unit exceeds the value for Max Price. The sale quantity for week 2 is adjusted to 0.

Where, Include Inner Zeros = Include and Include Values Outside Phase In/Out = Include:

Sale Quantity Total Week Count Adjusted ROS
14 30 0.466666667

Second stage

The system then determines the time frame to use for each product. First, the system looks for defined phase dates.

Only product 3 has a phase in (week 1) and a phase out (week 3) defined.

Product 1 and product 2 have no defined phase dates. The first and last weeks with a sale are used for these two products.

An additional criterion is Days From Phase In. If set, the upper boundary or last week to be considered is determined by the value of Days From Phase In.

In this example scenario Days From Phase In is set to 42 which means the weeks taken into consideration is 6 weeks from the first week of sale (42 days / 7 weeks). Thus, for product 2 the weeks taken into consideration would be weeks 2 to 7 inclusive.

Thus, for the 3 products the algorithm uses the following time frames:

3 (Phase dates defined) 1 3
