Defining an export process for Promotion export

An export configuration is created by first defining an export process.

To define an export process:

  1. Click New in the Export Definition grid.

  2. Complete the columns as required.

  3. Click Save.

Once an export process has been configured, create an export configuration. For more information, see Creating an Export Configuration for Promotion export.

Export Definition field descriptions

Field Description
Export Type Select an export type from the dropdown list:

  • API – For this export type, the Module ID, Export Name and Method Called columns are required.

  • File – For this export type, the Module ID, Export Name, Base Output File Name, Output Path/Table, and Output Method columns are required.
Module ID Defaults to Promotions.
Export Name A descriptive label for the export process.
Method Called Select Export Promotions FileOutput from the dropdown list unless a custom method has been created. This column is required for File export types.
Base Output File Name The base file name for File export types. A time stamp is added to this name.
Output Path/Table The path that the export file is written to for File export types. Do not include a closing \.
Output Method Select either Dispatch Broker or xp_cmdShell from the dropdown list for File export types. The former is the preferred method.
Active (Optional) Select this column to indicate that the export is active.
Column Delimiter (Optional) Enter the single character column delimiter to be included in the output. Ensure that the column delimiter is a unique character that is not used elsewhere such as in field names or values.
Max Detail Record Batch Size (Optional) Enter the maximum number of detail records to output per batch for API export.
Output User Select an SQL Server login that has been configured for exporting from the dropdown list.
