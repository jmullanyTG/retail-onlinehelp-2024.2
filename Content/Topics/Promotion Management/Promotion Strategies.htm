Strategies

The Strategies screen is used to assign Calculation Sets, Validation Sets, and Pricing Details to a Promotion Strategy. Mandatory and required fields can be setup that are seen in the Promotion Pricing grid and Promotion Strategies set as active (i.e. available for use).

Fields and descriptions

Field Description
Name The Strategy’s name used to select a Promotion Strategy when creating a Promotion.
Type Select the Strategy Type from the dropdown list to determine how the Promotion Strategy displays.
  • Promoted Price - A promotional price.
  • Promoted Discount – An amount off or percent off Promotion.
  • Buy Get – A buy X get X or buy X get Y Promotion.
  • Spend Get – A spend X get points, dollars, or percent off.
  • Rebate – A regular retail or promotional price with a rebate.
Is Group Synced (Optional) Must be selected for all BOGO AA type Promotion Strategies.
Validation Set (Optional) Populated with Validation Sets defined in the Promotion Validation Configuration grid. Validations are performed after Calculations.
Calculation Set (Required) Populated with Calculation Sets defined in the Promotion Validation Configuration grid. Calculations are performed after methods.
Price Output Set Select a Promotion Pricing Detail from the dropdown list. Populated with Pricing Outputs defined in the Promotion Price Output Set Configuration grid.
Price View Collection The dropdown is populated by available pivot template to display pricing information. Select a display to use in the Promotion – Price Details tab.
Analysis View Collection (Optional) The available pivot template to display analysis information. Select a display to use in the Promotion – Analysis tab.
Edit Users By default, all Roles and Users have access to all Promotion Strategies. Clicking on this field opens the Remove User Rights dialog in which access to a Promotion Strategy can be removed from particular Roles and Users.

The list is filtered to only display Users that have User Right Allow Creation of Promotion.
