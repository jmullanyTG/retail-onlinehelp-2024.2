Planner tab

Promotion plans are created and managed using the Planner tab in the Promotion module. Promotion plans help determine what variants to promote based on the metrics provided within the filter meta data framework.

Promotions created in the Planner tab can be considered as pending until a plan is submitted. Until a plan is submitted, its promotions do not display in the List tab. Once the plan is complete and all the variants are assigned to the promotions in the plan, then the plan is submitted. At this point promotions are created and display in the List tab.

Once a promotion plan is submitted it cannot be edited or deleted and no additional variants can be assigned.

The following promotion types are supported:

  • Fixed
  • Amount Off
  • Multi Buy

Most fields in the variants grid are read-only. However, a promotion can be selected in the Promotion field and notes added in the Notes field.

The Site Exclusions tab displays all sites based on the Cluster Set assigned to the promotion plan. Site exclusions are by Promotion ID. Therefore, saving an exclusion on one variant excludes other variants that have the same promotion.

Additional data can be added to the Planner tab to assist in determining what items to promote. For more information, see Adding additional data to the Promotions module Planner tab.

Editing unsubmitted promotions and plans

If there is a need to change an input for a saved promotion that has not yet been submitted, clearing the name of the promotion unlocks the input fields allowing users to change the existing values and re-save.

Variants assigned to an unsubmitted promotion can be edited. The modified variant will either be added to an existing promotion with the same input values or added to a new promotion.

The fields in the Manage Plans window are read-only. To edit a promotion plan, click the Edit button to display the Edit Plan dialog.

Click Refresh to update the bottom pane. This displays when promotions have been created and have a Queue Status of Completed.

Promotion naming convention

Promotion names are generated using the following elements separated by underscores:

  • The Plan Promotion Name

  • The selected Strategy Name

  • The input values

If system policy Allow promotions with duplicate names (Module: Promotion) is set to True and a generated promotion name is a duplicate of another promotion in the plan that has already been submitted, a new promotion with the same name is generated. If set to False, a new promotion name with a numeric identifier appended is generated.

For example, a promotion name for any variants in Plan One, assigned a Fixed Price Strategy with a value of $10.00.

The generated name would be Plan One_Fixed Price_10. If submitted for another variant:

  • before the promotion has been submitted, an additional record is added to the existing promotion.

  • after the existing promotion has been submitted, a new promotion is created for the variants under the same name (if system policy Allow promotions with duplicate names is set to True) or using a numeric identifier added to the name, for example, Plan One_Fixed Price_10_1.

Editing promotions before they are submitted

  • Promotion names can be edited in the Manage Plans screen.

  • A promotion can be deleted prior to being submitted by selecting it and then clicking the Delete button in the Manage Plans screen.

  • Site exclusions can be set for a promotion by selecting a variant assigned to an unsubmitted promotion and checking the Exclude checkbox for the sites to be excluded. Exclusions apply to the whole promotion, regardless of the variant that is selected.

Creating promotion plans

The promotions use the start and end dates of the promotion plan.

  1. Click the Manage Plans button on the toolbar.

  2. Click the Add button on the toolbar.

    The Create Plan dialog displays.

  3. In the Promo Name Prefix field, enter a name that will be used as the base of the promotion name for any promotions generated for the plan.

    If this field is left blank, then the default used is the plan name. This field can be completed when creating the plan or any time after. If modified after promotions have been submitted, then the new name is applied only to new promotions.

  4. Complete the fields as required.

    The Cluster Set and Workflow must be common to all the items in the promotion. The Exclude Markdowns checkbox is an instructional flag used for exports.

  5. Click Ok.

    The Create Plan dialog closes and the new plan displays in the Plans grid.

    The next stage is to add promotions to the plan.

Adding promotions to promotion plans

  1. Click the Manage Plans button on the toolbar.

  2. Select a plan in the Plans grid.

  3. Click Ok.

    The Manage Plans window closes. The selected plan’s name displays in the toolbar.

    The grid displays the variants and High and Low Prices and Costs. The default price and cost types are defined using promotion system policies.

  4. Select one or more variants in the variants grid.

    A variant can only be assigned to one promotion within a plan.

  5. Optionally, select sites to be excluded from the promotion in the grid on the Site Exclusions tab.

  6. Select a strategy using the dropdown menu.

    The Strategy field is a dropdown with promoted price types and promoted discounts.

  7. Depending on the selected strategy, complete the strategy’s input fields.

    The strategies do not all use the same input fields. To display the assigned values for each of the variants, click Choose and select the relevant columns to display.

  8. Click the Save button on the toolbar.

    Within the Manage Plans window, selecting the plan displays the new promotion in the bottom grid.

Adding variants to an existing promotion within a plan

Different variant-site filters can be used to select variants to add to the same promotion. Variants that have already been added to a promotion but do not display in subsequent selected filters may not be visible but are still part of the promotion plan.

  1. Open a promotion plan.

  2. Select a variant-site filter using the Filter field.

  3. Click Execute.

  4. Select a variant.

  5. In the variant’s Promotion field, select an existing promotion using the dropdown menu.

    A selected promotion’s details can be edited in the promotion pane on the left. Update the values as required.

  6. Optionally, select sites to be excluded from the promotion in the grid on the Site Exclusions tab.
  7. Click the Show Plan checkbox when finished selecting variants.

  8. Click Execute.

    The filter is ignored and only the variants selected for the promotion display.

  9. Click Submit.

    Content and site exclusions can be further manipulated in the promotions module.
