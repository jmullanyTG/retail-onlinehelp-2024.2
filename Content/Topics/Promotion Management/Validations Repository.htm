Validations Repository

The Validation Repository section displays all the validations available to be included in Validation sets. A validation cannot be deleted from the repository while it is being used in an active set.

Creating validation rules

Within the Validations tab, click New in the Validation Repository grid toolbar to create a new validation rule. Complete the required fields and click Save.

Modifying validation rules

Modifications are made directly to the grid.

Deleting validation rules

  1. Select a validation in the Validation Repository grid and click Delete.

  2. Click Yes to confirm.

     The validation is removed from the grid.

Field descriptions

Field Description
Name A descriptive label to summarize the purpose of the validation. This identifies the validation when creating validation sets.
Description (Optional) A summary of the validation.
Type When the validation will be run in relation to the calculation. Select the Post option from the dropdown menu. Post should be selected in all cases and will run the validation after price and cost retrieval and Calculations have been run.

Pre and Both are not yet available. They are to be developed as future functionality.

Level Select a level from the populated dropdown. All levels act on the Site Cluster level.
  • Promotion Level – To be developed as future functionality. For now, selecting this option means that Calculations perform at the Promotion Group level.
  • Promotion Aggregate Level – Calculations perform at the Promotion Group level.
  • Group Level – Calculations perform at the Product Group level.
  • Product Level - Calculations perform at the Product level.
  • Item Level – Calculations perform at the Item Master Level.
  • Variant Level – Calculations perform at the Variant Level.

Product level definitions

  • Inventory – A product at a particular Site so it has a single price.
  • Item Master – A product across all Sites that may have different prices at different Sites. An Item Master may have a max price for example.

  • Variant – A variant of Item Masters based on a shared attribute i.e. size, color, style, etc.

    Men’s Polo Shirts.
Group Type From the dropdown options, select one to identify which Group Association Type this validation will be run on:
  • Buy Group - The validation only runs on the Buy Group.
  • Get Group - The validation only runs on the Get Group.

  • Buy Get Group - The validation runs on both the Buy and Get Groups.

    This option should also be selected for all promotions that are not of the Buy Get type.
Method (Optional) The available methods (i.e. advanced processes applied at the database level) configured for use. Select a method to apply for the validation. This selection supersedes any formula.
Formula (Required/Optional) A mathematical expression for the system to evaluate. Configured variables can be found in the right panel. This formula is used if the Method column is blank.
Severity Select a severity from the populated dropdown:
  • Information and Warning - the message displays but users can continue.
  • Error - the message displays and users are not allowed to continue. Corrections must be made.
Error Message Enter a message to display if the validation fails.
Is Verified Whether a price calculation or validation has been verified. IsVerified does not apply to Method-based Calculations or Validations.
