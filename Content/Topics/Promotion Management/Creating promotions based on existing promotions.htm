Creating promotions based on existing promotions

There are two ways to create a promotion based on an existing promotion. To create a new promotion based on an existing promotion, click either the Save As button or the Clone button located on the toolbar.

Save As versus Clone

Selecting Save As is almost the same as selecting Clone. Both functions default to the same price cluster set and other details.

The Save As dialog is used to make changes to the promotion header, dates, workflows, and so on. Also, the cluster set can be changed and a promotion generated with the same items but different clusters and site assignments. If the current cluster set is not changed, then the sites and clusters based on the latest snapshot for the cluster set are retrieved.

The Clone Promotion dialog can be used to make changes to the promotion header, dates, workflows, and so on. However, the Price Cluster Set field is read-only preventing the cluster set from being changed. The new promotion will have the same clusters, sites, and items as the original promotion.

Click Ok to save and load the new Promotion. This Promotion can be modified as needed.

Privacy options from the original promotion

If the Include Promotion checkbox is selected when cloning an Event with associated Promotions, then the privacy options are carried over from the original Promotions. Also, the last Promotion Owner is retained in the new Promotion being created by the cloned Event.

Field descriptions

Action Description
Promotion Name Modify the default name of the new Promotion.
Date Range Start and end dates of the new Promotion can be modified by clicking on the … button.
Description (Optional) Enter a summary of the new Promotion.
Priority (Optional) Select a Priority value from the configurable dropdown list.
Effect (Optional) Select an Effect value from the configurable dropdown list.
Tags (Optional) Click Tags to associate Tags with the Promotion. New Tags can be created with the appropriate rights.
Requirements (Optional) Click on … and select one or more Requirement in the window.
Price Cluster Set (read-only for Save As) Update the Promotion Site Cluster Set if needed. The version of the Site Cluster Set loaded to the Promotion is determined based on the Site Cluster Set start date and Promotion Start Date.
Workflow Update the Promotion Workflow from the dropdown list if needed.
Private, Public, Shared The default privacy option is based on the original promotion being cloned. The default privacy option for cloned promotions can be changed.
Created By (read-only) The name of the user that cloned the Promotion and the creation date and time.
Last Changed By (read-only) The name of the user that last modified the Promotion and the date and time when that change was made.
