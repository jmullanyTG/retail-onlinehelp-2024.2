Promotion manager system policies

System policies are configured to set up default settings. They are configured by Professional Services and shouldn’t be modified without assistance from ToolsGroup JustEnough Professional Services or Support. These policies are configured by clicking Policies from the Maintenance tab.

Module: PromoPlanner

System Policy Description
Days elapsed following a Promotion assignment to the Plan, after when the Plan becomes Delete-eligible Days elapsed following a promotion assignment to a plan, after which the plan becomes eligible for deletion.
Days elapsed following the Plan's end-date, after when the Plan becomes Delete-eligible Days elapsed following a plan's end-date, after which the plan becomes eligible for deletion.
Days of no Plan modification activity, after when the Plan becomes Delete-eligible Days of no plan modification activity, after which the plan becomes eligible for deletion.
Plan Exclude Markdowns Default Value Sets the default value for the Exclude Markdowns checkbox within the Manage Plans UI.
Promotion Planner View Collection Set A View Collection Set that can be configured to display product information pertinent to Promotion Planning.

Module: Promotions

System Policy Description
Allow Multiple Dates Lets users set a promotion to run multiple times across the calendar by setting more than one start and end date. Should be set to False.
Allow Promotions with duplicate names Promotions are generally uniquely identified by a combination of their names and dates. Allows multiple promotions with the same name and dates to exist. Should be set to False.
Auto select first group and load items when opening a promotion Auto select first group and load items when opening a promotion.
Clone Filters On Link to Promotion If a filter is linked to a promotion, a clone of the filter is created if set to True. Lets users make promotion level modifications to a filter without affecting the original filter.
Default Price Strategy used for new Promotions If a default promotion strategy is to be defined for new Promotions, enter the promotion Strategy ID. If the promotion Strategy should not be defaulted, set to 0.
Default PriceType (Default is Price Type 1 if left blank) Assign the default PriceType for Promotions by its ID.
Item Calendar View Collection Set Assign the View Collection Set to display the Item Calendar by its ID. If this is not set to a valid ID, a blank screen displays in the Item Calendar tab.
Limit permanent Purge Batch data deletion The number of days after the last date-time when a Promotion Purge Batch was touched, after which Purge Batch data and associated table-data can be permanently removed from the database. Defaults to 2 years.
Limits DEADLOCK retries The number of DEADLOCK Retries that can be attempted per DML statement - overrides the default system policy.
Limits Promotions to delete per run The number of Promotions that can be deleted in a single execution
New Promotion View Collection Set Set ViewCollectionSetID for creating a new promotion

Assign the View Collection Set to display as a second tab in the Create New Promotion wizard by its ID. If this is not set to a valid ID, the second tab does not display.
Promotion Calendar View Collection Set Set ViewCollectionSetID for displaying a promotion calendar.

Assign the View Collection Set to display the Promotion Calendar by its ID. If this is not set to a valid ID, a blank screen displays in the Promotion Calendar tab.
Promotion Inventory Meta Data View Assign the View Collection Set to display the Promotion Inventory in a promotion by its ID. If this is not set to a valid ID, the default data view displays.
Promotion Product Price and Cost Level The source level of Price and Cost values retrieved for products in a promotion.

  • Item\Site Pricing (Default) pulls Price and Cost from the Inventory Price and Inventory Cost tables.

  • Variant\Site Pricing uses Variant-Site Price and Variant-Site Cost tables. This setting is only supported via the Promotion Public API set.

Promotion Product Selection Level Select Item Level if the implementation will be Item Master based, or Variant Level if Variants are preferred.
Promotions Activity Log Retention Days Number of days to retain Promotions Activity Log data.
Recalculate Locked Promotion If set to True, even promotions that are marked as locked can be recalculated by an overnight process.

This could result in system pricing differing from advertised pricing if the promotion is in production.
Show Bids in Promotion View If set to True, Bids functionality is available for promotions.
Show Companion Items in Promotion View If set to True, Companion Items functionality is available for promotions.
Show View Collection Set for Promotion Calendar If set to True, show the View Collection Set defined in the Promotion Calendar View Collection Set policy, otherwise use the static Promotion Calendar.
Show View Collection Set for Item Calendar   If set to True, show the View Collection Set defined in the Item Calendar View Collection Set policy or the Variant Calendar View Collection Set policy depending on the Promotion Product Selection Level setting, otherwise use the default Item Calendar or Variant Calendar.
Track Promotional Pricing Changes When set to True, promotion pricing change records are created that can be reported to see the differences over time. The change records are retained until the promotion has been deleted and the scheduled Promotion Cleanup has been run.
Variant Calendar View Collection Set Assign the View Collection Set to display the Variant Calendar by its ID. If this is not set to a valid ID, a blank screen displays in the Variant Calendar tab.
