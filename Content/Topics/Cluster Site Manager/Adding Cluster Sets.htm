Adding cluster sets

  1. From the Management tab, click Site Clustering.

  2. Click Manage to display the Cluster Set Management window.

  3. Click New.

  4. Complete the fields as required and click Ok.

  5. Click Ok on the Cluster Set Management window.

  6. Click Calculate on the Cluster Set toolbar to calculates the cluster assignments for all stores included in the cluster set.

Cluster set field descriptions

Property Description
Name The name of the cluster set/template.
Description An optional description of the cluster set/template.
Group The group that the cluster set/template belongs to. Click the Change button to select a different group.
Security Indicates if this cluster set/template is available to all users (Public), some users (Shared), or just the logged-in user (Private).

If Shared was selected, click the Change button to select the users and roles to share with.
Modules The modules that this cluster set/template is available to.

Notes:

  • The cluster set/template is automatically filtered out of modules that are not selected in this property. This prevents having to sort through the clutter of cluster sets that are not intended for use.
  • Allocation and Assortment include support for their respective scenarios in the Rate of Sale module.
Cluster Type The cluster set/template type.

For the Wholesale Account Cluster cluster type, sites are selected the same way as with the other cluster types. Users do not select or create this cluster type within the UI as these are automatically created within the import process. However, users need to open new Wholesale Account Cluster sets and click Calculate to assign the correct Clusters. The result is Cluster Sets that have the correct Clusters assignments with a 1:1 Cluster to Site relationship. Snapshots continue as normal.

Editing a Cluster Set will show the Classifications created.
Cluster Partition Site Filter By Field When the Cluster Type property is a Site Item Partition or Site Partition, this property sets the site property used to partition cluster sets.

All site properties defined in Site Manager are available for selection, including basic store information, site attributes, hierarchy levels, site group, capacity, schedules and type.
Site Filter Type The method used to determine the total group of stores to be clustered:
  • Site Hierarchy Level
  • Site Filter
  • Intersection Site Hierarchy Level And Site Filter
Site Hierarchy The site hierarchy to use to define the stores to be clustered.
Site Level The level of the selected Site Hierarchy to define the stores to be clustered. This property is not visible when Site Filter is selected as the Site Filter Type.
Site Filter The site filter to use for defining the stores to be clustered.
Cluster Partition ItemMaster Hierarchy When the Cluster Type property is an Item Partition or Site Item Partition, this sets the product hierarchy to use for clustering data selection and cluster set partitioning.
Cluster Partition ItemMaster Level When the Cluster Type property is an Item Partition or Item Site Partition, this property defines the product hierarchy level of the selected Cluster Partition ItemMaster Hierarchy to be used in clustering data selection and cluster set partitioning.
Cluster Items The clusters to assign stores to when the cluster set/template is calculated.

Click Maintain to create, delete, and edit clusters.
Cluster Settings The priority ranking of each cluster selected in the Cluster Items property. Adjust ranking up and down using the arrows on the right side of the cluster list. The cluster with the lowest number has the highest ranking and contains the stores with the highest relative value.
Attribute Settings The priority ranking of each attribute used to calculate cluster sets. Adjust ranking up and down using the arrows on the right-hand side of the attribute list. The attribute with the lowest number has the highest ranking and will contain the stores with the highest relative value.

For more information about attributes, see Managing attributes.
Distance Function The distance function to use when calculating cluster sets.
  • Euclidean Distance calculates distance in a straight line (as the crow files).
  • Manhattan Distance calculates distance between two points as a sum of horizontal and vertical lines on a grid (no diagonals).
Allow Cluster Count to be Resized Indicates if all clusters in the cluster set must have sites assigned to them.

When selected, stores rank naturally according to the results of the selected Cluster Type algorithm. This may leave some clusters without any stores assigned to them.

If this option is not selected, at least one store is assigned to each cluster in the cluster set.
