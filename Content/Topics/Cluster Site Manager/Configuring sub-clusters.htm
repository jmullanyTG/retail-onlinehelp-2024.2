Configuring sub-clusters

After configuring the Cluster Set or editing an existing Cluster to add a Sub-Cluster, click Calculate on the Cluster Set toolbar to display the Sub-Cluster details in the grid.

The Sub-Cluster comes into effect when a snapshot is created. If an attempt is made to create a snapshot but Sites within the Sub-Cluster have no attribute setting, an error displays requesting classifications for each site.

Adding Sub-Clusters to Cluster Sets already in use in AP

It is possible to have regular Cluster Sets in the Plan as well, but these will appear as overlapping Cluster Sets. Users cannot assign a Placeholder to both the Cluster AND the specific Sub-Clusters.

However, there is a very important provision. USERS MUST NOT ADD A SUB-CLUSTER TO A CLUSTER SET ALREADY IN USE IN AP.

When the Cluster Set is recalculated with new Clusters (the Sub-Clusters being considered new clusters), and the new Clusters are snapshotted, then AP can no longer reference the old assortments to the non-attribute-based Clusters. The old assortments disappear from view, and there is no way of surfacing sales plans made to old Clusters. Data will not migrate from old Clusters to new.

To use Sub-Clusters after AP plans are already established, configure a new Cluster Set with the attribute-based Sub-Clusters and snapshot it. In the Cluster Details screen, unassort from the old cluster set and clusters and assort to the new cluster set and clusters.

Sales plans between old and new Cluster Sets will not be preserved. A custom script will need to be written to migrate and split sales plans from the old cluster set / clusters to new.

Assigning Sub-Clusters in AP

Within Assortment Planning, the Sub-Clusters are selected in the Manage Assortment Defaults screen by clicking in the Additional Allowed Cluster Sets field and selecting them. The Cluster Details side pane allows users to manage the assortment of placeholders and collections to Clusters and Sub-Clusters.

For information on using Sub-Clusters in AP, see Using Sub-clusters to adjust Phase Periods based on site attributes.

For information on using Sub-Clusters in AP, refer to the Assortment Planning User Guide.
