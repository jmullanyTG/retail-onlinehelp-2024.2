Managing attributes

Use the Cluster Set Management window to create, clone, edit, and delete attributes.

Adding an attribute

  1. From the Management tab, click Site Clustering.

  2. Click Manage to display the Cluster Set Management window.

  3. Select a cluster set and click Edit.
  4. In the Attribute Settings section, click Add.

    The Add Attribute window displays.

  5. In the Add Attribute window, enter a name and description for the attribute.

  6. Select the Attribute Type.

    The Sale Attribute, Forecast Attribute and Financial Plan Attribute types have additional properties that display when these attribute types are selected.

  7. Click Ok.

    The attribute is added.

Attribute Settings section toolbar

Action Description
Add Displays the Add Attribute window.
Edit Edits the selected attribute.
Delete Deletes the selected attribute.
Adjustment Factors Displays the Adjustment Factor window, where users can specify ratio modifiers for each attribute.

Attribute grid columns

Property Description
Attribute Name The name of the attribute.
Attribute Description The description of the attribute.
Attribute Type Options include:
  • Site Attribute: calculate cluster sets based on numeric site attributes, such as selling square footage or average temperature.
  • Sale Attribute: calculate cluster sets based on historical sales data.
  • Forecast Attribute: calculate cluster sets based on forecasted sales data.
  • Financial Plan Attribute: calculate cluster sets based on an MFP financial plan.
PropertyDescription
Sale Metric Type The sale metric to use to calculate cluster sets. Options are:
  • Unit: uses sales unit volume to cluster stores.
  • Revenue: uses sales revenue volume to cluster stores.
  • Cost: uses cost of sales (COGS = Sales units * average cost per unit) to cluster stores.
  • Gross Margin: uses gross margin revenue (Sales Revenue – COGS) to cluster stores.
  • Gross Margin Percentage: uses gross margin percentage ((Sales Revenue – COGS) / Sales Revenue) to cluster stores.
  • Gross Margin Return on Inventory Investment (GMROII): uses gross margin return on inventory investment (Margin Revenue / COGS).
Financial Plan The hybrid financial plans listed are those that are in status Initialized, In Progress, or Authorized.

The hybrid financial plans must have the exactly the same product hierarchy levels down to the selected clustering level. These plans can have lower levels of the product hierarchy as well but can’t be at a higher level than the required clustering.

Time FrameThe time frame such as Last 90 Days to use to calculate historical sales metrics. Options include all available time frames. For information on create or editing time frames, see Time Frame Maintenance window.refer to the Time Frame User Guide.
Use Item Filter Indicates if a product filter is used to select the product data to use in cluster calculation.

Item Filters supersede the selected product hierarchy level for each cluster set.

Item FilterThe item filter to use to calculate sales metrics.
Sales TypesThe type of sales to use in cluster calculation. Sales types are defined during software implementation.
Include ReturnsIndicates if returns (negative sales) are included in the sales data used in cluster set calculation.
Priority The priority ranking of each attribute. Adjust ranking up and down using the arrows on the right side of the attribute list. The attribute with the lowest number has the highest ranking and contains the stores with the highest relative value.
