Cluster Set Site Comparison report

The Cluster Set Site Comparison report compares multiple cluster sets against each other and displays how sites rank across them. This is most useful when comparing cluster sets that contain the same sites, classification structure, and clustering attributes.

For example, strengths and weaknesses of each site across a product spectrum can be identified by comparing cluster sets created by product category with the classification A, B, and C that were calculated based on the same historical sales time frame.

The conditional formatting in this report is determined by the Most Frequent field in the Cluster Set Site Classification report. Any cluster set classification rankings that deviate from the Most Frequent classifications are highlighted in orange.

The report is read-only.

For details on displaying cluster sets see Viewing cluster sets.

Display a cluster set

To generate the Cluster Site Comparison report

  1. Click Manage on the Cluster Set toolbar.

  2. Select a cluster set in the Cluster Set Management window.

  3. Click the Report button.

    A blank report displays as a separate tab.

  4. Select a cluster set from the Cluster Set List dropdown menu.

    The blue icons below the Cluster Set List field can be used to manage the selections.

  5. Click Execute.

    The Cluster Set Site Comparison Report and Cluster Set Site Classification Comparison Report display.

Toolbar functionality

Object Description
Cluster Set List The cluster sets to compare in the report.
Parameters Toggles displaying or hiding the Parameters area of the report.
Execute Generates the report and compares the cluster sets selected in the Cluster Set List.

Field descriptions

Column Description
Site Code The unique code for the site in the system.
Site Description The description for the site.
Classification/ Cluster Set Count The number of times the site was ranked in the corresponding classification across all cluster sets being compared.
Most Frequent The most frequently occurring classifications for the site based on the data in the Classification/ Cluster Set Count columns. If there is a tie for most frequent classification, all tying classifications are listed.
