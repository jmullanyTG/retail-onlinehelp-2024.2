Viewing a cluster set snapshot or template

The Cluster Set Management window displays a list of all the cluster set templates, and their related cluster sets, to which they have access. Select templates or sets to view, edit, delete, and create cluster set snapshots, as well as initiate the creation of new templates and sets.

To view a cluster snapshot

  1. From the Management tab, click Site Clustering.

    The Cluster Set Management window displays.

  2. To view a cluster set document for a given cluster set, select the cluster site in the grid.

    The Snapshot section displays below the Cluster Set section.

Cluster set toolbar

Action Description
New Displays the Add Cluster Set window to create a new cluster set or template.
Clone Displays the Update Cluster Set window with all parameters and criteria from the selected cluster set or template pre-populated and cloned (pre-pended to the beginning of the cluster set name).
Edit Displays the Update Cluster Set window for the selected cluster set or template.
Delete Validation exists to prevent the deleting of cluster sets that are currently in use by assortment plans.

Cluster sets are not truly deleted until the garbage collection procedure runs on the database. The frequency and timing of this procedure is determined during software implementation.
Allow Multiple Toggles displaying or hiding checkboxes in the grid. When enabled, select multiple cluster sets or cluster set partitions in the grid and then click Delete, Snapshot, or Calculate to perform the action on all of the selected cluster sets.

Notes:

  • Selecting a cluster set partition automatically selects all of its cluster sets.
  • Multi-cluster set operations can be submitted as tasks to the workload queue. A notification is generated when the operations are finished. For more information, see Workload Queue overview. refer to the Workload  Queue User Guide.
Snapshot Creates snapshots of selected cluster sets.
Calculate Calculates the cluster assignments for all stores included in a selected cluster set, based on the parameters and criteria specified during cluster set configuration.
Report Displays the Cluster Set Site Comparison Report.
View Collection Set Configuration Displays the View Collection Set Configuration window.
Refresh Refreshes the list of cluster sets displayed in the grid below.

Cluster set grid fields

Field Description
Name The name of the cluster set.
Description

There is a description for the cluster set. Click the icon to display the description.

There is no description for the cluster set.
Hierarchy The site hierarchy that the cluster set is based on. Blank if a site filter was used instead of a site hierarchy.
Level The specific level of the site hierarchy that the cluster set is based on. Blank if a site filter was used instead of a site hierarchy level.
Filter The site filter that the cluster set is based on. Blank if a site hierarchy was used instead of a site filter.
Last Modified The date that the cluster set was last changed.
Last Computed The date that the cluster set was last calculated.

Allocation

 Whether the cluster set is configured for use with the Allocation module and allocation scenarios in the Rate of Sale module.

Assortment

 Whether the cluster set is configured for use with the Assortment Planning module and assortment planning scenarios in the Rate of Sale module.

Events

 Whether the cluster set is configured for use with the Events module.

Promotions

 Whether the cluster set is configured for use with the Promotions module.

Price

 If the cluster set is configured for use with the Price Manager module.

For more information, see Cluster set Configuration screen

Snapshot section

Select a cluster set in the grid to open the Snapshot pane.

Action Description
New Creates a new snapshot of the selected cluster set.
Edit Edits the snapshot selected in the snapshot grid.
Delete Deletes the snapshot selected in the snapshot grid.
Report Launches the Cluster Set Snapshot Comparison Report.
Refresh Refreshes the data in the snapshot grid.

The snapshot grid displays the following read-only snapshot details.

Column Description
Cluster Set Name The name of the cluster set for which the snapshot was taken.
Cluster Set Description Click the icon to view any description.
Snapshot Date Time The date that the snapshot was taken.
Snapshot Note Click the icon to view any note.
Classification columns For each classification column, the count of stores in each classification at the time the snapshot was taken.
Effective Date Time The date that the cluster set snapshot became or will become effective.

Creating a new cluster set snapshot initiates recalculation of all assortment plans in which the snapshot cluster set is active. This recalculation occurs as a part of a regularly scheduled overnight processing. The frequency of overnight processing is determined during software implementation.
