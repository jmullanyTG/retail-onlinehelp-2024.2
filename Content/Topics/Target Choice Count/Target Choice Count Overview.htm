Target Choice Count Overview

Target Choice Count is a standalone version of the Target Creation functionality found within Assortment Planning (AP). The main differences between Target Choice Count and AP is that for Target Choice Count, the settings come from system policies and the Variant entity is being used. For AP, the settings come from the AP plan and it is the Variant Placeholder entity being used.

While this capability is included within the standard Assortment Planning solution, Target Choice Count can be licensed separately. This gives customers the ability to take advantage of intelligent recommendations when creating assortments even if they do not use the Assortment Planning module.

The main view is read-only. Changes are made by selecting the Edit button and using the Edit Target Scenario screen to make desired changes.

Traditional planning tools are typically rigid in the way users are allowed to plan: by product and site hierarchy only. This has led to many retailers attempting to gain detail by further extending their hierarchies, but this approach still does not offer true flexibility to adjust strategy as often as desired based on the changing needs of the business.

Target Choice Count offers the ability to segment hierarchy levels by a wide array of product attributes, both qualitative and quantitative, and then further subdivide and define plan targets and desired new choice counts all the way down to a site group, cluster, or site level.

Target Creation allows for:

  • creating scenarios in as simple or complex as manner as desired.
  • linking scenarios to existing merchandise financial plans, or basing plan targets on specified historical period.
  • defining business segments by product hierarchy or based on innumerable variables defined through product filters.
  • modeling new areas of business, adjusting percent contributions and planning targets at aggregate or more granular levels through the Pivot Grid functionality, and calculating recommended new choice counts based on targets through Rationalization (a marginal return curve calculation that suggests optimal assortment breadth and depth based on cleansed historical data).

Parameters

Use the following parameters to filter the displayed Targets:

Parameter Description
Product Hierarchy The product hierarchy levels a Target Creation scenario must contain to be shown in the Target Creation Scenarios section. Selectable levels are based on the intersection of the product hierarchy assigned to the current assortment plan and the product hierarchy permissions of the active user.
Site Hierarchy The site hierarchy levels a Target Creation scenario must contain to be shown in the Target Creation Scenarios section. Selectable levels are based on the intersection of the site hierarchy assigned to the current assortment plan and the site hierarchy permissions of the active user.
Window of Time The window of time a scenario must intersect with to be shown in the Target Creation Scenarios section. Selectable periods are based on those configured in the system down to the Period set for the current assortment plan.

For more information, see Creating floor set updates for varying time frames.
Display Currency The currency to display for all revenue based fields in the view. Defaults to the Home Currency for the active Assortment Plan.

Toolbar and grid functionality

  • New: Launches the Target Creation (New) window to create a new scenario.

  • Copy: Launches the Edit Target Scenario window.

  • Edit: Launches the Edit Target Creation window which displays the selected scenario’s details in the left pane. All fields are editable in scenarios of status Created, Calculated, or Rationalized.

  • Delete: Deletes the selected Target Scenario.

  • Compare: Launches the Target Comparison window used to compare scenarios.

    • Standard pivot grid functionality, including the ability to graph selected data elements and Export or Open analysis in a variety of popular formats, including Microsoft Excel and Adobe PDF.

    • Site count is a SUM across cluster sets and clusters, but an AVERAGE across segments and scenarios.

    • Only prerequisite for scenario comparison is that scenarios be in Calculated or Rationalized status.

  • Smart Start: If Target Choice Counts is used within AP, scenarios can be used to create a foundation for assortment plans through a process called Smart Start. For more information on Smart Start, see Smart Start . refer to the Assortment Planning User Guide.

  • Cluster Config: Launches the Target Configuration screen allowing users to determine which clusters should always have more choices than others. For more information, see Target Choice Count Cluster Configuration.
