Supply Chain Overview

Users can determine the supply chains that provide inventory to distribution centers (DCs) and stores.

ERP data integration

Supplier and site data from an Enterprise Resource Planning (ERP) system is used as the building blocks for the supply chains. To view this site information, see:

  • Supplier Manager Overview - displays information about external suppliers.
  • Site Manager Overview - displays information about stores and DCs.
  • Inventory Manager overview - displays the supplier for an item at a location.

Flexible supply chain designs

Supply chains can be simple or complex.

Supplier to DC/store supply chains

In these flat supply chains, stores get their inventory directly from external suppliers.

  • Displaying a direct-to-store delivery supply chain

Supplier to DC to store supply chains

This is a traditional supply chain design, with external suppliers feeding DCs that distribute inventory to local stores.

  • Setting Supplier-to-DC-to-Store Supply Chains

Multi-level supply chains

This design has multiple levels of distribution centers (primary and regional) supplying stores.

  • Displaying a Multi-Level Supply Chain

Redistribution supply chains

Stores or DCs can be grouped into redistribution pools using the Redistribution supply chain. These pools determine where inventory can be redistributed when there are inventory overages or shortages in sites. Customize lead times and source priorities within each pool as needed.

  • Assigning Sites to Pools of Stores (Redistribution Supply Chain)

Supply chain overrides

Supply chain overrides are used for situations when certain products at certain sites need to be supplied from a different source than their regular supply chain.

For example, during November and December suppliers might deliver Christmas related merchandise directly to stores instead of DCs. Overrides can be planned for specific time periods (such as holiday rushes) or set to run indefinitely.

  • Creating a Supply Chain Override

Customizable source priority

If a site experiences a potential shortfall in inventory, rank the preferred order of other sites it receives stock from. For example, prioritize geographically close sites, but protect flagship stores. To determine site priority, the following values are checked in order:

  1. Priority defined for the site in the supply chain
  2. Priority group (if present)
  3. Global Site Priority (for more information, see Prioritizing Stock Distribution.refer to the Site Manager User Guide.)
  4. Site ID
  5. Wait time days

Automated supply chain resolution

Create a task in Job Manager to re-process supply chains as part of a nightly or weekly job. This ensures that any additions or modifications made to supply chains get propagated throughout the system. For more information, refer to the Job Manager User Guide.
