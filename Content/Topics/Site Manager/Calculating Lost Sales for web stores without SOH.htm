Calculating lost sales for web stores without SOH

Lost sales for both physical and web-based stores that maintain their own stock on hand (SOH) is calculated using the store's SOH. This isn't possible for web-based stores which do not maintain their own SOH. In this case, the SOH of the primary source site is used to calculate lost sales.

Sites accumulate lost sales for a period if:

  • there is a proper supply chain

  • the primary source site has zero historical SOH

  • the site has a positive effective forecast

  • lost sale is greater than forecast sales.

To select a site that needs the SOH of the primary source site to be used, check the checkbox in the Calculate Lost Sales Using Source Site? field in the Site Manager Document grid and click Save.

This field does not display by default. To add this field, click Choose on the Site Manager Document toolbar and include the Calculate Lost Sales Using Source Site? option from the Site Lost Sales metadata group.

A filter can be created to display only those sites that are using the primary source site to calculate lost sales.

The default supply chain that is used is configured using a nightly task or by setting a default using system policy Supply Chain Default Active Supply Chain ID (Module: Supply Chain). A supply chain specific to web stores that do not maintain their own SOH can be set.
