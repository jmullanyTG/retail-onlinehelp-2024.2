Creating site models

An existing store or distribution center (DC) can be selected using the Site Modeling tab in the Site Manager Document to model forecasts after opening a new store or DC. This creates a store supersession, which uses the product level sales history from the model site to forecast demand for the new site. Orders for the new site can then be generated based on the modeled forecasts.

The Site Model Options section is used to create site model headers and then determine the effect of the models through the Site Model Details section. The Site Model Options section is also used to create site level supersessions by determining a start date, an end date, and an active status. Any supersession without a start date goes to the beginning of the history in the system. Any supersession without an end date continues in perpetuity.

After the product level hierarchy has been set for the system (through use of a system policy), the applicable product levels display as rows in the Site Model Details section.

For each line, select a model site to determine the set of historical sales to be copied and a factor. The factor determines the percentage of sales to be applied in the model. A factor of 1 copies 100% of the sales quantity. A factor of 0.5 copies 50% of the sales quantity.

To create a store supersession

  1. From the Management tab, click Site Manager.

    The Items window displays.

  2. Select sites using the dropdown menu in the Items dialog.

    The Site Manager Document screen displays.

  3. Optionally, click the Choose button to add additional columns.

  4. In the Site document, select a suitable store model for the new store.

  5. In the Site Modeling tab, click New in the Site Model Options toolbar.
  6. In the Site Model Options grid, enter the following details for the new site model row:
    • Supersession Start Date: Choose a date at least one year in the past. This is the date from which sales history is copied from the model site to the new site.
    • Supersession End Date: Choose a date two years from today. This is the date on which sales history stops being copied from the model site to the new site.
    • Click the Active checkbox to mark the supersession as active.
    • Click Save.
  7. Define which item sales histories from the existing store to copy through this supersession.

  8. Click Save.

Sales history from the model store is used to generate demand forecasts for items in the new store. Either generate these forecasts manually or wait for the forecasts to generate automatically as part of the scheduled overnight tasks.

Once generated, review the demand forecasts. For more information, see Reviewing new store forecasts.

Site Model Options section fields

Field Description
Superssesion Start Date The start date for the model. This is the first period of historical sales that is considered.
Superssesion Start Date The end date for the model. This is the final period of historical sales that is considered. If this date is beyond the date on which the new store has accumulated actualized sales, the model continues to be used to represent sales history.
Active A flag used to determine the active status of the supersession.
Site Opening Date (read-only) The date on which a site is set to open. To be used for new store openings.
Created Date Date of creation for the supersession.
Created By User who created the supersession.
Last Updated Date of the most recent update to the supersession.
Last Updated By The last user to update the supersession record.
Model Last Process The supersession model must be reprocessed periodically. This task must be done as part of a job that coincides with sales history updates, such as nightly jobs.

Site Model Details fields

Site model details help determine the product levels that will contribute to the creation of the model and their relative contributions.

Field Description
Product Level The level in the product hierarchy that can be modeled. The available hierarchy levels are controlled through a system policy setting.
Model Site The site used to create the model.
Model Site Opening Date The opening date for the modeled site. This is not a user-editable field.
Factor The ratio applied to the sales history of the modeled site. A value of 1 applies 100% of the sales to the model.
Active A flag used to determine the active status of the product level in the model.
Created Date Date of creation for the model line.
Created By Who created the model line.
Last Updated Date of the most recent update to the model line.
Last Updated By Who last updated the model line record.
