Adding entries to filter types

Once views are created (or data tables have been populated), entries must be made to tbl_FilterType. This will determine how the data contained within the view is organized and accessed within the user interface and by the filtering machinery.

Field Description
FilterTypeID Unique identifier that is referenced as the primary key by the tbl_Common_Filter_MetaViewData table.
FilterTypeDescription The grouping for the filter data.
FilterViewName Name of the view (or table) that was created and contains the reference data.
MetaDataViewName The reference table containing the metadata. Best practice is to set this value to tbl_Common_Filter_MetaViewData.
PrimaryKeyFieldName> The primary key referenced in the view.
DuplicateFlag Not used.
DefaultFlag If set to 1, the data from this view displays in the grid when of all document types for the filter entity are launched.
FilterEntityTypeID The ID of the filter entity type.
