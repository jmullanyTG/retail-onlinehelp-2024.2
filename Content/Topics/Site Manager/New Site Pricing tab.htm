Selecting new sites and copying pricing from similar sites

The New Site Pricing tab incorporated into Site Manager lets users select a new site and copy pricing from a similar site. Select the site that is to be priced, the model site to copy pricing from, and the price type to be copied. Pricing can be reviewed and then submitted to generate needed price actions in a designated plan.

The New Site Pricing tab is visible only if a user has been assigned User Right Allow Access to New Site Pricing. However, if system policy Show New Site Pricing is set to False (default), then the New Site Pricing tab is hidden regardless of whether the User Right was assigned or not.

Copying pricing from similar sites

  1. Within a Site Manager Document, select the new store in the top grid.
  2. Select the New Site Pricing blade.
  3. Complete the parameters as required.

  4. Click Execute.

    A read-only grid displays.

  5. Click Submit to create a Price Action.

    The Price Action is created for the specific new site’s Pricing Plan.

Parameters

Parameter Description
Model Site The site to copy pricing from.

The list of available sites is still filtered by the currency of the target site.
Start Date The date the copied price values are effective.
Price Type The price type value to be copied as the price. Current is the most current price of any Price Type.

The Price Types available for selection is determined by the following system policy Allowed New Site Pricing Price Types (Module: Price Planning).
Strategy A predefined set of calculations and validations used to provide pricing for new stores.
Copy From, Copy To The time period for which price values are copied.
Page Displays the number of pages of variants. Use the dropdown menu to select a particular page to look at.

The number of variants returned is determined by system policy New Site Pricing Displayed Variant Page Limit. The default for this system policy is 1000.
