Prioritizing stock distribution

Global site priority and priority group settings alter the way in which the replenishment and allocations ordering engines prioritize stock distribution. There are two separate modes based on whether any values are present in priority group fields for any site.

Using the Global Site Priority blade on the Site Manager Document screen, sites can be ranked based on priority as it relates to receiving stock in potential shortfall situations.

Site prioritization logic

In a constrained distribution center (DC) stock situation, with more than one store requesting stock:

  • if priority groups exist (meaning that the values are null, not set to 0) the stores are prioritized by their global site priority on a day-by-day basis when their stock requests are fulfilled.
  • if priority groups do not exist, stores are grouped by their respective priority group value, then ordered by the requested demand quantity on a day-by-day basis when their stock requests are fulfilled.
  • if all stores are in a single priority group, they are ordered by the requested demand quantity on a day-by-day basis when their stock requests are fulfilled. This can be used to create a demand prioritization version of the replenish engine.

To set global priority values

  1. From the Management tab, click Site Manager.

    The Items window displays.

  2. From the Filter field, select the filter that contains the sites manage.

  3. Click Ok.

    The Site Manager Document screen displays.

  4. Select a site in the grid.

    The Details pane displays.

  5. Click the Global Site Priority blade.

  6. In the Global and Group Priority grid, locate the site to be updated and change its Global Priority value.

  7. Click the Save button on the Global and Group Priority toolbar.

     

Field descriptions

Field Description
Site Code Site code for the store distribution center.
Site Type The site type (DC, Store, Account).
Site Name The name of the site.
Global Priority Unique, globally applied priority. Used to resolve shortfall situations.
Priority Group Associated priority group used to resolve shortfall situations. Protects a specific set of sites before referring to global priority to resolve shortfall situations. Takes precedence over global priority.
