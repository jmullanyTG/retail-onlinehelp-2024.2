Site Manager configuration

To add or remove views (blades) in the Site Manager Document screen, click the Configuration button located at the top on the right side of the screen. Use this button to set up a variety of views.

Adding or removing views

If a view is added or removed, the plugins must be refreshed or the application relaunched to see the changes. Each site detail blade is comprised of a view that can be set with individual user permissions to determine access.

To display or hide views:

  1. Within the Site Manager Document screen, click Configuration.

    The Views Maintenance screen displays.

  2. As required, check or uncheck views.
  3. Click Ok.

Site Modeling

The hierarchy used to populate the product dimension on the Site Modeling tab is determined by the Site Modeling: Product Hierarchy system policy.

Sales History

Site modeling creates entries in tbl_SalesHistoryByType as supersession sales history. To process the site modeling data, usp_Site_Model_ProcessModelData must be run. It is recommended to add this procedure as a task in the daily job, or as frequently as sales history is updated in ToolsGroup JustEnough.

Site Opening Date

The site opening date is read from tbl_Site_Status_Temporal. It is the minimum of start dates where site status is 1 (open).
