Breaking Changes

Inventory

The exceedingly old and replaced pair of tables, tbl_ArchiveInventory and tbl_ArchiveInventoryStatus, having been deprecated since JE 3.5.2.2, are explicitly excluded from any Purge activities. In addition, the database Foreign Key linking the former to the core tbl_Inventory table has been removed (DROPped) and will not be reinstated. Any custom code still referencing these tables ought to be eliminated entirely.

Any custom or TG JE support developed Inventory Purge procedures must be removed immediately and never reinstated. The new Inventory Purge supports purge of data from both TG JE core and any custom tables using InventoryID. Using any other purge mechanism is unsupported and is undertaken at the user’s own risk.
