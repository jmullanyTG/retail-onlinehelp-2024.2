Size Model

Reviewing size model exceptions

Typically, a ToolsGroup JustEnough module generates and regenerates size models regularly with the given parameters. Sometimes, the list of inventory provided to generate size models has no sales history or the filter becomes empty due to the passage of time. When this happens, very poor size models are generated due to the provided bad data. An enhancement has been made to add three new Exception messages to the Size Models view and one new Exception message to the Allocation Strategy view.

Size Model view

If a size curve’s item master filter is empty, the items in the filter have no history during recalculation, or all history has been filtered out due to sales model criteria, users are notified via a new column labeled Exception Status. If an Exception is present, the field is formatted red. Within the field, users click View to see the details of the issue.

There are three new bad data related Exception messages:

Exception message Description
No Valid Items in Size Model Filters There are no items matching the criteria specified in the item filter. Please ensure that the item filter is not empty or consider adjusting the filter criteria.
No Qualified Sales No historical sales were found within the specified time frame. Please ensure that the items specified in the filter have valid sales records between the dates set by the start date, end date, and Last X Days parameters.
No Qualified Sales Due to Thresholds No qualified sales that fit the sales model criteria. No sales history within the specified time frame and item filter fit the Percent Covered and Weeks Covered criteria.

Allocation Strategy view

Allocation strategies now detect if the Size Model used contains any Exceptions. The following new warning message displays in the existing Exception Status column:

Exception message Description
Bad Size Model Detected An exception has been detected in the used Size Model. Please navigate to the Size Models module to review the model or select an alternative size model. Recalculate the allocation strategy once the exception has been addressed.
