Assortment MPV Monetary Sales Values Are Not Recalculated When Making Periods Active Selling Periods External Bug 2024.1.1.0
Reported Behavior

In the main planning view, monetary sales values were not being correctly re-calculated when the Selling Period checkbox for a period is ticked having been previously unticked.

Steps to Verify

  1. Start with a fully populated future plan with selling periods ticked and both sales units and sales monetary values correctly populated.

  2. Untick Selling Period, and save, and sales values are cleared (units and monetary values) as expected.

  3. Re-tick Selling Period, and save, and sales units are re-calculated, however sales values are not re-calculated.

Target Choice Counts New Choice Count Rec incorrect when using Use Collection for Intra-Style Curve? External Bug 2023.4.3.0

2024.1.1.0
Reported Behavior

The new choice count recommendation when using Use Collection for Intra-Style Curve? was recommending choices that are 60% less than the choices recommended when using “style” for the intra-style curves despite the fact that the variance in the intra-style curve is minimal.

Steps to Verify

  1. Create a target creation scenario.

  2. Leave the checkbox for Use Collection for Intra-Style Curve? unchecked?.

  3. Calculate the scenario.

  4. Rationalize the scenario.

  5. Review the recommended choices.

  6. Go back and check the checkbox in order to use Use Collection for Intra-Style Curve?.

  7. Rationalize the scenario.

  8. Compare the recommended choices with the choices recommended when leaving the box unchecked.

Modified Behavior

The total new recommended choices will now be nearly the same with the variance in the scenario coming from the distribution of the recommended choices across the styles
