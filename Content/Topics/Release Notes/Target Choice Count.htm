Choices by Sequential Update

How users identify multiple updates (floor sets) within a Target Scenario window of time. The workflow has been revamped with the intention of providing a more user workflow-centric experience. This workflow allows users to define update periods and will generate choice recommendations beginning with the first update period. The user can then define the phase out date for choices recommended, including phase out dates with correspond to a future “update end” period. This allows the second update period to be aware of defined choices that are already accounted for within that update period.

Edit Target Scenarios view

Curves added to Segment Definitions on Edit Target Scenario view

The following existing metrics were added to the Segment Definitions section: Seasonality Curve, Lifecycle Curve, Markdown Curve, and Seasonality Curve Weight Factor. The curves can be automatically assigned from hierarchy defaults.

Hierarchy defaults, but not Cluster defaults, are automatically assigned if they have been set (for placeholders created both manually within the main planning view and through the Smart Start function).

 

Segment Floor Set Updates widget DMDEV-91523

The following fields were added to the Segment Floor Set Updates grid:

  • Default End Period

  • Default Markdown Period

  • Recommended New Choices

  • Active New Choices (To modify, only one segment can be selected.)

  • % New Choices

  • Carryover Choices

  • % Carryover Choices

  • Total Choices

  • Rationalized Choices

  • Comparison Choices

  • Pareto Choices

The same fields appear when the Target Choices By Update view view is launched (with some fields hidden by default) and the Edit Target Scenario view which launches when copying or editing Target Scenarios. All the configurable fields are visible and editable in the main planning view.

The markdown date is a user-enterable default markdown period for items in an update. The selection available in the dropdown depends on the configuration of End Date Definition for the scenario. If Phase Out Date is selected, it show periods between Phase In and Phase Out. If Last Regular Date is selected, it shows any period between Phase In and the scenario's last regular period.

Target Choices By Update view

The Target Choices By Update view, which displays by selecting toolbar option View Choices By Update on the Edit Target Scenario view, has been reconfigured as follows. Users can:

  • click between updates in the top view to display the to-date complete set of recommended choices in the bottom view

  • modify collection values and dates and view the resulting changes to the sales plan

  • change which locations each choice is assorted to

Phase date changes automatically trigger a re-calculation of the sales plan and Carryover numbers in the upper grid.

Sales Plan fields are frozen left and the cell color formatted if the item is in phase. Fields formatted in blue are editable.
