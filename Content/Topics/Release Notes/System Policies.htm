The following system policies have been added.

System Policy System Policy Group Description
Algorithm To Match Static Curve to Placeholder Module: Assortment This policy determines what mechanism is used to copy a placeholder from like style where the style has a static curve assigned. The two options are: False - this will keep the same existing static curve that is already copied and assigned to the like style, True - Re-copy the original dynamic curve as a new static curve.

When creating a new item via hindsight view the newly created item will now utilize the default seasonality curve assigned to the placeholder’s product hierarchy level or the default value of the pruduct/cluster
Assortment Engine Supplier-to-DC Lead Time Engine: Commitments Default Lead Time Days from the Supplier to the DC that is used to determine eligible receipt dates. This will be rounded to the nearest period. Default value is 7.
Attribute Display Module: Assortment Determines which fields to show in the attributes columns. 0 - Description, 1 - Code, 2 - Code and Description
Config: Code/Description Display Module: Target Choice Count This policy describes how to display properties that have a code and description option, like hierarchy levels or attributes.

The options are:

  • Code

  • Description

  • Code (Description) (e.g. COH001 (Auburn))

  • Description (Code) (e.g. Auburn (COH001))

The default is Code.
Enable VAT Conversion In AUT Calculation Module: Assortment Determines whether to use VAT rate conversion steps when deriving currency-specific AUT values from the given home AUT value.

The default is True.
Price Conversion Method Module: Assortment Determines which Price Conversion Method to use when deriving currency-specific AUT values from the given home AUT value. 1 - Will use the original Price Exception method only, 2 - Will use the Price Matrix method only, 3 - Will use neither method.
