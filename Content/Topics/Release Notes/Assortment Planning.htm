Assortment Planning

Using a price matrix with Smart Start within AP

TG JE Assortment Planning v2024.1 introduced the TG JE Price Matrix to better support planning in multiple currencies. TG JE v2024.2 extends this functionality into Assortment Planning Smart Start. (Smart Start is not enabled for the standalone Target Choice Count module.)

Please refer to the TG JE v2024.1 release notes for an extensive description of TG JE's ability to plan in multiple currencies via a Price Matrix.

With the introduction of TG JE Price Matrix, the AP Smart Start process has been updated as follows:

  • The AUT Discount % step has been changed to occur earlier in the process

  • derived values are not seeded

  • The Apply procedure runs instead of Reconcile to derive Blended AUT and AUR

  • there are no VAT conversions if Price Matrix functionality is enabled

Blended AUT and AUR are derived from a Home AUT. This means that the Smart Started AUR value is not guaranteed to match the planned scenario AUR.

Additionally, the Blended AUT metric has been added to the following:

  • Historical section on the Hindsight Management screen.

  • Image view (View Options > Quick Text drop-down menu).

Managing attribute-based sub-clusters across views

Existing sub-clustering functionality allows users, via Cluster Sets view, to grade and cluster (ABC) store cluster sets based on sales performance and then sub-cluster by a site-specific Lookup attribute type (such as Cold, Hot, Warm), including user defined attributes. The result is adjustments to phase periods based on site attributes. The sub-clustering functionality is now extended to the following:

  • Receipt Plan view

  • Sales Plan view

  • Top Down Core Plan view

An in-depth description on sub-clustering, including configuring AP to use attribute sub-clusters, is available by referring to the TG JE v2022.2 Release Notes.
