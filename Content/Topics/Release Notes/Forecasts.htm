Forecasts

Forecasting cost and retail ticket price

Planners can now use two new series, Forecast Cost and Forecast Retail, to forecast the cost and retail ticket price of selected inventory. The result is insight into the profitability of future sales originally priced in different currencies. The cost and retail of the inventory is calculated using the default cost and retail price of the inventory. Pricing displays in the planner's system or home currency. Currency conversion is automatic and is defined by existing system policy Currency ID (Engine: Replenish).

If system policy Currency ID is set to 1, then currency conversion does not take place.

The two series appear as color coded lines in the graph with the data being from the currently selected inventory in the Forecast view.

Currency values are greater than unit values; where 1 unit can correspond to a high currency value (e.g. 1 unit = $100) If both unit and currency forecast data series are selected, the unit values on the graph appear stacked across the bottom of the graph. This is because the y-axis is extended to display the currency values. A best practice is to select either units or currency forecast series when viewing values in the graph. This will apply the appropriate scale to the y-axis for proper viewing.

Series Name: Forecast Cost

Color: Blue

Calculation: Forecasted units * cost of the unit in the system currency
