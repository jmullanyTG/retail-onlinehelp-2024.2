Execute action messages immediately

The Execute Action Messages Immediately strategy allows the abrupt execution of Action Messages as soon as they are available for processing. This strategy can be considered the rapid fire or message spray approach and is the most resource intensive. In fact, if there are enough action messages using this execution policy to fill up the windows services’ available thread pool, it may starve other sub-systems of threaded resources reducing throughput.

This strategy has the ability, under the correct circumstances, to over subscribe the ToolsGroup JustEnough Systems (ToolsGroup JustEnough Engines and Web Service Methods) available resources. Hence, this approach should only be used where near real-time processing is a non-negotiable requirement.

To use this execution scheme, users need to compose an Action Message where the Action is the following constant string:

DISPATCH_SERVICE_SEND_EXECUTE_ACTION

Example Action Message

The following is a simple action message to determine if the service is currently running using the Execute Action Message Immediately strategy:

<ActionMessage> <Sender> <string>DATABASE</string> </Sender> <Recipients> <ArrayOfString xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"> <string>DISPATCH_SERVICE</string> </ArrayOfString> </Recipients> <Action> <string>DISPATCH_SERVICE_SEND_EXECUTE_ACTION</string> </Action> <ActionParameters.Count> <int>2</int> </ActionParameters.Count> <ActionParameters> <ActionParameters.Item> <ActionParameters.Item.Key> <string>DISPATCH_SERVICE_EXECUTE_ACTION_ACTION_MESSAGE</string> </ActionParameters.Item.Key> <ActionParameters.Item.ValueType> <string>ToolsGroup JustEnough.Common.Action.ActionMessage, ToolsGroup JustEnough.Common.Action</string> </ActionParameters.Item.ValueType> <ActionParameters.Item.Value> <ActionMessage> <Sender> <string>DATABASE</string> </Sender> <Recipients> <ArrayOfString xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" /> </Recipients> <Action> <string>COMMON_SEND_PING</string> </Action> <ActionParameters.Count> <int>0</int> </ActionParameters.Count> <ActionParameters /> </ActionMessage> </ActionParameters.Item.Value> </ActionParameters.Item> <ActionParameters.Item> <ActionParameters.Item.Key> <string>DISPATCH_SERVICE_EXECUTE_ACTION_ACTION_MESSAGE_CALL_IDENTIFICATION_TOKEN</string> </ActionParameters.Item.Key> <ActionParameters.Item.ValueType> <string>System.String, mscorlib</string> </ActionParameters.Item.ValueType> <ActionParameters.Item.Value> <string>33CAECFF-2419-4E78-A4BD-9612953EC2D0</string> </ActionParameters.Item.Value> </ActionParameters.Item> </ActionParameters> </ActionMessage>
