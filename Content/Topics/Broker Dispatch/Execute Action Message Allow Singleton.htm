Execute action message allow singleton execution on distinct action

The Execute Action Messages Allow Singleton Execution On Distinct Action is a strategy that only allows one singleton instance of an Action Message to be executed if Action Messages have identical Actions specified. This is the strictest of all the schemes in terms of granular control, ensuring that only the minimal amount of Action Messages are being processed.

If there is an identical Action for an Action Message currently executed and a new Action Message with identical Action arrives for processing, then the new Action Message is discarded and this discarding of the Action Message is reported.

Example scenario

Action Message A is not identical to Action Message B but does have the same Action.

Action Message C is not identical to A and B and does not have the same Action.

All 3 Action Messages arrive at the Dispatch Broker for processing. In addition, assume that the messages arrive in order A, B and C.

With the Execute Action Messages Allow Singleton Execution On Distinct Action strategy, A and C are processed immediately while B is discarded or thrown away.

To use this execution scheme, users need to compose an Action Message where the Action is the following constant string:

  • DISPATCH_SERVICE_SEND_EXECUTE_ACTION_SINGLETON_DISTINCT_ACTION

Example Action Message

The following is a simple Action Message to determine if the service is currently running using the Execute Action Messages Queue On Distinct Action strategy:

<ActionMessage> <Sender> <string>DATABASE</string> </Sender> <Recipients> <ArrayOfString xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"> <string>DISPATCH_SERVICE</string> </ArrayOfString> </Recipients> <Action> <string>DISPATCH_SERVICE_SEND_EXECUTE_ACTION_SINGLETON_DISTINCT_ACTION</string> </Action> <ActionParameters.Count> <int>2</int> </ActionParameters.Count> <ActionParameters> <ActionParameters.Item> <ActionParameters.Item.Key> <string>DISPATCH_SERVICE_EXECUTE_ACTION_ACTION_MESSAGE</string> </ActionParameters.Item.Key> <ActionParameters.Item.ValueType> <string>ToolsGroup JustEnough.Common.Action.ActionMessage, ToolsGroup JustEnough.Common.Action</string> </ActionParameters.Item.ValueType> <ActionParameters.Item.Value> <ActionMessage> <Sender> <string>DATABASE</string> </Sender> <Recipients> <ArrayOfString xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" /> </Recipients> <Action> <string>COMMON_SEND_PING</string> </Action> <ActionParameters.Count> <int>0</int> </ActionParameters.Count> <ActionParameters /> </ActionMessage> </ActionParameters.Item.Value> </ActionParameters.Item> <ActionParameters.Item> <ActionParameters.Item.Key> <string>DISPATCH_SERVICE_EXECUTE_ACTION_ACTION_MESSAGE_CALL_IDENTIFICATION_TOKEN</string> </ActionParameters.Item.Key> <ActionParameters.Item.ValueType> <string>System.String, mscorlib</string> </ActionParameters.Item.ValueType> <ActionParameters.Item.Value> <string>33CAECFF-2419-4E78-A4BD-9612953EC2D0</string> </ActionParameters.Item.Value> </ActionParameters.Item> </ActionParameters> </ActionMessage>
