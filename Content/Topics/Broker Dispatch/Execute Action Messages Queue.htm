Execute action messages queue on distinct action message

The Execute Action Messages Queue On Distinct Action Message is a strategy that queues Action Messages if they are identical in every way. While this level of granular control is stricter than the Execute Action Messages Immediately strategy it is still a scheme that optimizes for near real-time throughput at the cost of resource utilization.

The queue preserves execution order of when messages arrive for processing.

Example Scenario

Action Message A is identical to Action Message B.

Action Message C is not identical to A and B.

All 3 Action Messages arrive at the Dispatch Broker for processing. In addition, assume that the messages arrive in order A, B and C.

With the Execute Action Messages Queue On Distinct Action Message strategy, A and C are processed immediately while B is queued to be processed after A has completed.

To use this execution scheme, users need to compose an Action Message where the Action is one of the following constant strings:

  • DISPATCH_SERVICE_SEND_EXECUTE_ACTION_QUEUE

  • DISPATCH_SERVICE_SEND_EXECUTE_ACTION_QUEUE_DISTINCT_ACTION_MESSAGE

Example Action Message

The following is a simple action message to determine if the service is currently running using the Execute Action Messages Queue On Distinct Action Message strategy:
