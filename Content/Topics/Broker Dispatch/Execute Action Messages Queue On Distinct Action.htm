Execute action messages queue on distinct action

The Execute Action Messages Queue On Distinct Action is a strategy that queues Action Messages if they have identical Actions. This strategy optimizes resource utilization by queuing Action Messages that have the same Action defined.

The queue preserves the execution order for all action messages for each queue bucket.

Example Scenario

Action Message A is not identical to Action Message B but does have the same Action.

Action Message C is not identical to A and B and does not have the same Action.

All 3 Action Messages arrive at the Dispatch Broker for processing. In addition, assume that the messages arrive in order A, B and C.

With the Execute Action Messages Queue On Distinct Action strategy is that A and C are processed immediately while B is queued to be processed after A has completed.

To use this execution scheme, users need to compose an Action Message where the Action is the following constant string:

DISPATCH_SERVICE_SEND_EXECUTE_ACTION_QUEUE_DISTINCT_ACTION

Example Action Message
