Configuring cluster sets

  • Name – The name used to identify the cluster set.
  • Description – A description of the cluster set.
  • Group – A folder style file structure for organizing cluster sets. Cluster sets groups are maintained using the Change button.
  • Security – Allows specific users or roles to view or maintain the cluster set.
    • Public – All users with access to the cluster manager can view and maintain the cluster set.
    • Private – Only the user responsible for creating the set can view and maintain it.
    • Shared – The cluster set can be shared with selected users. This is done by including or excluding individual users or roles using the Change button.
  • Cluster Type – Determines the clustering method used to build the cluster sets.
    • K-Means – K-Means clustering groups the data points into logical clusters based on the nearest mean. The centroid is a point in the center of the cluster, regardless of where the data points are.

    • K-Medoid – K-Medoid clustering groups the data points into logical clusters while restricting the centroid to a data point. K-Medoid still uses the minimum mean distance to group the logical clusters.

      This is a more computationally intensive process than K-Means.

    • K-Means Collection – A template used to create similar K-Means sets in which a single parameter (partition filter by value) changes to create new K-Means cluster sets.
    • K-Medoid Collection – A template used to create similar K-Medoid sets in which a single parameter (partition filter by value) changes to create new K-Medoid cluster sets.
  • Entity Type – The entity type determines the granularity of data which will be clustered. Setting the entity type at a higher level excludes lower level data.
    • Inventory – An item and site combination.
    • Item Master – An item across all sites.
    • Site – A physical or virtual location capable of holding inventory records.
    • Customer – The party at the receiving end of the sales transactions.
  • Filter by Type – The available filter types contained in or parented by the entity type. Sets the level of filters which appear in the filter by filter field.
  • Filter by Filter – Allows users to restrict the cluster set data to a specific filter (most commonly an inventory filter). This can be thought of as a cross-section of the data with an intersecting filter.
  • Group By – The group that represents the actual data points displayed and grouped in the cluster. The available options in the group are based on the supplied entity type (e.g. site groups display if site is selected as the entity type).
  • Partition Type – Determines the mechanism by which cluster sets are segregated.
    • Filtering – Uses standard filtering.
    • Filtering Lite – Limits the filtering to only a single criterion, and only the equals clause (i.e. the filter must be for distribution center site).
  • Partition Filter by Type – Determines the type/level of filter used for partitioning.
  • Partition Filter by Filter (if using traditional filtering) – Determines which filter is used to segregate the cluster set. Selecting the Change button displays a filter list.
  • Partition Filter by Field (if using filtering lite) – Allows users to choose the specific field which must be equal to the partition filter by value supplied. This is restricted to the available fields specified in the partition filter by type.
  • Partition Filter by Value (if using filtering lite) – The value which must equal the data to be included in the cluster set.
  • Classification Items – Classification items determine the actual data to be classified by the clustering engine.
  • Classification Priority Order By – The way in which the classifications are ranked.
    • Ascending – Ranks each classification in ascending order using consecutive alphabetical letters to designate each cluster.
    • Descending – Ranks each classification in descending order using consecutive alphabetical letters to designate each cluster (suggested use).
  • Classification Settings – Classification settings list the selected classification items and allows users to alter their priority. It is important to remember the priority order setting when determining the order.
  • Attribute Items – Attribute items are included in the classification engine as inputs.
  • Attribute Priority Order By – The way in which the attributes are ranked.
    • Ascending – Ranks each classification in ascending order using consecutive alphabetical letters to designate each cluster.
    • Descending – Ranks each classification in descending order using consecutive alphabetical letters to designate each cluster (suggested use).
  • Attribute Settings – Allows users to edit the priority, determine the aggregation function, and assign a weight to the attribute if desired.
    • Attribute – The attribute name.
    • Entity Type – The associated entity type for the attribute (Site, Item Master, etc.)
    • Priority – The ranked priority determines how the clusters are built. This is affected by the attribute priority order by.
    • Order By – An override for the attribute order by which may be changed for a single attribute.
    • Aggregate Function – Determines the mathematical means by which the attribute values are aggregated.
      • Average
      • Count
      • Maximum
      • Minimum
      • Summation
      • Statistical Standard Deviation
      • Statistical Standard Deviation for the Population
      • Statistical Variance
      • Statistical Variance for the Population
    • Weight – A weight value for the attribute. Will be considered against all the other listed weight values.
  • Distance Function – How the mean distances are calculated.
    • Euclidean Distance – The distance as drawn on the Euclidean plane (straight lines).
    • Manhattan Distance – The distance as drawn on a plane while restricted to 90 degrees right and left turns (often referred to as the taxicab distance).
  • Allow Cluster Count to be Resized – Lets the clustering engine resize the overall number of clusters.
  • Compute Optimal Cluster Count – Lets the clustering engine compute and optimize the number of logical clusters.
  • Maximum Allowed Cluster Count – A practical cap on the number of clusters.
  • Number of Iterations – For use with K-Medoid clustering, this caps the maximum number of computational iterations with which the clustering engine has to process exact fits for the centroids. Infinite if left NULL.
