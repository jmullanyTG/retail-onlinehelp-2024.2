Creating new content versions

Content versions are created using the Content Versions tab of the Content Manager Configuration window.

  • To associate a new or modified content version with a promotion in an event that is currently in planning, first remove the promotion from the event slot and then add the promotion to the event again to reference the new content version. Any prior modifications to event specific content will be lost.
  • Promotions added to events after a new content version is created automatically inherit the new content version.

Creating new versions

  1. Select the Content Versions tab.
  2. Click New.

  3. In the New Content Version dialog box, enter values as required.

  4. Click Ok.

Editing content versions

  1. In the Content Versions tab, select the content version to modify.

  2. Click Edit to open the Edit Content Version dialog box.

  3. Modify values as desired.

  4. Click Ok.

Field descriptions

Field Description
Content Version Name A descriptive label to summarize the purpose of the Content Version. This identifies the Content Version for Promotion Content.
Default Content Version If this checkbox is selected, the new content version is the default content version. If selected, the new content version automatically includes all geographies when saved. When a promotion is added to a Slot, if there are any Geographies that are not associated with a specific Content Version, they are assigned the default Content Version.
Description Optional.
Clusters pane In the Selected column, select the site clusters to assign to the content version.
Content Field pane In the Display column, select the content fields to assign to the content version. It is possible to:
  • set the order in which selected content fields display by dragging them up or down the list.

  • control edit permissions for a content field by double-clicking in a selected row and using the Remove User Access window.

    Users and roles listed in the Keep Role\User access list can edit this field. Those in the Remove Role\User access list can see the field but not edit it (it is read-only).
