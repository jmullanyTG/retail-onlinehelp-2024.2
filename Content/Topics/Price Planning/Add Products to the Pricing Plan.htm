Managing the products included in a pricing plan

The Variants grid is used to manage the products included in a pricing plan.

The grid can be used to:

  • add variants to a plan at any time
  • remove variants from a plan at any time
  • apply newly added variants to existing active price actions or new price actions
  • automatically remove variants from existing active price actions when the variants are removed from the plan.

Adding variants to pricing plans

There are two ways to add additional products: using filter criteria or using the system clipboard.

Adding products from a filter

Click Add > from Filter to choose an existing filter or click the Maintain Filters icon to create a new filter. Once a filter is selected, click Ok to append any new products to the existing product listing and then click Save to commit changes.

If active pricing actions exist, a dialog displays to select which products to add to the price actions.

If products cannot be added, the pricing action may be in a status that does not allow modification with the user's assigned rights.

Adding products from the clipboard

Add products by selecting the right-click menu option Copy Column Cell(s) in the Product grid in one plan and then selecting Add > from Clipboard in a second plan.

All entries are validated and all failed products are logged for investigation. Successful entries are added to the pricing plan and the new products can be added to a price action.

Removing products from plans

There are two methods to remove products: manual selection or using the clipboard.

Manual selection

To remove products from a plan, select and mark the checkbox adjacent to the appropriate products in the Pricing Plan Variant grid and click Remove Selected.

Products cannot be removed from a plan if they are being used in a price action.

Removing from clipboard

In the Variants grid toolbar, select Remove > from Clipboard.

The copied list should have one product (i.e. Variant or Item Master, global policy) per line.

All entries are validated and all failed products are logged for investigation. For successful entries and upon confirmation, the products are removed and the price actions recalculated.
