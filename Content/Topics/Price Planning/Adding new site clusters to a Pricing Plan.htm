Adding new clusters to a pricing plan

Site clusters can be created and added to pricing plans. New clusters must include at least one site.

For new site clusters:

  • the currency is derived from the sites selected in the filter.
  • if multiple currencies exist, the site cluster name is reused by currency (for example, EMEA(GBP) and EMEA(EUR)).
  • only sites not already associated to the plan’s clusters are added to the new site cluster.

In the Sites grid on a pricing plan’s Information tab, sites are broken out by currency and country. Added sites are assigned to an existing cluster with the same country and currency. If no matching clusters exist, a new one is created. The defined country displays in the Country Name column.

To add a new cluster:

  1. Select New Cluster in the toolbar of the Sites grid.
  2. Enter a name that is unique to the pricing plan in the Site Cluster Name field.

  3. Click the dropdown arrow in the Site filter field to choose a set of sites.

  4. Optionally, to return a subset of sites from the Site filter, select the checkbox labeled Select only some Sites from the current filter.

    1. On the Site Selection dialog, select the checkboxes for the sites to be added.

      Only those sites meeting the filter criteria and not already associated to the pricing plan’s pre-configured site clusters display.

    2. Click Ok.
  5. Click Ok on the New Pricing Site Cluster dialog.

  6. Click Ok on the following confirmation dialog.

  7. If the Activate the New Site Cluster(s) to Existing Action(s) window displays, then select the price actions, not already exported, to include the new site cluster.

    The Activate the New Site Cluster(s) to Existing Action(s) window does not display if:

    • there are no future price actions

    • all future price actions are in a core status

      A core status is one that is owned by the system and is not user configurable. Core statuses do not display in Workflow. The following are core statuses:

      • Preparing Data for Export
      • Ready For Export
      • Exported
      • Export Confirmed
      • Exported With Errors

  8. If required, click Save on the Activate the New Site Cluster(s) to Existing Action(s) window.

    When saved, the site cluster and associated sites are added to price actions as included and the Pricing Strategy Variable default values are applied to the new site cluster and pushed down to variants\sites. Also, the following are marked as Calculation Required:

    • Site Cluster Variant Sites

    • Site Cluster Variants

    • Site Cluster

    • Price Action

    • Price Action's Plan

    If a price action has gone into a calculating state, the site cluster and associated sites are added as Excluded and display a dialog indicating that not all price actions were updated.

    For price actions that were not selected, the site cluster and associated sites are excluded. Also, any default values are not applied nor any associated plan, price action, product, or site objects marked as Calculation Required.
