Price validations

Corporate pricing validations are configured via formula expressions along with validation error messages using the Price Validations screen. The grid on the right is used to copy and paste formula values to the Formula field to reduce manual entry error. The grid also provides insight into which variables are being used in the formula.

Creating and maintaining validations

Within the Price Validations screen, click New to create a new Calculation or Calculation Rule, enter the required fields and click Save. Modifications are made directly to the grid.

Validations cannot be deleted if associated to a Price Strategy and once deleted cannot be restored.

Validation fields and descriptions

Field Description
Name A descriptive label to summarize the purpose of the Calculation or Rule.
Description An optional summary of the Calculation or Rule.
Method (Optional) A list of preconfigured database stored procedures for more complex mathematical formulas or validations. Selection in the Method field overrides any entry in the Formula field.
Formula (Required/Optional) The mathematical expression for the system to evaluate. Configured variables can be found in the right panel. Validation examples include:

If Cost1 is NULL, $0.00 or $0.01

  • ISNULL([Variable1],Replacement Value)<.02
  • Variable1 = Cost1
  • Variable1 is being wrapped by the ISNULL function to replace any NULL values, in Cost1, with a value of zero
  • ISNULL([Cost1],0)<.02

If [Cost1] is less than .02, the response is the configured severity level in the Pricing Strategy which can also be viewed in the Validation Log. The Price Action does not continue validation and calculation if the severity level is configured as Error.

If [Cost1] is greater than .02, the validation is marked as passed and continues calculation and evaluation.

If Retail Sales Price is lower than MSRP

  • Variable1 < Variable4
  • Variable1 = [RSP]
  • Variable4 = [MSRP]
  • [RSP]>[MSRP]

If [RSP] is greater than [MSRP], the response is the configured severity level in the Pricing Strategy which can also be viewed in the Validation Log. The Price Action does not continue validation and calculation if the severity level is configured as Error.

If [RSP] is less than [MSRP], the validation is marked as passed and continues calculation and evaluation.

System Error Message A message to be included in the validation log for validation failures. Error messages cannot be configured for calculation types.
Active (Optional) Unchecking the checkbox removes the ability to configure the calculation in any new price strategies.
Validated If checked, a validation is run against the formula. Only calculations that have been validated are executed during the calculation process.
