Price reversion types

System Administrators can use the Reversion Types tab to configure Reversion Type profiles. These profiles are then used to create Price Actions that revert prices to a previous state. Selecting the type during the Price Reversion process determines the Price Strategy, Workflow, and the Price Type used to populate the System Input variable in the Reversion Price Action.

The Price Types are limited by system policy Price Reversion Allowed Price Types.

Creating Reversion Types

Within the Reversion Types tab, users can:

  • click New to create a Reversion Type directly in the grid
  • select Reversion Types and delete them.

Reversion Types tab: fields and descriptions

The following fields are all required.

Field Description
Reversion Name A unique name for the Reversion Price Action.
Reversion Description A description of the Reversion Price Action.
Source Price Type Price Type from which the previous value will be derived.
Price Method Price Strategy to be used for the reversion.
Workflow The Workflow to be used to create the Reversion Price Action.
