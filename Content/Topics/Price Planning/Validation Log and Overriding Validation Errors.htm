Validation icons, the validation log, and overriding validation errors

A message displays indicating if a Price Action has successfully complied with Validation rules. Validation Severity failure icons on the Price Actions grid provide convenient indicators of whether a Price Action is validated or not while.

To display the Validation Log, double-click the Validation Severity failure icon in the Validation field on the Price Actions, Variant Summary, Items, or Item Location grids.

Overriding a validation failure

Overriding Validation failures reduces the amount of time spent reviewing failures. To override a Validation failure, select the Override checkbox and click Ok.

 The override of a Validation failure causes an update to any higher level affected by the Price Action’s worst case Validation Severity up to the Site Cluster, Price Action and Plan levels. These levels still display worst case aggregation when an override is made. If there is another worst case (error) the higher levels do not update. If the override is at the:

  • Inventory level, then there are updates to the Inventory Manager, Variant, Site Cluster, Plan, and Action Header level, if appropriate.
  • Item Master level, then there are updates to the Variant, Site Cluster, Plan, and Action Header level, if appropriate.
  • Variant level, then there are updates to the Site Cluster, Plan, and Action Header level, if appropriate.

Order of validation severity levels

The order of the Validation Severity levels (from best to worst) is as follows:

  • Validated

  • Reviewed

    • Information Overridden
    • Warning Overridden
    • Error Overridden
  • Information Provided
  • Warning Found
  • Error Encountered

The recalculation and revalidation of a price action

When recalculating and revalidating a price action, the validation process:

  • keeps the current severity state when the failure matches the previous failure (e.g. an Error remains an Error).
  • updates the current severity state if the failure does not match the previous failure (e.g. an Error becomes a Warning).
  • discards the current severity state if the validation process no longer fails (the price action is now validated).

The Validation Log provides messages based on failed validations configured in the Price Strategy screen. Depending on where the Validation Log displays, the results are filtered as appropriate. The Validation Log can viewed from any Price Actions grid (Price Action Header, Summary, Item and Item Location). Double-click on the icon in the Validation column in the Price Actions grid.

A failed validation that has been reviewed is now considered no longer failed and is considered for export. Also, not all Validations can be overridden. The ability to override is based on system configuration which in turn is based on a review of the customer’s business process.

Validation log

The Validation Log has advanced filtering that lets users review and mark as reviewed validation failure records allowing Promotions to continue to downstream systems. The Validation Log does not limit the number of records to be returned.

Product Validation failures are filtered based on where the Validation log originated:

Level Filter
Price Action All Site Cluster Validation failures, at the Inventory level, for ALL Site Clusters
Site Cluster All Variant's Item Master's Inventory Validation failures for the specific Site Cluster
Variant All associated Item Masters' Inventory Validation failures for the specific Site Cluster
Item Master All associated Inventory Validation failures for the specific Site Cluster
Inventory Validation failures for the specific Site Cluster

Advanced Validation Log

Reduce the number of records that need evaluating by evaluating Pricing Validation failures at a Variant and Site level. To enable this functionality, system policy Advanced Validation Log needs to be set to True.

If a plan’s Product Pricing Level is set to Variant Site Plan, then system policy Advanced Validation Log is ignored and the Advanced Validation log displays regardless.

If a plan’s Product Pricing Level is set to Item Site Plan, then system policy Advanced Validation Log is evaluated and either the default or advanced validation log displays.

Exporting price actions with a validation severity type of error

Price actions with a failed validation severity type of Error cannot be exported to downstream systems (such as reporting, ERP, POS) until all such validation failures have been overridden.
