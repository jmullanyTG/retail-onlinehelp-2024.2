Price variables

The Price Variables tab is used to define parameters (system derived as well as user configurable) which are used in calculations and can be configured for export.

Giving pricing variables different numeric precisions to be inline with local currencies

System policy Variable Numeric Precision addresses the need at the time of implementation to configure pricing variables to have different numeric precision for different geographic locations to be inline with local currencies. This system policy is hidden and is only set once at the time of implementation. This system policy is not changed after initial implementation.

Variable Numeric Precision sets the database field precision to ensure efficient updates and selects and maintenance of database growth for configured pricing variables and their assignment to pricing strategies. Further refinement of each pricing variable's formatting is implemented via the Price Manager Configuration screen.

Creating and maintaining variables

Users can:

  • click New to create a new variable, enter the required fields, and click Save.
  • modify variables directly in the grid.

  • select variables and delete them.

    Variables cannot be deleted if associated to a Calculation.

Variables fields and descriptions

Field Description
Variable Name A descriptive label to summarize the purpose of the variable.
Variable Description An optional summary describing the purpose of the variable.
Format String (Required) A designation that determines how the system formats manual entry:

  • Currency (C2) – data is formatted using the configured currency with 2 decimal digits

    • Database Value: 5.990000
    • Cost (C2): $5.99

  • Decimal (D6) – data is formatted as a whole number using a minimum of 6 digits

    • Database Value: 5.990000
    • Cost (D6): 5.990000

  • Number (N2) – data is formatted as a numeric field with 2 decimal digits

    • Database Value: 5.990000
    • Cost (N2): 5.99

  • Percent (P2) – data is multiplied by 100 with 1 decimal digit

    • Database Value: 0.5990000
    • Percent Off: 59.90%

Detailed information can be found on Microsoft’s MSDN website:

https://msdn.microsoft.com/en-us/library/dwhawy9k(v=vs.110).aspx\
Output Field Name A read-only label created when the record is saved. This label is used when designing calculation formulas as well as the export staging tables.
Input Type The option selected determines the available options in Input Price / Cost Field.
  • Cost, Price – imported read-only fields for reference for calculation formulas and validations.

Cost and Price are configured during implementation and determined by actual fields configured to be imported.

  • User – A manually entered or calculated field
Input Price / Cost Field Which Input Type is selected determines which options are available in the Input Price / Cost Field. Select an option to be associated to the variable.
Allow Output Override (Optional) The checkbox is only editable when Input Type is User. If checked, users can enter data to be associated to the variable.
>Active (Optional) Unchecking the checkbox removes the ability to configure the variable in any new calculations. When a price action is calculated the price strategy calculations and validations are affected.
Active (Optional) Determines if the rounding criteria is used when a rounding rule is configured with a calculation.
