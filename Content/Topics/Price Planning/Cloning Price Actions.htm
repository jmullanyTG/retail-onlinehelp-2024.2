Cloning Price Actions

  1. Click Clone to display the Clone Price Action dialog.

  2. Update editable fields as required.

    The system appends (Copy) to the original price action name. Overwrite with a new name as required.

  3. Click Ok to create the new price action.

    The Price Actions grid automatically refreshes with the newly cloned price action.

Clicking Cancel discards the creation of the new price action and closes the Clone Price Action dialog.

Field descriptions

All fields are editable.

Field Description
Name The original price action name.
Description The original price action description.
Start Date The original price action’s start date. If the start date is in the past, then if the user:
  • has the Right Emergency Price Action the start date defaults to today.
  • does not have the Right Emergency Price Action the start date defaults to tomorrow.
Strategy The original price action strategy.

  • If not modified, the Strategy Variable values are copied and any lower level exception overrides.

  • If modified, the system will not copy over the Strategy Variable values nor any lower level exception overrides.
Workflow The original price action workflow. The system applies the first status for the selected workflow.
Variant The system applies the selected Variants from the original price action. Additionally, the system applies any product exclusions across all levels.
Site Clusters The system displays the selected Site Clusters from the original price action. If a Cluster is added, that was previously not included, then the system applies the default Strategy values for the appropriate currency.
Price Matrix Optional. If the Plan is configured to use a Price Matrix, then this field allows the Price Matrix to be changed or removed from a Price Action.

This field displays if system policy Default Pricing Plan Price Matrix Usage is set to True.
