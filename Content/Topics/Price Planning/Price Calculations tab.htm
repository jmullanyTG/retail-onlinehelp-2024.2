Price calculations

The Price Calculations view provides the ability to configure calculations via formula expressions as well as the rounding set and destination fields. The grid on the right can be used to copy and paste values to the Formula field to reduce manual entry error. The variable grid also provides insight into which variables are being used in the formula.

Creating and maintaining calculations

Within the Price Calculations screen, click New to create a new Calculation or Calculation Rule, enter the required fields and click Save. Modifications are made directly to the Calculation grid.

Calculations cannot be deleted if associated to a Price Strategy and once deleted cannot be restored.

Calculation fields and descriptions

Field Description
Calculation A descriptive label summarizing the purpose of the Calculation or Rule.
Description An optional summary of the Calculation or Rule.
Method (Optional) A list of pre-configured database stored procedures for more complex mathematical formulas or validations.

A selection in the Method field overrides any entry in the Formula field.
Formula (Required/Optional) The mathematical expression to be evaluated. Configured variables are found in the right panel.

Example

Retail Sales Price is % Above Cost1
Variable1 = (1+Variable2)* Variable3
Variable1= [RSP]
Variable2 = [PercentAbove]
Variable3 = [Cost1]
[RSP] = (1+[PercentAbove])*[Cost1]
$5.988 = (1+.20)*$4.99

[RSP] populates to the Targeted Output Field. Validation and calculation of the Price Action Strategy continues.
Targeted Output Field (Required) A list of configured variables (i.e. user input only) that the configured mathematical formula populates. Output fields cannot be specified for ‘validation’ types.
Rounding Set(Optional) The Rounding Rules to be applied to the calculation. Rounding Rules cannot be applied to ‘validation’ types.
Active (Optional) Unchecking the checkbox removes the ability to configure the Calculation in any new Price Strategies.
Active If checked, the Calculation process executes that Calculation or Validation. Users can deactivate or uncheck for testing and so on. If unchecked, the Calculation process ignores the Calculation or Validation.
Validated If checked, a Validation is run against the Formula. Only Calculations that have been validated are executed during the Calculation process.
Use Matrix Allows for a Method based Calculation to designate that its method is a Matrix Calculation. Then that method can be configured using the Price Strategy Matrix Config tab in the Price Matrix configuration view. For a method designated as a matrix calculation, the price action calculation checks the configuration for that method.
