Price matrixes

For Pricing Plans and Price Actions that are assigned Price Matrixes, set the decimal precision that values are rounded to and default Price Matrixes using the Price Matrix tab in the Price Manager Configuration views.

Price point matrix rounding

Use the Price Point Matrix Rounding tab to set, based on channel, country and currency, the decimal precision that a value is rounded to before it is sent into the Price Matrix.

Default price matrix configuration

Set the default price matrix being used in the Default Price Matrix Config tab. Based on the selected channel, feature (the source operation that is creating the Price Action) and Price Method, the Price Action is assigned the Price Matrix as a default.

The default can be overridden after the Price Action is created.

Selections available in the Feature field include:

Feature Description
Auto Price Auto Generated Price Action for Automated Pricing
Price Reversion Auto Generated Price Action for Price Reversion

Price matrix strategy configuration

The Price Matrix Strategy Configuration tab supports the configuration of Price Strategies and the needed properties for resolving computed prices through the Price Matrix.

Field Description
Strategy A dropdown list with the available Price Strategies.
Method The method name assigned to the selected Price Strategy.
Current Price A dropdown list of variables. Selected variables are used as the source of the Current Price for the Price Matrix calculation.
Base Price A dropdown list of variables. Selected variables are used as the source of the Base Price for the Price Matrix calculation.
Validate Full Price If checked, the Price Matrix calculation for the strategy evaluates the Allow Full Price field.
