Configuring auto pricing to automatically create price actions

  • the following two user rights allow users to view and edit Plan Auto Pricing Configuration parameters:

    User Right Description
    Allow Access to Plan Auto Price Configuration Allow Access to Plan Auto Price Configuration.
    Allow Access to Plan Price Reversion Allows users to view and edit.

  • the following two system policies determine the available Base and Target price types:

    System Policy Description
    Auto Pricing Base Price Type The Price Type ID to be used as the Base Price in Auto Pricing computations.
    Auto Pricing Target Price Type The Price Type ID to be used as the Target Price in Auto Pricing computations.

  • the following system policy allows Administrators to designate price types to be ignored during the Auto Pricing analysis process:

    System Policy Description
    Auto Pricing Excluded Price Types Comma Delimited List of Excluded Price Type IDs that should not be considered during the Auto Pricing Process. Example: 1,2,3.

    The Auto Pricing feature compares newly imported Base pricing (usually MSRP) and resets a designated Target Price via Price Action, assuming there are no other active prices based on established criteria. In the case where a customer has multiple Base Price Types, one or more of these prices may be interpreted as an active price that cancels out the creation of the required Price Action.

  • the following system policy allows Administrators to choose start dates for Auto Pricing Actions:

    System Policy Description
    Auto Pricing Start Date Days Out The Number of Days to use from the Date of Processing to use to set the Price Action Start Date for Auto Generated Price Actions.

  • the following two Features can be configured in the Default Price Matrix Config and Default Workflows screens:

    Feature ID Description
    Auto Price 4 Auto Generated Price Action for Automated Pricing
    Price Reversion 5 Auto Generated Price Action for Price Reversion

The Auto Price process is controlled by a procedure which can be configured in Job Manager as a recurring job.

The following need to be configured:
  • system policies
    • Auto Pricing Base Price Type
    • Auto Pricing Target Price Type
  • Default Workflow with FeatureID = 4 (Auto Price)
  • Strategy that’s using the System_input variable
