Adding sites to existing pricing plan clusters

Sites can be added to existing Pricing Plan Clusters.

To add an existing site to a pricing plan:

  1. Select Add Site in the toolbar of the Sites grid.
  2. In the Add a Site to an Existing Pricing Site Cluster dialog, click the dropdown arrow in the Site Filter field to choose a set of Sites.
  3. Optionally, to return a subset of Sites from the Site filter select the checkbox labeled Select only some items from the current filter.

    1. On the Site Selection dialog, select the checkboxes for the Sites to be added to the new Site Cluster.

      Only those Sites meeting the filter criteria and not already associated to the Pricing Planâ€™s pre-configured Site Clusters display.

    2. Click Ok.

      If the newly added Site does not share the same currency the Site Cluster name is re-used by currency. For example, EMEA(GBP) and EMEA(EUR).

  4. Click Ok on the Add a Site to an Existing Pricing Site Cluster dialog.

  5. Click Ok on the confirmation pop-up window.

  6. If the Activate the New Site(s) to Existing Action(s) window displays, then select the Price Actions, not already exported, to include the new Site.

    The Activate the New Site Cluster(s) to Existing Action(s) window does not display if:

    • there are no future Price Actions

    • all future Price Actions are in a core status

      A core status is a Status that is not user configurable. Core statuses do not display in Workflow. The following are core statuses:

      • Preparing Data for Export
      • Ready For Export
      • Exported
      • Export Confirmed
      • Exported With Errors

  7. If required, click Save on the Activate the New Site Cluster(s) to Existing Action(s) window.

    Upon clicking Save, the Site is added to the Site Cluster and associated Sites are added to Price Actions as Included and the Pricing Strategy Variable default values are applied to the new Site and pushed down to Variants\Sites. Also, the following are marked as Calculation Required:

    • Site Cluster Variant Sites

    • Site Cluster Variants

    • Price Action

    • Price Action's Plan

    If a Price Action has gone into a calculating state, the Site Cluster and associated Sites are added as Excluded and a dialog displays indicating that not all Price Actions were updated.

    If a Price Action exists where the Site Cluster is Excluded, then the new Site is included as Excluded to the excluded Site Cluster.
