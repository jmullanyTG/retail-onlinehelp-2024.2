Adding sites to existing pricing plan clusters

Sites can be added to existing pricing plan clusters.

To add an existing site to a pricing plan:

  1. Select Add Site(s) in the toolbar of the Sites grid to open the Add a Site to an Existing Pricing Site Cluster dialog.
  2. Click the dropdown arrow in the Site Filter field to choose a set of sites and, if desired, check the Select only some items from the current filter checkbox to refine the list to be returned.

    1. On the Site Selection dialog, select the checkboxes for the sites to be added to the new site cluster.

      Only those sites meeting the filter criteria and not already associated to the pricing plan’s pre-configured site clusters display.

    2. Click Ok.

      If the newly added site does not share the same currency the site cluster name is re-used by currency, e.g. EMEA(GBP) or EMEA(EUR).

  3. Click Ok on the Add a Site to an Existing Pricing Site Cluster dialog.

  4. Click Ok on the confirmation pop-up window.

  5. If the Activate the New Site(s) to Existing Action(s) window displays, then select the price actions, not already exported, to include the new site.

    This window does not display if there are no future price actions, or all future price actions are in a core status (a core status is not user configurable and does not display in Workflow). The following are core statuses:

    • Preparing Data for Export
    • Ready For Export
    • Exported
    • Export Confirmed
    • Exported With Errors

  6. If required, click Save on the Activate the New Site Cluster(s) to Existing Action(s) window.

    Upon clicking Save, the site is added to the site cluster. Associated sites are added to price actions as Included and the Pricing Strategy Variable default values are applied to the new site and pushed down to Variants\Sites. Also, the following are marked as Calculation Required:

    • Site Cluster Variant Sites

    • Site Cluster Variants

    • Price Action

    • Price Action's Plan

    If a price action has gone into a calculating state, the site cluster and associated sites are added as Excluded and a dialog displays indicating that not all price actions were updated.

    If a price action exists where the site cluster is Excluded, then the new site is added as Excluded to the excluded site cluster.
