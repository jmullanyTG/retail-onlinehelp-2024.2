Price action status, status changes, and messages

The Price Action Status provides the ability to drive corporate configured processes and associated permissions by roles and users, for each Status, as well as provide communication between Statuses that can be reviewed to understand the life cycle of the Pricing Action.

Changing price action statuses

Price Action statuses can be changed by clicking the Change Status(es) button to display a dropdown menu to change the status of a single Price Action or change the status of multiple Price Action at one time.

The ability to simultaneously change multiple Price Action Statuses greatly reduces the repetition associated with opening and changing the Status for each and every Price Action.

The menu options are:

  • Change Status
  • Change Multiple Statuses

Change Status

The Change Status menu option is based on the Price Actions selected in the List grid. Selecting this option opens the Change Status dialog.

This menu option is enabled when the:

  • start date is today or in the future.
  • state is not Calculation Required.
  • appropriate User Rights have been assigned to modify the status in its current Workflow Status State.
  • user has permissions to the next Workflow Transition.

Change Multiple Statuses

The Change Multiple Statuses menu option is based on ALL the Price Actions displayed in the List grid. Selecting this option opens the Change Statuses dialog.

Field Description
Price Action ID (Read Only) A unique identifier
Price Action (Read Only) The Price Action name
Workflow Name (Read Only) The Price Action Workflow
Price Action Status Displays a dropdown with the next available Status for the assigned Price Action's Workflow for which the user has been provided access.
Owner Displays a dropdown populated with all Users and Roles that are not disabled and not deleted. The list is filtered by users that have the right to edit Price Actions.

The purpose of changing the Price Action Owner is to assign the Price Action to another user for additional work or review and approval.
Message An editable text field used for capturing the reason for the Status change or a message to the newly selected owner.

This menu option is enabled unless any one of the following criteria is true:

  • all Price Actions have a start date that is in the past.
  • all Price Actions have a state of Calculation Required.
  • permissions have not been assigned to modify Price Action status during the current Status State for all Price Actions.
  • the user does not have permissions to the next Workflow Transition, for all Price Actions.

Status changes and messages

To view the Price Action status changes and messages, double-click the icon in the Messages field to display the Messages window.
