Creating Price Actions

Within Price Manager, navigate to the Actions tab in a Pricing Plan. The Actions tab is next to the Information tab:

  1. Click New.

  2. Complete the fields in the Create New Price Action dialog as required.

  3. Click Ok to create the new price action.

Field descriptions

Field Description
Name A descriptive label.
Description A summary regarding the purpose of the price action.
Start Date Select a future date that the price action will set new product pricing. The system will default and restrict to no earlier than the start date.
Strategy The price action Strategy determines the fields to be used for user input, how the pricing is calculated and validated based on the priority of Price Strategy Rules and the types of validation failed severities configured to determine pricing.

The strategies available to be selected are filtered based on the Price Action Strategy’s restricted User and Role access.

Workflow The life cycle of the price action from creation to export to downstream systems.
Variant Actions can be configured for a subset of products configured in the price plan. Additional product groups can be added to the action only after being added to the plan.
Site Clusters Site Cluster Sets are available based on Site Clustering configuration.
Price Matrix Optional. If the Plan is configured to use a Price Matrix, then this field allows the Price Matrix to be changed or removed from a Price Action.

This field displays if system policy Default Pricing Plan Price Matrix Usage is set to True.
