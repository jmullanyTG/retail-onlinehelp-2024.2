Add a price strategy rule

After a Price Strategy has been selected, click New to add a Price Strategy Rule. Complete the fields as required and click Save.

Field Description
Order Enter the priority order of the Strategy Rule. The existing priorities re-order accordingly, if appropriate.
Type Whether a calculation or a validation is configured. The selection populates the Rule Calculation dropdown.
Description A descriptive label to summarize the purpose of the Price Strategy.
Rule Calculation The calculation or validation to be used in the Price Strategy from the dropdown. Only active calculations and validations display in the list.
Calculation Description An automatically generated read-only description.
Validation Severity How Calculation Rule failures should be evaluated. Severity can only be set for Validation types.
  • Upon failure (Low Priority), provide notification in the validation log and continue calculation and evaluation of any following Price Strategy Rules.
  • Upon failure (Medium Priority), provide notification in the validation log and continue calculation and evaluation of any following Price Strategy configured Calculation Rules.
  • Upon failure (High Priority), provide notification in the validation log and discontinue calculation and evaluation of any following Price Strategy.
  • Error Override (Optional) – If checked, users with permissions can override validation failures to continue calculations and validation evaluation.
Allow Validation Override If checked, users such as Merchant or Buyer Managers can override (approve) Validation failures. This reduces the number of user reviews of Validation failures throughout the life cycle of the Price Action. All Validation Severity Types (Information, Warning, Error) can be overridden.

The ability to override is also in conjunction with the Workflow Status State configuration.
Active (Optional) If unchecked, the configured Calculation Rule is not evaluated when calculating and evaluating Price Strategies.
