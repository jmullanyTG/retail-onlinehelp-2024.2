Price planning system policies

The following system policies influence the behavior of various functionalities within the Price Manager module.

System Policy Description
Advanced Validation Log When this policy is set to False, the system displays failed Validations at the Inventory level.

When the policy is set to True, the system groups Inventory failed Validations up to the Variant level and Site (i.e. suppressing Item Master). The system also allows overrides to be acted upon at the Variant level if configured.

If a plan’s Product Pricing Level is set to Variant Site Plan, then this system policy is ignored and the Advanced Validation log displays regardless.

If a plan’s Product Pricing Level is set to Item Site Plan, then this system policy is evaluated and the system displays either the default or advanced validation log.
Allowed Auto Price Base Price Types Comma Delimited List of Allowed Price Type IDs for New Site Pricing. Example: 1,2,3. No value means all price types are allowed.
Allowed Auto Price Target Price Types Comma Delimited List of Allowed Price Type IDs for New Site Pricing. Example: 1,2,3. No value means all price types are allowed.
Allowed New Site Pricing Price Types Comma Delimited List of Allowed Price Type IDs for New Site Pricing. Standard PriceTypes: 1 = Regular, 2 = Promo, 3 = Markdown, 4 = MSRP, -1 Current (Current Active Price Regardless of Price Type). Example: 1,2,3. No value means all price types are allowed.
Allowed Price Planner Price Types Comma Delimited List of Allowed Price Type IDs for New Site Pricing. Standard PriceTypes: 1 = Regular, 2 = Promo, 3 = Markdown, 4 = MSRP, -1 Current (Current Active Price Regardless of Price Type). Example: 1,2,3. No value means all price types are allowed.
Auto Generated Price Action Limit Sets Variant batch size for auto-generated Price Actions
Auto update Price Action on import changes Controls the system to implement automatic updates for imported product changes. By default, the system is set to ‘False’ requiring you to review changes before updating the Price Action with the new information from the user interface.

If set to ‘True’ the system will automatically update Price Actions with imported changes that affect Pricing Actions with the same Product and Site combination.
Auto Pricing Base Price Type The Price Type ID to be used as the Base Price in Auto Pricing computations
Auto Pricing Excluded Price Types Comma Delimited List of Excluded Price Type IDs that should not be considered during the Auto Pricing Process. Example: 1,2,3
Auto Pricing Start Date Days Out The Number of Days from the Date of Processing to use to set the Price Action Start Date for Auto Generated Price Actions.
Auto Pricing Target Price Type The Price Type ID to be used as the Target Price in Auto Pricing computations.
Config View Collection Set This policy determines the user interface to be used for the Price Manager’s Configuration.
Data Initialize Batch Size This policy determines the number of records to return, at one time for performance reasons, when initializing a Price Action. The system default is 1000 Item Masters records. This will need to be modified based on the available server configuration.
Default Pricing Plan Price Matrix Usage Sets the default Price Matrix Usage value for New Pricing Plans
Disable Action Level Calculate If True the Calculate button at the Price Action Level will be disabled and users will need to select a single Cluster for Calculation.
Filter unpriced Variants for New Site Pricing By default the New Site Pricing Variant Select will return all Variants for the Selected New Site. When this policy is set to “True” only Variants that have a value for the selected Price Type will be returned by the Select. This policy only filters the number of Variants returned to the UI, and only Variants with a Price Value will be processed on execution regardless of setting value.
Inventory Meta Data View Sets ViewID for Price Manager inventory view meta data
Limits DEADLOCK retries The number of DEADLOCK Retries that may be attempted per DML statement during Price Action Creation - overrides the default System Policy.
Maximum Price Action Detail Tab The maximum number of Price Action Detail Tabs a Plan can have open.
New Site Pricing Displayed Variant Page Limit Sets the limit of the number Variants to be displayed per page within the New Site Pricing UI. The value of the setting should never exceed 200000.

The default for this system policy is 1000.

An error is generated if this system policy is set to 0 as this would mean trying to send zero items to be selected. An error is thrown because the Filter requires at least 1 item to be sent.

If this system policy is set to 1, then you are only allowing 1 variant to return to the screen at a time. This would result in a very high page count as there would be 1 page for every variant available at the site.
New Site Pricing Plan ID The New Site Pricing Plan ID designates which plan to use when creating price actions that initialize prices at a new site from a model site.
New Site Pricing Start Date Lead Time The New Site Pricing Start Date lead time specifies how many days in the future the new site pricing should start. Minimum value is 1.
Plan Start Date Number Of Days Out Set default number of days out from current for start date field in Price Manager. This can only be overridden (i.e. backdated) by users with the Allow Emergency Pricing Action user right which allows a date to be selected as early as tomorrow. For more information, see Price manager.
Price Action Calendar View Collection Set This policy determines the user interface to be used for the Price Manager’s Calendar.
Price Action View Collection Set A View Collection Set that may be configured to display information in context of a Price Action (e.g. reports).
Price Manager VAT Rate Product Hierarchy ID Product Hierarchy ID used for Price Manager VAT Rates
Price Planner Configuration: stale for N days before deletion occurs. Count of days to have elapsed since the last activity was recorded for, or scheduled for, a Price Planner Configuration, after which it can be deleted.
Price Planning Period Calendar The Calendar that is to be used to select periods pertinent to Price Planning.
Price Planning Period Calendar Period Type The Calendar Period Type that is to be used to select periods pertinent to Price Planning.
Price Planner View Collection Set A View Collection Set that may be configured to display product information pertinent to Price Planning.
Price Reversion Allowed Price Types

The Price Type IDs Allowed for use as the Source Price in Price Reversion Price Actions.
Price Reversion Tab Visibility If set to True the Price Reversion Tab will be accessible within Price Manager Plans. If False (default) the Tab will be Hidden.
Pricing Plan View Collection Set Sets whether or not to display a pricing plan view collection set as a tab in the Price Manager's pricing plan screen.

Value = 0 does not display a view collection set tab.

Value = n (where "n" = a view collection set ID) displays the corresponding view collection set as a tab in a pricing plan screen. The tab name is the name of the selected view collection set.
Product Pricing Level Determines the level at which the system will allow generation of pricing (execution of calculations and validations) within the solution.

Setting the value to Variant\Site and Item\Site Pricing allows users to manipulate the generation of pricing at a high level (Variant\Site) and have greater control and degree of manipulation of pricing at a lower level (Item\Site).

All Price Actions within a Pricing Plan will be limited to either Variant\Site or Item\Site pricing.
Show New Site Pricing Is New Site Price tab in Site Manager visible
Variable Numeric Precision The system policy is used to set the database field precision to ensure efficient updates\selects and maintenance of database growth for configured Pricing Variables and their assignment to Pricing Strategies. Further refinement of each Pricing Variable's formatting can be implemented via Price & Markdown Configuration screen.
