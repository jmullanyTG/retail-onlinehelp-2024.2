Editing price actions

The Actions tab provides the ability to create and manage price actions in price plans. As there is limited ability within the Actions tab to change data specific price actions need to be opened in a separate tab for more extensive edits. The Edit button only allows changes to the price action header. The Open button opens the price action in a separate tab where detailed changes can be made.

Opening price actions in a separate tab

To open a price action in a separate tab, either:

  • double-click a price action in the Price Actions grid on the Home screen which displays the Actions tab.
  • double-click a plan in the Plan grid on the Home screen. Navigate to the Actions tab.

In the Actions tab, select a price action and click Open.

Detailed information is available in the separate tab. In addition to the Price Actions grid and the Clusters grid, there is a Details section.

Within the Details section, the data that displays in the Variants grid is at the Variant-Site level (the Filter Entity Type is Variant-Site). For more information on Filters, see Creating a new filter. For more information, refer to the Filters User Guide.

Modifying pricing in the Variants grid

New values for applicable Variant variables can be applied within the Variants grid. The modified value is applied to all lower levels (i.e. Item and Item Location Detail).

Any differences within a level are denoted by a shaded cell. Lower level differences are noted by a marked Exception checkbox.

Additional detailed information

To view and edit the details for multiple variants in a single screen, highlight the desired variants and click the Variant Details button on the toolbar.

Modifying price action header information

A price action’s header information can be modified directly within the Actions tab by clicking Edit in the Price Actions section.

For more extensive editing, click Open to open selected price actions in a separate tab.

Within the Price Manager screen:

  1. Double-click a Pricing Plan in the Home screen to open the Plan in a new tab.
  2. Select the Actions tab.
  3. Select a Price Action in the Price Actions grid.

  4. Click Edit in the toolbar to display the Edit Price Action dialog.

    The Price Strategies available to be selected are filtered based on the Price Action Strategy’s restricted User and Role access. If a Price Action Strategy was selected in the past that is no longer valid, the system retains the invalid Price Action Strategy. The invalid Price Action Strategy is no longer available once a different Price Action Strategy is selected.

    The Price Action screen refreshes. If a new Strategy or Start Date is selected, then the system ensures that the Price and Cost values are still valid.

  5. Revise the field values as required.

  6. Click Ok.

    The Actions tab automatically refreshes and displays the new values.

Running calculations

The Calculate button on the Price Actions toolbar runs calculations on all clusters for the selected price action.

The Calculate button on the Clusters toolbar runs calculations on the selected cluster only.

Once a price action has been configured, click Calculate to execute the Price Strategy’s configured prioritized calculations and validations, which are run at the lowest level, and displayed as a minimum and maximum in the Price Action Summary grid.

If there are any differences in entered variable information the Exception checkbox is marked for easy identification. To view lower level calculated pricing information, click Detail for the appropriate Variant details.

You cannot edit a Price Action while it is being calculated and validated by the system.

Applying Price Matrixes to Clusters in a Price Action

A Price Matrix can be applied to all the Clusters in a Price Action. To apply a Price Matrix, select the checkbox in the Apply Matrix To All field in the Clusters grid.

If a Price Matrix is applied to all the Clusters in a Price Action, then values that are not percents in all the other Clusters within the Price Action use the foreign currency equivalent from the exchange matrix that was imported into Price Point Matrix module. For example, if the chosen Cluster is priced in Belgium/EUR at 4.99, then a second Cluster priced in Canada/USD updates to 6.00. (See following screenshot.)

If the variable is a percent value, then the percent value applies as is to all sibling Clusters.

Deleting price actions

The Delete button is hidden without the User Right to delete the Price Action based on the Price Action’s current status, if a workflow has been configured, or if the Price Action has already been exported.
