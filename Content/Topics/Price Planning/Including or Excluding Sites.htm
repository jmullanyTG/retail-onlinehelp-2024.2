Excluding or re-including sites and site clusters in a pricing plan

Sites can be excluded from or re-included in plans even after price actions have been created. This does not change existing price actions. However, new price actions will have the sites excluded or re-included.

To include or exclude sites or site clusters in a pricing plan, navigate to the Sites pane on the plan’s Information tab.

The checkbox in the Inactive field is unselected by default meaning the site or site cluster is included. Excluded sites or site clusters are marked by a strikethrough.

Once Save is clicked, the plan's start date is evaluated (displayed in the Start Date field) to determine if a new site cluster set snapshot exists. If a new site cluster set snapshot does not exist, then any inclusions or exclusions of sites and site clusters are saved to the plan.

If a new site cluster set snapshot does exist, then a message dialog displays to inform that an updated site cluster set exists and will be loaded. Once Ok is clicked, the Site Cluster grid is refreshed with the site cluster set snapshot applicable to the plan’s start date and exclusions and re-inclusions must be re-selected.

For more information on Snapshots, see Creating and Managing Financial Plans.

For more information on Snapshots, refer to the online help.

Excluding sites and site clusters

In the Sites grid:

  1. Select the checkbox in the Inactive field by the name of the site cluster to be excluded or select checkboxes within a site cluster to exclude particular sites.

  2. Click Save.

    The screen automatically refreshes and the names of excluded sites and site clusters are denoted by a strikethrough.

    There must be at least one site and site cluster included in a plan. Therefore, the following message displays if there is an attempt to exclude all site clusters.

    Clicking Ok closes the dialog and refreshes the screen.

Re-including sites and site clusters

In the Sites grid:

  1. Select the checked checkbox by the name of the excluded site clusters (indicated by a strikethrough) to be re-included or select checkboxes within a site cluster to re-include particular sites.

  2. Click Save.

    The screen automatically refreshes and the names of the re-included sites and site clusters are no longer denoted by a strikethrough.
