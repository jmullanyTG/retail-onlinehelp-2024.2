Creating price actions within Variant by Site plans

Use the Planning tab to determine which variants / variant sites are prime candidates for markdowns and to create the necessary price actions. The Planning tab provides visibility to what the price is at a future date, stock on hand, sales, revenue, etc.

The Planning tab is located between the Price Action Information and Actions tabs and is only available for Variant Site Plans.

The Planning tab is only visible with User Right Add Price Action.

Set the dates to plan for and the strategy to implement, and review all the variants in the plan, or use an available Variant Site Filter to intelligently identify variants in the plan based on filter metadata. The data can then be reviewed and markdowns applied by entering a percentage or directly entering a new price at the variant or the variant site level. The plan can then either be saved with further markdowns applied at a later date or submitted to commit the changes and generate the needed price actions.

Another feature for Price and Markdown is the New Site Pricing tool incorporated into Site Manager. This tool is used to select a new site and copy pricing from a similar site. From Site Manager, select the site that is to be priced, the model site to copy pricing from, and the price type to be copied. Pricing can be reviewed and submitted to generate price actions in a designated plan.

Adding custom views in tab form

For Variant by Site plans, custom views can be added as tabs on the Planning view. For more information, see Adding custom views to the Price Manager Planning tab. For more information, refer to the Views User Guide.

System policy Price Planner View Collection Set must be set to the View Collection Set ID. When the system policy is set to a valid View Collection Set ID, then that associated View is visible as a tab on the Planning view next to the Site Detail tab. When the system policy isn’t set, the tab is not visible.

Grids

The top grid is an aggregated view of all the prices of the variant site, up to variant. This grid displays style/color attributes, markdown price, recommended price, store inventory and sales history specific to the selected stores in the Store Selection pane. Users can enter a markdown price, or override a recommended price, directly or calculated as a percent discount of IP or CSP.

Prices can be further refined in the Site Detail grid which displays the sites associated with the variant. In this grid, enter a style/color-store specific markdown price.

Click the Submit button to create new price actions for the modified entries to reflect the new prices.

Notes

  • Resubmitted new sites have Resubmitted + Date & Time appended to the Price Action Name.
  • New sites must have existing variant site records.
  • If a Plan Cluster already exists based on the new site’s CountryID and CurrencyID, the new site is added to that Cluster. If a Cluster for the new site does not exist, a new cluster is added and the new site is assigned.
  • The new site will get Model Site prices for variant site records that already exist for the new site. This process does not add variant site records that exist for the Model Site but do not exist for the new site.

Grid functionality

Choose: Launches the Choose Fields dialog in which additional columns can be selected to provide more information.

Toolbar functionality

Functionality Description
Execute Saves the selected parameters. The Execute button is enabled once all mandatory parameters are selected. The selected parameters are saved as a set and can be selected later from the Select Configuration dropdown menu.
Submit Saves changes and creates a new price action based on the configuration and selected variants (variants with assigned price changes). The Submit button is only active when unsubmitted price changes exist.
Save Saves all changes since the last save.
Apply Recs If there are any recommended prices with a start date between the price planning configuration’s Period Start and End date, then clicking Apply Recs copies those price recommendations for the items currently displayed to the User Input Price and/or Percent Off fields, and the New Price for those items is calculated based off that recommended price.

Recommended Price and Percent off values are also visible as columns in Price Planner displaying what those prices are prior to applying the recommendation.

Refresh Pricing

Recalculate new prices based on the keyed in value. Refreshing the pricing does not overwrite any user keyed values.

This is a quick and convenient way to check for and refresh price and cost data based on the date and recalculate all items for a price action or price action cluster regardless of the calculation status.

Users can display a particular price action and refresh the whole price action or select and refresh a specific cluster. The entire price action is recalculated whether or not it's been modified.

Refreshing can be repeated as necessary. For example, should all the prices or costs change later on, the Refresh Pricing functionality can be rerun. All the prices or costs for all of the items and all the clusters update and the calculations redone.
Clear Deletes any previously saved data in progress (data that has not been used to create a Price Action) and clears the UI of all input values.
Select Configuration Once the parameters are selected and at least one change made, the set is saved (except for the Variant Site Filter selection) and can be selected at a later date to further modify the Plan prior to the Plan being submitted.

Parameters

Parameter Description
Period The current price period. Based on a system policy.
Strategy The Price Strategy to be used to create Planned Price Actions.
Base Price Type The Price Type that discount inputs will be used to calculate against to get the new Price. Current is the most current price of any Price Type.

Selecting the most recent Variant Price for the specified StartDate regardless of Price Type

Select Current to choose the most recent Variant Price for the specified StartDate regardless of Price Type. This option only displays if -1 is specified in the following system policies:

  • Allowed New Site Pricing Price Types
  • Allowed Price Planner Price Types
Variant Site Filter A dropdown menu to filter which Plan variants display in the price planner grid.
Null Selecting the Null checkbox located beside the Variant Site Filter parameter causes the Variant Site Filter to be ignored. All Variants in the plan are returned.
Price Action Name A descriptive label.
Start Date A future date that the price action will set new product pricing.
Use EoS Sets Price Actions to use the “End of Season” time frame where applicable.

A metadata labeled Season Time Frame can be set in the Site Manager module to define an End of Season date that takes precedence over the Start Date. This requires a time frame be defined in the Timeframes module and values be set via Site metadata labeled Season Time Frame in Site Manager.

The Season Time Frame column displays the defined time frame. Use the Choose Fields functionality to select Season Time Frame from the Time Frames group.

A column in the upper grid labeled EoS Date shows the actual resolved End Of Season date, this is the date that will be used when creating a price action through the Planning tab if Use EoS is checked.
Currency The user selected currency.

For Plans that have sites with multiple currencies, values for sites that do not match the selected UI currency are converted to the corresponding currency.
Addt’l Price, Find Missing Search for up to 2 additional Price Types via dropdown menus, each paired with a checkbox to indicate that only records without a value for the selected price type should be displayed. Users can toggle between viewing only those records with missing prices and all records returned by the search parameters.

Within the search results:

  • When the 1st Addt’l Price checkbox is checked, then only records without a value for the Price Type selected in 1st Addt’l Price display.
  • When the 2nd Addt’l Price checkbox is checked, then only records without a value for the Price Type selected in 2nd Addt’l Price display.
  • When both checkboxes are checked, then only records with missing values for both selected Price Types display.
Show All Sites When selected, the Sites grid displays all Sites within the Plan and related metadata. This allows users to view all Sites and metadata related to a Variant regardless of the filter selected. All Sites are returned regardless if Find Missing is checked.

This does not affect the aggregation in the upper grid as the aggregation is based on the selected Variant-Site filter.
Show Plan If checked, returns only Variants that have price changes and all sites in the Site Detail grid regardless of how Show All Sites is configured.
