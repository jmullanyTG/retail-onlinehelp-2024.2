Policy Information

The Policy Information tab displays identical information to that which is found in the Policy module. The policy settings for a specific inventory can be adjusted overriding any higher level settings for the inventory.

For more information, see Policys Overview. refer to the Policy User Guide.

The Policy Information and Effective Policy Information tabs are blank for items managed by the Allocations module.

Changing Policy Types

Use the Type column in the top grid in Inventory Manager to update the Order Policy Type for multiple items. However, the Allow Item's Replenishment Strategy To Be User Controlled checkbox on the Policy Information blade must be selected if changing the Type. Otherwise, a message displays and updates are prevented.

If the Allow Item's Replenishment Strategy To Be User Controlled checkbox is checked and the Type is changed to Bin [B] or Showroom [S], then default Bin Min and Bin Max values are set, and a Demand Transference Method added.

Defaults

If Bin Max has a value that is less than Bin Min, it will be set to be equal to Bin Min. By default, Bin Min = 0.

The Showroom Demand Transference Method default is Split Factor.

Target Service Level

A dynamic quantity based on a service level target. It is a percentage of the items that are fulfilled in a location out of the maximum demand.

The Target Service Level field is configured to prevent values below 50%. This is intended to avoid a scenario where values below 50% generate negative safety stock levels. Upon clicking Save, any values below 50% are converted to 50%.
