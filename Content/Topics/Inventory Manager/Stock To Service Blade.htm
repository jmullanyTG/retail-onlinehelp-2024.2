Stock to Service

Users can easily view how different target service levels affect inventory Safety Stock days by navigating to the Stock to Service blade in Inventory Manager and reviewing the Stock to Service Curve graph.

The Stock to Service blade only displays if there has been an optional integration (via a Webhook) with the Stock to Service calculation. This calculation is an analytical curve developed by TG Service Optimizer 99+ (SO99+) affecting the safety stock days of applied inventories. For more information regarding the Stock to Service calculation, see Target Service Level and Stock-to-Service (STS).

Accessing the Stock to Service Curve blade

To access the Stock to Service Curve graph, navigate to Inventory Manager > Select Inventory > Stock to Service Curve blade.

Stock to Service Curve graph

Users can change the graph minimum and maximum X values to view a portion on the graph by zooming in and use the curve to see the exact value of any given point on the graph. The curve rises or falls in 1% increments. For example, users can put the curve at the 90% service level and move along the graph to see service levels 91%, 92%, and 93%.

Axis definitions

Axis Definition
Y - Safety Stock Days Minimum value of 0 and maximum value is Safety Stock days value of 100% Service Level.
X - Target Service Level Minimum value of 0 and maximum value of 100.

Stock to Service curves

Stock to Service (STS) curves are analytical curves that relate each service level value with the average stock that is statistically required to achieve it.

Statistical demand characteristics are generated in terms of: volume (expected demand), variability (volatility of demand), frequency (typical market/customer order profile).

The STS curve therefore determines the relative average stock and the associated control stock values for each item/warehouse and for the desired service level.

Logistic Parameters

The following parameters represent the input for STS modeling.

Logistic parameters and constraints Description
Review period [periods] Generally corresponds with the planning frequency and the relative ability to react to events
Minimum lead-time [periods] Nominal or contractual lead-time, calculated from the moment in which the need to order is perceived to the moment when the material is available for picking at the warehouse
Average delay [periods] Average supply delay added to the minimum lead-time
Minimum supply lot [units] (*) The minimum quantity that can be ordered from the supplier or produced .

For modeling, TG SO99+ uses whichever of the 2 parameters offers greater time coverage.
Incremental supply lot [units] Size of the incremental quantities allowed, in addition to the minimum lot
Minimum reorder interval [periods] (*) Target minimum coverage of each order. It is effectively a replenishment lot expressed in periods (generally >= stock review period)

For modeling, TG SO99+ uses whichever of the 2 parameters offers greater time coverage.
