Classification module field definitions

There is a grid and several tabs in the Classification Task screen. Details related to highlighted tasks display in the various tabs.

Grid field definitions

The grid displays all of the tasks in their respective execution order.

Field Description
Task Name The assigned name for the task.
Task Description The optional description for the task.
Default Task Select this checkbox if the task is the default task.
Task Enabled If unchecked, the task will not be processed upon execution.
Task Execution Order The order in which tasks are executed. In case of any conflicts, the task with the higher execution order overrides the previous settings.
Task Start Date Defined If the task is dynamic (time sensitive) and the start date is specified, the Task Start Date Defined checkbox must be enabled and a date entered in the Task Start Date field.
Task Start Date The start date for a dynamic task.
Task End Date Defined If the task is dynamic (time sensitive) and the end date is specified, the Task End Date Defined checkbox must be selected and a date entered in the Task End Date field.
Task End Date The end date for a dynamic task.
Task Allowed To Execute If checked, there is a final check to ensure the task can be executed. Task Enabled must be selected and the task be in the specified date range if it is a dynamic policy.

General tab

The General tab displays the information that appears in the task listing at the top of the screen, as well as the filter associated with the task.

Default Priority tab

It is important to set up the default priority to establish a baseline for the task. There are several options for doing so, as follows.

Field Description
Do Not Set Priority The task does not change the default priority.
Set Priority To Specified Value Assigns the priority of all items to a single priority.
Calculate Priority Based On Input Values Uses the selected parameters to define the priority.
  • Prioritize Based On ___ Days: the number of days of historic demand or projected demand. It is common to set this to 365.
  • Entity: the entity type for classification (inventory, item master, or site).
  • Contribution: historic demand or forecasted demand contribution.
  • Classify Based on: determines the inventory metric by which the classification is made. It can be one of the following:
    •  Classify Based On Unit Contribution: based on gross volume of units. Expressed as (Value * 1).
    • Classify Based On Cost On Cost Of Sales Contribution: based on the cost of the volume of units. Expressed as (Value * DefaultCostPrice).
    • Classify Based On Sales Revenue Contribution: based on the sales revenue of the volume of units. Expressed as (Value * DefaultSellingPrice).
    • Classify Based On Margin/Profit Contribution: based on the profit margin of the volume of units. Expressed as (Value * (DefaultSellingPrice – DefaultCostPrice)).
  • Group By: determines the entity by which the classification is grouped.
  • Include Site Transfers: if selected, site transfers are included. This is important to consider if the site is a distribution center. If prioritizing based on sales, the previous receipted site transfers are included in the calculation. If prioritizing based on forecast, the projected site transfers and existing outstanding orders are included.

The percent contributions for each priority can be changed and a running total displays at the bottom of the table. Immediate changes are also displayed in the pie graph.

Priority tabs (1 to 5)

These additional priorities display the same fields found on the Default Priority tab.

The values can be changed as needed.

Relative Deviation tabs (1 to 3)

Relative deviation sets the X, Y, and Z criteria for inventory.

Field Description
Do Not Set Relative Deviation The task does not change the relative deviation.
Set Relative Deviation To Specified Value Assigns the relative deviation of all items to a single relative deviation.
Calculate Relative Deviation Based On Input Values Uses selected parameters to determine the relative deviation.

Determine Relative Deviation Based On ___ Days: the number of days of historic demand or projected demand calculations. It is common to set this to 365.

Contribution: historic demand or forecasted demand.
