Creating classification tasks

Classification tasks work in conjunction with policy tasks to set appropriate ordering policies for inventory. New classification tasks can be created or existing tasks cloned.

Use the industry best practice settings to set up the base task which can then be cloned and modified or disabled if another task replaces it.

Base tasks and default tasks

A base task is the initial task that must be set up as a part of creating classification tasks, while a default task is set manually by the user to default to a specific task that may or may not be the base task. Base tasks are usually the default tasks, but there can be exceptions depending on how the user wants to set up their classification tasks and which tasks they want to run as a default.

To create a task

  1. Click New.

    The Add Classification Task screen displays.

  2. On the General tab, enter a unique name, description, and filter.
  3. Select the Default Task checkbox.

  4. Click the Default Priority tab.

  5. Select the Calculate Priority Based On Input Values radio button, and:

    • enter the number of days in the Prioritize Based On field.
    • from the first dropdown menu, select a level.
    • complete the remaining dropdown fields as required.
  6. In the Percentage % column, select the appropriate percentages for each priority.

    The total must equal 100%.

  7. Click the Relative Deviation1 tab.

  8. Select the Calculate Relative Deviation Based On Input Values radio button.

  9. Add the number of days to the Determine Relative Deviation Based On field.

  10. Select a demand type in the dropdown field.

  11. Complete the Upper Range Value and Lower Range Value columns as required.

  12. Click Ok.
