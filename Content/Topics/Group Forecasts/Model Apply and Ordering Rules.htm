Model apply and ordering rules

Model apply and ordering rules are used for conflict resolution in instances of model overlap for an inventory by creating and maintaining a set of deterministic rules that define the inventory to model mapping.

These rules are maintained via the Maintain Model Apply Rules And Apply Order dialog.

Accessing the Maintain Model Apply Rules And Apply Order window

  1. In the Group Forecast window, click Manage.

    The Forecast Group Model Management window displays.

  2. Click the Maintain Model Apply And Ordering Rules button.

    The Maintain Model Apply Rules And Apply Order window displays.

Changing the application order for the models

Choosing to edit the application order for the models creates deterministic rules for how the inventories are affected by group forecast models.

To edit the application order:

  • Click the Edit Apply Rule button.

These rule orders are read when the group forecast values are moved into the live forecast tables which define the projected demand.

Fields and descriptions

  • Model Name – The name of the model.
  • Order – The order of application to the inventory. The rules are applied in numerical order (e.g. rule 1 is applied and overridden by rule 2, etc.)
  • Apply Type – Determines when the rule is applied.
    • Do Not Apply – Will not apply the group forecast model to any inventories.
    • Automatic – Will apply group forecast changes automatically to associated inventories.
    • Manual – Will only apply group forecast changes if the user requests a manual push.
  • Apply From – The effective start date of the rule.
  • Apply To – The effective end date of the rule.
  • Edit Apply Rule – A button which launches the apply rule editor.
