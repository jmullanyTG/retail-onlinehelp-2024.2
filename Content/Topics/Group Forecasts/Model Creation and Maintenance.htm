Working with group forecast models

When building a model, all items in the model must share the same bucket type. Models are created and maintained through the Add / Update Group Forecast Model window.

To create or update a group model

  1. From the Execute tab, click Group Forecast and then Manage.

    The Forecast Group Model Management window displays.

  2. Click New.

    The Add / Update Group Forecast Model window displays.

  3. Complete the fields as required.

  4. Click Ok.

Field descriptions

  • Name – A name used to identify the model.
  • Description – An optional description for the model.
  • Group – The group forecast groups function as a container or folder structure and may be used to organize models. Folders may be nested if desired. A group must be assigned to a new model.
  • Filter – The selected filter defines the set of items which are participating in the model. The filters which are available were created using the Filters module (a preliminary step to building a successful model is to first build an appropriate filter).
    • Hierarchal – Filters based on product or site hierarchies would be built out of the item master or site group values.
    • Attributes – Filters may be built to include or exclude certain attributes.
  • Security – As group forecast models can be built to ignore item assign filters. Users creating the models should take care to set security rights which are appropriate for the model.
    • Public – Shared with all users.
    • Private – Shared with no other users.
    • Shared – Shared with specific users.
  • Sales Types – If sales history by type data has been supplied, users may choose to include or exclude typed sales when building the model.
