Strategy Activation view

The Strategy Activation view is a single column pivot grid that can be used to create Allocation Strategies and start working on them before starting to allocate. Use the Item Filter dropdown field to filter which variants (style/color) to display.

Assigned permissions determine which channels a logged-in user can activate. For the site based permissions, if the logged-in user has permission for at least one site, then they have permission to the entire channel (i.e. see the entire column for that channel). If all the sites belong in only one channel, then only one column displays.

Allocation Strategies that have already been created are check marked in the Activate? column and grayed out. Existing strategies cannot be deleted in the Strategy Activation view. (There is an existing nightly job that deprecates strategies.)

The cells for Channels that have not been activated against a particular variant are formatted white with the checkboxes in the Activate? field being unchecked. To activate a Channel, select the Activate? checkbox and click Save. The Strategy Activation view automatically refreshes and the cell is formatted to indicate the strategy has been created.

Multiple Channels can be selected to activate against the same variant.

Reviewing the newly created strategy

Note the Level ID and navigate to the Allocation Strategy view to see the newly created strategy in the Allocation Strategy grid.

Newly created strategies have an Exception Status of Missing Configuration. Clicking View in the Exception Status field displays the Allocations Strategy Exceptions dialog. To resolve, select the Calculate Allocation checkbox and click Execute.

Related strategies

The related strategies are reviewed using the Strategy Relations blade.
