Size models field reference

Size model configuration allows users to create the curves, define the ranges, and determine their constituent items.

Top grid

  • Size Model Name – User generated name for the size model.
  • Size Model Description – Optional description for the size model.
  • Size Range – The pre-defined set of acceptable sizes to be included in the curve. The size range data must be imported.

  • Size Curve - The size percentage contribution by store, based on historical sales.

  • Exception Status - Required configuration elements are missing.

    If a size curve’s item master filter is empty, the items in the filter have no history during recalculation, or all history has been filtered out due to sales model criteria, users are notified via an Exception message. If an Exception is present, the field is formatted red. Within the field, click View to see the details of the issue.

    There are three possible Exception messages:

    Exception Description
    No Valid Items in Size Model Filters There are no items matching the criteria specified in the item filter. Please ensure that the item filter is not empty or consider adjusting the filter criteria.
    No Qualified Sales No historical sales were found within the specified timeframe. Please ensure that the items specified in the filter have valid sales records between the dates set by the start date, end date, and Last X Days parameters.
    No Qualified Sales Due to Thresholds No qualified sales that fit the sales model criteria. No sales history within the specified time frame and item filter fit the Percent Covered and Weeks Covered criteria.
  • Filter – The product filter which is used as a base of historical data to generate the size model.
  • Access Level – Access levels determine which users may review or edit the size model.
  • User Select – Displays the specific users for a shared access level size model.
  • Sales Type – Type of sales in historical sales which will be used to generate the curve.
  • Start Date – The beginning history date from which sales data is used to calculate the size model.
  • End Date – The ending history date from which sales data is used to calculate the size model.
  • Last X Days – Number of preceding days of sales history data used to calculate the size model.
  • Percent Covered - The percentage of sizes in a range needed to consider a period in stock, Sizes are ideally weighted in importance based on sales data percentage to total.
  • Calculate – If checked, the size model calculates upon application.
  • Override Allowed – If checked, users can override calculated values within the size model detail screen.

  • # Variants Qualified - The number of variants for a site that must pass data cleansing for the size curve to pass.

  • % Variants Qualified - The percentage of variants for a site that must pass data cleansing for the size curve to pass .

  • Qualified Sales Units - The number of sales units for a site that must be left after data cleansing for the size curve to pass.

  • # Sizes Under - The number of sizes for a site that can have a curve percentage contribution below % Size Under for the size curve to pass.

  • % Size Under - The curve percentage contribution beneath which a size must fall to count in the # Sizes Under threshold.

  • # Sizes Over - The number of sizes for a site that can have a curve percentage contribution above % Size Over for the size curve to pass.

  • % Size Over - The curve percentage contribution above which a size must fall to count in the # Sizes Over threshold.

Thresholds can be added to the grid measuring data-cleansed variants, sales units, and sizes at a site within the item filter selected for the size model. For more information, see Validating enhanced size models.

Middle grid

Aggregate Size Curves display in the middle grid. For more information, see Aggregate Size Curves across all sites.

Bottom grid

Double click a size model to display its details in the bottom grid. The grid displays the contributions of each size to the variant, by store.

  • Size Model Name – Name of the highlighted curve in the size model configuration grid.
  • Site Name – Name of the site referenced in the site specific contributions.
  • Site Code - The Site Code of the Site.
  • Channel Description – Configurable site grouping.
  • Clear Override – A highlight indicates that there are override contributions within the curve. If checked, the calculation clears the override contributions, but does not alter the eligibility flag. For more information, see Clearing substitutions and other size model overrides.
  • Size Listings – Each size in the selected size range appears as a column within the size model detail screen.
    • Is Eligible – If selected, the size is considered to be eligible for allocation to the listed site. All eligible sizes have an associated contribution percentage. For more information, see Recalculating size models when overrides are present.
    • Contribution – Contribution for each site.
  • Total Contribution – Total contribution will sum the contribution of all eligible sizes, and will re-normalize to 100% upon application and recalculation.

  • Override Site - A site to use as a substitute site. For more information, see Substituting size models.

  • Sync Override - Designates whether the substitution site values are continuously updated each time a change is made. For more information, see Syncing size model substitutions.

  • Pass? - For a curve to Pass, all thresholds and tolerances set to non-null values must be passed. A single failure results in Fail.
