Allocations Overview

The Allocations solution contains a workflow to execute orders for the transfer of merchandise throughout the supply chain at a level above item master (defined as variant).

Components of an allocation

There are a few building blocks for Allocations to determine the demand or need of a store. The main components to help calculate the demand for a store are product model, size model, location strategy, and sales curve.

Product model

Product model provides the contribution values to help determine the percent of contribution that each site in the Location Strategy should receive of the total demand. For more information, see Product Model.

Size model

Size model provides the value to further break down the store demand from product model by size. There can only be one size type defined per size model. For more information, see Size Models.

Location strategy

Location strategy provides the sites that will always be considered when calculating demand. Excluding sites can influence other site’s demand.

The following system policies determine if demand is shifted:

  • Forecast - Total Available Quantity - Include All Sites

  • Forecast - Total Sales Quantity - Include All Sites

Setting to Yes includes all location strategy sites and setting to No only includes eligible location strategy sites.

Phase in and phase out overrides can influence demand. For more information, see Location Strategy.

Demand type and phase in/out

Demand type and phase in/out are also components but are part of the allocation strategy.

Demand type

Demand Type determines how the system calculates total demand which is always calculated from Phase In to Phase Out even if the Phase In Date has passed. For more information, see Allocation Strategy grid field reference.

Sales curve

After the size demand is determined for a site the sales curve provides the information to see how to spread the size demand by bucket.
