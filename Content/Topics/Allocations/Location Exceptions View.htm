Location Exceptions

Any overrides applied to a specific location or set of locations within the location strategy are considered to be location exceptions. Location exceptions are viewed and managed in the Allocations Strategy Location Exception view.

To access this view, click the Overrides button within the Location Strategy field in the Allocation Strategy grid.

Once configured, conditional formatting (bold) is applied to the Location Strategy field in the allocation strategy view to indicate the existence of exceptions.

In the Allocations Strategy Location Exception view, users can apply static override values to any of the listed fields on a site-by-site basis or through a mass change.

Additional location information such as site grouping and hierarchical information can be optionally returned to the grid to assist with grouping.

Changing multiple rows simultaneously

  1. Select the first field to be updated.
  2. Press the Shift key and select the last row to be updated.
  3. Right-click to display a menu.
  4. Select Multiple Changes.
  5. Complete the Multi Data Change Form dialog as required.
  6. Click Save.

Once saved, overridden values are conditionally formatted with an orange highlight as is the Clear Override? checkbox. Selecting the checkbox and saving clears all overrides for the site.

A blank or null value in a field does not equate to a zero (i.e. an actual 0). Thus, a user override value of 0 in the weeks of cover field is an explicit user override specifying the desire to hold exactly zero weeks of cover in safety stock for the site, not that the strategy level value should be held.
