Allocation Strategy field reference

The following fields are available on the Allocation Strategy screen:

  • Active Flag – Checked if the strategy is currently active.
  • Actual Rate of Sale – Actualized rate of sale based on sales history.
  • Allocate Strategy ID – A unique identifier for the strategy line. Any active product groups are assigned a unique allocate strategy ID.

  • Allocation Strategy Selection – Defines the strategy bucket used to classify the strategy. Typically, these strategy selections are determined by product life cycle status. This field drives which exceptions are triggered, how some of the settings are applied, and which process (Allocations or Replenishment) manages the item.

    • New Item – An item without sales history, or with a minimal volume of sales history data which has been deemed insufficient to drive allocation measures. New Item can be used to drive initial load supply chains (e.g. circumventing regional distribution centers). The individual supply chain mapping configuration for the different strategy selection values is maintained in the Supply Chain Map Configuration maintenance view. New Items are managed by the Allocations engine.
    • Ongoing Item – An item which has multiple points of sales history data. This item can have a re-trending method applied to the sales projection. Ongoing items are managed by the Allocations engine.
    • Replenishment – An item which no longer needs to be managed at the strategy level and can be put on an automated replenishment strategy. Replenishment items are managed by the Replenishment engine, which takes over responsibility from the Allocations engine for executing processes and providing item-related information. The strategy line is read-only when this option is selected.

  • Allowed Channels - Whether there is one or more channels depends on whether an omni-channel strategy is allowed. If allowed, then one strategy can have multiple channels. If an omni-channel environment is not allowed, then the strategy is broken down into multiple strategies each with 1 channel.

    A new strategy is always created when a strategy is activated using Strategy Activation view. To add channels in existing strategies without going into the Strategy Activation view and creating unwanted strategies, click within this field and select the ellipsis to display the Allowed Channels dialog. The Allowed Channels dialog is used to add and remove channels as needed.

  • Allow Reservation – Ensures that all stores that were set up to have reservation quantities receive their reserved stock quantities.
  • Calculate Allocation – Formatted in orange if there are changes made to the allocation strategy.
  • Channels In Use - The channel or channels associated with the strategy. Which channels are allowed displays in the Allowed Channels field.
  • Cluster Sets – (For users with the Site Clustering module) Location strategies may be defined by selecting specific site grades within cluster sets (cluster sets must be configured for use with Allocations). A location strategy value of Cluster based all sites must be set to use cluster sets to define the effective location strategy. Location exceptions are still applicable. For more information, see Configuring a cluster collection set.

  • Code – The identifying code for the level ID associated with the strategy. This is the item master group code of the hierarchical level defined in the allocation hierarchy.

  • Create Save Point – Creates a Save Point.

    All the information relating to the strategy including location exceptions is saved. Users can then go to the Allocate Strategy Save Point screen and mass apply a Save Point to all the variants in the Item Filter.

  • Demand Rate of Sale – The rate of sale currently being applied to the phase period (defined as the period between the phase-in date or current date and the phase-out date).

  • Demand Type – Determines the driver for the demand signal. This setting also determines which fields within the allocation strategy are and are not relevant to shaping the demand signal.

    • Total Sales Quantity – Input the Total Sale Demand Units for the Phase In / Phase Out period. This value is used to determine the average weekly ROS by spreading it across the stores and weeks between the Phase In / Phase Out.
    • Rate of Sale – For use with rate of sale type plans.
    • Total Available Quantity – Allocation based on total available quantity while protecting the size curve.

    • Total Push – Allocates all available inventory in the selected inventory sources at one time. A specific quantity to push can be designated by entering a value in the Push Qty field.

      Total push demand type prioritizes the movement of stock from the DC and violates store level constraints or ideal stock quantities such as phase in, phase out, and max stock.

    • Rate of Sale by Grade – (For users with access to the Rate of Sale Calculator module) A rate of sale value by cluster grade can be applied to individual locations. A location strategy of cluster based must be specified, and site clusters must be selected to define the location strategy for this demand type. Included grades by cluster set must be configured within the allocate cluster rate of sale view.
  • Description – A description accompanying the code. This is the item master group description of the hierarchical level defined in the allocation hierarchy.
  • Dynamic SS Policies - (read-only) If the checkbox is checked, then the strategy has dynamic safety stock policies.
  • Eligible Channel – A configurable named site group, or the explicit channel attribute. Additional site groups may be returned as required. Informational only.
  • Engine Run Date – A date field showing the last time the allocation strategy was calculated.

  • Exception Status – If required configuration elements are missing from the strategy or if it is ready to allocate. If there are no blocking exceptions, then Ready to Allocate displays.

    To view exceptions, click within the Exception Status field and then click View to launch the Allocations Strategy Exceptions dialog.

    Exceptions

    Exceptions include:

    Exception Description
    Channel Migration Overlap Detected An allocation strategy had overlapping channels and were not migrated.
    Missing Channel Configuration An allocation strategy has no associated channels; therefore, the strategy cannot be processed.
    Overlapping Channel Configuration  

    Allocation strategies detect if the Size Model used contains any Exceptions. The following message displays:

    Exception Description
    Bad Size Model Detected An exception has been detected in the used Size Model. Please navigate to the Size Models module to review the model or select an alternative size model. Recalculate the allocation strategy once the exception has been addressed.
  • Has A Related Strategy - Displays which strategies have a relationship. The relationship is either a Prepack or a Channel.
  • Holdback % - Defines a quantity of stock to reserve in the warehouse or distribution center.
  • Holdback Qty – A holdback quantity instead of a holdback percentage if desired. The holdback quantity is applied at a variant level allowing Allocators to reserve a fixed quantity of stock in reserve for new store openings or other related purposes. Holdback quantity is not applicable to product groups.

  • Inventory Source Rule – Defines the priority for the sourcing of stock.

    • No Priority – No source priority should be applied. This applies to selected DC and PO stock restrictions as well (i.e. absolutely no restrictions are placed).
    • PO Only – Only use purchase orders.
    • PO then WH – Source from purchase orders first, then warehouse stock.
    • Warehouse Only – Only use warehouse stock.
    • WH then PO – Source from warehouse stock first, then purchase orders.
    • PH Only – Use only placeholder stock.
    • PH + Available – Use placeholder stock, then any available stock.

  • Item Exceptions – Any and all item level exceptions which have been triggered appear in this field. Multiple exceptions can be viewed in the exception detail view.

    New item exceptions

    • Undefined Product Model – An item without a selected product model.

    • Unallocated Weeks Before Set – For new items, the difference in time between the current week and the scheduled set week are less than the defined lead time.

    Both new item and ongoing exceptions

    • Missing Actionable Inventory - The allocation strategy does not have any available inventory.
    • Phase Window Exceeds Limit – The phase window (defined as the number of days between the phase-in and phase-out dates) exceeds the allowed quantity as defined by the system policy for the phase window.
    • Size Model to Range Mismatch – The allocation strategy has a size model that does not cover all of the sizes for the selected item or product group.
    • Supply Chain Incomplete – An Inventory does not have an associated DC or Store as a source site priority.
    • Undefined Size Model Option – An item without a selected size model.
    • Undefined Strategy Location – An item without a selected location strategy.
    • Undefined Temporal Plan – An item which has been defined as using an imported plan sales type curve, without a plan selected.

    Allocation item exceptions

    Item level exceptions which may or may not be blocking.

    New item exceptions

    The Item is defined as New Item:

    • Item Recording Selling – The Item has actualized selling of more than 3 periods.
    • In DC Missing Cutoff – The goods will not arrive in time to be ordered and arrive in store by the Phase In date. There is no DC OH or store OH and the Projected PO Receipt Date is greater than the Phase In date minus the lead time.
    • Purchase Order, No Allocation – The Item has purchase orders at DC and no Stock OnHand in the stores and no orders have been generated.
    • Stock On Hand, No Sales – The Item has warehouse OH and no Stock OnHand in the stores and no units in transit to the stores.

    Ongoing allocations

    • Out of Stock – There is not enough inventory to fill the projected weeks of cover between today and the Phase Out date.
    • Below Min SOH – There is not enough inventory to fill the projected weeks of cover. At least 1 store will be below the minimum stock OH.
    • Markdown Upcoming – the specified MD Date is within the defined weeks of cover, or the MD Date is the week after the current week.
    • Phase-Out Upcoming - the specified Phase Out Date is within the defined weeks of cover, or the Phase Out Date is the week after the current week.
    • Incoming MD, Below Sell-thru - The projected sell thru % at the item defined Markdown Date is less than the item defined sell thru %.
    • Purchase Order, No Allocation – Item is defined as “Ongoing Allocation” and the incoming Purchase Order at DC has not been allocated and the receipt date is with the weeks of cover.
    • Aged DC Inventory – Item is defined as “Ongoing Allocation” and the Warehouse stock OH is greater than 0 and there has been sales recorded for 13 weeks and no reduction in DC Inventory.
    • Flow Expected, No Receipt – The Store Purchase Order receipt date is before the current week, but the goods have not arrived.

    Both new item and ongoing exceptions

    • Actual Qty Above Expected Qty - The actualized PO receipt for the week is greater than 105% of the total expected units from the purchase order.
    • Actual Qty Below Expected Qty – The actualized PO receipt for the week is less than 95% of the total expected units from the purchase order.
    • Unable to Fulfill Pending Orders – Pending Orders will not be fulfilled due to a lack of stock OH and/or incoming order.
  • Last AP Push - When the Assortment Planning module last updated the Allocations module.
  • Location Strategy – Displays the selected location strategy for the allocation. If the location strategy column is bolded, an override has been applied to that allocation strategy’s location strategy. The location strategy contains a group of sites which are the intended destination for the allocated stock.
  • Markdown Date – The planned date on which the product is to be marked down.
  • Max Retrend WOC: Shows the MAX Value of (Total Retrend Forecast within the WOC End Date - Total Forecast within same date range) / Total Forecast for phase window WOC End Date is pretty complex but it boils down to today's date + RC + Days of Cover.
  • Max Stock – A maximum stock constraint for each site contained within the location strategy. Set for the allocation strategy level (typically style-color).

  • Maximum Prepack Overage Allowed – An editable number between 1 and 100 as a percentage. Implements a ceiling for incomplete prepacks that the system recommends.

    Consider the following scenarios of varying prepack demands for a prepack of 5 units and a threshold of 41%:

    • 1 single unit of demand = 1/5 prepack = 20% < 41%, therefore round up and send the full prepack
    • 2 units = 2/5 = 40%, therefore send a full prepack
    • 3 units = 60% which is greater than 41% therefore a prepack is not sent
  • Min Stock per Size – Item master level safety stock.
  • Phase In Date – The date on which the items will be set or placed on sale within the stores. The phase-in date is the starting anchor point for the demand signal.
  • Phase Out Date – The end-of-life or clearance date of the product. Strategy will no longer be considered or calculated if it is outside of Phase Out date. If not entered, the default calculation is six months from the Phase In date.

  • Placeholder Stock – Allows for Allocators to specify a stock quantity that is expected to be ordered for allocation. This allow Allocators to determine the result of an allocation based on a mock quantity to ensure the sizing and quantity is sufficient to meet the allocation constraints of a given strategy.

    • Direct Ship Functionality – Users wishing to execute orders based on opportunity buys may create direct supplier orders by using placeholder stock. When placeholder stock is selected as an inventory source and users have keyed in a placeholder quantity, orders will be generated as direct supplier orders to the sites contained in the location strategy. These order recommendations may be approved and actioned upon within Order Manager when selecting a “from strategy” allocation order.

  • Prepack Run Mode – Default is NULL. Allows for the selection by strategy line of the mode for a prepack Assignment. The default is set by the system policy Allocate Prepack Run Mode.The three possible modes are:

    • Match Process By Match Process:
    • Period by Period:

    • Skip Perfect Fit: Similar to Period by Period mode, but ignores the perfect fit step.

      Thresholds need to be set to 51% or higher to get desirable results.

  • Presentation Priority – Defines what to do in a constrained situation:

    • Order Multiple > Presentation > Demand – Prioritizes coverage to ensure all locations receive units, then fulfills minimum presentation requirements for all locations, then addresses demand. If there is not enough stock available to cover all stores, then global site priority is used to determine which stores get coverage.

      It is important to note that the Engine uses the term Need which is comprised of multiple components such as Coverage, Safety stock, and Demand to determine what each site is requesting to generate recommended orders.

      For example, the following recommended orders are generated for 2 sites with a Need of 25:

      Site Need Order Multiple (Coverage) Presentation (Safety Stock) Demand
      1 25 5 8 12
      2 25 8 15 2

      This is the only option for Replenishment Runs

    • Demand > Presentation – Prioritizes the planned demand for all locations over presentation minimums.
    • Presentation > Demand – Fulfills minimum presentation requirements for all locations before addressing demand.

    Constrained situations

    In Allocations, users can decide what to do in a constrained situation with the following:

    Coverage (aka order multiple) → Safety Stock (aka presentation) → Demand

    Demand → Safety stock

    Safety Stock → Demand (Also this is the only option for Replenishment Runs)

    It is important to note that the Engine uses the term Need which comprises of multiple components such as Coverage/SS/Demand to determine what each site is requesting to generate recommended orders.

    Site 1 could have a Need of 25 but the Engine sees that as a Coverage = 5, SS = 8, and Demand = 12, where Site 2’s Need = 25, but Coverage=8, SS=15, Demand=2

  • Primary WH Only – Restricts sourcing to the primary warehouse as defined by the supply chain.

  • Priority Sequence - If Priority Groups will be used to protect specific stores before referring to Global Priority Group:

    • Use Priority Group – when looking to fulfill orders, Priority Group will be considered.

    • Ignore Priority Group – when looking to fulfill orders, Priority Group will be ignored.

  • Product Models – The selected model or models. These may be adjusted by clicking on the ellipses to launch the product model selection wizard.
  • Push Qty – A specific quantity of stock with which to execute a “total push” demand type allocation.
  • Rate of Sale Override – A user override value for the strategy rate of sale.
  • Rate of Sale Variance – Percentage difference between the demand rate of sale and the actual rate of sale.

  • Re-Trend Method - The method to use for obtaining a retrend forecast for the allocation strategy.

    • Default - Uses the option set in the Default Forecast Retrend Method To Use system policy.
    • PID - Use the standard retrend calculation to create a retrend forecast.
    • Monte Carlo - Uses the Monte Carlo retrend method to create a retrend forecast. For more information, see Monte Carlo Re-Trend Overview.
    • Demand Forecast - Uses a generated inventory-level demand forecast as the retrend forecast. Requires the Inventory Forecasting module and an existing demand forecast. (Demand forecasts are not generated as part of Allocation execution.) For more information, see Inventory forecasting overview.
  • Rollup – The View button embedded within the Rollup field launches the Rollup (Strategy) window containing information on the highlighted strategy. This window presents a more narrow focus when compared to the Allocation Rollup screen.
  • Safety Stock – The minimum required quantity of stock to allocate. Sometimes referred to as presentation stock.

  • Sales Curve Basis – Determines the type of sales curve which will be used to shape the demand signal of the allocation strategy.

    • Last Year – Curve based on last year’s historical sales. This information comes from the Curves module.
    • Imported – Use an imported sales curve.
    • Imported Plan – Use an imported temporal plan.
    • Manual Selection – Allows for the manual selection of a sales curve as generated in the Curves Module. If selected, the curve must be specified within the sales curve selection field.
  • Sales Curve Selection – The curves available for manual selection. The selected curve will only be applied to the demand forecast if the sales plan basis is set to “manual selection”.
  • Sales Plan Name – A plan with a time period based strategy which is used to define the sales curve. Effective if Imported Plan is selected in the sales curve type field.
  • Sellthrough Target – This is an input which designates a planned sell-through percentage.
  • Setting Source – Whether Allocations or Assortment Planning is setting the source. The default is Allocation.

    If set to Assortment Planning, then the following are pushed from the Assortment Planning module to the Allocations module:

    What data is not pushed to Allocations

  • Setting Source Description – A description of the origin of where the data came from.
  • Site Count – Site count lists the number of sites (stores) contained within the location strategy.
  • Size Model Assignment – The selected size model for the strategy. A listing of available size models presents as part of a dropdown menu.

  • Strategy State – Determines how the strategy is managed and if it is visible to users.

    • Active – Default state. The strategy may be actively managed and allocations may be created using the strategy. Type 1.
    • Hidden – Strategies with no actionable stock may be set to hidden to ease navigation. Type 2.
    • Inactive – User driven status to deactivate a strategy. Type 3.
    • Deleted – System setting for inactive strategies which may be removed. Type 4.

  • Strategy State Management – Determines if users are able to action on the strategy state field.

    • Open – No restrictions (may be updated based on system policies and execution).
    • Manual – Only users are able to change the field.
  • Strategy Stock – The available stock count given the strategy parameters. Uses the available to allocate calculation and is inclusive of any stock availability restrictions which have been applied and saved.
  • Supply Chain – The supply chain used in the allocation strategy line. Allocation Strategy Selection determines the supply chain used, and each Strategy can have its default supply chain configured in the Configure Engine Chain view.
  • Total Sales Demand – Quantity of sales units defined as demand for strategies utilizing total sales quantity as the demand type.
  • Use Re-Trend – If selected, the allocation strategy recalculates the allocation strategy quantities using the re-trend calculator.
  • Variant Code - The unique code describing the variant.
  • Variant Description - The optional description for the variant.
  • Weeks of Cover – The number of weeks of demand for which stock is allocated and is an aggregation of days of cover.

  • Weeks of Supply – The effective replenishment cycle for this product. In other words, it is the effective length of time (in weeks) in which stock supply will be depleted to safety stock, before being restocked.

    Overrides the Allocation:Replenish Cycle system policy value.
