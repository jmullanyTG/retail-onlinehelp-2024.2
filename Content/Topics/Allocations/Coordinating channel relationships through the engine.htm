Coordinating channel relationships through the engine

The Allocations module does not coordinate channel relationships through the engine. It is imperative that this is resolved with the CLIENT’S PROCESS.

For example, Strategy One - Retail and Strategy Two - Outlet share DCs and/or SDCs. The last strategy to run the engine removes any OVERLAP inventory from the related strategies.

If Strategy One - Retail runs first and has 200 units on order for a related DC, Strategy Two - Outlet running second removes the 200 units from that related DC which can cause the entire order for Strategy One - Retail to not be realistic.

The client’s process must be that they run their highest priority channel first through the engine, then set it to a state that sets Pending Stock (i.e. pre-authorized). Then they can run the next priority channel, then set the Order Manager State. This process will be repeated until their channel priority list is exhausted.
