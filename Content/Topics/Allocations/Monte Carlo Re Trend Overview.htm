Monte Carlo Re-Trend Overview

The re-trending of forecasts is an important step to correctly manage the continued allocation of products as they transition from a pre-season plan to in-season. As products begin to sell, their sales patterns inevitably trend away from the initial plan (sometimes in the positive, sometimes in the negative). The included re-trend mechanisms in ToolsGroup JustEnough account for the trends in sales and form an amended go-forward sales plan against which continued allocations may be structured.

With the initial launch of the Allocations solution, a PID mechanism was included to provide a reactive result set based on potentially limited data. With the introduction of the Monte Carlo simulation re-trend mechanism, organizations now have additional tools at their disposal to provide flexibility when dealing with irregular sales patterns.

Using historical demand for an item, a distribution for the observed demand values can be calculated using Monte Carlo simulation.  Then a dynamic confidence level is calculated based on the variability of the item’s demand and the number of observations available.

Using the confidence level and the demand distribution, a total demand value is calculated and compared to the total forecast over the same time frame.

A re-trend factor is then calculated based on a desired demand satisfaction percentage.

Model inputs

There are seven inputs into the model:

Input Description
Min multiplication factor (min mf) Lower bound for re-trend factor
Max multiplication factor (max mf) Upper bound for re-trend factor
Evaluation window The max number of historic demand periods that will be used to calculate historic demand statistics
Forecast Period The number of forecast periods used to calculate total forecast value. If value < 0 past forecast periods are used. If value > 0 future forecast periods are used (relative to the current date). It’s recommended to keep value greater than 0 with a suggested default of 12 weeks/periods.
Min observations Items where total observations are less than minimum observations are penalized when calculating a final confidence level.

Needs to be greater than Evaluation window.

Values between 4 and 12 weeks are preferable. The recommended default is 6 weeks.
Number of Simulations The number of Monte Carlo simulations that will be run to generate a demand distribution. More simulationsgive a more continuous distribution but increase calculation time.

The recommended default value is 100 simulations (in testing 100 simulations have been sufficient for producing consistent results).
Desired demand satisfaction percentage The percentage of total demand that needs to be satisfied. For example, if total demand = 40, and Desired demand satisfaction percentage = 95% then total forecast needs to be equal to at least 40*0.95=38.

Higher values (90% and upwards) are recommended for more consistent results.

The recommended default value is 100%.

System policies

There are several system policies specific to PID. There are also re-trend system policies that are used for both re-trend methods, which can be found below:

  • Forecast Model - Maximum Forward Horizon Days
  • Sales Day For Selling Item
  • Allocations Minimum Retrend Multiplicative Factor
  • Allocations Maximum Retrend Multiplicative Factor
  • PID - Evaluation Window Size

Although Monte Carlo and PID are different re-trend methods, they both share system policies that help frame where to operate and what bounds to set. There are PID specific parameters (Kp, Ki and Kd) which are not used by Monte Carlo.

Monte Carlo receives its inputs from the Allocation Strategy as well as through system policies.
