Product models

Product models are built using historical data from groups or levels of similar products over a specified time period and support the ability to create indexing of stores to support allocations. When specific products for modeling are selected, the index is created not as an indication of overall store performance, but as a representation of how a product or set of products will perform across the stores.

Building product models based on hierarchy product levels

Allocators can build product models based on desired hierarchy levels. This allows product models to be optimized for desired business distributions. When new items begin allocation, a product model can first be built off higher-level hierarchies such as seasons and departments. As more sales history becomes available to the new items, the hierarchy can be lowered to variant levels to really reflect the item’s individual sales history.

This is vastly powerful as which hierarchy level to use for each product model can be chosen based on availability of data and desired accuracy.

We do not recommend setting the product model hierarchy lower than the allocation strategy hierarchy. For example, if allocations are at variant level, don’t set the product model hierarchy to item master level.

The item master level hierarchies that display in the Hierarchy column are defined using the Hierarchy Filtering view.

Model rule selection

There are two methods for model rule selection: fixed and set period rolling. A fixed period supplies start and end date which creates the fixed period. The set and rolling period rule specifies a set distance in the past (365 days in the past from today) plus a defined rolling period (+90 days). This creates the period as the next 90 days from exactly one year ago.

Product model calculation

By default, the product model calculation uses the MEDIAN to produce the average values.

System policy Product Model - Use Advanced Calculation is set to False by default. If set to True, then the calculation uses the MEAN.

Product model recalculations

Product models are prevented from recalculating if they meet one of the following criteria, the product model:

  • is in a deleted state

  • is NOT linked to an allocation strategy

  • was calculated in the last 36 hours

    The time duration is governed by system policy Product Model: Time Interval To Skip a Product Model Calculation (Module: Product Model).

Defining a product model

  1. On the Product Model screen, click New.
  2. Enter a unique product model name in the Product Model field.
  3. Provide a description in the Product Model Description field.
  4. Click the Model Time Frame Selection field and choose a Time Frame.
  5. Select the appropriate field and choose the model metric and model type.
  6. Optionally, select the appropriate checkbox to include inner zeros and lost sales.
  7. Click on the Sales Type field to select sales types.
  8. Click on the Product Selection field to choose the Item Master filter that specifies the items to use during the calculation.
  9. Optionally, click on the Default Product Selection field to choose an Item Master Filter to be used if the first item master filter does not produce values.
  10. If a location plan related model metric was selected, then select a location plan in the Location Plan field.

  11. Click Save.

    The contributions are automatically calculated and display in the bottom detail grid.
