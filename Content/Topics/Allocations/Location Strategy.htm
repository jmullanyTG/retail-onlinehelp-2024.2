Location strategies

Location strategies are groups of sites which are used to ensure that the products are allocated to the correct locations.

Location strategies may be created and applied to allocation strategies to define the location dimension of both the inventory cache and the allocation orders. They can be saved and re-applied to multiple strategies by multiple users. Location strategies may be based on hierarchical relationships or attributes (when available).

Strategies can be duplicated to reduce repetitive work by selecting the Duplicate checkbox and clicking Save.

The location strategy grid lists the location strategies.

The bottom grid displays the sites the logged-in user has access to based of the site assignment filter. Selected sites are denoted by a check mark in the Site Eligible checkbox.

Defining a location strategy

  1. Click New.
  2. Provide a unique name in the Location Strategy Name field.
  3. Click on the Site Filter field and select a filter in the Items window.
  4. Select Save to populate the bottom grid.

  5. In the bottom grid, select the checkbox in the Site Eligible column for each site that is to be part of the location strategy.

    Only selected sites are considered part of the strategy.

  6. Click the Save button in the bottom grid.

Location strategy fields and descriptions

  • Location Strategy Name – User supplied name for the location strategy.
  • Location Strategy Description – An optional description for the location strategy.

  • Site Filter – A filtered list of sites to be included in the location strategy.

    These are the sites to which inventory is distributed. It is not intended that the distribution centers be included in this list. Location strategies are populated based on the filter contents at the time of the strategy creation and do not automatically update based on filter content changes.

    All sites contained in the store selection must be of type “store” (type 1). No DCs (type 2) or “store/DC” (type 3) should be presented in the store selection.

    The Is Contained in Location Strategy? field is used in site filters to identify sites which are (or are not) included in location strategies.

  • Included Channels - Indicates the channels defined for the selected sites in the lower grid. For example, if at least one selected site is configured for Retail, then Retail displays in this field. If no selected sites were configured for eCom, then the eCom channel does not display in this field.
  • Referenced Strategy Count - Indicates how often the location strategy is used in the Allocation Strategy tab. The number that displays may not exactly match the number of occurrences in the Location Strategy column in the Allocation Strategy tab. This could be due to strategies not being visible or being tied to a group.
  • Included Site Count - The number of sites selected in the bottom grid.
  • Access Level – Access levels determine which users may use or edit the location strategy.
  • User Select – Editable if Shared was selected for Access Level. Displays the specific users for a shared access level location strategy.
  • Duplicate – If checked, the strategy is duplicated upon application. This allows users to easily build similar groups of stores.

Dynamic Store Eligibility does not work with product groups, causals, and total available quantity. When a threshold is set to 0 or NULL, it defaults to 0.0001 (not actually zero).
