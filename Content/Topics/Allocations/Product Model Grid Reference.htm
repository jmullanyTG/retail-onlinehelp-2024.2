Product Model screen fields and descriptions

Product models are built using calculation parameters and product based (item master) filters.

  • Product Model – The name for the product model.
  • Product Model Description – Optional description for the product model.
  • Hierarchy - An item master level hierarchy filter. Specifies which hierarchy to use.
  • Model Time Frame Selection – The applied time frame for model calculation. A time frame must be defined for each product model. Click in this field to select a time frame. For more information, see Selecting a time frame. refer to the Time Frames User Guide.
  • Model Metric – Defines the measure used to calculate the model. For more information, see Types of model metric.
  • Include Inner zeros – When selected inner zeros are retained versus being discarded during the calculations for Sales History.
  • Include Lost Sales – When selected Lost Sales Quantity is added to Sales History. Selecting this checkbox potentially prevents a situation where a store is out of stock and negatively affects the product model.

  • Model Type - Model Type may be static or dynamic.

    • Dynamic – calculates the contribution down to the Store and Level ID for the Hierarchy ID

    • Static – calculates the contribution only down to the store.

  • Sales Type – Included sales types (listed in sales history by type) for model creation. Clicking within the field and selecting the ellipsis displays a dialog that allows for the including and excluding of sales types. Right-click within the dialog's column header field to display a dropdown menu with additional options.

    For customers that use non-standard sales types in the Product Model grid, a column hidden by default provides an additional user friendly Sales Type description. To display, right-click in the column header fields in the Excluded and Included sections and select Column Chooser from the dropdown menu.

  • Product Selection – Product selection is built off of a product specific (item master) filter. These filters may be built using the Filter Manager to incorporate the desired items used to build the model.
  • Default Product Selection – A supplemental filter to ensure that product models do not have “zero” values for a store. Should be set to a high enough level of the product hierarchy to ensure that values are present, or all item masters.
  • Access Level – Access levels determine which users may use or edit the model.
  • User Select – Displays the specific users for a Shared access level product model.
  • Created – Revision history metric showing the creation date of the model.
  • Created By – User ID of the model creator.
  • Last Updated – Revision history metric showing the date of the most recent revision.
  • Updated By – User ID of the most recent editor.

Types of model metric

History Related Metrics

Uses Sales History by Type Data. The time frame should be in the past.

  • Units – Uses Sales Quantity Field
  • Revenue – Uses Total Revenue Field
  • Margin – Uses Total Revenue minus Total Cost Fields

Hybrid Plan Related Metrics

Uses Financial Plan or Snapshot Hybrid Detail Plan. The time frame should be in the future.

  • Plan Revenue – The fields are dependent on Sales Types and there are many permutations but consists of Sales related fields.
  • Plan Units –The fields are dependent on Sales Types and there are many permutations that use Sales Units related fields.
  • Plan Receipts – Uses Target Receipt Units Field.

Inventory Forecast Related Metrics

The time frame should be in the future.

  • Forecast Units – Uses Effective Forecast Field
  • Forecast Revenue – Uses Sales Value (Effective Forecast times Price Value) Fields
  • Forecast Margin – Uses Sales Value minus Cost Value (Effective Forecast times Cost Value) formula
