System policy settings

Several system policy vales must be set for the Allocations Module to function correctly. These policy values must be addressed as part of the business solution design process, or the IT solution design process.

R Engine: Call to get the URL

The use of the R Engine within the Compute Node Manager was deprecated with the call to get the URL reverting back to the previously existing system policy The RScript Service URL (Module: Allocation). The reason is that R Engine did not go through the Compute Node Manager. It goes through a separate dispatch broker call.

This system policy was previously hidden but is now visible.

Configuring when to skip Allocation Strategy Inventory Cache calculations

System policy Time Since Last Allocation Strategy Inventory Cache Calculation for Batch Exclusion can be used to set when to skip an Allocation Strategy Inventory Cache calculation if it occurred within the specified time limit. The system policy is configured with the desired number of hours. If the value is set to a negative number, the default time interval of 18 hours is used.
