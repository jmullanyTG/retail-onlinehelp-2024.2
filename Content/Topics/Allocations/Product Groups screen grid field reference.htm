Product Groups screen fields and descriptions

The Product Groups screen is used to build and configure product groups.

  • Group Name – The user defined name for the product group.
  • Group Description – Optional description for the product group.
  • Group Allocation Priority – Determines how the variants within the group are prioritized for allocations.
    • Lowest SOH – Products with the lowest stock on hand will be granted higher priority.
    • Highest SOH – Products with the highest stock on hand will be granted higher priority.
    • Sellthrough – Products with the highest sellthrough % will be granted higher priority.
    • Margin Rate – Products with the highest margin rate will be granted higher priority.
    • Aged Inventory – Products will be prioritized based on last receipt date. This is meant to grant higher priority to stagnant items.
    • High ROS – Products with the highest rate of sale will be granted higher priority.
    • Low ROS – Products with the lowest rate of sale will be granted higher priority.
  • Max SOH per Variant – Allows users to define a maximum quantity per variant. This will allow users to maintain color maximums when variant level is set to style-color.
  • Product Eligibility – The Item Master filter used to determine the Variants available for inclusion in a group. Check or uncheck the Include checkbox to include or exclude Variants.

  • Allowed Channels - Displays the channels that are or were previously included. Channels associated with a strategy continue to display even if the strategy is later removed.

    When users create Product Groups, the allowable channels is dependent on their assigned Rights. Therefore, if they are only allowed to work with Retail, then Allowed Channels defaults to Retail. If there are no restrictions to channels or sites, then the user has access to all of them.

  • Included Strategy Channels - The channels associated with the selected strategies in the bottom left grid.
  • Included Strategy Count - The number of strategies associated with a channel.
  • Access Level – Access levels determine which users may use or edit the group.
  • User Select – Displays the specific users for a Shared access level product group.
  • Strategy Access Level – Access levels determine which users may use the group within the Allocation Strategy screen. May be different from product group creation and modification permissions. 
  • Strategy User Select – Displays the specific users for a Shared access level product model.
  • Created – Revision history metric showing the creation date of the group.
  • Created By – User ID of the group creator.
  • Last Updated – Revision history metric showing the date of the most recent revision.
  • Updated By – User ID of the most recent editor.

Strategies grid

  • Include – If checked, the variant is included in the product group.
  • Allocate Strategy ID - The Allocate Strategy ID associated with a strategy within a Product Group.
  • Code – The variant name as specified in the item master group which has been defined as the variant level.
  • Description – Optional description of the variant.
  • Configured Channel - The available channels.
  • Effective Channel -
  • Location Strategy - The location strategy associated with a particular strategy within a Product Group.
  • Available SOH – Total stock on hand quantity across all DCs (will use the available to allocate calculation). Does not include placeholder stock.
  • Total Inbound – Sum of inbound orders to all DCs.
  • Actual ROS – Actualized rate of sale. Will display as “0” if item has not begun selling. Must be attached to an allocation strategy to populate values.
  • Sizes with SOH – Total count of sizes with a company-wide stock on hand greater than zero.
  • Max Size SOH – The maximum of stock on hand across all sizes with a stock on hand greater than zero.
