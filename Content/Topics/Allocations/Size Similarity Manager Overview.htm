Grouping same real sizes

It is not uncommon for the same real size to be present in a database under multiple names (for example, size small could be represented as S, SM, SML, etc.). This could be due to issues with the way data imports are set up or even problems with the master data being fed into ToolsGroup JustEnough. Regardless of the cause, these additional data points can cause unintended dilution when calculating and working with size models.

The Size Similarity Manager view allows users to identify sizes that should be grouped together as one, specify a single name under which to unify and display them, and ultimately work with them as a single entity without the need to manually unify the relevant data points.

The Size Similarity Manager view is accessed through the Views module – System Utilities > Views > Size Curves. Select Size Similarity Manager from the Navigation Menu.

Saving and restoring layouts

Changes made to the screen layout through use of the Fields button or pivot grid manipulation can be saved and quickly recalled later. Right-click to display a menu. Select Save Layout to launch the Save Layout dialog.

Enter a name for the layout, an optional Description, and a privacy option: Private, Public, or Shared (if Shared, click to define the users with which to share the layout).

To make the layout being saved the default system layout for the current view, check Default. Click Ok and the layout is saved for future recall. Layouts can be restored through a right-click menu option: Restore Layout.

Opening and exporting files

The ability to open and export files in various formats is available via a right-click menu option.

Open opens the data from the view directly in the application associated with the chosen format whereas Export requires the user to name and save the file and does not immediately open the data in the associated application.

Functionality and fields

Toolbar

New: Adds a new similarity set.

Delete: Deletes the selected similarity set.

This does not reverse any data migrations that have already been completed. Sets are prevented from being used for any future migrations or data imports. Deleting the set will also make any Similar Sizes that were displayed under the Display Size visible throughout ToolsGroup JustEnough once more.

Fields: Launches the Customization dialog to customize the fields shown and layout of the current view.

Save: Saves to the database. Changes cannot be automatically undone with once saved.

Cancel: Undoes all changes made in the current view and restores deleted similarity sets provided the deletion has not been saved.

Field Descriptions

  • Size Similarity ID: The unique system ID number associated with the similarity set.
  • Size Similarity Set Name: The name for the similarity set.
  • Size Type: The size type to define the similarity set for. This field populates with Size Types defined in the Allocation module.
  • Display Size: The size name under which all similar sizes in the similarity set display after migration has occurred. This field populates with sizes contained in the selected Size Type.
  • Similar Size: The sizes to be grouped together in the similarity set, populates with sizes contained in the selected Size Type.
  • Queue For Migration: Indicates that changes made to the similarity set have been saved and queued for migration in the next migration script run. Similarity sets that have been queued for migration but which contain sizes that have yet to be migrated are formatted in red.
  • Has Migrated: Whether migration has been completed for the similarity set.
  • Last Migrated Date: The date the similarity set was last migrated.
  • Last Migrated: Who last ran the migration script for the similarity set.
  • Create Date: The date the similarity set was created.
  • Create User: Who created the similarity set.
  • Last Modified Date: The date the similarity set was last modified.
  • Last Modified User: Who last ran the modified the similarity set.
