Allocation Rollups

The Allocation Rollup screen displays demand and stock projections at several different levels based on supplied parameters, including:

  • Product Hierarchy and Product Level
  • Location Hierarchy and Location Level
  • Time Window (historic, present and future)

To enhance performance, the maximum number of rows to be returned can be defined using system policy Allocation Rollup Maximum Rows Returned. The default is 500,000 rows.

The Allocation Rollup screen contains a pivot grid where allocations data can be customized and reports generated.

Behavior of product groups within Allocations Rollup

Although data from allocation strategy lines can be used into product groups that might not be at the same aggregation level, this data may not aggregate correctly. In this case, aggregation is done to the lowest common level. All common levels display the product group name instead of an aggregatable level. In the product groups, size is treated like an attribute instead of a product dimension level.

If an item filter contains items which are in a product group and some which are not in product groups, one row is displayed for the product group and the remaining items display the expected hierarchical value names for that item.
