Strategy relations and prepack relationships

The Strategy Relations blade gives visibility into the relationship, if any, between strategies and between strategies and any configured prepacks. The blade includes two sections: Strategy Relationship and Prepack Configuration.

If the same rainbow prepack items are a part of more than one Allocation Strategy, users can identify if a linked strategy needs to be calculated or not.

A nightly process associates a strategy to other strategies that are affiliated based on multi-SKU prepacks. Linked items are already grouped together (via a back-end process) in the Allocation run.

How thresholds are calculated to release multi-SKU prepacks

Thresholds originate with the Allocation Strategy itself or are determined by system policies. Two system policies linked to these calculations are:

  • Allocations CLR Prepack Run Mode
  • Allocations Maximum Prepack Overage Threshold

Strategy Relationships

Where omni-channel is allowed and there are multiple channels using the same strategy, then the Strategy Relationship grid lists all the channels using that strategy.

It is important to note that there is only one stock-on-hand for the multiple channels sharing the same Variant. Therefore, it is recommended to first run the strategy with the most important channel and verify the order. Once satisfied, repeat the process with the other strategies to avoid over allocation.

The information in this section can be used to ensure the prepack strategies are configured correctly to align within the Allocation run.

Prepack Configuration

The Prepack Configuration section lists all related prepacks linked to the selected Allocation strategy, based on ItemMasters linked to the Strategy. This provides a better understanding of what is within a prepack. This section identifies what ItemMasters are contained within a prepack.
