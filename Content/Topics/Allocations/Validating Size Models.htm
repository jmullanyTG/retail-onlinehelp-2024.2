Validating enhanced size models

Setting up pass or fail tests ensures each site had enough qualified data left over after data cleansing to calculate a valid size model. For more information, see Data Cleansing Thresholds.

Size model validations are for informational purposes only and do not affect whether a curve is used.

Thresholds

Validations are based on the following user-defined thresholds, which appear as columns in the Size Models grid.

These thresholds measure the data-cleansed variants, sales units, and sizes at a site within the item filter selected for the size model.

Threshold Description
# Variants Qualified The minimum number of variants required at a site for its size model to be valid.

After data cleansing, the curve validation fails for a site if the number of remaining variants at that site is less than this value.

% Variants Qualified The minimum percentage of variants at a site that must pass data cleansing for the size model to be valid.

After data cleansing, the curve validation fails for a site if the remaining percentage of variants at that site is less than this value.

Qualified Sales Units The minimum number of sales units required at a site for its size model to be valid.

After data cleansing, the curve validation fails for a site if the number of sales units at that site is less than this value.
# Sizes Under The maximum number of sizes allowed at a site whose contribution to the size model falls below the value defined in the % Sizes Under threshold.
% Sizes Under The % contribution value to test against when validating # Sizes Under. (A size's % contribution value is shown in the Size Model Detail pane's Contribution column.)

For example, if this value is set to 10 and # Sizes Under is set to 2, a validation checks the number of sizes with contribution values below 10%. If more than 2 sizes at a site have contribution values below 10%, then the validation fails for that site.

# Sizes Over The maximum number of sizes to allow at the site whose contribution to the size model is greater than the value defined in the % Sizes Over threshold.
% Sizes Over The % contribution value to test against when validating # Sizes Over. (A size's % contribution value is shown in the Size Model Detail pane's Contribution column.)

For example, if this value is set to 25, and # Sizes Over is set to 2, a validation checks the number of sizes with contribution values greater than 25%. If more than 2 sizes at a site have contribution values greater than 25%, then the validation fails for that site.

Not all of these thresholds need to be used to test size models, just the ones that make sense for the data. Also, validations per curve can be customized, with different sets of thresholds and values for each curve as desired.

If more than one threshold is needed to test a size model, then a curve must satisfy all thresholds to pass validation. Testing stops once any threshold test fails.

Once changes to a size model's thresholds are saved, the system automatically validates the site-specific curves against those thresholds and displays the results in the Size Model Detail pane.

Setting up validations for the enhanced size model

This section describes how to set up validations for enhanced size models.

  1. From the Execute tab, click Allocations to open the Allocation window.
  2. Click Size Models on the toolbar to open the Size Models grid.

  3. Find the row of the enhanced size model to be validated.

    To tell if a size model is enhanced or not, look at the curve's read-only Used Advanced Calc column. If the checkbox is marked, it is an enhanced size model.

  4. In the size model row, enter the desired threshold values in the threshold columns described in the table above.

    Any combination of thresholds can be entered. However, if defining Sizes Under or Sizes Over thresholds, enter values in both the # Sizes and % Sizes columns.

  5. Click Save to save the thresholds and execute the validations against the site-specific size models.

    Results are shown in the Size Model Detail pane. Size models are not automatically recalculated prior to validation. They are run against the current or existing size model.

  6. Repeat steps 3-5 for each enhanced size model to be validated.

Validation results

Select a size model in the top grid to view its validation results in the bottom grid.

The Pass? column indicates whether the size model for each site passed validation.

Value Description
The size model for this site passed validation.
The size model for this site failed validation.

In addition, the site's Total Contribution column value appears color: #ff4500; red. (This is a visual indicator only and does not reflect any issues with the total contribution value itself.)

When size models fail validation

The following options are available if a size model fails validation:

  • Manually overriding size models
  • Substituting size models
  • Use the size model as is.
