Creating a new filter

New filters are created using the Filters module and wherever filters can be selected in other modules. The process is the same except that an Entity type must be selected when creating filters in the Filters module.

When creating filters in other modules, the entity type is pre-defined based on the current context. For example, when creating a new filter in the Supplier module, the Entity type is pre-defined as Supplier.

Creating filters in the Filters module

  1. Open the Maintain Filtersdialog.
  2. Choose one of the following:
    • Create a New Filter, click Ok, and select the entity type.
    • Clone from Existing Filter , click Ok, and edit the filter details, criteria, and conditions.
  3. Click Ok to save the new filter.

Creating filters in other modules

Filters can be created in other modules without having to switch to the Filters screen.

  1. Click the Maintain () button next to the filter list.

    The Maintain Filters window displays.

  2. Choose one of the following:
    • Create a New Filter, click Ok, and select the entity type.
    • Clone from Existing Filter , click Ok, and edit the filter details, criteria, and conditions.
  3. Click Ok to save the new filter.

About cloned filters

Cloned filters have the default name: Cloned From: <cloned filter name>. Edit this name as desired. Cloned filters inherit the entity type of the filter they are cloned from.

About entity types

Filters are distinguished by the type of entity they support (inventory, site, supplier, etc.). Because different entity types are used for different functionality and analysis throughout the system, the filters that support this functionality and analysis are also based on entity type.

When selecting filters, only the filters created for the applicable entity type are available for selection.

When selecting a site filter in the Site Manager screen, only site filters are listed and not inventory filters, supplier filters, etc.

The entity type of the filter can be manually selected when creating filters in the Filters screen. In all other locations, the entity type is pre-defined based on the context of the location.

For example, the entity type of a new filter is pre-defined as Site for filters created in a site filter selection screen.

The following filter entity types are available:

Entity Type Description
Inventory Item-site combination. A common filter that will drill down to the lowest level in most instances.
ItemMaster The item across all sites.
Site Any location that is capable of holding inventory.
Supplier Outside entities that supply the business with inventory.
Customer Any outside entity that is buying or receiving inventory.
Promotion Promotions as created and maintained in the Promotions solution.
KeyItem Deprecated.
Event Events as created and maintained in the Events solution.
KeyItem Variant Deprecated.
File Used for repository files. Not to be confused with assembly files used to operate the ToolsGroup JustEnough client.
ItemMaster Group Used in Content Manager to filter ItemMaster groups.
CustomerInventory Do not use.
Variant Used in Allocations and Assortment Planning.
Variant Group Used in Content Manager to filter variant groups.
Variant Placeholder Used in Assortment Planning.
Variant Site Used in the Price Manager module.
Variant Customer Used in the Variant Customer Manager module.
Causal Used in the Causals module. This allows users to create a filter using Causal metadata, including Causal types:
  • Causal
  • Event
  • Information Causal
  • Manual Causal

Rebuilding filter caches

When a new filter is created, its cache is built immediately.

In most instances, rebuilding filter caches is an extremely important task and should be run on a daily basis. This can take a considerable amount of time during the overnight run depending on the volume of data and the number of filters that need to be rebuilt.

Custom code must no longer reference tbl_FilterCache because population of tbl_FilterCache is optional when filters are rebuilt.
