Filters Overview

Filters display only the subsets of data to be worked on in a given screen (or that the system works with in a given process). Filters can be as simple or as complex as needed and then saved for future use. Once created, they can be shared across the entire organization, with select others, or kept for private use.  

Flexibility

Filters are extremely flexible and are able to support:

  • Static data: the data the filter displays never changes ensuring the same data displays whenever the filter is used.
  • Static conditions: the conditions used to filter data cannot change, but the data displayed can change ensuring that the data is current. 
  • Dynamic conditions: users select the condition values used to filter data ensuring the filter is relevant to their current needs.

Entity-specific filtering

Because different modules involve different data entities (such as suppliers, inventory, and sites), the available filters apply to the relevant entity.

In the Suppliers screen, there is a list of available supplier filters as opposed to a mix of supplier, inventory, and site filters.

Creating filter groups

Filter groups make it easier to organize and find filters. To create a new filter group, click the three dots next to the Filter Group dropdown menu.

Filter criteria and conditions

Filter criteria and conditions work together. Filter conditions determine the field values that a filter searches for, and filter criteria determine whether an entity that meets those conditions is included or excluded as a result. When managing filters with multiple conditions, it is important to pay attention to the order in which the conditions are listed.

Filter criteria

Filter criteria can be one of the following, where:

  • all of the following must be met.
  • any of the following must be met.
  • none of the following must be met.

Filter conditions

Adding or editing filter conditions is done through the Filter Condition dialog.

Filter conditions involve the following settings:

Allow dynamic condition

To select the condition value for a filter, select Allow dynamic condition in the Filter Condition window. For example, a site filter with a dynamic Site Region value condition displays the option of regions to work on at the time the filter is selected. This can be much more convenient than creating separate filters for each region.

Filter conditions and data implications

The following should be considered when creating filters to determine how and what data displays.

Should the entire filter be static?

A static filter always displays the same data set. Select Filter is Static in the filter details to make a filter static.

A filter is created in January 2024 at the end of the planning period Block 1 for the class Ladies Shirts. The data set displayed by this filter contains all variants in Ladies Shirts at the end of Block 1. This data set does not change after the filter is saved.

What conditions should be static?

Static filter conditions do not change when the filter is used. However, the actual data displayed by the condition can change as time goes on.

To make a condition static, add the condition and do not select Allow dynamic condition.

A filter is created to display all variants for the class, Ladies Shirts. A condition is added (class value = Ladies Shirts). As new styles are created for the class Ladies Shirts, the data set displayed by the filter is updated to include the new styles.

What conditions should be dynamic?

Conditions that are dynamic can be changed when the filter is selected. The data displayed by the filter is determined by the user's selection.

A filter is created with a dynamic condition for Class. When using this filter, the class to filter on is chosen. For example, the class Ladies Shirts displays all styles of ladies shirts and the class Mens Shirts displays all styles of mens shirts.

To make a condition dynamic, add the condition and select Allow dynamic condition.

Filters are color coded so that when selecting a filter users know which filters are static and which have dynamic conditions. For more information, see Selecting a filter.

Filter with multiple criteria and conditions

Filters can have multiple criteria and multiple conditions. The criteria display in a tree format to show the order in which each criteria and condition is applied by the filter. The filter starts at the top of the list and works its way down.
