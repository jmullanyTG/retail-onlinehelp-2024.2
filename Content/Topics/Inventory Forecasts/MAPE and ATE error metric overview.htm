MAPE and ATE error metric overview

Error metrics can be used to determine the winner of a forecast tournament.

Mean Absolute Percentage Error (MAPE)

Historically, the default error metric has been Uniform MAPE. This is a good metric for measuring the success of each individual point of the forecast, but analysis has found that it is less than optimal at maximizing the statistics most customers actually desire. Generally, what is most important to customers is a forecast which minimizes the total error, summed across an order period. To this end, the Absolute Total Error (ATE) metric is preferred.

Absolute Total Error (ATE)

Absolute Total Error (ATE) is based on a simple summation of the errors between the forecast and the actual history. Thus, errors in opposite directions in neighboring periods (overestimating one week and underestimating the next) cancel out. There are numerous other case examples of how ATE outperforms MAPE, outlined below. The summation is transformed into a percentage, and then into a model fit, which is presented to the user. This mirrors the existing interface for using and displaying MAPE error.

The application's data science team recommends the usage of ATE in all scenarios (see below). That said, it is a semi-experimental new design choice, with far-reaching ramifications, so caution is urged as well. Inform ToolsGroup JustEnough of any problem situations that arise with the new forecasting metric.

Functionality

ATE forecasting can be enabled for all inventories by changing system policy Forecast Error Method (Engine: Forecasts) to ATE.

At present there is no way to use MAPE on some forecasts and ATE on others without manually switching the system policy for each forecast.

Differences and notes

  1. ATE will often prefer smoother, flatter forecasts than MAPE, especially in high noise histories (this is desirable).
  2. ATE forecasts tend to be somewhat less responsive to sudden changes then MAPE (this can be an advantage or disadvantage).
  3. As with MAPE, model fit can be compared between products to determine where problems are occurring.
  4. Due to how the mathematics work out, model fit for ATE will be higher across the board than model fit for MAPE. For this reason, new limits on what is considered a ‘problem’ may need to be considered, and model fits cannot be meaningfully compared between ATE forecasts and MAPE forecasts.

Usage Recommendation

The data science team has concluded that ATE produces comparable or superior forecasts to MAPE in nearly all scenarios considered, including long term products, high sales products, low sales and intermittent sales products, highly seasonal products and products with strong trends. In particular, it was found that intermittent forecasts were vastly better when ATE was used. For this reason, it is recommended to use ATE for all inventories. The only exception found, where MAPE was significantly better than ATE, was on inventories whose behavior changed suddenly in a non-predictable, non-seasonal manner, particularly when such changes happened to line up with the parsimony cutoff in the history (see Examples below for more details). However, this is considered to be an unlikely edge case, compared to the across-the-board superiority of ATE elsewhere.

Math

Given a history H→, a certain subset of the most recent history (where possible, a full year) is classified as ‘parsimony.’ Models are trained on the non-parsimony section of the history, then forecasted against the parsimony. The errors between the forecasts of various models and the actual history in the parsimony region are compared, and the model/parameter set with the lowest error is selected. This model is then retrained using the entire history, and that trained model is used for the final forecast. The model fit is calculated from the error in the parsimony region as follows:

Which forecast is actually preferable in this scenario likely varies by customer and by product. However, it is important to note that, at present, no ToolsGroup JustEnough forecast method will attempt to change the phase (i.e. period) of a spike. Whether there is a spike at all depends on whether a given method is seasonal, includes causals etc., but if there is a spike, no method will outperform any others at lining it up.
