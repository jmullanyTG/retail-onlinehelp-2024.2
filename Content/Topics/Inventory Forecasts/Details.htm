Viewing detailed forecast information

To scrutinize the forecast beyond the method fit, click the Details button to display detailed forecast information.

Forecast section

The forecast information displays information only related to the forecast method currently applied.

  • History Start Date – The first (oldest) time period of demand history data for the inventory item used by the forecasting engine. Default is approximately two years previous to the forecast start date.
  • Forecast Start Date – The date that the forecast tournament was last run on the item or the date of the last saved forecast.
  • Accuracy – This field is a placeholder for configurable accuracy measurement displays. It does not populate by default.
  • Lock Expiration – The expiration date for a lock if the forecast methods have been locked.
  • Forecast Message – The reason a forecast method was disqualified from a tournament in addition to the forecast method dropdown list.

Details section

The details section lists information related to the inventory item.

  • Seasonality – Seasonality (different from Seasonality Constant) is a value assigned to each item to specify the number of consecutive buckets to consider when determining seasonality trends.  If it is set to 12, a seasonal trend only qualifies if it repeats itself in two sets of consecutive 12-bucket periods.
  • Creation Date – The date on which the inventory item was added to the database.

  • Supersession – Displays the details of any supersessions linked to the item.

    For more information, see Supersession overview.

    For more information, refer to the Supersessions User Guide.

Settings section

This section displays some of the information from the System Policy module which will affect the forecasts.

  • Backtrack/Simulation Ratio – The ratio for weighting the optimistic backtrack to the pessimistic simulation during the forecast engine run. It is set to 0.5 by default (50%) and is considered best practice to keep it there.
    • 0 = Only consider simulation
    • 0.5 = Equal split
    • 1 = Only consider backtrack
  • Prevent Negative Forecast – A system policy (Disallow Negative Forecast Data) parameter setting which prevents or allows Negative Forecast values to be calculated and displayed. It is set by default to True and it is considered best practice to keep it that way.
  • Error Method – The method by which the forecast engine grades the tournament winners. It is set by default to Uniform MAPE and it is considered best practice to keep it that way.
