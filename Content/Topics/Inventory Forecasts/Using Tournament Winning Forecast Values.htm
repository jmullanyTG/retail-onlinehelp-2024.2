Using tournament winning forecast values to build MFP plans

Users can generate forecasts based on product level history and projections and not tied to specific skus/locations. In addition to LY, LLY or a defined comparison time period, planners now have visibility to a sales projection with a very high level of accuracy. The forecast results are the winning forecast method from the forecast tournament with AI and gradient boosting included in that forecast tournament.

MFP forecasting creates a forecast at the lowest MFP hierarchy level. Meaning, for an MFP plan that includes Mens > Mens Tops > Mens Tees > Mens Graphic Tees, with graphic tees being the lowest product level, then the forecast runs using the graphic tees grouping as the forecast source.

Lowest hierarchy level applies to Channel as well. For example, if the Channel hierarchy is Total > Retail > West, then the forecast is performed at the Channel level and not by individual sites within that Channel.

Forecasted values can be seen within MFP with the following existing metrics: Forecast Sales and Forecast Sales Units. Use the Fields Customization functionality to display these fields.

Forecasts are future looking. As such, LY Forecast Sales and LY Forecast Sales Units only populate for future periods. For example, if there is a forecast for 2024, then LY Forecast will populate for the same period in 2025.

Running forecasts for a plan or for all plans

Forecasts can be generated at a financial plan level for a particular plan or for all plans. These forecasts do not go down to the inventory level. Two standard job tasks are available:

  • (MFP) (Per MFP Plan) Merchandise Financial Planning Forecast Task can be imported to run a forecast task for a single MFP master plan.

  • (MFP) (All MFP Plan) Merchandise Financial Planning Forecast Task can be imported and run immediately to generate a forecast, with the same forecast parameters, for all MFP master plans.

Tasks to generate forecasts for MFP plans are built using a new type of forecast type called Merchandise Financial Plan Forecast Engine Task.

To generate forecasts for particular plans or to run a forecast for all plans, click Add to create a new task and select the appropriate option from the dropdown menu in the MFP Plan field on the Forecast Task dialog or click Edit next to the Engine Task field to update an existing task.

MFP forecast frequency

The MFP forecast frequency can be configured to run on a schedule such as weekly or monthly using two job manager tasks. These tasks need to be configured so that TG JE's implementation of Gradient Boosting can analyze MFP plan level data.
