Inventory forecasting

At its core, statistical forecasting is a process that displays a result containing some degree of error. Mitigating these errors and producing highly accurate forecasts is the main focus of the Tournament forecasting system. Understanding that there is always some degree of error in any projection gives users a realistic expectation for extracting useful information from the inventory forecasts.

Users create, view, and manage forecasts using the Inventory Forecast module.

Filters in the Forecast view

An inventory forecast generates the forecast engine at the inventory level (item at site, or SKU/location). After specific items are selected, the Forecast view displays all of the inventories for the selected filter.

A static set of items can be viewed based on certain item or site assigned filters (or any others), or dynamic exception filters used to identify exceptions which may require attention. These filters must be built using the Filters module.

In the Item field above the toolbar, the entire filter displays as a sequence of items (e.g. Item 1 of 100) and is navigated by clicking the dropdown arrow or sequentially using the navigation arrows to the right.

Grid view

The grid displays the actual data values for the forecast, demand and other series displayed in the graph. As such, the descriptions of the series are identical to those for the graph.

Use the grid view to work with the forecast series using specific values. Manual forecast values can be entered for the current period and any future periods. These changes affect the graph.

Clicking on the values in the grid highlights the associated series in the graph. Any series represented by a line becomes thicker and the bar graph series show additional shading.

The data displayed on the graph view is color coded to match the corresponding data series in the grid view.  The individual data series can be selected to be displayed on the graph by checking the Visible checkbox for the specific data series on the grid view.

Series Description
Core
Total Sales Historical demand from sales history. Matches the imported history totals of sales for the inventory item.
Total Lost Sales When selecting forecast methods, lost sales totals are represented by the purple bar if users elect to include lost sales in their calculations.
Effective Total The effective total for demand history. Determines the input to the forecasting engine when determining historical demand. The Effective Total series usually runs along the top of the total sales bar graph and total lost sales bar graph (if lost sales have been included).
Manual Forecast The manual forecast series is represented as a red or burgundy dot.

The manual forecast series overlaps the effective forecast series and only diverges upon manual manipulation of the series.

Manipulation is by either by dragging the dot in the graph or the addition of a value to the manual forecast row in the grid. Once changes are made, the dot changes to burgundy to differentiate the series from the effective forecast which was displayed upon entering the viewer.
Manual Level Forecast The Manual Level Forecast is represented by red dotted lines. The Manual Level Forecast functions the same as Manual Forecasts and is used to manually override Level Forecasts. This series is usually un-editable in inventory forecasts and is only used in Level Forecasts.
Effective Forecast The approved forecast which will be referenced throughout other parts of the application. The effective forecast series changes once Save has been clicked if manual adjustments are made.
Sales
Sales The sales history for the period.
Lost Sales The lost sales calculated for the period.
LY Total Sales The total sales for this period last year.
Sales Planning  
Supersession Lost Sales The lost sales for any superseded item.
Supersession Sales The sales history for the period for a superseded item.
Forecast
Original Forecast When selecting between different forecast models, the yellow bar graph shows the resultant winner of the forecast tournament. This can be a useful comparison tool when choosing a forecast method.
System Forecast The system forecast series represents the output from the forecasting engine.
Level Forecast The level forecast represents forecasting engine output for a forecast level. This would be 0 for inventory forecasts and is only used in Level Forecasts.
Profile Forecast The profile forecasts shows the applied profile for the forecast if a profile is applied. This field would be hidden otherwise.
Allocate Forecast  
Retrend Forecast Calculated for strategy lines whose Use Re-Trend column is checked.
Level/Inventory Causals
Causal Name The name of each linked causal appears as a data series.
Additional  
Forecast Cost Calculation: Forecasted units * cost of the unit in the system currency
Forecast Retail Calculation: Forecasted units * retail of the unit in the system currency
