Split modes

Higher level forecast types have multiple selectable split methods which can be applied by type through system polices.

  • Aggregate Item Sales – Aggregated inventory sales history (this may be set by inventory history or sales history by type, selectable through a system policy).
  • Aggregate Site Sales (Applies to Aggregate Forecasting only) – Aggregated Sales at all sites.
  • Temporal Forecast – Split using the forecast that varies by time period.
  • Custom – Custom aggregate factors provided by end users or through a custom procedure.

A default split type for all forecast types will be set as a system policy, and each forecast type may be assigned a specific split type beyond the default.

Aggregate Item Sales and Aggregate Site Sales are cached (usp_Forecast_Split_Calculate) whenever possible. In Group Forecasting, because a group forecast can vary depending upon which sales types a model uses, the factors are calculated real time. 

For Level, Aggregate, and Profile forecasts, the system expects cached entries. If no cached entries are found, a temporal forecast is used to fill in missing factors.
