Using machine learning to improve forecasting

The Gradient Boosting Forecasting Method is based on machine learning. The machine learning algorithm uses Gradient Boosted Decision Trees to analyze all relevant historical data (known as features) relating to demand patterns (e.g. seasonality, brands), business decisions (e.g. Promotions, changes in display and space), and external factors (e.g. holidays, footfall, weather). The historical data is then translated into an extremely accurate prediction or forecast.

One major feature of Gradient Boosting is that it outperforms the statistical systems on sparse data. For example, items with low sales are associated with other, similar items with low sales, and historic examples of items with low sales. Subtle or obscured connections between features can be uncovered leading to more accurate forecasts and improved sales. Additionally, machine learning requires fewer resources while leveraging all data for each series, is easy to scale, and can result in significantly improved forecasting at aggregate levels.

Deployment

ToolsGroup JustEnough's implementation of Gradient Boosting is designed for Cloud deployments only. With this Cloud implementation, there is the ability to horizontally scale to shorten calculation times.

The Forecast Method Gradient Boosting displays only for Cloud deployments and is for Inventory Forecast only since it is at an item site level.

High level process flow

The Trainer API generates fit models. The Inference API uses the fit models generated by the Trainer API to make a forecast. When users run an inventory forecast, the Forecast API gets the inference predictions and tournaments them against other statistical forecast methods to determine the best fit. The output forecast method is called Gradient Boosting in the Forecast viewer.

Analysis Installer

Gradient Boosting requires the Analysis Installer. This installer is required to install the Analysis API and is required to be run on its own compute node. These compute nodes can be configured remotely using dialog windows in which users configure the audience, instance name, and the base application server address. This removes the need to follow a manual configuration process for each compute node reducing the chances of errors.

Job Manager tasks

Two Job Manager tasks need to be configured to run the machine learning.

The Training and Inferencing Job Tasks clear previous run data to prevent stale data runs.

Task Description
(DF) (Monthly) Gradient Boosting Run Train Process Sets up the data that is necessary to run the Gradient Boosting Run Inference Process task. Should be run once a month or quarter.
(DF) (Weekly) Gradient Boosting Run Inference Process When run, clears the previous data. Should be run weekly.

Calculations are performed on weekly buckets for performance and accuracy reasons and thus running this task more than once per week will not yield different results. For most accurate and timely forecasts, check which day of the week buckets end. Inferencing run after that day will forecast the coming (or now current) week.

Execute the Gradient Boosting Run Inference Process task after the Gradient Boosting Run Train Process task. Otherwise, the Inference task will fail if it does not find the fit model necessary to run.
