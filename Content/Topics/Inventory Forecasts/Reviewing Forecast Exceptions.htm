Reviewing forecast exceptions

The process for reviewing forecast exceptions is straightforward. For each forecast, determine if the forecast needs to be altered. If so, pick a method to alter the forecast and regenerate the forecast. If the revised forecast is acceptable, mark it as reviewed and continue to the next forecast in need of review.

Reviewing forecasts

  1. From the Execute tab, click Inventory Forecast.

  2. Choose the level or inventory filter that corresponds to the exception types to be reviewed.

  3. A forecast document opens containing the items with forecasts that match the filtered exception type.

  4. Review the forecast and demand history to determine whether the currently displayed forecast is acceptable or if it should be altered.

    • If altering the forecast, it is best to do so iteratively (make one change and then regenerate the forecast to review its effect before making any further changes).
    • Forecasts can be regenerated manually by clicking Forecast (or Save and Forecast) on the forecast toolbar.
  5. When the forecast is ready, click Reviewed on the forecast toolbar to remove the forecast from the current exception filter.

Altering forecasts

The following methods are commonly used to alter forecasts. The specific method that works best for any given forecast may depend on the exception type and the unique circumstances of the item.

  • Selecting a different forecast method
  • Adjusting the lost sales quantities.
  • Using an item supersession.
  • Managing seasonal trends.
  • Creating and deleting causals
  • Creating and deleting causals (refer to the Causals User Guide)
  • Altering store-level split.
  • Manually adjusting a forecast
