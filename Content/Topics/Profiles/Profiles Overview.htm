Profiles Overview

The Profiles module is used to simulate effective sales history for new items. This is done where sales history or a supersession are not used. Curves are created based on sets of familiar items with well established product life cycles. These curves are then applied to new items.

Profiles are managed in a hierarchical manner with profile sets acting as the container for individual profiles.

In the Profiles module there are two main workflow screens: the Profile Management screen and the Curve Management screen. The Curve Management screen is identical to the full Curves module.

For more information on Curve Management, see Curves overview. the Curves User Guide.
