Causal Manager Overview

Causal Manager mirrors a lot of the Inventory Manager functionality for both flexibility and consistency.

To view causals, select them from the dropdown menu near the top of the screen, or select individually using the navigation arrows in the upper right side of the screen.

Select the Configuration button to display the historical and future periods (buckets), which are visible in the Series Value viewer.

Causals

Causals indicate when specific periods of increased demand for items should not be used to forecast normal demand. Typically, demand increases when a promotion is run on an item. However, if the forecasting engine is not configured to indicate that a promotion ran during the period of increased demand, the increase is treated as an increase in normal demand and higher future demand is forecast for the item accordingly.

Causals prevent this from happening by indicating that the increased demand was caused by a promotion. Consequently, the increased demand is not treated as a regular increase when the forecast is created for that item.

There are four types of causals:

  • (Normal) Causal
  • Event
  • Information Causal
  • Manual Causal

For descriptions of each causal type, see Causals screen fields and descriptions.

Causal information is stored in the database in the tbl_Causal namespace.
