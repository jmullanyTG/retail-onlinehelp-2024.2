Linking causals using Causal Manager blades

Causals are created based on supplied weather lift factor per product segment. Inventories can then be linked to these product segments allowing the weather lift factors in the causals to be considered as part of the inventories demand forecast.

Inventory Links

Click Add to display the Items filter dialog. The grid populates once a filter is selected. Multiple rows can be selected within the Inventory grid which displays the inventory links for the selected inventories.

If the items do not display in the grid, items with a mismatched bucket type to that of the causal have been selected.

Clicking View opens an Inventory Manager Document with more details. Within Inventory Manager, the selection from the Causal Manager module can be viewed by navigating to the Forecast Viewer blade. The Forecast Viewer displays the inventory links and forecasts.

To remove items in a filter from a causal, select the items and click the Remove button.

Item Master Links

The Item Master Link links the causal to an item master and applies it across all sites.

If the items do not display in the grid, the items have a mismatched bucket type to that of the causal.

For more information on a specific item, select the item and click View to open the Inventory Manager Document.

Filter links

The most common way to apply causals is to link them to a filter. These filters must be created using the Filter module, and then linked in the Causal Manager module. Inventory, item master, site, and variant filters can be linked to a causal.

Selecting a filter links all items contained within a filter to the specific causal. This is especially helpful with dynamic filters, as the items contained within the filter can change over time.

If the items are not appearing in the grid, the items have a mismatched bucket type to that of the causal.

  1. Navigate to the Filter Links tab.
  2. Click Add.
  3. Select the appropriate Filter Entity Type.

  4. Click Ok.

Inventory, item master, site, and variant entity filters can be linked with the actual causal link applying at the constituent inventory level.

Selected items from the filter can be removed from the causal link by highlighting them and pressing the Remove button.

For more information on a specific item, highlight the item and click View to open the Inventory Manager document.

Level Links

With Level Links, causals are applied to a value in the pre-configured level of the product and location hierarchies, as specified in Level Forecasting.

The values are limited to those contained in the configured level.

Signal Subscriptions

The Signal Subscriptions blade displays the available External Signal subscriptions and the current subscription status. This blade is primarily informational.

Requirements to view the Signal Subscriptions blade

The following is required to be able to view this blade:

  • Right (Right Group: External Signals) Allow Access to Signal Subscriptions

  • System policy (Module: External Signals) Display Signal Subscriptions Tab in Causal Manager? (set to True)

Field definitions

Field Definitions
Signal Source The name of the signal subscription.
Description The nature of the data being supplied through the subscription.
Subscribed Whether currently subscribed.
Manage A URL link that when clicked opens a browser to a page that allows the user to request sales information for the subscription.

Weather Signals

The Weather Signals blade is specific to the Planalytics weather trends subscription. Only the Segments that the customer has a subscription for display. The blade displays the status of product Segments and Markets for the subscription. It also allows the administrator of the subscription to set which subscribed Segments and Markets will be used to generate causals.

Selecting a segment in the left grid displays the markets available to it in the right grid. Checking or unchecking the Create Causal checkbox determines whether Causals for the segments and markets are created in response to the incoming data.

Checking or unchecking Create Causal in the segment grid automatically selects all the segment’s available markets. Particular markets can then be unselected if Causals are not wanted for them. Unchecking the Segment or Market checkboxes prevents future causals from being created for those Segment/Market combinations.

The bottom graph displays Planalytics weather trend data for the currently selected product Segment and Market for the next 14 days.

Requirements to view or edit the Weather Signals blade

The following is required to be able to view or edit this blade:

  • Right (Right Group: External Signals) Allow Access to Weather Signals

  • Right (Right Group: External Signals) Allow Editing of Weather Signals

  • System policy (Module: External Signals) Display Weather Signals Tab in Causal Manager? (set to True)

Segment grid field definitions

Field Definitions
Segment The name of each product segment.
Create Causal A checkbox that indicates if a causal is being created for the Segment. The default is checked (a causal will be created).
Last Modified Date The date of the last change to the Segment.

Market grid field definitions

Field Definitions
Market The name of each Market.
Create Causal A checkbox that indicates if a causal is being created for the Market. The default is checked (a causal will be created).
Last Modified By The name of the last user to make a change to the Market setting.
Last Modified Date The date of the last change to the Market.

Planalytics Variant Segment Link

Variants are subscribed to Segments using the Planalytics Variant Segment Link blade. To populate the grid, select a Variant Filter and click Execute. Segments can be added or updated using the dropdown in the Segment field. Select a blank Segment to clear the field. Only subscribed segments assigned to the logged-in user display and a segment cannot be removed once added.

Causals only affect the Variants that are assigned to a segment.

The Segment data is provided by Planalytics.
