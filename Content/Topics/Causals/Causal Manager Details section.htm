Reviewing a causal’s detailed information

Although causals are typically managed by filters, they can be linked to an inventory and/or an item master object. The Causal Details tab displays the causal over time. What these values mean varies depending on the causal type. They may be positive or negative and represented in units or a percent lift factor. The series value view scales on the y-axis depending on the values present in the grid.

Click Choose on the toolbar to add the following metadata to quickly see how a causal is linked to the inventory records:

  • Has Filter Links

  • Has Hierarchy Level Links

  • Has Inventory Links

  • Has ItemMaster Links

Navigate to the relevant tab for detailed information.

For more information on available metadata, see External Signal related metadata.

Causal Details tab

Selecting a causal in the top grid refreshes the Causal Details tab with the details of the selected causal.

Header level information related to the causal including the name and type displays above the grid.

Name – The user created name for the causal.

Type – The type of causal. Understanding the distinctions between the causal types and their correct applications is key to properly using Causal Manager.

  • Causal – An additive causal that adds or subtracts based on a non-explicit scoring factor (the default value). The values typically relate to an increase or decrease in the overall volume of units. Negative values can be entered.

  • Event – Used to flag a period that is associated with an event. This is useful if there is an obvious jump in the data series, but the cause of which is unknown. The forecasting engine matches it to the best fit.

    Series values are used as a unique identifier. Any number or letter can be used. Any events that span multiple periods must have a different unique identifier per period, if the desired effect is to completely remove the event’s effect. If the desired effect is to use the event across multiple future periods, any events with the same unique identifier will average the effect for the future periods.

  • Information Causal – For informational purposes only. This type of causal is rarely used.

  • Manual Causal – A multiplicative causal that adjusts up or down based on an explicit factor, such as a 10% increase or a 20% decrease.

  • Bucket Type – The type of container period for the series values of the associated causal. This must match the bucket types of the eventual linked items/inventories.

Grid values

Defines the nominal value for the causal set. In almost all cases, this should be set to 0. Applying a non-zero to the default value elevates (or decreases) the nominal value to the entire history of the linked items.
