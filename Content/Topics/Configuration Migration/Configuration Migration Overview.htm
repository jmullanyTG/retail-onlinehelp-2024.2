Configuration Migration Utility Overview

The Configuration Migration Utility helps automate migration of selected objects from a source system (such as a QA system) to a destination system (such as a production system). Through the migration process, configuration objects are exported from the source system to a file. That file can then be imported to the destination system, eliminating the opportunity for manual transcription errors.

Configuration Migration user right

Users require user right Access to Configuration Migration.

Supported configuration objects

The following configuration objects can be exported and imported through Configuration Migration Utility:

  • Promotion strategies, including:

    • User restrictions from specific strategies
    • Rounding rules (standard)
    • Rounding rules (price ending)
    • Calculations
    • Calculation sets
    • Promotion pricing details
    • Promotion validations
    • Promotion validation sets

Prerequisites

  • All customized database fields must be created in the target environment.
  • All stored procedures must be copied over manually to the target environment.
  • All view collection sets, views, and widgets must be copied over manually to the target environment.
  • The lower level environment must be a copy of the destination environment to ensure the configuration elements' UUID are aligned.
  • The lower level environment must be on the same version.

Toolbar functionality

Button Description
Import Imports a configuration file.
Export Exports selected objects to an export file.
Log / Details Toggles displaying log messages and configuration objects.

  • When configuration objects are displayed, click Log to view the log messages related to this system's imports and exports.

    Import and Export buttons are disabled when the log is displayed.

  • When the log is displayed, click Details to return to the configuration objects screen.

Exporting configuration objects from a source system

  1. In the Configuration screen, select the objects to be migrated to a destination system.

  2. Click Export on the Configuration toolbar.

  3. In the Save As dialog box, enter a name and directory location for the configuration file.

  4. Click Ok to begin the export.

    View a log of the export by clicking Log on the Configuration toolbar.

Importing configuration objects into a destination system

  1. In the Configuration screen, click Import.

  2. In the Open dialog box, select the configuration file to import.

  3. Click Open.

    If the configuration file is valid, the Confirm Changes dialog box opens displaying the list of objects that will be updated as a result of the import.

  4. Click Import to update the system with the changes or Cancel to cancel the import.

Viewing the log

The configuration migration log contains messages related to exports and imports of configuration files.

The log includes the following columns which can be sorted and filtered for convenience.

Column Description
Message Type The icon represents the type of message contained in the log entry. Message types are:

- Start of a task

- End of a task

- Information

- Warning

- Error
Date/Time The date and time of the message.
Task The type of task attempted (Import, Export, or Validating)
Element Either the type of element (calculation, rounding rule, etc.) involved or Start, which indicates the start of an import.
User Name Who initiated the import or export.
Message The log message.
