Dynamic inventory policies

Dynamic inventory policies can be set up with start and end dates to make specific modifications for a certain period of time. These policies can be layered to systematically affect only certain measures during eligible periods. By making adjustments into the future, the correct amount of product can be sent to stores.

Policy settings can be either replaced or modified.

Working with dynamic policies

Adding dynamic policies

  1. From the Execute tab, click Policy.

    The Policy Task screen displays.

  2. Select a Policy Task.

    The Update Policy Task screen displays.

  3. Select the Policy tab.

  4. In the Dynamic Policy section, click Add Dynamic Policy.
  5. Complete the fields in the grid as required.
  6. Click Ok.

Deleting dynamic policies

  1. Select a policy the policy to be deleted.

  2. Click Delete Dynamic Policy.

    A message prompts to confirm the deletion.

  3. Click Yes to confirm.

    The dynamic policy is deleted.

Dynamic Policy field descriptions

Field Description
Policy Date The effective start date for the dynamic policy.
End Date The end date for the dynamic policy.
Notes Optional notes field.
Policy Type Determines how dynamic policy values will affect the default settings. The options are:
OptionDescription
AdjustAdjustments can be additive or multiplicative. Not applicable to all policy settings (e.g. order policy type, target service level). An additive value applies a positive or negative quantity change to the default policy value. A Multiplicative value applies a multiplicative percent adjustment to the default policy value.
ReplaceA value in a replace policy type overwrites the default policy value during the effective period of the dynamic policy.
Percent AdjustA multiplicative percent adjustment to the default policy value.

Applying a multiplicative adjustment to a ratio or percentage policy value produces a multiplicative result. For example, if a back order ratio is set to 50%, applying a +5% change yields a result of 52.5% effective back order ratio, not 55%.
