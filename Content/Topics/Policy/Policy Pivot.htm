Defining policies by filter or hierarchy level

The Policy Pivot screen is used to define policies at either the filter or hierarchy level. Active policy settings can be set up through mass changes in the Pivot grid. The layout of the Pivot grid can be modified using item master and site groupings, as well as all policy settings. Dynamic policies cannot be set up in the Policy Pivot grid, only effective settings.

Access to the Policy Pivot screen depends on user access rights. Because small changes can affect critical values across large segments of the organization, it is highly recommended that only users with a full understanding of the consequences be granted access to the feature.

The parameters used to display information in the Pivot grid are controlled by user assigned filters. If items or locations are excluded from a user’s assigned filters, but kept in the hierarchical segmentation or selected filter, the inventories do not appear in the pivot grid and the inventory policy settings are not be changed.

Only the active policy settings can be controlled through the Policy Pivot screen. All changes made on the Policy Pivot screen are considered user overrides. User overrides are not overridden by policy tasks (similar to inventory level overrides in the Inventory Manager).

Accessing the Policy Pivot screen

  1. From the Execute tab, click Policy.

    The Policy Task screen displays.

  2. Click the Pivot icon on the right of the screen.
  3. Select the following parameters:
    • Filter
    • Item Master Hierarchy
    • Site Hierarchy

  4. Click Execute. The results for the selected parameters display.

Changing multiple policies simultaneously

To change multiple policies simultaneously, select multiple fields in the same column for the policies to be updated. Right-click and select Multiple Changes. Use the Multi Data Change Form For Column to redefine the policies.

Parameters

The parameters used to display information narrow the scope of inventories.

Filter: The inventory filter. If the Item Master Hierarchy and Site Hierarchy parameters are left blank, all inventories in the filter display in the grid.

Item Master Hierarchy: Segments of the product hierarchy can be selected to narrow the scope of the filter selected in the Filter parameter.

Site Hierarchy: Segments of the location hierarchy can be selected to narrow the scope of the filter selected in the Filter parameter.

To clear the Item Master Hierarchy and Site Hierarchy parameters, select NULL next to the dropdown menus.
