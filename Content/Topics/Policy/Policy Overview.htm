Policy Overview

Inventory policies determine the rules by which the replenishment mechanism operates. Inventory policies create the default settings for all inventories, but can be further manipulated for specific inventories in the Inventory Manager (workbench).

The inventory policy details mirror the policies available at the inventory level in the Inventory Manager. Inventory policy details are applied as the default to all of the items in the filter.

Inventory policies are created, maintained, and deleted within the Policy Task screen. This screen displays the current policy tasks and their respective details.

Policy tasks

Policy tasks are individual tasks that can be assigned both start and end dates, if required. Policy tasks are sorted by their respective execution orders in the Policy Task grid. The details of each task display below the Policy Task grid.

It is important to select the correct filters when building new policies to determine the application of the task to the intended items. New filters can be created through the Filters solution if appropriate filters do not exist.

All changes to policy tasks must be made from the Add or Edit screens.

Reordering tasks

  1. In the Policy Task screen, click Reorder to display the Reorder Policy Tasks screen.

  2. Select a task.
  3. Change the task execution order in the Task Execution Order field.
  4. Click Ok.

If changes are made to enabled policy tasks, the tasks may have to be executed outside of the Daily Job Manager.

General tab

The General tab displays general policy information and the filter associated with the policy task.

To display the items contained in filters

  • Click the View Items button to the right of Filter field. The Inventory Manager displays the details for the selected filter.

Policy tab

In the Logistics section of the Policy tab, a lead time in days can be added to a policy task to indicate the outbound handling lead time. This value accounts for the need of some distribution centers (DC) to prepare stock on hand for shipment.

In instances where a DC cannot accommodate an order request the day it was placed, an outbound handling lead time better represents the realistic ship date from the DC.

The Reset to Default checkbox resets the value in the Outbound Handling Lead Time field to the default.

Task grid field definitions

Field Description
Task Name The name given to the task. Cloned tasks are prefixed with Cloned From before they are changed.
Task Description A basic description of the task.
Default Task The default task is the fallback inventory policy for any inventories not covered by other tasks. There can be only one default task.
Task Enabled If the task is currently active.
Task Execution Order The order in which the task rules are applied. This is important because some task rules may supersede others, depending on their order.
Task Start Date Defined This checkbox must be selected if a task is intended to begin on a certain date with the start date specified in the next column.
Task Start Date If a task is intended to begin on a certain date, the start date must be specified and the checkbox must be selected in the preceding column.
Task End Date Defined This checkbox must be selected if a task is intended to end on a certain date with the end date specified in the next column.
Task End Date If the task is intended to end on a certain date, the end date must be specified and the checkbox must be selected in the preceding column.
Task Allowed To Execute An informational checkbox that displays the effective status of the task. If the task is enabled and falls within the specified date range, the Task Allowed to Execute checkbox must be selected.
Filter The filter used to determine which inventory the policy is applied to.
