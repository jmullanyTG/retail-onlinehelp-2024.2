Creating and updating policy tasks

Policy tasks apply replenishment and stocking strategy related settings to the items contained in the task's filter.

Policy tasks are individual tasks that can be assigned both a start and end date if required. They are listed by task execution order in the policy task grid and the details of each task may be viewed. The tasks can be reordered by clicking the Reorder button in the ribbon. This launches the Reorder Policy Task window in which the task execution order can be changed.

Setup

The following object should be set up before creating a policy task:

  • a filter that includes the items and/or sites the policy task is to apply to.

To create a policy task

  1. From the Execute tab, click Policy.
  2. In the Policy Task toolbar, click New.

  3. In the Add Policy Task dialog box, enter a name and optional description for the policy task.

  4. Select the filter for this policy task by clicking Change and choosing the desired filter from the Items dialog box.

    • Click View Items to see which items are included in the filter.

  5. Define settings for the rest of the policy task parameters in the General and Policy tabs.

    For more information on the parameters, see Policy task parameters.

  6. Click Ok to create the policy task.

Results

After the policy task is created, the policy task appears on the Policy screen. The policy task can be executed manually or be executed by the daily job manager.

To update a policy task action

  1. From the Execute tab, click Policy.

    The Policy Task screen displays.

  2. Click Edit.

    The Update Policy task screen displays.

  3. Click the Policy Tab.

    The policy task actions details display.

  4. Update the necessary information.
  5. Click Ok.
