Policy task parameters

Policy task parameters appear on two tabs: General and Policy.

General tab parameters

Parameter Description
Name The display name of the policy task.
Description An optional description field.
Filter The filter containing the items that the settings in the policy task will apply to.
Default Task Whether the policy task is the default policy task for all items.
Task Enabled Whether the task is enabled. If not enabled, it is not applied to items.
Task Execution Order The order in which the task executes among all other active tasks. Execution order is important because tasks executed later in the order supersede those executed earlier.
Task Available From Select these options to set start and end dates for tasks. The task is automatically enabled on the start date and disabled on the end date.
Task Available To

Task Available From Date

 Select Task Enabled From to enable this property.
Task Available To Date Select Task Enabled To to enable this property.

Policy tab parameters

Users are able to determine manual order policy and manual policy settings.

Policy Task Actions

The Policy Task Actions section displays in the Add Policy Task and Update Policy Task windows.

Parameter Description
Manual Order Policy Indicator Whether the Order Policy Type of items included in this task can be automatically changed by this or other policy tasks.
  • Set All Items As Auto - An item's Order Policy Type can be automatically changed by a policy task.
  • Set All Items As Manual - An item's Order Policy Type cannot be automatically changed by a policy task. It must be changed manually.
  • Do Not Update - Preserves the current setting of this property in each item.
Manual Policy Setting Whether the policy settings of items included in this task can be automatically changed by this or other policy tasks.
  • Set All Items As Auto - An item's policy settings can be automatically changed by a policy task.
  • Set All Items As Manual - An item's policy settings cannot be automatically changed by a policy task. It must be changed manually.
  • Do Not Update - Preserves the current setting of this property in each item.

Policy section

Parameter Description
Order Policy Type Sets whether and how order recommendations are generated for items in this task.

  • [Do Not Set]: Preserves an item's current Order Policy Type. Allows for non-order policy specific settings (order multiple) to be updated without disrupting the order policy driven behavior.
  • Bin: Applies a fixed-quantity order methodology to items (for example, if one is sold another one is ordered). Best suited for items with intermittent, low demand. See, Configuring bin settings.
  • Non-Stocked: Prevents replenishment recommendations for these items, except where customer orders cannot be filled with current stock on hand. See, Configuring non-stocked settings.
  • Normal Stocked: Items that require a demand, inventory, and replenishment plan. These items are everyday selling items which should be continually stocked and replenished. Order recommendations are generated for these items based on inventory policy and item-specific ordering criteria. See, Configuring normal stocked settings.
  • Redundant: Items that are never likely to sell or be in demand. This classification prevents these items from being ordered.
  • Showroom: Items which are sold out of one location, but are by design fulfilled from another location. This can include display model items, catalog items, or items that are scheduled for delivery. See, Configuring demand transference for showroom items.

Setting which order policy types display

The system policy Allowed Inventory Order Policies defines which order policy types display. To review this system policy, navigate to Module: Inventory Policy. The system policy is configured using order policy indicators.

Order policy indicator

The order policy indicator corresponds to a single letter that identifies the policy type.

Order Policy Indicator ID Order Policy Indicator Name
B Bin
E Economic Order Quantity
N Non-Stocked
P Normal Stocked
R Redundant
S Showroom

Replenishment Strategy box

This table describes all possible replacement strategy parameters. Not all of these parameters appear in a given policy task. The parameters that appear in a specific policy task are determined by the task's Order Policy Type.

Parameter Description
Back Order Ratio The percentage of lost sales which are expected to result in a customer returning to purchase the item.
Bin Min Units The minimum quantity of units required in the bin.
Bin Max Units The maximum quantity of units allowed in the bin.
Review Period Days The number of days between inventory reviews for this item. Inventory reviews determine whether the item needs to be re-ordered to comply with stocking parameters.
Replenishment Cycle Days The number of days that must pass between orders for this item. Must be greater than or equal to Review Period Days.

Safety Stock box

This table describes all possible safety stock parameters. Not all of these parameters appear in a given policy task. The parameters that appear in a specific policy task are determined by the task's Order Policy Type.

System policy Max Safety Stock as Percent of Max Stock caps safety stock quantities at a specified percentage of an item's Maximum Stock Units value. Unit-based minimums (for example, Minimum Safety Stock Units) that exceed the Maximum Stock Units value are respected.

Parameter Description
Safety Stock Days A dynamic quantity based on a number of days forecast. As the number of days changes, so the safety stock requirements change.
Target Service Level A dynamic quantity based on a service level target. It is a percentage of the items that are fulfilled in a location out of the maximum demand.
Minimum Safety Stock Units A static quantity based on a minimum number of units as the safety stock.

Optimized Target Service Level using a Stock to Service (STS) calculation

Returns box

Parameter Description
Return Rate The percentage of items which are expected to be returned.
Days Before Return The expected average number of days between the sale of an item and its return.
Reuse Rate The percentage of returned items that are expected to be salable stock again.
Days Before Reuse The expected average number of days between an item being returned and it being considered salable stock again.
