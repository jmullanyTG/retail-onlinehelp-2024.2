Order line priority and locking

Select Locked for Constraints to prevent the constraint logic from over-aggressively constraining a single order line to meet the goals of a constraint set. During the constraint process, order lines are prioritized.

Constraining up

  1. The order line is not locked
  2. The replenish order quantity (user order quantity) is a positive number
  3. The order line was not constrained within the previous loop or iteration
  4. Ordered by replenish order quantity, descending
  5. If replenish order quantity is identical, ordered by default priority, ascending
  6. If default priority is identical, ordered by OrderLineID

This allows for the constraining up of items with a higher volume of future orders, whereas the current period order is relatively small.

Constraining down

  1. The order line is not locked
  2. The replenish order quantity (user order quantity) is a positive number
  3. The order line was not constrained within the previous loop/iteration
  4. Ordered by replenish order quantity, ascending
  5. If replenish order quantity is identical, ordered by default priority, descending
  6. If default priority is identical, ordered by OrderLineID

This allows for the constraining up of items with a higher volume stock contained in the current period order, whereas the future period order is relatively small.
