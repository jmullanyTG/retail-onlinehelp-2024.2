Constraint parameter descriptions

Values change as users highlight different constraint sets within the header. When building a new constraint it is easy to think of the constraint as the following template sentence:

Constrain so that “Constraint Type” is “Operator” the value “Constraint Value”.

  • Priority – Constraints are applied in order of priority within context of the constraint set. Priority 1 is applied last and can override lower prioritized constraints.
  • Constraint Type – The inventory property to be constrained. All monetary constraint values apply to the system default currency. One of the following attributes:
    • Default Cost Value – A standard cost value.
    • Default Sell Value – A standard value based on retail price.
    • Quantity – The quantity of units.
    • Any volume metric data imported into the system.

    • Any cost data imported into the system.

  • Operator – The mathematical operator for the constraint. Can be one of the following:
    • Greater Than
    • Less Than
    • Equal To
    • Equal to Multiple of – Used for basic case pack logic.
  • Constraint Value – Numerical value of the constraint.
  • Tolerance Above – Should the system be unable to constrain the value exactly to the correct amount, the tolerance above the value which is considered acceptable. This may be expressed as a percentage, or quantity of units.
  • Tolerance Below – Should the system be unable to constrain the value exactly to the correct amount, the tolerance below the value which is considered acceptable. This may be expressed as a percentage, or quantity of units.
  • Rounding – Acceptable rounding rules must be selected when the operator is Equal to Multiple of:
    • Round Up
    • Round Down
    • Round to Nearest

Additional fields can be added by including groups of items within the selection.
