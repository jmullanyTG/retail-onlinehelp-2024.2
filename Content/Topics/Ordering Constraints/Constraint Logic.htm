Constraint logic

As order lines are determined to be eligible for constraint, the relative amount to which they are constrained varies based on several factors including order multiple, the calculated constraint window, and the calculated fixed constraint package unit.

  • Order Multiple – The order multiple value as specified in inventory policy.
  • Calculated Constraint Window – A sampled period of time within which order recommendations are considered to be adjustable based on constraint logic. The calculated constraint window is determined as (LT + RC + SS days x2).
  • Fixed Constraint Package Unit – The quantum number of units by which a constraint adjustment may be made (either up, or down). This is largely applicable to monetary (i.e. cost or price) based constraint adjustments.

If unit cost is $10 and order multiple is 6, then adjustments must be made with “$60 worth of items” as the adjustment increment.

As the constraint logic loops through each iteration, when constraint related adjustments are made, a flag internal to the logic updates denoting that the order line has been constrained within the loop/iteration.

If additional constraining is required and it does not find an order line which is eligible, then all flags reset to eligible. If on the following iteration it cannot find any eligible order lines, then the constraint logic terminates.
