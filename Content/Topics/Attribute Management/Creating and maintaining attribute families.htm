Creating and maintaining attribute families

The Attribute Families section in the Attribute Management screen allows for the creation and maintenance of attribute families. Each attribute family contains attributes that pertain to that family and can be imported from host systems if available.

The Attribute Management screen contains two sections: Attribute Families and Attribute Values.

Parameters

There are two parameters:

Entity Type: Which attribute families are returned based on the attribute families which are associated with the entity. Only attribute families that are associated with the chosen entity are returned.

Attribute values can be assigned by entity and there are seven entities to which they can be assigned

  • Inventory – Item/location level for attributes that relate to an item at a specific location.
  • Item master – Item across all locations for attributes that are the same for each item across all locations.
  • Site – Site based attributes that can be specified at a site level.
  • Supplier - Supplier specific attributes
  • Customer – Attributes specific to customers.
  • Variant – Attributes that are applied to the defined variant. A variant is grouping of items for which plans are created and maintained. An example of this is a style and color. Variants are used to define real products in assortment planning.
  • Variant Placeholder – Variant placeholders are created in Assortment Planning to allow users to generate a plan for a variant that does not exist yet in any external system. Attributes can therefore be assigned to the variant placeholders before they become real variants.

Attribute Sets: Returns the sets users want to assign attribute families to. Allows users to bring back multiple attribute sets at a time.

Newly created Attribute Sets do not immediately appear for selection in this dropdown if they are for the currently selected entity type. To refresh the dropdown and show new sets, select a different entity type then select the entity type of the new set once more.

Attribute Families section

The Attribute Families section has columns that define how attributes can be assigned and which families are available in the appropriate sets.

  • Family Name – A descriptive name for the family of attributes.
  • Family Description – A description of the attribute family and the attributes it contains.
  • Entity types – The entity types selection restricts the data entities to which the attributes within the family can be applied.

    The Variant type must be assigned for an attribute family to be definable on variants. The Variant Placeholder entity type must be assigned for an attribute family to be definable for placeholders within Assortment Planning.

    By only assigning the Variant Placeholder entity type to a family, that family is not assignable to variants and is exclusively used within Assortment Planning. This is useful in organizations where a master data team defines the primary attributes but there are others that users within the merchandising team wish to assign while planning.

  • Data Type - The associated data type.

  • Allow Multiple Values – Whether users can select multiple attributes from a family to assign to an entity.

    If an attribute type is converted from allowing multiple values to a single value only, then the following message displays warning that all the previous values saved for the attribute are deleted.

  • AP Updates Variants – Whether changes are allowed in Assortment Planning to attributes attached to assigned variants. If checked, the corresponding attribute families are editable in Assortment Planning for placeholders with assigned variants, and any changes made to them update the assigned variant’s attributes. Unchecked means the corresponding attribute families are grayed out and read-only in Assortment Planning for placeholders with assigned variants.

    This setting displays if system policy Update Variant Attribute Values In Assortment Planning is set to True and is only valid for attribute families with the Variant Entity Type assigned. For more information, refer to the Assortment Planning User Guide. For more information, see Attributes view.

  • Attribute Set Assignment – The attribute sets selected in the filter described above will dynamically form columns which users can check to specify whether an attribute family is available for that set.

Attribute Values section

On selecting the attribute family from the widget above, the Attribute Values widget updates.

The Attribute Values widget allows users to create new widgets that have not been imported from a host system.

  • Value Code – A unique code that identifies the attribute value.
  • Value Description – The value description describes the attribute value.
  • Value External ID – (read-only) Populates with attribute values from external systems. Users may not edit attribute values on imported data.
  • Behavior – For each attribute set to which the attribute value may be assigned, there are three behaviors for the value:
    • Blank/None – Users cannot select this attribute value to assign to a product associated with the attribute set.
    • Allowed – The attribute value is allowed to be set against a product associated with the attribute set.
    • Default – The attribute value or values (if multi values are allowed) that will default against any product associated with the attribute set.
