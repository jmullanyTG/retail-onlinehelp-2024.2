The Attributes supporting view in Assortment Planning

The Attributes supporting view in the Assortment Planning Placeholder and Collection views is used to view and manage product attributes for placeholders and collections. The Attributes view slides in from the right side of the screen by default.

The attribute families available to select display based on the set they were assigned to and how the attribute set was assigned to a level of the product hierarchy.

Two selection options per parameter family is not allowed.

  • Single Values – Select from this column if the parameter family was specified to only allow single values. The Multiple values column is not editable. Click on the dropdown list and select the attribute value to assign to the placeholder variant or placeholder collection that is currently selected.
  • Multiple values – Select and click on the ellipsis if the parameter family was specified to allow multiple values. A pop-up displays for users to select multiple attributes to assign to the placeholder variant or placeholder collection that is currently selected.

Multiple lines can be selected to assign attributes. Users are not required to assign attributes one line at a time in either the Placeholder view or the Collection view.

Note that attributes can be associated with placeholder variants and variants. Attributes can also be maintained on variants in the Variant Manager module.

Users can segment the assortment across attributes to ensure the correct mix of assortment.

In addition to slicing up assortments that have been built, attributes are an effective way to rationalize future assortments. The target creation process allows for attributes to be used in the estimation of the width and depth of future assortments needed to meet financial targets.

For instance, use the price point range attribute to segment future assortments using the marginal return curve to estimate the number of options to offer in each price point range.
