Assigning attributes to variants

Product attributes can be assigned to variants through the Variant Manager module.

On launching Variant Manager, a filter selection dialog appears allowing users to specify a filter of variants. The Variant Manager Document screen displays once a filter has been selected.

Once a variant manager document is open, users can select variants in the grid section of the document. Details for the selected variants appear in the Variant section.

Users can work with a variety of aspects of variants within a variant manager document by clicking the appropriate tab. To assign or edit product attributes for selected variants, click the Variant Attribute Management tab.

Both the upper and lower sections of the view can be displayed or hidden by clicking the corresponding Grid and Details buttons in the upper right of the view.

Variant Attribute Management tab

Functionality

Parameters: Shows or hides the parameters area.

Execute: Updates the view according to the selected Variant Hierarchy and the variants selected in the grid section.

Parameters

Variant Hierarchy: The variant hierarchy for which to assign attributes.

Because it is possible to create alternate variant hierarchies which contain the same variants, it is also possible to have different attributes assigned to a variant across those hierarchies. Ensure the correct variant hierarchy is selected before proceeding.

Field descriptions

  • Family Name: The name of the attribute family
  • Family Description: The description for the attribute family
  • Single Values: The attribute value of the adjacent single-select attribute family for the selected variants. When variants are selected that have different attribute values assigned, this field shows (Multi) to indicate this.
  • Multiple Values: The attribute values of the adjacent multi-select attribute family for the selected variants. When multiple variants are selected that have different attribute values assigned, this field shows the combination of all values present.

Data Population

The view populates with all attribute families that any of the selected variants is eligible for. Attribute values available for selection include all values that any of the selected variants is eligible for.

Applying Changes Across Multiple Items

When multiple variants are selected in the Grid section, any conflicts that exist between the applied values and the attribute value eligibility of the variants the change is being applied to are automatically resolved.

  • Single-select attribute value changes are applied to all variants that are eligible for them. Ineligible variants remain unchanged
  • Multi-select attribute value changes are applied additively to variants that are eligible for them. If only some of the applied values are eligible, those will be applied. Ineligible variants remain unchanged

Dual-directional Attribute Flow

Depending on where attributes are assigned (to placeholders in Assortment Planning or to variants in Variant Manager) attributes are transferred when a variant is assigned to a placeholder.

Assigning a variant

Any product attributes attached to the variant are transferred to the placeholder. Any attributes that were already assigned to the placeholder but do not conflict with the variant attributes (i.e. single-select families that the variant already had assigned) remain on the placeholder and are transferred to the variant.

Attribute changes in Assortment Planning

Any changes made to placeholder attributes in Assortment Planning are transferred to the assigned variant for the product hierarchy assigned to the active assortment plan. They are not transferred to other placeholders that variant is assigned to.

Attribute changes in Variant Manager

Changes made to variant attributes in Variant Manager are transferred to all placeholders that variant is currently assigned to across all existing assortment plans that have the selected Variant Hierarchy assigned.
