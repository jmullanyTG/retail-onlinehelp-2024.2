Schedule sets

Schedule sets are containers for schedules. The sets are applied to sites and suppliers to control order behavior.

Schedules are applied to a set based on their order in the set. The application order is top-down through the set. The mode of the top-most schedule dictates the expected behavior for all other dates that are not specifically mentioned.

Schedule sets are stored as almanacs in the database. They can be found in the tbl_Common_Schedule_Almanac table.

Schedule relationships are stored in the tbl_Common_Schedule_AlmanacLink. table. This table also holds the behavior as ScheduleMode and the priority. For a particular schedule set (almanac), the priority 0 determines the highest priority. Its associated ScheduleMode determines the behavior of all non-specified dates.

Navigation pane

  • Set Name: The name for the schedule set. The set name displays in Site Manager and Supplier Manager solutions for selection.
  • Mode: The mode dictates the intended behavior of a schedule in the schedule set.
  • Enabled: Allows the schedule set to be selected in Site Manager and Supplier Manager.

Schedule set details

The schedule set details display when a schedule set is selected in the navigation pane.

  • Name: The schedule set name.
  • Active Flag: Controls the enabled behavior for the schedule set in the Details view.
  • Description: An optional description field for the schedule set.
  • Schedules: Information related to the schedules in the schedule set.
    • Enabled: Controls the enabled behavior for the schedules in the Details view.
    • Schedule Name: The name of the schedule.
    • Mode: The current behavior of the schedule in the schedule set.
    • Summary: A summary description of the schedule behavior.
  • Resolved Dates: A listing of the resolved dates in the schedule set. The range for the resolved dates is from today’s date to ten years out.
