Policies Overview

The Policies module is the control panel for ToolsGroup JustEnough. The user interface is meant to interact with the system policy table and should not be manipulated by anyone other than a ToolsGroup JustEnough technical consultant or an extremely well trained administrative level user.

System policies are grouped together for ease of navigation. They may apply to one of the .NET engines, or to the behavior of a specific module, or to the application as a whole.

System policy settings

System policy settings are segregated by module, engine, or overall system. They should only be manipulated with a full understanding of the potential consequences. System policies are organized by system policy type. These are typically grouped together by module, or within context of system specific functions or tasks.

  • Engine – Relates to an engine specific set of policies and parameters.
  • General – Relates to common application settings.
  • Import Defaults – Relates to assigned defaults for imported data.
  • Module – Policy settings specific to the context of the listed module.
  • Process – Policy settings common across the application.

List of available system policies

The list of available system policies displays in a navigation pane.

Users can expand these folders to view the available system policies.

Highlighting these populates the system policy settings screen.

Required Configuration

For certain modules, system policy settings will need to be configured as part of the setup and initialization process. Details related to the specifics of these policies should be referenced within the functional help documentation of the associated modules.

System policy settings

All policies which appear in the user interface have a SystemPolicyIsVisibleFlag = 1

Policy types control the display groupings of system polices. These may be referenced in tbl_SystemPolicyType.

Policy Data Type

A single character is used to designate the data type which may be entered within a system policy setting field.

  • B – Boolean. A bit/flag indicating an on/off setting. May be a 1 or a 0.
  • C – Char. A single letter character.
  • D – Date. A datetime value.
  • E – Enumerator. A value selected from the available values listing.
  • I – Integer. An integer value.
  • N – Number. A numeric value.
  • S – String. A Unicode text value. May be a list.
