Supersession Grid screen reference

Supersessions are managed using the Supersession Grid screen.

To access the Supersession screen

From the Execute tab, click Supersession.

The Supersession Grid screen displays the active and inactive supersessions.

Fields and descriptions

The following is a description of the fields available on the Supersession Grid screen.

Field Description
Old Item The item that supplies the supersession history. In many instances, this is the item that is being replaced (last year’s model).
New Item The item that supersedes the item described in the Old Item column. This is the item that is replacing the old item (this year’s model).
Superseded for Replenishment If selected, this supersedes the new item over the old item for all replenishment orders.
Superseded for Forecasting If selected, this brings in the demand history of the old item into the forecast engine to calculate the projected demand for the new item
Replenishment Ratio Value of the ratio to 1 that is applied to the old item stock on hand quantities. A value of 1 gives a 1:1 ratio of old item quantities to new item quantities. A value of 2 gives a 2:1 ratio of new item demand to old item demand, a 100% increase. A value of 1.2 is for a 20% increase.
Forecast Ratio Value of the ratio to 1 that is applied to the old item sales history amounts. A value of 1 gives a 1:1 ratio of old item quantities to new item quantities. A value of 2 gives a 2:1 ratio of new item demand to old item demand, a 100% increase. A value of 1.2 is for a 20% increase.
Sales History Start Date The beginning of the measurement period for the old item sales history.
Sales History End Date The end of the measurement period for the old item sales history.
Supersession Start Date Determines when the supersession becomes active. If blank, the supersession is considered active from the go-live date of the system.
Supersession End Date Determines when the supersession becomes inactive. If blank, the supersession is considered to continue in perpetuity.
Active The active flag displays in the Active checkbox if the supersession is currently active.
Old Item Type Determines the replenishment status of the old item.
Redundant The item remains in stock, but will no longer be ordered.
Non-Stocked The item will no longer remain in stock (not on the shelf).
User The UserID of the creator of the supersession. Useful for audit purposes.
Date Created The date on which the supersession was created.

Sites affected

When a supersession is selected in the Supersession Grid, the sites affected by that supersession display on the Sites Affected panel on the right of the screen.

Notes

A place for additional notes along with a time stamp and the user ID of the person who added the note.

Toolbar functionality

Action Description
New Creates a new supersession. See, Creating supersessions.
Edit Updates the sites for the selected supersession.
Delete Deletes the selected supersession.
Save Saves property changes made on the Supersessions grid.
