Filters and hierarchies

The Hierarchy Filtering view is accessible from the Views solution on the System Utilities tab and is used in the configuration process during an implementation.

Parameter

  • Entity Type: The hierarchy type

Hierarchy Configuration view: editable fields

  • Active: Checked by default after the hierarchy is created. When unchecked and applied, the hierarchy is no longer accessible from the front end.
  • Available for AP: Check this field if the hierarchy must be available to select in Assortment Planning.
  • Is used in active AP Plan: This field is checked if the hierarchy is used in an active assortment plan.

Hierarchy Configuration II view: editable fields

  • Order Number: The lowest number represents the highest level of the hierarchy.
  • Enabled: By default, after the hierarchy is created this is unchecked. If checked, then a group is part of the current hierarchy being configured.

Variant and Site entity types must be configured in Assortment Planning (AP) and must be created prior to creating any assortment plan. The combination of the lowest levels of both hierarchies make the lowest level of the assortment plan and the level at which placeholders are created.

Site hierarchy

A full description of the location hierarchy must be entered. Using the Site Groups available (Friendly Names are maintained from the Setup Table Definition Friendly Names view) as well as the explicit channel attribute, configure the site hierarchy to have levels of commonality between Merchandise Financial Planning and Allocation Planning.

Item Master hierarchy

Item master hierarchies are the hierarchies of the product dimension. They are used throughout ToolsGroup JustEnough to define hierarchical relationships and levels for both aggregation and dissemination of information.

Item master hierarchies are built out of:

  • item master groups
  • the variant entity
  • the explicit style,  and
  • size attributes (where applicable).

Variant hierarchy

Assortment Planning uses a variant entity hierarchy for its product hierarchy. This hierarchy is used for all rollups and reporting in Assortment Planning.

The variant hierarchy must be structured to extend to one level above the variant (equivalent to placeholder in Assortment Planning, referred to as style-color), unless a level for style (referred to as a collection in Assortment Planning, a group of variants – often a single design in multiple colors) exists, then the variant hierarchy must extend to one level above that.

When using the Segment By Hierarchy function of Target Creation within Assortment Planning, the lowest level of this hierarchy is the level at which target segments will be created.

To use targets from a merchandise financial plan in a given assortment plan, the product and site hierarchies assigned to both plans must share common levels from the top down to the lowest level of the merchandise financial plan hierarchies (assortment plan hierarchies generally extend to at least one level lower). The plans must also share common time structures, such as calendar and period type.

Cluster sets to be used in assortment plans must be contained in a single leaf level of the site hierarchy assigned to those plans.
