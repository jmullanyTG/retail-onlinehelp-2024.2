Translate a language or language and culture

The Translate a language or language and culture view allows users to create new string values for the culture. This is where strings are translated. The prefix of StringName indicates which parts of the application will be translated. 

  • String Name – The string name as used by the application itself. Without a translation, this is the value which appears in the User Interface.
  • Default String Value – The translated name as defined in the default string maintenance section.
  • String Value – The translated value or friendly name.
  • Note – Any applicable notes.
  • Date Time – The date on which the translation was created or edited.

Locating individual strings

Users can use the search bar to locate individual strings (or use a contextual keyword). New entries may be made by clicking on the Click here to add a new row section in the grid.

String values can be edited by clicking on the String Value field.

Importing or exporting translated language files

The translated languages files can be imported or exported as a ToolsGroup JustEnough Translated Language (.tlan) file type.
