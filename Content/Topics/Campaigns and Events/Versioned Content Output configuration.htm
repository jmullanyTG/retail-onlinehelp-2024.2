Versioned content output

Users with administrative rights can version the composition of any Versioned Content Export file. The settings used to export versioned content for use with Adobe InDesign or other creative layout applications are located in the Versioned Content Output Configuration tab.

To access the Versioned Content Output File Export Settings tab, open an Event and navigate to Event Workbench > Output.

Versioned content output file export settings

The settings of the output file are configured in the Versioned Content Output File Export Settings tab. These settings are configured by Professional Services and should not be modified without assistance from ToolsGroup JustEnough Professional Services or Support.

Field Description
Output Root The root destination directory for exported output files. Subfolders are created for the Event and its associated Slot Sets. The output files are stored under the Slot Set subfolders.
Image DTP Path Attribute The repository attribute that will be exported for every Image in an Event.
Non-Export Prefix Specifies a Slot Set prefix that identifies that the Slot Set should not be included in the output file. This should not be set to any value that is likely to be used in Slot Set naming.
Database This setting is used to connect to the database during the export process.
DBServer Name This setting is used to connect to the database during the export process.

Fields included in the output file

The fields to be included in the output file are configured by Professional Services and should not be modified without assistance from ToolsGroup JustEnough Professional Services or Support.

There are multiple rows identified with the Source Type System that are required to ensure the desktop publishing software plugin works properly. They are displayed for informational purposes only and cannot be modified. The remaining rows can be configured for the plugin to populate desktop publishing software as needed or additional rows can be added.

Issues with versioning content ouput

If versioning is not happening for a Slot Set, the Non-Export Prefix setting could be the source of the problem. This setting checks the Slot Set Name to see if it starts with the designated prefix, and excludes it from version calculations if it does. If Slot Sets are not getting versioned even though they should, checking this setting may reveal that this is the case because of how the Slot Set is named, or a bad prefix value for the setting.

If the prefix is not to be used the easiest way to ensure it will not interfere with versioning is set the value to Export all Slot Sets (which must be manually keyed).
