In this example, the end result of the versioning process identifies five distinct versions, composed of three Base Versions. Two of Base Versions are further split into two Plate (black) Versions each.

Versions are strictly based on the order of the Geographies in which changes occur and not a majority rule. This means that even if ‘Zone 1’ was the only Geography to have content values that differed, while all the other Geographies shared common values, version ‘AP1’ would still be based on the values of ‘Zone 1’. Geographies ‘Zone 2’ through ‘Zone 10’ would be assigned to a second version. This allows the first, or Base Version of each Slot Set to be driven by key stores that have most likely been assigned to the highest ranking Geography.

In this example, the version ‘CP1’ was created not by Promotional content, but rather by the lack of any assignment at all for Geography ‘Zone 9’ in Slot 3. When a Promotion with any attribute values is assigned to at least one Geography within a Slot, then any gaps caused by unassigned Geographies, or missing attribute values, must be included in the comparison of content and pricing during the versioning computations.

If a Slot has been versioned to display 2 or more Promotions with different Promotion names, Base Versions will be generated based on Promotion name even if the content and pricing for each Promotion in the Slot is identical.
