Versioning process

Versioning can occur at many steps along the way to completing an advertising Event. Versioning is the association of a set of information (such as Promotion Strategy, price, copy, and images) to one or more sites while associating different information to other sites.

A Price Cluster is a grouping of sites for the purposes of promotional price versioning. A site may only belong to one Price Cluster in a Price Cluster Set.

A Geography is a grouping of sites for the purposes of shared message versioning. A site can belong to more than one geography (distribution point) allowing retailers to tailor content to distributions based on language, images used, or adjusted wording to better target customer segments receiving the distributed version of the event.

Promotions can be versioned at the Price Cluster level to specify different Promotion Strategies or pricing.

Event slots can be versioned to display a different Promotion for specified geographies.

Whether added to an Event or not, all Promotions can have content versions that specify copy and images assigned to geographies.

Versioning is optional.

Promotion versioning

At the Promotion level, users can alter the product mix, Promotion Strategy and pricing for each associated Price Cluster. In addition, Promotion content, copy and images, can be unique for each content version.

Event slot versioning

When a Promotion is placed in an advertised slot of an Event slot set, it is possible to assign the Promotion to all or some of the Event geographies. This allows for the assignment of alternative Promotions to the slot for remaining geographies thereby creating a regional product version. A geography can only be associated to one Promotion in a slot. Geographies are linked to Promotion Price Clusters through their respective site assignments.

Content versioning

After a Promotion is placed in an Event slot, the Promotion’s content is copied to create an Event content instance that is specific to the slot the Promotion was placed in. As with Promotions, Event content (both copy and images) can be unique for each content version.

Content versions are linked to geographies. It is possible to associate a geography to more than one content version. For example, a geography may require that content be provided in both English and French.

Event versioning

The geography in the Event represents a single message that describes all the Promotions assigned to it within the Event (the Event slot sets). Performed at the slot set level, versioning compares the message for each geography and identifies the true set of distinct messages or Event versions.

Calculating Event versions is a two-step process in which each slot set is first broken down into Base (color) versions according to image assignments and any non-black copy, price, or Promotion attribute values. After which, each Base version is further broken down into additional Plate (black) versions. The result is a distinct set of content data by slot set and slot needed to create a complete Composition that is applicable to one or more geographies.

The versioning order is based on the order of the geographies in the Geography Set used for the Event. The first geography in the set provides a constant for determining the first color version for the slot set. Further versions are grouped by their highest ranked geography.

Base versions

Base versions are named using alpha characters beginning with the letter A. When the number of bases exceeds 26, the base name becomes a 2-character string starting with AA and continuing as AB, AC, etc.

Plate (black) versions are named numerically (preceded with the letter P) within their computed Base version, and resets back to 1 for each base afterwards.

The naming convention for the complete version is the <Event Name>_ <Slot Set Name>_<Base Name>P<Plate Version number>. For example, MyEvent_Page1_AP1.

If an Event only contains one geography and if every Promotion only uses one Price Cluster, then the only version that should be created is AP1.
