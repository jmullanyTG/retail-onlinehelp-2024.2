Configuring engine chains

The engine chains can be configured using the Configure Engine Chain view. This view allows someone with the correct permissions to enable or disable the engine chains.

Enabling the engine chains changes the logic in SQL to route to different procedures that create the proper engine chain tasks for each of the engines that are part of that engine, and calls the appropriate engine chain endpoint on the Compute Manager.

In the case of Allocations, when the AllocationLogisticProjection (ALP) engine chain is enabled, the call from the UIL to the Calculate procedure reroutes to use the ALP Calculate procedure instead. The procedure uses different engine subtypes for the engine chain.

Disabling the engine chains allows the data to be run on the engine as they are currently implemented. Engine Chaining does not affect previously created jobs or tasks in Job Manager that may be run nightly or weekly unless they call the specific Calculate procedures as part of the task. The logic for those jobs and tasks can be changed to utilize engine chaining if it is needed.

To configure an engine chain, navigate to System Utilities > Views > Maintenance > Configure Engine Chain.
