About custom notifications

Custom notifications can be configured and received through the Default delivery method.

Custom Notifications are initiated by a scheduled task in the Job Manager or through a customized stored procedure that can only be configured by ToolsGroup JustEnough Professional Service or Support.

Creating custom notifications

  1. Click Notification Maintenance.

    The Notification Maintenance screen displays.

  2. Under the respective solution, click the [+ Add Custom Notification] row.

    A new row displays.

  3. In the Notification Properties panel:
    1. enter a notification title.
    2. select the notification type.
    3. enter the notification message.

  4. Click Preview on the toolbar to preview the custom notification message.

  5. Click Save to create the custom notification message.

To initiate the custom notification, see Initiate Custom Notification Configuration.

Field Description
Solution The notification solution depends on where the custom notification is initiated. The selection:
  • determines which icon is used in the notification message that is sent to users.
  • can be used to sort messages in the Notification Inbox.
Title Describes the purpose of the message. Follow corporate standards naming conventions, if applicable. The title is not displayed in the message sent, but it is updated in the Notification subscription window.
Type The notification type displays an icon according to the message type selected. The system defaults all custom notifications as a General notification type. The types are:
  • Assignment
  • Completed
  • Error
  • General
  • Information
  • New Task
  • Warning
Message The notification message is the content in a message, which also displays in the Notification Inbox.
UUID (read-only) The Universal Unique ID number is a system-generated identifier used when configuring the notification to be initiated in Job Manager or through a Customized Stored Procedure (CUSP hook).
