Notifications Overview

Notifications report when processes (calculations, forecasts, validations, etc.) have been initiated or completed and if any errors occurred.

Notification delivery settings are system configured. However, these settings can be changed on the Notification Preferences screen.

There are three Notification views:

  • Notification Maintenance: Used by system administrators to subscribe to system generated notifications, by module, and configure the default delivery method for each subscription. Additionally, administrators can create custom notifications, by module, which can be scheduled and sent by Job Manager or through customized stored procedures (CUSP hooks) in the respective solution.
  • Notification Preferences: Located under the User Settings tab, allows users to override system configured delivery methods.
  • Notification Inbox: Displays the notifications as they were configured. The Notification icon on the right side at the top of the screen indicates the number of unread notifications. Click the Notification icon to display the Notification Inbox.

Depending on the view, notifications can be subscribed to and configured in the following:

  • Allocation
  • Assortment Plan
    • Order Manager - Prepack
  • Assortment (Receipt) Plan
  • Site Clustering (custom notifications only)
  • Content Manager (custom notifications only)
  • Curves
  • Events (custom notifications only)
  • Inventory and Level Forecast
  • Group Forecast
  • Price Manager
  • Promotions (custom notifications only)
  • System (custom notifications only)
  • Workflow (custom notifications only)

Standardized jobs in the Notification Nightly Purge can be configured to delete messages that are older than a certain number of days. In the standardized job, a parameter is available to modify the number of days the messages are stored in the database. By default, notification messages are stored for seven days.

The job must be scheduled to run outside of the regular work hours. For example, a Saturday from 3 AM to 11 PM.

Notifications Configuration

Notifications must be configured only by ToolsGroup JustEnough Professional Services, Support, or a trained on-site super user.

Notification user rights

The Access Notification System Maintenance global permission allows a super user to access and manage user permissions from the Notification Subscription screen.
