Initiate custom notification configuration

In a scheduled job’s task or in the CUSP, the token parameters must be configured to initiate a custom notification.

For more information on creating jobs, associated tasks and schedules, see Job Manager.refer to the Job Manager User Guide.

Field Description
iMessageUUID (Required) Enter the system generated UUID for a specific custom notification.
iSenderUserID (Required) This parameter must always be set to a value of one (i.e. IOAdmin).
nRecipientUserIDList (Required) Enter the user or role that will receive the custom notification. If multiple users or roles will receive the notification, enclose the user ID list in single quotes.
nMessageToken (Optional) The message tokens will be passed into the notification call as a single string for the system to parse appropriately into the notification message.

For correct configuration, refer to Custom notification implementation examples.
cTokenIdentifier (Optional) If a value is entered, the system attempts to retrieve tokens from the table notification .TokenStore. This table stores tokens for the queue so that the XML does not have to be perpetuated throughout the process.

If both @nMessageToken and @cTokenIdentifier are supplied, @nMessageToken Tokens overrides tokens present in [notification].[TokenStore], but union tokens that do not exist in both.

bDebug (Optional) This must always be set to a value of zero unless ToolsGroup JustEnough Professional Services or Support are investigating an issue.
