About core notifications

The Notification Solution Subscription list displays core notifications that are sent to a user or user role. The description of each notification solution subscription type and when it should be sent is described as follows:

  • Queue Submission/Preparing/Coalescing/Ready: after a process is submitted to the Workload Queue, a notification displays the current number of processes ahead of the process that was submitted.
  • Queue Completion (without error): The Workload Queue has successfully completed the submitted process.
  • Queue Completion (with error): The Workload Queue has completed the submitted process with an error.
  • Queue Removal: The process is removed from the Workload Queue by a Super User.
  • Queue Expired: The configured job time for a submitted process on the Workload Queue has expired.
  • Queue Failure: The submitted process has failed to be placed on the Workload Queue.
