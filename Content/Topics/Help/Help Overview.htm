Help Overview

The Help module manages the location and type of help documentation which the application can access. Details of the help context are displayed and maintained within the single help context management screen of the Help module.

  • Help Context Key – The name used to identify the location in which the Help launches. The help context key is one of the module UIL files.
  • Help Type – The file type of the associated help document.
    • Open File – A plain text file which is opened by an outside application on the client computer such as Microsoft (MS) Word or Notepad.
    • HTML Help Pages – A website which is accessible through a web browser.
    • Microsoft Help – A windows compatible help file such as a .chm file.
  • HTML URL – The destination URL if using an HTML type help document.
  • MS Help File Location – The file name if using an MS help file. All .chm files should be loaded into the database (using the files module) and can be referenced only by their full file name.
  • MS Help Navigator Type – The MS Help Navigator type assists the Help module in directing Help actions to the appropriate section within a .chm document. There are two types which should be used for most cases. The Help file opens to:
    • a specific topic if the topic exists. Used to link to a specific segment of a larger .chm file (such as a sub-heading). Requires an accompanying MS Help Parameter to work.
    • the table of contents. Defaults to the table of contents for the .chm file. Default for module specific help.

    The following are not intended for use with current Help files.

    • the index. Opens to the index.
    • the search page. Opens to the search page.
    • the index entry for the first letter of a specified topic. Opens to the index entry for the first letter of a specified topic.
    • the topic with the specified index entry if one exists. Otherwise, the index entry closest to the specified keyword is displayed. Gives multiple options for how the file is opened.
  • MS Help Parameter – If the help navigator type required a parameter (if using “the help file opens to a specific topic, if the topic exists”, a parameter is required), this field holds the parameter.
  • Other File Location – If using Open File, this is the file path of the file.

Importing a help configuration (.jehc) file

For most installations, the Help files are pre-loaded into the user interface by way of the database. During implementations, it may be necessary to import a Help Configuration (.jehc) file, which contains all of the appropriate file links.

Creating or modifying entries

If desired, users can create new entries or modify entries to point to a new location. This is useful for custom workflow screens which would benefit from specific Help documentation.

A new line may be added by clicking the Add button. A highlighted row is deleted by clicking the Delete button. Changes enable the Save and Undo buttons.

Creating entries for MS help files (.chm)

When building a Help configuration entry to accompany a .chm help file which has already been added to the database using the Files Module, there are a few tricks and tip that make the process easier:

  1. Getting the Context Key – The context key is the module file name. As some of these can be quite verbose, an easy way to collect this syntax has been included.

    1. Navigate to the Module and click the help button or press F1. The following window displays informing the user that no Help file currently exists.

    2. Click Yes.

      The context key syntax copies to the clipboard.

  2. Return to the Help Module and paste in the context key.
  3. Getting the MS Help Parameter – If using the “help file opens to a specific topic, if the topic exists” option to link to a certain sub-heading within a .chm file, the required parameter must be included.
    1. Open the .chm file and browse to the desired sub-heading.
    2. Right click and choose properties.
    3. Under the Address (URL) section, the end of the entry displays the file name and link for the specific sub-heading.

  4. Copy this syntax and paste it into the MS Help Parameter.

Importing and exporting help configuration files

The most common way in which the Help is configured is via a full import of a .jehc file. This contains the links to the appropriate help file locations. However, it does not contain the Help files themselves. These must still be imported using the Files Module.

Loading .chm help files into the database

To load .chm help files into the database, the functionality included in the Files Module is the optimal tool (reference the Help in the Files Module).
