Order Document details – break by size

The Break By Size tab allows users to split order line quantities across assigned size curves, or specific sizes selected from curves, based on planned sales and anticipated size inventory across clusters. Sizing analysis is done for all order lines on a document simultaneously.

Break By Size functionality

In addition to the standard functionality, this blade has the following:

Functionality Description
Calculate Calculates the ideal units per size to order by variant and cluster, based on the Size Curve and Selected Sizes for each order line on the order document. See below for examples.
Approve Approves the Break By Size information for the order document and transitions the Workflow Status of the document to Break By Size.
Approve & Route Approves the Break By Size information for the order and launches the Commitment Status dialog. Select the Workflow Status the document will transition to from available states, based on the active workflow.
Enforce Size Minimum (related to Receipt Size Min set in Receipt Plan view of Assortment Planning) When checked, enforces the total minimum quantity per size defined by that constraint and increases the order quantity to meet it, if necessary.
Partition Option Provides additional partitioning metadata by which break by size can be further broken down.

This field is controlled by system policy Default Size Break Partition Table.

Calculations that display the ideal units per size to order by variant and cluster

 

Variant grid (left pane)

  • Variant: The variant code for the corresponding order line.
  • Cluster Set: If applicable, displays the cluster set for the corresponding order line.
  • Cluster: If applicable, displays the cluster for the corresponding order line.
  • Receipt Date: The effective delivery date for the corresponding order line.
  • Size Curve: Displays/sets the size curve to be applied to the corresponding order line. Defaults to the size curve set for the variant in the assortment plan from which the order line was created.
  • Selected Sizes: Displays/sets the sizes to include in the corresponding order line. Defaults to all sizes included in the selected Size Curve.
  • Site Count: The number of sites the corresponding order line is for.

Pivot grid (right pane)

  • Avg U/Site: Sets the desired average units per site for the corresponding size as a constraint for calculation of sizing needs.

    Entering Avg U/Site can cause an increase in the original order quantity for the order line

  • Act Avg U/Site: The calculated average units per site for the corresponding size by dividing the corresponding Order Units by the Site Count for the cluster
  • Quantity: The raw system suggested units to be ordered for the corresponding variant/cluster/size

    This quantity does not include changes for the corresponding user specified Avg U/Site constraint, if any.

  • Constrained Qty: Displays the constrained system suggested units to be ordered for the corresponding variant/cluster/size

    This quantity includes changes for the corresponding user specified Avg U/Site constraint, if any.

  • Final Qty: Displays/sets the units to be ordered for the corresponding variant/cluster/size. Defaults to Constrained Qty value, but can be overridden by the user. User overrides are formatted in orange.

    Final Qty can also be entered in the subtotal and total rows of the Pivot Grid. Doing so proportionally splits the number entered down to the lower levels based on their current values
