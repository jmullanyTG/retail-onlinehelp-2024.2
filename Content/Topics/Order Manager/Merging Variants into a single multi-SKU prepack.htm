Merging Variants into a single multi-SKU prepack

Prepacks are a combination of related SKUs packaged together. In the Order Manager module, the Rainbow Pack Calculator view creates prepacks consisting of multiple Variants (style-color). For example, white shirts in large and black shirts in large both in the same prepack.

Such multi-SKU prepacks containing different style-colors (white and black shirts all at the same size) cannot be created within the Assortment Planning (AP) module as within AP everything is done at the Variant or Placeholder level. In AP, prepacks are created at the Style (6 units, all large white shirts) and Style-Color-Size (4 units, S, M, L, XL white shirts) levels.

The multi-SKU prepacks in Order Manager are really combinations of Style packs created in AP. The existing Style packs are combined into larger prepacks.

The multiple Variant or multi-SKU prepacks:

  • cannot be smaller than an individual Placeholder packing instruction.

  • can contain multiple Placeholders, so long as the original packing multiple for each is maintained within the multi-SKU prepack.

    For example, a box with a maximum capacity of 20 units could hold a combination of 4 SKUs that have an order multiple of 5 each. Therefore, the value entered in the Max Box field must be large enough to accommodate the order multiples for the selected Placeholders.

    Regarding excess space within a box, excess space is not filled if filling the box causes the ratio of Placeholder packs to be out of alignment. The box is not filled in order to hold the alignment among Placeholders in the pack.

  • only be generated by combining the initial multiple style-color single size orders, prior to sending the orders to the ERP.

Macro Assortment Commitment Rainbow Packs must be executed to display the Rainbow Pack Calculator. The Rainbow Pack Calculator is a prepack combination selection tool that uses user input parameters to search for available Variant order consolidations that would make viable multi-SKU prepacks. The parameters define the desired plan to work with, the product filter for Variants within that plan with delivery dates within the window of time, the window of time, and the maximum units or upper limit for inclusion in a box.

The multi-SKU prepacks are grouped by:

  • identical receipt dates

    The Style packs created in AP cannot be edited nor the delivery dates shifted within the the Rainbow Pack Calculator. Thus, for orders to be combined, they must have the same receipt date. However, receipt dates can be changed elsewhere within the Order Manager UI. The dates can also be updated by any changes being fed from the ERP. Any changes to date alignment breaks apart any multi-SKU prepacks that are no longer aligned.

  • common supplier

    A common supplier ensures that the order document and any multi-SKU prepacks are prepared for a single supplier, in case variants are sourced by more than one supplier.

  • and either Placeholder Collection code or Variant Style code (controlled by system policy Rainbow Pack PlaceholderCollection Grouping).

To group Variants as potential multi-SKU prepacks, the Rainbow Pack Calculator may have moved Commitment Variant lines from one order header to another.

For selected multi-SKU prepacks which should not be created, simply uncheck the checkbox in the Selected field in the grid. Once the desired multi-SKU prepacks have been chosen, click the Process Packs button on the toolbar. The Rainbow Pack Calculator then uses predefined packing instructions to create multi-SKU prepacks as prepacks within ToolsGroup JustEnough.

Navigate to the Prepack tab in the Order Document to view the new multi-SKU prepack's details.
