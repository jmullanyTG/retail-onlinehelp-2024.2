Prepack Engine Metric Definitions

The following table provides definitions:

Metric Definitions
Crossover Rate The amount of information that can be shared between parents when creating new offspring (packs) to test with in the genetic algorithm.
Elitism Ratio How many of the best packs from the current population can be carried over to the next generation without any mutations or crossovers.
Max Generations Maximum number of total loops before stopping calculation to prevent infinite loop errors.
Max Overage The explicit value we are allowed to have for an overage when determining a prepack.
Max Pack Size The maximum number of items in each prepack.
Max Packs Per Location The maximum number of packs that can be sent to an individual site.
Max Total Overage (internal engine definition) – Pack Tolerance * Total Need
Min Allocation Ratio The percentage of total packs that any one prepack must have.
Min Pack Size The minimum number of items in each prepack.
Mutation Rate The amount of information that can be changed between generations to create new packs to test with the genetic algorithm.
Number of Prepacks The number of different prepack configurations allowed.
Overage Threshold The percentage overage value we are allowed to have when determining a prepack.
Overage Weight Used for error calculation. It will tell the engine that we are ok with having overage when determining pack sizes and number of packs to each site.
Pack Tolerance The percent overage of total need allowed for allocating to all sites.
Population Size The amount of different initial combinations of packs that start out in the genetic algorithm. Each member of the population can have different or similar combinations of pack sizes.
Termination Generations Number of loops with the same error value before stopping calculation.
Termination Threshold The value compared against of the current generation fitness and the best generation fitness to determine if the fitness value is below the threshold to stop calculations.
Total Pack Weight Used for error calculation. It will tell the engine that we value giving each site as many packs as we can to meet their need.
Underage Weight Used for error calculation. It will tell the engine that we value not having underage when determining pack sizes and number of packs to each site.
