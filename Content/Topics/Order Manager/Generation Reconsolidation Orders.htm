Generating reconsolidation orders

Reconsolidation orders pull excess or leftover stock from stores, back to a distribution center through a semi-automated process. Reconsolidation recommendations can be made based on similar criteria to redistribution and rebalance orders.

Reconsolidation orders recommend a stock transfer up one level in the specified supply chain. This can be used to recall stock at the end of a season or reconsolidate excess stock back to a distribution center, if the stock is unwanted at a store.

If more than one source stock is selected, the minimum value (including zero) is considered as the eligible stock quantity when the reconsolidation order is built. If a source stock is selected with a zero value for the inventory, the reconsolidation run will not return a recommendation for that inventory.

As best practice, select a single source stock per reconsolidation order.

To execute a reconsolidation order

  1. In the Order toolbar, click Reconsolidation.

    The Items window displays.

  2. From the Filter field, select the filter with the eligible items to be included in the reconsolidation order.

  3. Click Ok.

    The Add Reconsolidation Order: Reconsolidation window displays.

  4. Click Ok.

Fields and descriptions

Field Description
Source System generated information used to create the order header document
Workflow The workflow attached to the order header. Defaults to Normal following the standard order document approval workflow.
Name The name of the order document. This field pre-populates with Reconsolidation. If this is not changed, it is appended with the user name of the creator and the date and time at the time of creation.
Description Optional description for the order document.
Owner The user name associated with the creation of the order document.
Security The security setting for the order document.
  • Public: visible and editable to all users.
  • Private: visible only to the document owner and system administrator accounts.
  • Shared: visible and editable to only the specified users in the shared list.
Source Stock Determines eligibility in the filter and the quantity of stock to be reconsolidated.
  • Leftover: the quantity of stock on hand in a store (controlled by site type) if today is on or beyond the planned phase-out date. Specific to allocations and based on the allocations default supply chain.
  • Projected Leftover: the projected quantity of stock on hand in a store (controlled by site type) at the end of the horizon period (accounts for customer orders beyond the phase-out period). Specific to allocations and based on the default supply chain. Projected leftover is inclusive of leftover.
