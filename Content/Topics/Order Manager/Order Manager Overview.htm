Order Manager Overview

Order Manager provides a central location to review, modify, view by status, and authorize orders. New orders can be based on allocations or Ad Hoc, and the summary of the order details viewed in a pivot grid.

Regular supplier orders, direct supplier drop-ship orders, transfers, allocations, commitments, and distributions can all be controlled in a document-centric format while following workflow rules established to conform to the business requirements.

Order Manager documents are available depending on user access and rights. Because these document types are related to different processes, their fields and associated views are likely to differ.

The contents in the header of an Order Manager document depends on the order document type. The header may contain information, such as ownership, security rights, workflow details, and narrative status.

After creation, Order Manager documents can be viewed and updated (based on the header status) through the Navigation pane. Depending on assigned rights, users can view or edit documents in varying states of approval, or advance the document through the pre-defined workflow steps.

Viewing additional information to make better decisions around order quantities

Several Order Document columns provide additional information into units remaining in the DC after an order is generated. This allows for taking the remaining quantities into account when making decisions to update User Order Quantity and consume final units.

The columns are:

Field Description
Source SOH Qty Today DC OH (order line resolved source)
Source OO Qty Today DC OO QTY (order line resolved source)
Source SOH+OO Qty Today DC OH+OO Quantity (order line resolved source)
All Sources SOH Qty Today DC OH (whole supply chain - default)
All Sources OO Qty Today DC OO (whole supply chain - default)
All Sources SOH+OO Qty Today DC OO+OH (whole supply chain - default)
Source Remaining Qty DC Remaining Qty (DC OH-total order)
Total Order Qty from Source Total Order Qty (Sum of user order line quantities. Updates on save.)

Combining order lines to consolidate duplicate lines and reduce processing effort

A macro labeled Combine Order Lines is available when users select the option of This Weeks Orders or Future Recommended Orders (From Strategy, Redistribution). This macro consolidates duplicate lines into the earliest delivery dates per source site and reduces processing effort needed for both DC and stores.

The system is prevented from retrieving multiple order lines by delivery date for the same style/color/size/site/source site. Often the duplicate lines have very small recommended order units and processing such multiple small orders results in increased effort for both DC and stores.

Whether zero order lines should be preserved is determined by system policy Macro:CombineOrderLines - Preserve Zero Lines.

External supplier orders and prepack orders will not be combined.

Ensuring changes to Order Quantity do not exceed quantity of units available for distribution in the DC

The Over Consumption alert eliminates the risk of an order being rejected due to the quantity exceeding available units for distribution in the DC. Order quantities are summed up at a SKU-size-source level and the sum is checked against the available quantity (SOH-stock pending) to see if it exceeds the quantity in the DC. This ensures changes made to User Order Quantity do not exceed quantity of units available for distribution in the DC.

To take advantage of this functionality, select the checkbox labeled Enable Overconsumption Check in the Add Order wizard. The totals can be reviewed in two Order Document fields: Source Rec Order Qty and Source Remaining Inc Rec Order Qty.

If system policy labeled Overconsumption Check is set to Yes, then the checkbox is selected by default. If assigned User Right Allows override of Overconsumption check, users can enable or disable the Over Consumption check at the Order Document Level.

Order Types Not Supported by this feature:

Allocate: From Stock, From Order, From Scenario

Assortment: Commitment

Automatically checking for valid orders

Within the Policy module’s Policy tab, a checkbox labeled Enable Auto Authorized Order Check in the Auto Authorization section allows for automatic checks to be done on the inventory.

The default for new inventory can also be set using system policy Inventory Policy Default: Enable Auto Authorized Order Check.

Performing a data validation check

If selected, a checkbox labeled Enable Data Validation Check at the Order Manager header level performs a data validation check when an Order is Pre-Authorized or Authorized. If a single Inventory fails the check, then the order does not proceed to the next state.

A field labeled Data Validation Check Passed has a checkbox that indicates whether a row passed the check.

Authorizing orders that failed the data validation check

A macro labeled Auto Authorize Order automatically authorizes orders when lines fail the data validation check. These failed lines move to a new order header named [Existing Order Header Name]-Failed Data Validation.

Related system policies

This functionality is controlled by the following system policies:

  • Inventory System State: Days to consider for Last Updated Stock On Hands Validation
  • Inventory System State: Days to consider for Last Updated Supplier Order Validation
  • Inventory Policy Default: Enable Auto Authorized Order Check
