Working with order documents

The majority of work in Order Manager occurs in the Order Document viewing area to the right of the Navigation pane. This area is usually blank when initially accessing Order Manager. The exceptions being:

  • an order was opened after it was created using the Create Commitments option in the Receipt Plan view, or

  • the user clicked a link to a specific order in the Assortment Upcoming Commitments Dashboard.

Double-clicking an order line in the order document displays the same information found in Inventory Manager.

User Rights control access to Order Manager and all the functionality within Order Manager. For a listing of the Order Manager User Rights, see Order Manager User Rights.refer to the online help.

Viewing an order document

Double click an order document in the Navigation pane to display the order document as a separate tab.

Displaying or hiding the Column View name in a document’s caption

A system policy labeled Remove Column View Name from Document Caption lets users stop the Column View name from being appended to the document name in the document’s caption. As the Column View name is important, the name is shown as a Tooltip when the system policy is set to True.

Limiting the total number of columns in order documents

Order documents have a configured default column count. In addition to the default number of columns, users can add additional columns to their order documents. However, documents may become unusable if there are too many columns. To prevent this, system policy Column Max allows for limiting the total number of columns that can be included in an order document.

For example, if Column Max is set to 100 (the default), then users can have a maximum of 100 columns beyond the default column count.

If using the Choose Fields dialog to add columns to a document results:

  • in a total of 100 or fewer columns then there is no change in existing behavior and all columns display.

  • in more than 100 columns then the following message is generated. (The number in the bracket is the total number of columns that can be included in the order document.)

    Clicking OK cancels the operation and no extra columns are added. Users can retry and choose fewer additional columns.

System Policy Description
Column Max (Default 100) This policy limits the number of additional columns in an Order Manager document. Only applies for the number of columns beyond the configured default column count. This prevents the document becoming unusable through having too many default columns.

Order header functionality

Edit

Launches the Update Order Header dialog in which users can change the workflow (given sufficient permissions), name, description, and security settings for an order document.

Delete

Launches a dialog asking the user for confirmation to delete the active order document.

Narrative

Launches the Narrative dialog displaying the history of document status changes for the order document. The User ID of the user who adds a note displays in the Modified User field.

The user right Allow user visibility of order manager document when participating in order narrative stack is required to view order manager documents within the narrative.

Selecting the Redacted checkbox allows the user who added the note to hide it from viewing for other users.

  • Order: Indicates the order in which the status changes in the narrative occurred.
  • status fields: The date the order document moved to the status indicated, status description, and the user ID of the user owning the document after the status change occurred.

Add Note

Launches the Add Order Note dialog, allowing users to add standalone note entries to the order document narrative.

Sets the text for the note to include in the narrative entry.

Pre-Authorize

Launches the Pre-Authorize Order dialog if allowed by the workflow permissions assigned to the order document. Otherwise, disabled or hidden. Authorizes the order only within the context of ToolsGroup JustEnough, it does not forward the order on to the host system or system of record.

Therefore, it logically partitions the stock separated out as if it has been already authorized but it does not trigger the communication to actually action the order. This communication only happens when the order is actually authorized.

Users can pre-authorize an order and then return later and decide whether to continue with the order or not.

Selecting Ok updates the document status to Pre-Authorized, locking it for editing, and adds a corresponding entry to the order document narrative.

Authorize

Launches the Authorize Order dialog if allowed by the workflow permissions assigned to the order document. Otherwise, disabled or hidden. Same as the Pre-Authorize dialog, except clicking Ok updates the document status to Authorized.

Authorized orders are uploaded to external systems per the upload schedule defined during software implementation.

Forward

Launches the Forward Order dialog. Relates to structured workflows where an approval from other parties is required. The other parties can forward the order.

  • Forward To: Selects the User ID of the person to whom the order document will be forwarded.
  • Cancel: Closes the dialog without forwarding the order document.
  • Ok: Updates the document status to Forwarded, and adds a corresponding entry to the order document narrative.

Retract

Launches the Retract Order dialog if allowed by the workflow permissions assigned to the order document. Otherwise, disabled or hidden. Allows users to bring back an order they forwarded and either edit or delete the order. However, the original user cannot retract the order if the order has been forwarded on by the next party.

Clicking Ok updates the document status to what it was prior to being forwarded and adds a corresponding entry to the order document narrative.

Reject

Launches the Reject Order dialog if allowed by the workflow permissions assigned to the order document. Otherwise, disabled or hidden. This allows a user to reject an order document that has been forwarded to them with a note explaining why.

Clicking Ok updates the document status to what it was prior to being forwarded and adds a corresponding entry to the order document narrative.

Status > Edit (Assortment Plan Commitment order type only)

Launches the Commitment Status dialog. Users can change the current order status using the available options as defined in the workflow assigned to the order document.

Grid button

Shows (default) or hides the Grid view (order lines view) in the order document viewing area.

Grid view is locked to show unless the Details view is also showing, in which case it can be hidden.

Details button

Shows or hides (default) the Details view in the order document viewing area. The Details view displays the same information found in Inventory Manager.

Orders generated in Assortment Planning

For orders generated in Assortment Planning, the Details view contains the Break By Size and Prepack views.

For more information, refer to the Assortment Planning User Guide.

Refresh button

Refreshes the order document Grid view.

Grid parameters

Name: The order document name.

Description: The order document description.

  • Indicates there is a description present for the order document
  • Indicates no description is present for the order document

Workflow: The workflow currently assigned to the order document.

Status: The current workflow status for the order document.

Source: The order document type (always Assortment Plan Commitment for order documents created from assortment plans).

Owner: The User ID of the user who created the order document.

Security: The security setting for the order document.

Grid functionality

Save: Saves changes made to the order document in the Grid view since the last time it was saved.

Undo: Reverses any changes made to the order document in Grid view since the last time it was saved.

Columns

Choose: Launches the Choose Fields dialog in which users can select and include additional columns to provide more information on the items in a given order.

Calculated: Launches the Create Calculated Column dialog.

  • Column Name: Sets a name for the new calculated column
  • Column Type: Selects the data type for the new calculated column
    • Integer: use for data that will consist only of whole numbers
    • Decimal: use for data that requires decimal precision
    • DateTime: use for data that specifies, dates, times, or both
    • String: use for data that will be formatted and treated as text only
    • Boolean: use for binary (true/false) data

  • Format String: Specifies the way data in the new calculated field is to be formatted.

    Entering #,###.00 yields numeric values with commas every three places to the left of the decimal and rounds to a static two decimal places to the right of the decimal. 1000.9 would be formatted 1,000.90 and 10.891 would be formatted 10.89. A format string of 0,000.00 instead yields 0010.89.

    Entering mm/dd/yyyy yields date values with two digits for month and day and four digits for year, with month, day, and year separated by slashes (11/19/2021 for example).

The Expression Editor is used to write custom expressions with which to populate new calculated columns.

Order line

To select multiple order lines, use CTRL + Click or Shift + Click.

Add (Not applicable to the Assortment Plan Commitment order type)

Adds order lines based on an inventory filter (Item code by site code).

Order lines are returned even if they have zero orders as long as they are contained within the filter. Once order lines with zero orders are returned, then users can put in an order quantity. This can be useful if a user wants to structure the document to include what’s necessary based on the requirements of a vendor, an ERP, or a WHMS.

Copy

Launches the Copy dialog allowing users to copy selected order lines to another order document in the same workflow. Only highlighted order lines are copied. Copied order lines are not deleted from the original document. Therefore, the order lines will be in the system twice. In this case, there is the possibility that both lines will be approved and transmitted.

Users can create a brand new order document as the target order document using the Copy dialog.

To create a new order:

  1. Leave <Create New Order> as the selection in the Copy To field. The Add Ad Hoc Order dialog displays.

  2. Complete the fields as required.

  3. Click Ok. The new order document is added to the user’s workspace in the Navigation pane and also displays as a separate tab.

Move

Launches the Move dialog allowing users to move selected order lines from the current order document to another order document. This function is essentially the same as cutting and pasting the order lines. The order lines are removed from the original order document and added to another document.

The workflow for creating a new order document is the same for both Copy and Move.

If a user plans on executing an order, then they should only use the Move function. Otherwise, there is the risk of running some lines twice.

Copy + Authorize and Move + Authorize

These are single actions that accomplish repetitive actions. Highlighted selections are automatically copied or moved and then authorized.

Delete

Deletes the selected order lines. This removes the order line from the order document. However, the order line still exists in the recommendation and can be refreshed, it just is not in the context of the order document anymore. The order line can be re-added using the filter.

The system does not ask for confirmation prior to deleting the order lines. However, this action can be reversed with the Undo button.

Merge, Unmerge: (Assortment Plan Commitment order type only.)

Merges two or more order lines that share the same variant into a single line. Merged lines can be unmerged.

CheckBox Mode

Toggles between displaying or hiding a checkbox. This mode requires the user to check individual order lines. Only selected lines are actioned.

Grid right-click menu options

The grid right-click menu options include:

Export: Exports grid data to a file in the selected format.

Export Options: Sets text export options (Value or Display Text).

Import Excel Data: Allows users to import data directly from MS Excel spreadsheets. A confirmation dialog indicates how many rows were updated. For more information, see Importing data from spreadsheets.

Grid fields

The columns and labels that appear depends on the order document type selected. The columns that display or can be added are a subset of all the fields available in the Inventory metadata (fields can be added or removed using the Choose function).

Fields prefixed with user and fields prefixed with Recommended

The grid will contain two similar groups of fields: one group with a prefix of User and one group with a prefix of Recommended.

Fields prefixed with Recommended are system generated and read-only. Fields prefixed with User are editable and over-write the system generated equivalent (these fields are read-only for pre-authorized and authorized order documents).

For example, Recommended Source Site and User Source Site. The Recommended Source Site field is populated with DC - US East. Users can choose a different source site by entering it in the User Source Site field.

Constraints

If constraints have been added, users can run an order constraint macro that will enforce the constraint.

For example, a user’s budget is $15,000 and the sum of the User Order Cost Value is $16,000. In this case, the user can run a pre-configured constraint macro to reduce the cost value and stay under $15,000.

For more information, see Ordering Constraints Overview. For more information, refer to the Ordering Constraints User Guide.

Reviewing whether inventory orders meet constraints

The Constrained Status metric identifies those orders that did not meet a constraint. There are three possible statuses based on applied constraints:

  • Failure
  • No Constraint
  • Success

  • Code, Description: The code and description for the order line. The label is prefixed with:

    • Variant for Assortment Plan Commitment order types
    • Item for inventory order types
  • Color Code: The color code for the variant.
  • Color Description: The color description for the variant.
  • Cluster Name: The cluster name the order line was generated for. Only applicable for commitment orders generated from Receipt Plan view in Assortment Planning.
  • Cluster Description: The cluster description the order line was generated for. Only applicable for commitment orders generated from Receipt Plan view in Assortment Planning.
  • Cluster Set Name: The Cluster Set Name.
  • Cluster Name: The Cluster Name.
  • Grouping Identifier: Allows the user to enter text to create ad hoc pre-pack groups for variants contained in order lines allowing for prepacks that span more than one variant to be created. Used only in the Prepack tab of the Details view.
  • User Order Date: The order date for the line. Defaults to the date the order was generated.
  • User Ship Date: Allows user to manually enter shipping dates communicated from suppliers if systemic transfer of shipping dates is not possible. Defaults to the User Order Date.
  • User Delivery Date: The target receipt date for the order line. Defaults to the Sunday of the receipt period defined in Assortment Planning.

  • User Order Quantity: The total order quantity in units for the order line. Defaults to the receipt units quantity for the corresponding receipt period in Assortment Planning.

    Orders generated from Receipt Plan view have order lines by Cluster in which case the order quantities are specific to the corresponding receipt units quantity for the cluster specified by the order line. Orders generated from within Order Manager are consolidated by variant by default but can be unmerged into their cluster/order line counterparts.

  • User Cost Price: The cost price per unit for the order line. Defaults to the receipt AUC for the corresponding receipt quantity and period in Assortment Planning.

    Orders generated from within Order Manager that represent multiple cluster level orders with differing AUC values display a weighted average in this field.

  • User Order Cost Value: The total cost for the order line (calculated as User Order Quantity * User Cost Price).
  • Effective Order Date: The date the order is confirmed received by external systems or the User Order Date if no confirmation has been received.
  • Effective Ship Date: The most current ship date information sent by external systems.
  • Effective Delivery Date: The most current delivery date information sent by external systems.
  • Effective Order Quantity: The most current order quantity information sent by external systems.
  • Effective Cost Price: The most current order cost per unit information sent by external systems.
  • Effective Order Cost Value: The most current total cost for the order line (calculated as Effective Order Quantity * Effective Cost Price).

Viewing roll-ups and grand totals using a pivot grid

Click the Pivot button to replace the grid view with a pivot grid. The pivot grid allows users to viewing roll-ups and grand totals. If viewing the pivot grid, then click the Pivot button to see the grid view.

Grouping order lines by metadata

Click and drag columns to the area above the column headers to group by any column in the order document grid view.

Adding or removing columns

To add or remove columns from the grid, right click on any column header and select Column Chooser from the context menu.

This launches the Customization dialog.

Drag and drop the column headers from the dialog to the desired locations in the grid to add them to the view.

To remove columns, drag the column headers to the Customization dialog or right-click the column header and select Hide Column.

Columns can also be reordered within the grid by dragging them to the desired location

Closing and floating tabs

The following right-click options are available on tabs:

  • Close: Closes the order document
  • Close All But This: Closes all open order documents except the one right-clicked (only available when multiple documents are open)
  • Float: Converts the document tab to a window that can be sized and positioned on the screen (or other screens, in multiple monitor configurations)
  • Viewing Order Documents

Floating can also be achieved by clicking and dragging the document header without right-clicking and selecting Float. After floating, click the document header and drag the document to the desired location. Right click the document header and click Dock as Tabbed Document to return the document to its tabbed position. Hover over the directional buttons of the positioning reticle that appears while dragging to see possible docked locations for the floated window.

New Vertical Tab Group: Configures all documents currently being viewed vertically side by side. After clicking this option, right click the document header again and select Move to Previous Tab Group to return to the last screen configuration.

New Horizontal Tab Group: Configures all documents currently being viewed horizontally stacked. After clicking this option, right click the document header again and select Move to Previous Tab Group to return to the last screen configuration.
