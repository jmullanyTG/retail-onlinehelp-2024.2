Generating rebalance orders

A rebalance order is a redistribution order that is run between distribution centers (DCs). These orders pull excess or leftover stock from a DC and redistribute the stock to a DC with unfulfilled sub-branch demand.

Rebalance orders are primarily filter driven to determine eligible items. After the desired Inventories have been selected, the quantities to rebalance are driven by the source stock selection options.

Rebalance orders can only be run across a sibling level tier within the normal supply chain (that is a DC may not redistribute to a store, as that is a normal distribution. A store may not redistribute to a DC as that is a reconsolidation action. To resolve these tiers, the Replenish Engine references the default supply chain defined using system policy Default Supply Chain (which is of type site/supplier).

Setup

The following objects must be set up before generating a distribution order:

  • a filter that includes the items and/or sites to distribute.
  • the supply chain to use.
  • demand forecasts for the items to distribute.
  • stocking strategies for the items to distribute.

To generate a rebalance an order for an item

  1. From the Execute tab, click Order Manager.

    The Navigation menu displays.

  2. In the Order toolbar, click Rebalance.

    The Items window displays.

  3. From the Filter field, select the filter with the items and sites to be included.

    The Add Rebalance Order: Rebalance window displays.

  4. Select:

    • the item to be rebalanced from the Supply Chain dropdown menu.
    • the Source Stock.

    • what type of orders to rebalance (for example: Today's Orders, This Week's Orders).

      If more than one source stock is selected, the minimum value (including zero) is considered when building the rebalance order. As such, it is best practice to select a single source stock per rebalance order.

  5. Click Ok.

    The recommended orders are calculated and display on the screen. Each row represents a different recommended order and contains basic details of the order.

Once a rebalance order has been generated, double-click rows to view the demand forecast, stocking strategy, stock holding projection, and other details that drove the order recommendation.

Rebalance order wizard field descriptions

Field Description
Source System supplied information used to create the order header document.
Workflow Once created, the workflow attached to order header. Defaults to “normal”, which follows the standard order document approval workflow.
Name The name of the order document. Pre-populates with Rebalance. If unchanged, is appended with the user name of the creator and the date and time of creation.
Description Optional description for the order document.
Owner The user name associated with the creation of the order document.
Security The security setting for the order document:
  • Public - Visible and editable to all users.
  • Private - Visible only to the document owner and system administrator accounts.
  • Shared - Visible and editable to only the specified users in the shared list.
Supply Chain The supply chain used to determine stock transfer links. This supply chain must be of redistribution type (the selection options will be filtered to only include the correct type of supply chain). A redistribution supply chain may be configured within Supply Chain Manager.
Source Stock The source stock is used to determine eligibility within the filter and the quantity of stock to be rebalanced.
  • Leftover – Quantity of stock on hand in a store (controlled by site type) if today is on or beyond the planned phase-out date. Specific to Allocations and based on the allocations default supply chain.
  • Projected Leftover – Projected quantity of stock on hand in a store (controlled by site type) at the end of the horizon period (accounts for customer orders beyond the phase-out period). Specific to Allocations and based on the default supply chain. Projected leftover is inclusive of leftover.
  • Excess – Excess stock quantity defined as stock on hand greater than LT + SS + RC. Specific to Replenishment.
  • Excess for Redistribution – Excess stock quantity defined as stock on hand greater than LT + SS + RC + a factor as a function of forecast quantity. Specific to Replenishment.
This Week’s Orders Selects the time frame context for the engine orders. If selected, returns orders for the current week.
Today’s Orders Selects the time frame context for the engine orders. If selected, returns orders for today.
Future Recommended Orders Selects the time frame context for the engine orders. If selected, returns orders for the specified time frame in the context of orders placed or orders delivered.
Use Pre-Calculated Orders If selected, creates the order document with the most recently cached order values for the selected inventories. These may be from an overnight task run or from a user driven action. If this is unselected, a Replenish Engine calculation runs creating the order document.
Exclude Uploaded Orders If selected, any uploaded orders within the filter context will not be included within the order document.
