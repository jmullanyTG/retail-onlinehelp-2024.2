Supplier production constraints

When an order line is selected, the Supplier Capacity Report blade, which displays when the Details toolbar button is clicked, populates with all the suppliers that have the order line’s construction type. Planners can review how much production capacity is left for each supplier of the construction type and how much of the capacity is taken up by pending stock. Supply planners can also manage scenarios where all suppliers are unable to fulfill the order so that exceptions are managed effectively.

Often times a wholesaler’s constraint lies with supplier production. In this scenario and bearing in mind each supplier can produce X amount of construction types a month, a supply planner:

  • looks at all the unconstrained orders today

  • adjusts the order quantity based on constraints such as production capacity or orders for a particular vendor has been particularly low.

By default, the Supplier Capacity Report displays recommended orders for today without any supplier constraint applied. Planners can review and apply supplier constraints to recommended orders as needed by applying the supplier production limit of a construction type to the values in the User Order Quantity field in the top grid using macro Apply Supplier Production Limitation. The values update automatically with the results displaying in the Supplier Capacity Report blade.

In the above example, the value in the Pending Order Quantity field is the sum total (64) of the previously authorized orders (33, 31) shown in the User Order Quantity field.

The Supplier Capacity Report is read-only and color formatted as follows:

Color Remaining Percentage
Red < 30% (includes negative numbers)
YELLOW 31 to 70 %
GREEN 71 to 100 %

The default value that displays in the Remaining Quantity field is set in the Supplier Manager module against the bucket with the desired delivery date. Another product with the same Attribute contributes to the capacity. Failing to run the Apply Supplier Production Limitation macro may, upon an order being Authorized, result in the value in the Remaining Quantity field turning negative due to an over commit. In this scenario, Planners can use the Retract toolbar functionality to reverse the authorization and then run the macro against the order. This will add new order lines in the top grid and leave a positive value in the Remaining Quantity field.

Order lines that go beyond a supplier's production capacity split with the excess overflowing to the next ranked supplier of the same construction type. The resulting multiple order lines have the same order date, but different order quantities and source supplier. Order quantities that exceed all available supplier capacity default to a supplier called Not Known which has a limited capacity. Users should examine all Not Known suppliers and either push the order and/or delivery date to a future production date or find additional supplier capacity to satisfy the order quantity of that construction type. Suppliers that have not been configured have a limited capacity rather than no capacity. The process does not look ahead to take orders from the next bucket (time period).

Viewing Orders with Supplier constraints

Planners can see all the different suppliers for a selected order line based on the order line's construction type (construction types have a 1-to-1 relationship to items), the production capacity of each supplier over time, and how much of the production capacity is being used already by authorized orders.

The Order Date defines which recommended order lines of the same construction type are processed first with the earliest order date going first. The order in which suppliers are used to fulfill a construction type is dictated by the supplier construction priority provided during import. However, if all the orders have the same delivery date, construction type, and source supplier then order number is used.

Dates do not change. Therefore, policies such as schedules and lead time do not apply. However, policies Minimum Order Quantity and Maximum Order Quantity do apply.
