Scenario Plan grid fields and descriptions

The Scenario Plan grid manages the configuration of scenario plans.

  • Access Level – Defines user access to the scenario plan.
    • Public – All users within the application.
    • Private – Only to the creator.
    • Shared – Only specific users.
  • Choice Count – The available choice count models. Must be populated to pass configuration status requirements.
  • Configuration Status – The status of the scenario plan. Any missing configuration components display in a comma separated list in this field. Missing configuration elements prohibit scenario calculation.
  • Demand Plan – The available demand plans. Must be populated to pass configuration status requirements.
  • Location Strategy – A key component to calculation of both choice count and allocation volumes, the location strategy specifies the stores which are eligible to receive stock as part of the allocation. For more information, see Location strategies. For more information on Location Strategies, refer to the Allocations User Guide.
  • Max Pick Ratio – The maximum pick quantity for the draft engine based on a percentage of the ideal pick value (calculated as need target/choice count). This field is hidden by default and defaults to system policy specified value (initializes to 200%).
  • Min Pick Ratio – The minimum pick quantity for the draft engine based on a percentage of the ideal pick value (calculated as need target/choice count). This field is hidden by default and defaults to system policy specified value (initializes to 50%).
  • Plan Description – An optional description for the scenario plan.
  • Plan Length – The expected duration of the stock coverage from the resulting plan. Though not restricted to aligning with the selected demand plan time frame, it is considered best practice not to deviate from the already specified length of time.
  • Plan Name – The unique name for the scenario plan.
  • Plan Status – Defines the workflow options available for the scenario plan. Updates based on changes made to the plan. Statuses include:
    • Requires Configuration – Configuration elements are missing. Refer to the Configuration Status field for additional information. No workflow options are available until configuration is complete.
    • Configuration Changed – Configuration elements have changed and a plan calculation is required. Any results currently displayed within the scenario details section are related to the previous configuration and are considered to be stale. Workflow options are identical to those of the Ready to Calculate status.
    • Ready to Calculate – Once configuration is complete, the plan status updates to Ready to Calculate and the draft engine may be run. Workflow options are identical to those of the Configuration Changed status.
    • Ready to Execute – Once calculation has been completed, the scenario plan results are available to view within the scenario details section and the plan status changes to Ready to Execute. From this workflow status, users can create orders if the results are satisfactory or alter the configuration to produce different results. This updates the status to Ready to Calculate.
    • Scenario Finalized – Once orders have been created, the scenario is finalized and is read-only to prevent order duplication.
  • Product Set – The available product sets. Must be populated to pass configuration status requirements.
  • User Select – If set to a shared access level, the list of shared users.
