Location plans as demand source

If Location Plan is selected as the demand source, the following fields display in the Detail Configuration window.

There is no need for a product model as financial targets have already been set by store.

  • Product Hierarchy – (read-only) Displays the system policy defined product hierarchy. It is purely informational.
  • Product Selection – A mapping of the applicable levels within the product hierarchy which will be sampled to create the demand plan.
  • Plan Name – A selection of the available strategic merchandise financial plans.

  • Include Core Products – If core products are contained within the planned financial targets, they can have their contribution removed by de-selecting the Include Core Products flag.

    Core products are defined as any product-location combination with a demand forecast value greater than zero for a future period through management in Demand Forecasting and Replenishment or Allocations.
