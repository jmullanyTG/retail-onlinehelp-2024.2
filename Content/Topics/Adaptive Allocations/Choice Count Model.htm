Choice counts

Choice counts help determine the quantity of picks available to each store when meeting their allocation volumes. The choice count is determined by selectively sampling historical sales data. Choice count models are built to sample sales history based on a filter of items, a time frame, sales types, and then removing disqualified sales periods.

Choice Count screen

Choice count models can be created and configured for consumption during the scenario plan building phase within the choice count model grid.

New plans are created by selecting New. Fill in all fields before saving. The access level defaults to Public unless otherwise specified.

Choice count models (even linked choice count models) can be deleted for users with appropriate permissions by highlighting row and selecting Delete.

The Choice Count screen can be undocked and realigned or moved to a secondary monitor by clicking and dragging the tab to the desired location.

Fields

  • Choice Count Model Name – The unique name for the choice count model. Used to look-up choice count models within the scenario plan configuration screen.
  • Choice Count Model Description – An optional description for the choice count model.
  • Product Filter – The filter of items used to calculate the choice count. Though not required, a more consistent result is expected if this filter has some relevance to the product selection used within demand plan generation.
  • Time Frame – The time frame used to sample sales history.
  • Sales Volume % of Max Threshold – The percentage of the maximum sales volume of the sample which must be met for a selling period to be included. For more information, see Sales Volume % of Max Threshold .

  • Active Variant Threshold Factor – The percent of each store-variant’s average sales which will set the minimum threshold for de-seasonalized values. This value is available for use when calculating longer sales periods with highly seasonal products within the sample, but is otherwise not applicable to many common Adaptive Allocations scenarios.

    The default value for this threshold is 0. This field is hidden by default. Use the Choose Columns functionality to display this field. For more information, see Active Variant Threshold Factor.

  • Minimum % of Selling Periods Threshold – Defines the percentage of periods within the sample to have recorded a sale for the variant to be considered a part of the choice count model. Setting this to a high value eliminates low selling periods for variants which may have set late or had stock-out related issues. For more information, see Minimum % of Selling Periods Threshold.
  • Sales Types – The sales types included in the sampling. Options include regular, promotion, markdown, and all supersession sales types (should be used for modeled stores and new products).
  • Access Level – Used to define user access to the choice count model. This controls its visibility in this screen as well as restrict its selection for use within a scenario plan.
    • Public – Available to all users within the application.
    • Private – Available only to the creator.
    • Shared – Available to only specific users.
  • User Select – If set to a shared access level, the list of shared users.
