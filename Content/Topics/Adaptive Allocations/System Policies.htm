System policies used to support adaptive allocations

System policies are used to support Adaptive Allocations. Certain policies are shared with Allocations and others are exclusive to functions within Adaptive Allocations. All policies will need to be configured before operating Adaptive Allocations.

System policy Description
Accounting Type This policy appears in the General section as it is a global default for ToolsGroup JustEnough implementation. This will determine the accounting method of stock valuation used for the organization (cost or retail). Within Adaptive Allocations, this will determine if price values or cost values are used for target to unit conversion.
Average Inclusion Threshold Default value for average inclusion threshold, which is the percentage of a variant's maximum sales which must be met for a period to be considered in the de-seasonalization process. Periods of sales for the variant below the threshold will be removed before de-seasonalization. Defaults to 3%.
Active Variant Threshold Factor Default value for active variant threshold factor, which is the percentage of a variant's de-seasonalized average sales which must be met for that period to be considered "Active". Defaults to 0.
Active Variant Time on Offer Minimum Default value for active variant time on offer minimum, which is the percentage of the scenario time frame during which an item must have been on offer in order for it to be considered "Active". Defaults to 30%.
Calendar Calendar used within the module. Defaults to “1” (default calendar).
Cost Type The cost type value used when determining the inventory level target to units conversion factor. Defaults to “1” (default cost).
Currency Default base currency for display within the module.
Item Master Hierarchy Identifier for the product hierarchy used within the module. Should be configured as part of the initial solution design.
Maximum Pick Ratio Default Defines the maximum pick ratio if it is not supplied at within the scenario. Defaults to “2”.
Minimum Pick Ratio Default Defines the minimum pick ratio if it is not supplied at within the scenario. Defaults to “0.5”.
Period Type The period type (day, week, month) used within the module. It is highly recommended to keep this setting consistent with the period type used within Allocations. Defaults to “2” (week).
Price Type The price type value used when determining the inventory level target to units conversion factor. Defaults to “1” (default price).
