Creating a master plan

The first action when working in the Financial Plan Maintenance view is to create a master plan.  A master plan is a Financial Plan that is considered to be the single version of the truth.  Master plans should be used as approved plans within the planning process.  Master plans contain historic and plan periods.  Historic periods represent what actually happened within the business and plan periods are the approved plans for future periods. 

In managing a Master plan it is recommended that a super user or system administrator has control over this feature.

To create a master plan:

  • Click the New button on the toolbar.

    The Maintain Financial Plan window displays with tabs available for creating a new plan.

    When the screen is completed, click Ok to create the plan based on all the criteria that was defined in the tabs contained in the Maintain Financial Plan window that display when creating plans.  New plans display automatically.

To edit a master plan:

  • Click the Edit button on the toolbar.

    • The Maintain Financial Plan window displays with tabs available for editing an existing plan.

Initializing a plan

To initialize a plan, click the Initialize icon in the header of the Financial Plan Maintenance view. The Financial Plan Maintenance view displays and the plan is assigned the status of Initialized.

Tab descriptions

General tab

The General tab is used to configure the master plan's name and description, plan type, currency to work with, data values, calendar to plan with, the level of the calendar, and start period.  These general settings form the basis of the structure of the master plan. 

Name:  The name of the plan

Description:  What the plan is and any other important information about the plan that may be important to the user group.

Plan Type: The metric set and calculations that will make up the master plan.

  • Hybrid Cost-Based Plan: This plan provides the ability to plan by Channel/location groups at defined levels of the product hierarchy.  This plan contains the ability to plan sales, cost of sales, margin, inventory, receipts, and open to buy to support the budgeting process.  This plan is for clients who value their inventory at cost.

  • Hybrid Cost Dual-Markdown Plan (HCDM): This plan type allows organizations to plan their permanent and promotional markdowns separately. Promotional markdowns are tied directly to Promotional Sales and reduce AUR for periods in which they are realized, impacting margin through that reduction. Conversely, because markdown sales do not necessarily occur in the same period in which the actual markdowns are taken, Permanent Markdowns are managed strictly as a percentage of sales, and are subtracted directly from the Maintained Margin value for the period in which they occur.

    In this plan type, average unit cost (AUC) and average unit ticket (AUT) do not typically change from period to period because of changes to the sales plan. AUC changes as a result of incoming receipts or other changes to the inventory mix. AUT only changes if the user changes it directly or indirectly, through a change in IMU %.

  • Hybrid Retail Dual-Markdown Plan (HRDM): Behaves in the same way as the Hybrid Cost Dual-Markdown plan type. This new plan type applies a Cumulative Markup (CMU) percentage to retail values to determine margin, cost of goods sold, and ending inventory for cost. This allows planning teams to focus on retail values and AUR while the CMU drives the efficiency and accuracy of financial plans.

    As the name implies, there are two markdown metrics in the HRDM plan. The Markdowns (Perm) and Markdowns (Promo) fields are enabled for the Hybrid Retail Dual Markdown plan type, as well as the Markdowns (Total) field. All are plannable.

    Hybrid Retail Dual Markdown plan-specific metrics

    There are Current plan year, Comparison, and LY versions of all of the below metrics. The only user-editable metric described below is the Season Begin metric.

    Metric Name Calculation and Notes
    Season Begin

    • No calculation.

    • If enabled by a system policy, this metric is only editable by users in the Administrators role.

    • The Season Begin metric is always editable and never actualizes.

    • When a time-aggregate is modified (e.g. a month or quarter), the first period within the aggregate is modified while the other periods are left alone.

    • When Season Begin is changed, the change is applied to the same period for the whole plan, including product and channel hierarchy levels that are not selected.

    Season-To-Date Metrics GAFS (Goods Available For Sale) metrics

    Season-To-Date metrics recalculate from each Season Begin period, even if that period is actualized.
    GAFS Sellthrough % IF(Season-To-Date GAFS Units = 0, 0, Sales Units / Season-To-Date GAFS Units ) * 100
    Season-To-Date GAFS Cost BOP Cost from Season Begin + cumulative season-to-date Target Receipt Cost + cumulative season-to-date Inventory Adjustment Total Cost
    Season-To-Date GAFS Retail BOP Retail from Season Begin + cumulative season-to-date Target Receipt Retail + cumulative season-to-date Inventory Adjustment Total Retail
    Season-To-Date GAFS Units BOP Cost from Season Begin + cumulative season-to-date Target Receipt Units + cumulative season-to-date Inventory Adjustment Total Units
    Season to Date GAFS CMU% (Season to Date GAFS Retail - Season to Date GAFS Cost ) / Season to Date GAFS Retail

    Default GAFS (Goods Available For Sale) metrics

    The Default GAFS metrics have the same formula as Season-To-Date GAFS metrics. Where they differ, is that: Season-To-Date metrics calculate from the Season Begin period, even if that period is in the past. Default metrics calculate from the last actualized EOP value.
    Default GAFS CMU%

    (Goods Available For Sale)

    		 IF(Season-To-Date GAFS Retail = 0, 0, ( Season-To-Date GAFS Retail - Default GAFS Cost ) / (Season-To-Date GAFS Retail))
    Default GAFS Cost Period of Season Begin: BOP Cost + Inventory Adjustment Total Cost Rest of season: (previous period Season-To-Date GAFS Retail * (1 - EOP CMU % )) + current week Inventory Adjustment Total Cost

    Metrics disabled in Hybrid Retail Dual Markdown plans

    AUT and IMU% are both not editable in the HRDM plan type, because their formulae have changed. AUT is now equivalent to BOP AUR, and IMU% is equivalent to BOP Markup %. Both are now automatically calculated. Thus, the IMU % calculation toggle is disabled in HRDM.

Plan Sub Type: The two options for Plan Sub Type are Default and Original Plan. For more information, see Plan Sub Types.

Plan Currency: The base currency to work with.

Management Type: Set to Manual by default. This field can be edited after the creation of a Master or Branch plan and can be updated as and when required. Additionally, Master and Branch plans can have different Management Types.

Type Description
Open The plan can be modified both by imports and within the application. A “last in” rule is used here. If an import runs after user modifications are made, the imported values overwrite the user values, and vice versa. If values are intended to be locked for UI or import modifications only, the Management Type should be changed.
Manual The plan can only be modified through the application. This plan is not a valid destination data set for imports.

If an import post runs for a plan of type Manual, then the plan is not affected and an error displays.
Import The plan can only be modified through imports.

Further, actualized values can be imported via the API import process. If a historical period is provided, it must either be: a) for a master plan, or b) for an Original Plan (OP) branch plan type. Historical data is not valid for Default typed branches. Such data is discarded with a warning in import logs.

Note that some consideration must be paid between timing API imports and MFP maintenance. If the API import runs and imports actualized data for the last 4 weeks, but then the default MFP maintenance runs for the last 8 weeks, then all of the values imported in the first step is zeroed out and replaced with whatever values are found in the other standard data feeds (sales history, inventory history, etc). Some care needs to be taken to set up and sequence imports and maintenance so that the desired plans, metrics, and periods have data maintained by the desired feed.

When import posts run for plans of type Import, hierarchy levels are automatically added to the plan based on the import data. If desired, the API can be run in Window mode, deleting all existing data in a plan and re-loading it from the import data.

Seed Type:  Defines how future periods in the plan are seeded for the metrics offered within the plan type that has been selected. There are three standard seeding options to choose from. Seed Types cannot be combined to seed future periods when creating the plan.

  • Historic Sales LY: Seeds future periods with sales, sales units, and average unit retail from last year’s comparable periods.
  • Historic Sales LLY: Seeds future periods with sales, sales units, and average unit retail from two years ago comparable periods.
  • Forecasted Sales: Seeds future periods with sales units that are coming from the forecast module.  Sales are derived by multiplying the forecasted sales units times the average unit retail from last year’s comparable periods.

Regardless of the selected Seed Type, metrics that would be seeded with negative values based on the available data indicated by that type are seeded with zeros instead.

Calendar: The NRF retail planning calendar used in the master plan.  Users can select a 4-4-5 calendar or 4-5-4 calendar.  It is possible to import custom calendars during implementation of this module if necessary.

Calendar Level:  The lowest level of time used in the master plan.  There are three standard options: Year, Month, and Week. Most plans created in this module go to the Month or Week level.

Start Period:  The first period contained within the plan.  This can be a historical period or future period.

Stock Split Factor Type: (Only available at the Master Level. This functionality is not available at the Branch level). The default is Legacy MFP. Lets users select a custom DC split factor. Custom stock splits are editable. Legacy types are read-only.

Stock Split Supply Chain: The supply chain used to determine how inventory is split among DCs. (The Supply Chain Status field indicates whether the supply chain is valid for use with this plan.)

Stock Split Sales Types: The sales types to consider when apportioning stock to DCs.

Supply Chain Status: Whether the supply chain is valid. (For more information, see Validating a Supply Chain.)

Owner:   Disabled until the plan is created.  Once the plan is created the system displays the name of the user who created the plan.

Creation Date:  Disabled until the plan is created.  Once the plan is created the system displays the date the plan was created.

Status:   Disabled until the plan is created.  Once the plan is created the system displays the status the plan is in.  Plan status is covered in the Plan approval section of this document.

Product tab

Product hierarchical structures contained within plans are defined in the Product tab.  The product hierarchy levels selected define the levels users can work with.

Tab layout

Plan levels are selected in the top section of the screen. Users determine which categories are within the master plan in the bottom section.

Channel tab

Location hierarchical structures contained within plans are defined in the Channel tab.  The location hierarchy levels selected define the levels users can work with.

The top section of the screen is where the plan levels are selected. The bottom section is where users determine which channels are contained within the master plan.

For information relating to adding Wholesale to the hierarchy, see Including Wholesale In Channel Hierarchy.

User Access tab

The User Access tab is used to define user and role access permissions.  In the bottom section, setting up the permissions requires that the user navigate through three tabs: Product, Channel, Merge Measure Set. 

Access permissions by channel are defined in the Channel tab. In the Merge Measure Sets tab, measure sets can be merged into the Master Plan from a branch that belongs to that master plan.  Measure sets are defined in the Header Area controls section. 

View Collection tab

The View Collection tab defines the collection of metrics that are contained within the plan as well as the collection of metrics that are contained within any snapshot taken of the plan. 

View collections are set up by clicking Add for both the top and bottom window. Upon clicking Add, select the Plan Type and View Collection set that is to be associated with the plan. 

View collection sets defined in this tab are inherited by default by any Branch plans created from a Master plan. If the same view collection sets assigned to a Master plan are required for use in Branch plans beneath it, this section can be left blank. Changes to this section must only be made if different view collection sets are required.

Supply Chain Errors

The Supply Chain Errors tab displays any errors that were detected during validation. To fix errors, edit the supply chain and re-validate the supply chain.

For more information, see Validating a Supply Chain.

The Master Financial Plan that has been created updates to Created state.  This means the plan is ready to be initialized.  Initializing the master plan populates all the measures defined for historical and future periods.  A plan must be initialized to view and work with the Master Plan.

DC Stock Split Factor tab

The DC Stock Split Factor tab screen shows how the Cluster Set Clusters are being serviced by multiple DCs. A pivot grid displays with fields by DC, cluster set, and cluster.

Changing the percentage of receipts a DC receives

To change the percentage:

  1. Select the appropriate field within the grid.
  2. Click the dropdown arrow to display a calculator.
  3. Choose the new percentage.
  4. Click Save.

The figures displayed in the grid automatically recalculate.
