Plan Sub Types

The two options for Plan Sub Type are Default and Original Plan. The distinction between the two choices lies in the treatment of periods that are in the past (e.g. last week and prior).

Plan Sub Type notes

  • Master plans cannot be set to Original Plan Sub Type. Master plans must always be Default. Their branches, however, can be either Sub Type.
  • The Original Plan Sub Type is only available to Hybrid plan types.
  • Branches can be switched between Default and Original Plan throughout their life, if necessary.
  • Branches of various Sub Types can be Merged and Re-Seeded. For Default Sub Type plans, only future periods are allowed for these operations. For Original Plan Sub Type plans, all periods are allowed for these operations.
  • The Foreign Exchange Neutralization (FXN) functionality cannot be used in conjunction with an Original Plan Sub Type.

Original Plan Sub Type populating On Order metrics

The Original Plan Sub Type populates the On Order metrics as of the plan creation only, and holds those values (many with zeros) throughout. This is the case for all instances, including future dated OP plans (plans with the first period in the future).

The On Order metrics still update when a user re-seeds or merges plans of Original Plan Sub Type.

Characteristics of past periods in each Plan Sub Type

The characteristics of past periods in each Plan Sub Type are as follows:

Default:

  • disable editability of metrics in past periods
  • formats cells within the past periods to signify past
  • surfaces actual values instead of plan values

Original Plan

  • allow editability of metrics in past periods
  • cell formatting does not change between past and future periods
  • surfaces planned values in all periods

The above descriptions define behavior for the base metrics (e.g. TY metrics). LY metrics (and LLY metrics, if that is selected as the Comparison Type for display) DO actualize. The behavior for LY and LLY metrics in both Default and Original Plan types is the same.

Verifying which plans are default and which are Original Plan

The Plan Sub Type Name column is an available column that supplements Plan Name, Plan Description, Plan Type Name, and Plan Status Name. The column allows users to quickly and easily see which plans are default and which are Original Plan.

Plan Sub Type Name can be added using the Choose Column option on the right-click menu.
