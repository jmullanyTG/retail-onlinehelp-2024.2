Plan approvals

A process can be set up to review, approve, reject, and merge branch plans. A plan’s status changes to In-Progress once it has been adjusted. The plan can then be sent to an authorized user for approval. 

This process can support pre-season as well as in-season reviews.

The functionality available for the plan approval process is contained within the Financial Plan Maintenance view. 

Best practices

  • Before approving plans, it is a good idea to create an additional branch of the plan that takes on the merged changes. Once a plan has been merged to another, it’s difficult to revert back to the plans that existed prior to the merge or approval process. This makes it easier to revert back to the plans that existed prior to any approval or merge.
  • Create a plan review calendar to establish deadlines for Pre-season and In-season plan reviews. Users can take full advantage of the plan approval functionality if they know their due dates.
  • Limit the number of branches that need approved during the plan approval process. Creating branches that contain multiple product and location dimensions cuts down on the number of plans to be reviewed and approved. This helps to streamline the process, especially for In-season plan reviews.
  • Take advantage of the notes section during the approval or rejection process. This allows users to focus on the area of the plan that needs reviewed or adjusted.
  • Remind users that the notification inbox contains all the changes made in the Plan approval process. This is where they see the progress being made during plan reviews.

Sending a plan for approval

To send a plan to an authorized user for approval:

  1. Select the Branch plan.

  2. Click the Submit button on the toolbar.

    The Submit Financial Plan window displays.

  3. Complete the fields as required.

    Select the destination plan for which the source plan is to be approved and merged to.  Add notes to help the approver through the approval process. 
  4. Click Ok.

The approval process

The plan is sent to the authorized approver for approval.  The submitted branch plan then moves to a status of Awaiting Approval.  Approved plans are merged into the destination plan. 

The destination plan changes to a status of Approved and the merge history on the right hand side of the Financial Plan Maintenance view updates. 

Merge History tab

The Merge History tab is an audit trail of the approval process and displays when merges were made to and from the plans that are being worked with.

The branch plan or source plan remains in Approved status until changes are made at which time the status reverts back to In-Progress. 

Rejecting a plan

If the plan is rejected and not approved, then the plan is returned with any notes added by the authorized approver. 

To reject a plan:

  1. Click the Reject button to display the Reject Financial Plan window.

  2. Add any notes and send back to the planner to make the requested changes to the plan.

    The plan changes to a status of In-Progress.
