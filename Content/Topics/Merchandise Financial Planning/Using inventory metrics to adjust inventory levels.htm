Using inventory metrics to adjust inventory levels

Users can adjust inventory quickly and conveniently using metrics labeled Inventory Adjustment and Inventory Reduction. These metrics can be assigned friendly names using the Metric Configuration configuration screen.

Having separate individual metrics with user-friendly names gives greatly improved visibility to each, and better planning of AUR associated with each.

Configuring an inventory metric with a friendly name

  1. Click Manage in the main planning view.

  2. Click the Configuration option on the toolbar of the Financial Plan Maintenance screen.

  3. Select Metrics from the dropdown list.

    The Metric Configuration screen displays.

  4. In the Friendly Name field, enter a new name for the metric.

  5. Click Save.

If the plan is already open, click Execute or press F5 to refresh the screen.

Inventory metrics

Inventory Adjustment 1-4 are identical in how they are calculated. All 4 can be given different user-friendly names and used for different adjustment scenarios.

The Inventory Reduction 1-3 metrics can be used in the same way as the Inventory Adjustment metrics.

As both positive and negative values can be entered into Adjustments and Reductions metric fields, the following is important to note:

  • Reduction fields are always subtracted, so negative values entered in these fields result in an addition. 

  • Adjustment fields are always added, so negative values are subtracted.

Changes to Adjustments or Reductions affect rolling EOP and BOP.

Generalization Of component metrics

MFP has three sets of metrics called component metrics meaning they are corresponding parts of a whole other metric. The parts are:

  • the metric set Comp and Non-Comp are parts of Sales.

  • The metric set of Regular, Promo, and Markdown are parts of Sales.

  • The Target Receipts 1 and 2 are parts of Target Receipts.

The application allows configurability as to which of these components are enabled and what they are called.

Components can be repurposed and renamed in whatever way is desired. Once saved, the changes take effect in the MFP grid once it is reloaded. In the above screenshot, Comp and Non-Comp are repurposed to represent 2 sub-channels where stock is fulfilled either via the company’s DC or supply chain, or via a wholesaler. Thus, the fields are renamed to Warehoused and Factory Direct.

In addition, there is the concept of a primary component. This means that, if the total metric (e.g. Sales) is currently 0, and a non-zero value is added, then 100% of the value goes to the primary component. The value does not evenly split among the components. The primary component is indicated by a check mark in the Is Primary column in the Generic Metric Configuration view.

In the case of the 3-Part Sales Component Set (the Regular / Promo / Markdown set), then Regular MUST be enabled if the component set is enabled at all. In other words, users can disable all 3 components, or users can disable ONLY Promo or ONLY Markdowns, but Regular cannot be disabled while allowing the other 2 to be enabled. Metrics generated in the main planning view

When a given generic Adjustment or Reduction metric is enabled, 29 metrics are visible for the Adjustment metric and 26 are visible for the Reduction metric in MFP. The difference is due to 3 Retail Ex Vat metrics being applicable to Adjustment only.

Aside from Retail Ex Vat, each of the set of metrics is repeated for TY, LY, and Comparison, in addition to % +/- LY for Retail and Units.

If two or more Adjustments or Reductions are enabled, then a Total Adjustments and/or Total Reductions field displays as well. This is an aggregation of all enabled adjustment or reduction fields, but is not plannable.
