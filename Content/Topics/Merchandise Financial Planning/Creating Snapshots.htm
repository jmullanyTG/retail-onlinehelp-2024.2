Creating snapshots

Snapshots serve as the third main concept of plans and are managed in the Financial Plan Maintenance view. Snapshots are locked down versions of Master or Branch plans. The main differences between a snapshot and any other plan is they do not actualize as time moves forward. Snapshots can be accessed on the right hand side of view and are specific to the master or branch plan highlighted on the left hand side of the view. 

Accessing snapshot functionality

  • Click on the Snapshots tab on the right side of the view.

The Snapshots tab

The Snapshots tab is used to set up plan snapshots which occur on a regular schedule as well as to create and delete snapshots manually. Snapshots are captures of the state of the current plan at the time they are generated for the purpose of future comparisons as users work with the plan.

There are two main components that make up the Snapshots tab.  The top portion of the view is the MFP Snapshot Templates.

The MFP Snapshot Templates section

The MFP Snapshot Templates section lists all snapshot templates for the current plan which the current user has access to view. This section is used to create recurring snapshots of the current plan.

Functionality

  • New: Adds a new snapshot template line.
  • Delete: Deletes currently selected snapshot templates from the plan.
  • Fields: Opens the Customization dialog. Drag fields to or from the snapshot templates column header area and this dialog to customize the view. Custom layouts can be saved or restored by right-clicking in the Snapshot Templates area and selecting Save Layout or Restore Layout, respectively.
  • Save: Saves changes made to snapshot templates and saves newly created templates.
  • Cancel: Cancels changes made to snapshot templates and removes templates created since last save.

Fields

  • Name: The name for the snapshot template.
  • Description: A description for the snapshot template.
  • Created Date: The date the template was created.
  • Created By: Who created the template.
  • Remove By Days: The number of days from the generation of snapshots created by a template after which they are automatically deleted. This is used for maintenance purposes.
  • Snapshot Start Days: The number of days backward from the date a snapshot is generated to be captured.
  • Snapshot End Days: The number of days forward from the date a snapshot is generated to be captured.
  • Product Level: The product hierarchy level to include in snapshots generated by the template. Available choices are based on the assigned product hierarchy for the current plan.
  • Channel Level: The location hierarchy level to include in snapshots generated by the template. Available choices are based on the assigned location hierarchy for the current plan.
  • Schedule: The schedule to use for determining the frequency and timing of snapshots generated by the template. Schedules must be created in the Schedules module to appear in this list.
  • Snapshot Access: Sets user access to snapshots generated by the template: public, private, or shared.
  • Snapshot Users: If Snapshot Access is set to Shared, then this allows for the selection of specific users to have access to snapshots generated by the template.
  • Template Access: Sets user access to the template: public, private, or shared.
  • Template Users: If Template Access is Shared, then this allows for the selection of specific users to have access to the template.

The MFP Snapshots section

The MFP Snapshots section lists all snapshots for the current plan, whether created manually or by a template, which the current user has access to view.

Functionality

  • New: Adds a new snapshot line to the list.
  • Delete: Deletes selected snapshots from plans.
  • Fields: Opens the Customization dialog. Users can drag fields to and from the snapshots column header area and this dialog to customize the view. Custom layouts can be saved or restored by right-clicking in the Snapshots area and selecting Save Layout or Restore Layout, respectively.
  • Save: Saves changes made to snapshots and generates newly created snapshots.
  • Cancel: Cancels changes made to snapshots and removes snapshots created since last save.

Fields

  • Start Date: Sets the start date for the snapshot window of time. The date must be before the end date. This field locks after the snapshot is generated.
  • End date: Sets the end date for the snapshot window of time. The date must be after the start date. This field locks after the snapshot is generated.
  • Product Levels: The product hierarchy level to include in the snapshot. Available choices are based on the assigned product hierarchy for the current plan. This field locks after the snapshot is generated.
  • Channel Levels: The location hierarchy level to include in the snapshot. Available choices are based on the assigned location hierarchy for the current plan. This field locks after snapshot is generated.
  • Remove By Date: Sets the date on which the system automatically deletes the snapshot. This must be set after the current date and is used for maintenance purposes.
  • Access Level: Sets user access to the snapshot (public, private, or shared).
  • Access Users: If Access Level is set to Shared, then this allows for the selection of specific users to have access to the snapshot.
  • Created From Template: The name of the template the snapshot was generated from. NULL if manually generated.
  • View: Clicking View displays the snapshot in a separate window. Snapshot headers are a concatenation of the Plan name and the Snapshot name.

  • Last FXN Date: The last time FX neutralization (FXN) was recalculated for future periods. The date of the last time FXN functionality was run may not be the same date as the snapshot creation date.

    FXN updates future periods with the current run date’s conversion rates, which may vary over time. Past periods will use that period’s conversion rate, which will likely not change as the period is actualized. For example,

    • Future period 2024_wk52 uses today’s currency conversion rate for FXN values
    • Past period 2021_wk01 uses 2021_wk01’s currency conversion rate for FXN values

    For more information regarding FX neutralization, see FX neutralization.

    The metric updates whenever FXN Recalc is selected on the Financial Plan Maintenance screen or run nightly or weekly jobs.
