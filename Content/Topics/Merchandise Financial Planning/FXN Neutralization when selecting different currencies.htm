FX neutralization and selecting different currencies

If a user selects a different currency for display than the plan currency (e.g. the plan is USD but the user selects EUR), then a conversion rate must be selected in order to display the cached data in the chosen currency. The currency conversion rate chosen follows the same logical steps as those used to cache data. All future periods use the current period’s conversion rate for FXN LY and LLY metrics.

The following examples are for the Sales metric in a plan currency of USD, but apply to all default currencies and the metrics described in Metrics Impacted by FX Neutralization.

Data selection in FX Neutralized mode

Depending on display currency:

Sales = $2170 or €1855

LY Sales = $2487 or €2125

LLY Sales = $1895 or €1620

The system relies on the already converted and stored USD totals across all sites. It will not go back to site-specific data and reconvert all individual values from site currency to display currency.
