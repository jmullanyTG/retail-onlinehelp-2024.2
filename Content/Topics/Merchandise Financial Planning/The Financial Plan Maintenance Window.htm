The Financial Plan Maintenance window

The Financial Plan Maintenance window lists the existing plans users have access to. Plan documents can be opened and administrative actions performed such as creating, branching, merging, and initializing plans.

Getting here

  • Click Plan > Merchandise Financial Planning > Manage on the toolbar.

To open a plan document

  1. On the Financial Planning toolbar, click Manage.
  2. In the Financial Plan Maintenance window, select the plan to open.
  3. Click Ok at the bottom of the window to open a document for the selected plan.

The following functionality is available:

Functionality Description
New Creates a new plan. For more information, see Creating a master plan.
Edit Edits the settings of the plan selected in the plan list.
Initialize Seeds and populates data values in the selected plan according to the measure set selected for the plan. This action is only available to plans with the status Created. Once a plan has been initialized, its status changes to Initialized.

Regardless of the selected seed type, metrics that would be seeded with negative values are seeded with zeros instead.
Re-Initialize Re-initializes the selected plan. A plan should be re-initialized when its master data changes in the data feed (a re-class, for example).
Close Deletes the selected plan.
Re-Seed Overwrites and re-seeds planned values for the selected future time period within the selected plan.
  • A master plan is re-seeded from calculation as defined by its seed type.
  • A branch plan is reseeded from its parent plan.
  • Regardless of the selected seed type, metrics that would be seeded with negative values are seeded with zeros instead.

Re-seeding planned values for future periods where LY has not actualized

Users can select a period of time for re-seed that is beyond 1 year from actualized data and where the LY values are non-actualized. The non-actualized or planned values for those future LY periods are overwritten with the planned values for the LY planned period.

A common use case for this is long lead-time categories, where plans are built on top of planned values. For example, footwear might have a 18 month lead time. In this case, months 13-18 plans are being built on the plans for future months 1-6.

Users should ensure they are only re-seeding the future periods where planned values are desired to be overwritten.
Branch Creates a branch of the selected master or branch plan.
Merge Merges the selected branch plan into another branch or master plan.
FXN Recalc (Visible if system policy Enable FX Neutralization in MFP set to True) Updates the FXN values with planned values.

The FXN Recalc button is enabled when the user has rights to edit the plan.
Submit Submits the selected branch plan for approval.
Approve Approves the selected plan.
Reject Rejects the selected plan.
Rebalance Takes the desired values from the plan version to be continued with and homogenizes other versions to them.

For more information, see Homogenizing Plans using the Rebalance Functionality.

Configuration Displays a menu with the following functionality:

  • Global Groups - manages the site and product hierarchies used in financial plans.

  • User Access - controls user and role access to the selected plan including product, channel, and measure sets within the plan.

  • Merge Measure Sets - controls which set of metrics are merged from the selected plan.

  • Metrics - displays the Metric Configuration screen in which inventory adjustment and shrink metrics can be renamed. For more information, see Configuring inventory metrics with friendly names.
Refresh Updates the information displayed in the Financial Plan Maintenance window.
