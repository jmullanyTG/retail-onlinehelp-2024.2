Calculation toggles pane

The Calculation Toggles pane is used to define which dependent metrics recalculate and which ones hold when changes are made. Users can click on the metrics and change the calculation settings using a dropdown list.

Based on the planning method, Retail or Cost, some calculation toggles may not be available. This is due to the differences in how the calculation model must function together as a whole to properly balance within each method

When alternate (non-default) calculation toggles are selected, they appear in darker gray for easy identification at a glance.

Not all plannable metrics are shown in this view because not all plannable metrics have multiple calculation directions.

The Calculation Toggles pane can be collapsed by clicking the arrow at the top left adjacent the view title. Once collapsed, the view is visible only as a tab at the right hand side of the screen and can be restored by clicking the arrow near the view title again.

Notes

  • If a week's Markdown is changed, then that week's COGS holds but the following weeks to do not.
  • If a month's Markdown is changed, it spreads the change across the weeks. The first week in the month's COGS stays the same, but because the Markdowns changed, the first week's Sales Units and Total Original Retail change, which in turn changes EOP Units, EOP Cost, EOP Retail, and Markdown Cost. Because the first week's EOPs changed, the second week's BOPs changed.
  • COGS is based on the ratio between BOP Cost and BOP Retail, so it necessarily changes in the second week. Similarly in the third through nth weeks. Thus, the total COGS for the month changes.
  • Calculation toggles do not require saving to be in effect. The change is immediate once the selection is made.

Lock and spread related calculation toggles

When the following checkboxes are checked,

Remove Locks On Save: all locks present in the current plan are removed whenever Save is clicked in the main planning view.

Remove Locks on Merge: all locks present in the current plan are removed whenever the plan is merged into another plan manually or via the Approve action in the Financial Plan Maintenance dialog.

Auto Lock On Cell Change: each time the value of a sales-related metric is changed in the grid, the changed cell is automatically locked (and any related cells).

For more information, see Auto Lock on Cell Change.

Using calculation toggles to hold initial values

Once the first metric is entered, subsequent metrics must be entered using calculation toggles which hold the value of the first. Once the initial values are entered, fine tune is easy if a user is mindful of the metrics the toggles use are holding and calculating. Becoming familiar with the available toggles before beginning planning ensures a smooth planning process.

Promotion markdown values actualize using the following calculation: Promotion Markdown = Total Original Retail - Total Revenue.

The value that displays represents the total of markdowns (both Permanent and Promotion). This is because it is not possible to reliably determine which portion of the value represents Permanent or Promotion markdowns from standard data imports. Actualized values for Permanent markdown metrics are zeros.

Sales to Target BOP calculation toggle

This toggle is especially useful when the organizational direction is that Target BOPs are held and cannot change without approval. In this situation, holding Target BOP can be set as the default. If the Target BOP is allowed to change, there may be situations where the stock for a certain product hierarchy cannot be acquired but it is desired to increase the Sales Plan.

Users can toggle the calculation as a one-off where the Target BOP holds resulting in Stock to Sales being a little faster. However, if changing Sales users must consider the calculation toggle settings for both the Sales toggle and the Sales to Target BOP toggle.

Sales to Target BOP: HOLD Stock to Sales | CALC: Target BOP

The default calculation toggle for Sales to Target BOP.

Sales to Target BOP: HOLD: Target BOP | CALC: Stock to Sales

The alternate calculation toggle for Sales to Target BOP.

Dynamic renaming of particular metrics is in place to support the use of these values in either a Retail or Wholesale context. When viewing:

  • Wholesale channels only: Sales Units is replaced with Booked Units, AUR is replaced with ASP, and Total Original Retail is replaced with Booked At Wholesale.

  • Retail and Wholesale channels simultaneously: Sales Units is replaced with Sales Units / Booked Units, AUR is replaced with AUR / ASP, and Total Original Retail is replaced with Total Original Retail / Booked At Wholesale.
