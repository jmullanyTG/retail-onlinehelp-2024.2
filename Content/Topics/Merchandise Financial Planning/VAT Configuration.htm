VAT configuration

For organizations that include value added tax (VAT) in the planning process, there are a few required configuration steps that must be completed prior to beginning to plan.

Verbose product hierarchy

For VAT calculation functionality to work properly, a verbose product hierarchy is needed. This basically just means that data can be stored at all levels of the hierarchy rather than just the lowest leaf levels.

For example, if a hierarchy consists of the levels Company > Gender > Class, a verbose hierarchy allows data to also be stored against the Company and Gender levels, whereas a regular hierarchy would only allow data at the Class level. Because VAT rates can be set at any level of a product hierarchy, a verbose hierarchy is needed to allow the system to read the correct rates when calculating VAT related metrics.

Creating a verbose product hierarchy

    1. Click System Utilities > Views
    2. Locate and open the Hierarchy Filtering view.

    3. Select the Variant entity type and click Execute.

    4. If a hierarchy already exists, then select the hierarchy and check the Allow Data Storage at All Levels checkbox or create a new hierarchy:
      1. Click New.
      2. Enter a Hierarchy Name and Description.
      3. Check the Allow Data Storage at All Levels checkbox.

      4. Click Save.
    5. Set the Order Number and Enabled checkbox for the desired Item Master Groups and save.

      Check with a ToolsGroup JustEnough technical representative before making this change to hierarchies that are used in modules other than Assortment Planning and MFP. Other modules, such as Allocation and Group Forecasting are not designed to work with verbose hierarchies.

  2. While still in the Hierarchy Filtering view, set the MFP product hierarchy that VAT rates will be assigned to.
    1. Click Fields in the Hierarchy Configuration section and drag the Hierarchy ID field into the grid:

    2. Note the Hierarchy ID for use in the next step.

    3. Navigate to the following module: Maintenance > Policies.
    4. Select Planning VAT Rate Product Hierarchy ID from the Module: Financial Planning folder at the left.
    5. Enter the Hierarchy ID noted in the last step in the New Setting field.
    6. Save the changes.

  3. Enable the use of VAT in MFP.

    1. While still in the Policies module, select the Enable VAT in MFP policy from the Module: Financial Planning folder.

    2. Select True in the New Setting field.
    3. Click Apply Changes.

    The Enable VAT in MFP policy is a global setting meaning all MFP plans will have VAT enabled once this policy is set to True. For a VAT rate of zero set for the product/site levels included in a given plan, ex VAT metric values are identical to their regular counterparts and there is no mathematical effect on the plan.

After the above steps are complete and assuming VAT rates are set up in the Planning VAT Rates view, uses can plan with VAT included in MFP.

For more information on setting VAT rates, refer to Currency and VAT Management > Essential Business Concepts > Planning VAT Rates.
