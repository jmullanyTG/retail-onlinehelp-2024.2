Total company and average store modes

The View Mode parameter is used to choose between viewing data for the total company (all stores combined) or for the average store (the total company value divided by the number of active sites).

It can be useful to add Site Count fields to a plan document (Site Count, Comp Site Count, and LY Site Count). These metrics display the number of active sites in each period. (For more information, see Adding, removing, and rearranging fields.)

Viewing total company data

To view and plan in Total Company mode, choose Total Company as the View Mode parameter and click Execute.

In the screenshot above, the following total company values for the period 2022_wk27:

  • Site Count: 98
  • Sales Units: 1,670,486
  • Sales: $118,143,797
  • Sales % +/- LY: 28.31%

These values represent the combined sales for the stores for the period.

Viewing average store data

To view metrics in Average Store mode, choose Average Store as the View Mode parameter and click Execute.

In the screenshot above, the following average store values for the period 2022_wk27:

  • Site Count: 98
  • Sales Units: 17,046
  • Sales: $1,205,549
  • Sales % +/- LY: 28.31%

These values represent the total company values divided by the number of sites in the Site Count metric.

  • The Sales % +/- LY value does not change between Total Company mode and Average Store mode. This is because metrics that are already averages or ratios (such as AUR, AUC, and AUT) do not get re-calculated in Average Store mode.

  • The values displayed in Average Store mode do not show all significant digits stored in the database. Due to rounding, manual calculations may deviate slightly from displayed values.

Planning in Average Store mode

Plannable metrics can be edited while in Average Store mode. However, entries are interpreted as an average value for a single store, thus multiplying them by the site count to calculate the Total Company value.
