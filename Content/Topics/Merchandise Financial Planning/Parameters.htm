Plan parameters

Selections made in the parameters area define the data that is returned. Select the Parameters button to show or hide parameters. Clicking Execute displays data based on the parameters selected.

Viewing assortment plan data within the Merchandise Financial Planning module

Two comparison types can be used to review Assortment Plan data within the Merchandise Financial Planning module. This means Assortment Plan data can be referenced while working on Financial Plans without having to go back and forth between screens. This helps ensure that Financial Plans and Assortment Plans align for accurate forecasting and buying. The comparison types are:

  • vs. Assortment Plan
  • vs. Assortment Plan Snapshot

Parameters

The following parameters filter the plan data displayed in a financial plan document.

Parameter Description
Product The levels of the product hierarchy to display in the document. (Only active product levels are listed.)

System policy Product Parameter Sort determines whether the list is sorted by Code or Description. The default is Description.
Product Agg Level The lowest level of the product hierarchy at which data is pre-aggregated to optimize performance. This parameter should be set at the lowest level of the product hierarchy to view in the document. For more information, see Aggregation levels and top down planning.
Channel The levels of the site hierarchy to display in the document. (Only active site levels are listed.)
Channel Agg Level The lowest level of the site hierarchy at which data is pre-aggregated to optimize performance. This parameter should be set at the lowest level of the site hierarchy to view in the document.
Time The time periods for which data displays in the document. Options include all time periods including and after the Start Period defined for the plan. The duration of each period is determined by the Calendar Level setting defined for the plan.

If non-contiguous time periods are selected, the gaps are automatically filled and display data from all periods between the earliest period selected and the latest period selected.
Comparison Plan 1 Any plan type that is NOT TY (working plan) or LY (hard-coded to only return LY data). The two Comparison Plan parameters lets users simultaneously compare a TY, LY, and 2 additional Comparison plans.

Plan types are the first level in the selection. Click the arrow to see a list of relevant plans.

Options are:

  • None - do not show comparison data in this document.
  • vs. Master - compare this plan's metrics against the master plan selected in the Comparison Plan 2 parameter.
  • vs. Branch - compare this plan's metrics against the branch plan selected in the Comparison Plan 2 parameter.
  • vs. Snapshot - compare this plan's metrics against the snapshot selected in the Comparison Plan 2 parameter.
  • vs. LLY - compare this plan's metrics against the metric values from two years ago .

  • vs. Assortment Plan - compare this plan's metrics against a particular Assortment Plan.

  • vs. Assortment Plan Snapshot - compare this plan's metrics against a particular snapshot of an Assortment Plan.

When vs. Master, vs. Branch, or vs. Snapshot is selected in the Comparison Plan 1 parameter, this parameter sets the specific plan to use for comparison.

All comparison metrics are populated directly from the source plan specified rather than calculated. A value of zero displays if no data exists in the selected plan for a given site/product level and period.
Comparison Plan 2 The Comparison Plan 2 parameter returns a core subset of 86 metrics compared to the metrics returned by the Comparison Plan 1 parameter. Therefore, it is important to remember that Comparison Plan 1 returns the full metric set while Comparison Plan 2 returns the abbreviated metric set. With this difference in mind, users could consider scenarios such as whether the full metrics are required for a Snapshot consideration and the subset of metrics for LLY, or vice versa.
Currency The currency in which all revenue-based metrics display. (Only currencies assigned to active sites are listed.)
Currency Neutralization The currency conversion rate to be used. The parameters are:
  • Standard - Use the period’s currency conversion rate for each data period.

  • LY Neutralized - Use TY currency conversion rates for future-dated LY and LLY, and LY currency conversion rates for past-dated LY and LLY.

For more information on currency neutralization, see FX Neutralization.
View Mode Whether data displayed in the plan document represents Total company and average store modes.
  • Total Company - data values represent the total value aggregated from all active sites in the given time period.
  • Average Store - data values represent the average store value, calculated by dividing the total company value by the number of active sites in the given time period.
