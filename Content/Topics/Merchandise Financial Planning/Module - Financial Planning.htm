System policies related to the financial planning module

System Policy Description
Approved Snapshot Retention Days How many days approved snapshots are kept in the system. Default = 720.
DC Stock Include Pending Whether to reduce DC Available On Hand by Pending stock.
DC Stock Include Reserved Whether to include Reserved stock in the EOP Units DC Split and DC Available On Hand metrics.
DC Stock Splits future number of periods Number of periods forward from the current period to determine % DC stock splits by channel. Default = 8.
Deleted Plan Retention Days How many days plans are kept in the system after being deleted (closed). Default = 0.
Enable FX Neutralization in MFP This policy enables or disables currency neutralization functionality. Default(False)
Enable VAT in MFP Whether to use VAT. This should only be set once during implementation. Default = False.
  • True = Use VAT
  • False = Do not use VAT
Financial Plan Master Plan Max Future Days Number of days forward to populate the master plan where periods exist. Start date being the start of the current period. Default = 720.
Forecast Source Dictates which source of Forecast data to use - Inventory forecasts or aggregated level forecasts.

Level data should be chosen to expedite performance, but only if forecast jobs are set up to maintain hierarchy level data.
Include DC Stock Split into EOP / BOP Whether to include DC stock, which is split into channels during actualization, into the rolling EOP / BOP calculations.
Include On Order in EOP / BOP Whether to include On Order and On Order Past Due into the rolling EOP / BOP calculations.

NOTES:

  • If enabled, the following period BOP includes the On Order (under the assumption that it will be received as ordered).

  • Enable if the On Order is reliable, and also, the Target Receipt value does NOT include the On Order (or it will be double counted).
Inventory is Shared Whether or not DC inventory is shared among all channels. (Are receipts planned for specific channels or not?) Default = False.
  • True = Share DC inventory across channels.
  • False = Do not share DC inventory across channels.
MFP Plan Size Threshold Set an arbitrary size threshold for comparison to a PlanID's number of Product/Channel/Period combinations. Exceeding the threshold will result in running the process in the WorkManager Queue.
MFP Restrict Editability To Admin Restrict the editability of the following metrics to users in the Administrators group: Shrink % Of Sales, Target Receipt Freight, Season Begin. A value of True requires users to be administrators. A value of False allows editability of these metrics for all users.
Multi Plan Seeding Merging Whether to allow seeding or merging from other plans. Default = False.
  • True = Allow seeding and merging from other plans.
  • False = Do not allow seeding and merging from other plans.
Open Multiple Plans This policy enables or disables the ability to open multiple plans. Default(True)
Planning VAT Rate Product Hierarchy ID Product Hierarchy ID to use for planning VAT rates.
Product Parameter Sort Determine the sorting order of the Product hierarchy dropdown, by Code or Description.
Run module in Debug mode Whether or not to run Financial Planning module procedures in debug mode.
  • 0 = do not run in debug mode.
  • 1 = print records to application log.
  • 2 = return immediate results sets. This mode must be run through development tools; it will fail if run through the UI.
Save the LLY data for hybrid financial plans For hybrid financial plans, whether or not to save LLY data is saved on initialize and during maintenance.
  • True = LLY data is saved on initialize and during maintenance.
  • False = LLY data is not saved.
Snapshot ViewCollectionSetID Thee ViewCollectionSetID for the snapshots view collection set.
SqlServerPivotGrid Recommended Maximum number of SQL Data Rows Recommended maximum number of rows to return for any SQL Server Pivot Grid. If exceeded, a warning message is displayed in the UI. Default=2250.
Stock On Hand Actualization Batch Size This policy only applies to the Hybrid Retail plan type, when calculating Target BOP Cost during weekly maintenance. This policy determines which calculation direction should be used when COGS recalculates: hold Target Stock Cost to COGS and calculate Target BOP Cost (value of True), or hold Target BOP Cost and calculate Target Stock Cost to COGS (value of False).
Stock On Hand Actualization Source Dictates which source of stock on hand data to use during actualization - Inventory level data, or aggregated Variant level data. Variant level data should be chosen to expedite performance, but only if maintenance or imports are set up to maintain Variant-level data. For more information, see Inventory Level Archive (ILA) data stores.
Target BOP Calc Direction For Maintenance Use tbl_SupplierReceipts solely for historic receipts in MFP and not a combination of tbl_SupplierReceipts, tbl_SupplierOrderHeader and tbl_SupplierOrderLine.
Use Hybrid Markdowns and COGS in Retail accounting method Enable Hybrid Markdowns In Retail Accounting. Permanent, Promo and Hybrid Markdowns and COGS fields will be displayed.
Use Supplier Receipts Only for Historic Receipts Use tbl_SupplierReceipts solely for historic receipts in MFP and not a combination of tbl_SupplierReceipts, tbl_SupplierOrderHeader, and tbl_SupplierOrderLine.
