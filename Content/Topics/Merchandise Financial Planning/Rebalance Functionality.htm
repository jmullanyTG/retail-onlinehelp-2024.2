Homogenizing plans using the rebalance functionality

It is a common occurrence within companies to have multiple versions of a plan for the same product/channel levels and window of time for the purpose of exploring different possible outcomes and/or analyzing the business from different perspectives. Sometimes there is a version built from the location level up and another from the top site level down. Other times there are multiple scenarios built at the same level or even versions built using different hierarchies altogether.

Eventually, a reconciliation occurs where those within the planning organization decide which version to move forward with. The version to be moved forward with is the Source Plan version. The goal is to take the desired values from the Source Plan version and homogenize other versions to them.

If a particular Source Plan metric (Sales, Sales Units, COGS) is not selected, then the Source plan does not cause a change in the corresponding metric in the Destination Plan.

Rather than overwrite the values in the Destination Plan, a rebalanced plan is generated with the word Rebalance prepended to the Destination Plan name. If the Destination Plan is titled Branch of Hybrid Cost Plan then the rebalanced plan would be Rebalance Branch: Branch of Hybrid Cost Plan.

A very common scenario in the retail industry is where there are two sets of planners: Tops-down financial planners and store planners. These individuals create their ideal versions of the larger plan from their perspectives then meet to discuss which total volume outcome is most feasible given all the available data. Once complete, one version is rebalanced to the other.

Before initiating a rebalance, it is necessary to have two plans that share at least one common site level, one common product level, and a common window of time. It is not necessary for the common levels of the plans being rebalanced to exist in the same positions (the same number of levels deep). The system adjusts data as needed to ensure the proper values are transferred.

Once a rebalance is initiated, the system performs the rebalance from the Source to the Destination Plan at the lowest levels of commonality in the product and site hierarchies between plans.

To rebalance plans

  • Click Rebalance to launch the Rebalance Plan dialog.

Copying COGS between plans

COGS cannot be copied over between cost based plans and retail based plans during a rebalance. They are calculated and used differently in the different plan types. COGS is a ratio of BOP and Sales in the Retail method, but not in the Cost method. Inventory metrics are not copied during a rebalance.

All other dependent metrics

All other dependent metrics in the Destination Plan recalculate per the normal calculations defined for the appropriate plan type, Cost or Retail. For more information regarding the calculations, see Merchandise financial planning metrics list.

Rebalancing metrics based on configured measure sets

Users can rebalance metrics based on configured measure sets that display in the Measure Set dropdown. It is important to note that the Measure Set dropdown only displays measure sets relevant to the selected plan type and rebalancing should only occur between plans of the same plan type. Only the metrics selected for rebalance are affected within the destination plan. Any metrics unselected retain the values of the destination plan, therefore, it's important to select all related metrics within a metric set for rebalance.

 

For example, when rebalancing Receipts, a user should check all of the Target Receipt metric checkboxes for the full set of data to rebalance. Selecting only Receipt Retail and Receipt Units without Receipt Cost applies the Receipt Cost of the destination plan to the newly copied Receipt Retail and Receipt Units values from the source plan, resulting in a newly calculated Receipt margin. Similarly, by not selecting Receipt Units, the Receipt AUR and Receipt AUC values will not be direct copies as the Receipt Units values in the destination plan become the divisor.

Notes:

  • If a measure set is picked along with any of the Sales / Sales Units / COGS / Receipt Retail / Receipt Units / Receipt Cost checkboxes, it is the measure set that is copied and not the Sales and Receipt specific metrics even if the Sales and Receipt checkboxes are selected.

  • When the Sales within the Source and Destination plan are not equal, the ratio of Source Plan Sales and Destination Plan Sales is applied against other values being copied.

  • Rebalance plans are only created with the time periods specified in the Rebalance Plan screen.

  • Within the Hybrid Retail Dual Markdown plan type, there are metrics with rolling calculations using the designated (checked) Season Begin period. If the source and destination plans do not have the same Season Begin period, there will be some drift in these rolling metrics, two of note include, BOP Retail and COGS.

Rebalance Plan dialog field descriptions

Field Description
Source Plan The plan to which the Destination Plan is homogenized
Destination Plan The plan that is homogenized with the Source Plan
Destination Product Hierarchy The levels of the Destination Plan product hierarchy to be homogenized with the Source Plan
Destination Site Hierarchy The levels of the Destination Plan site hierarchy to be homogenized with the Source Plan
Destination Time Hierarchy The time periods of the Destination Plan to be homogenized with the Source Plan
Measure Set A collection of metrics saved as a set to be applied during future rebalances.

Measure sets are configured in the Manage > Configuration > Merge Measure Sets screen.
Sales, Sales Units, COGS Whether Sales revenue in the Destination Plan is homogenized with the Source Plan for the selected product/site levels and window of time
Receipt Retail, Receipt Units, Receipt Cost Whether the metric in the Destination Plan is homogenized with the Source Plan for the selected product/site levels and window of time
