Standardized jobs

Standardized jobs are found in Job Manager and are used to organize user and system jobs into daily, nightly, and weekend batches, according to the estimated workload. Historical logs that are no longer relevant (cleanup) can be cleared and a process priority determined.

For more information see,

  • Clearing historical logs

  • Optimization tuning

Standardized job types

Daily job

Daily batches with low workload run during daytime business hours. For example, Monday through Friday from 7am to 6pm.

Nightly job

Batches with medium workload run overnight after business hours. For example, Monday through Friday from 6pm to 11pm.

Weekend job

Batches with heavy workload run during the weekend. For example, Saturday from 3am to 11pm.

Viewing standardized jobs

Select a job in the Navigation pane. The job’s details display in the right pane.

Configuring job tasks

For convenience and to take advantage of parallel processing capabilities, configure daily, nightly, and weekly standard job tasks to include all or specific process types (forecasts, price actions, etc.).

  1. Select a job in the Navigation pane.

  2. Make the necessary changes in the right pane.
  3. Click Save.

Method parameter definitions

Method Parameter Name Default Value Description
@bDebug False Required. Whether the system should provide additional logging information.
@bEnforceWorkLimit True Required. Whether the system adheres to the Maximum Server Work Threshold system policy.
  • If set to True and the amount of work meets or exceeds the threshold set in the policy, any un-processed job tasks will go into delay cycles until the amount of work returns below the threshold value.
@bNightly False Required. Whether the job task processes only nightly jobs.
@ParallelBatching False Required. Applicable only when multiple job tasks are configured for a processing period (daily, nightly, or weekend).
  • If set to True, all job tasks work on the same job until it is completed. Then, the job tasks move on to the next job in the queue.
  • If set to False,each job task works on distinct jobs.
  • All job tasks must have the same value (True or False).
@bWeekly False Required. Whether the job task processes only weekend jobs.
@iProcessTypeID   Optional. Specifies the type of jobs that this job task will run. Leave this value blank to run all job types. Otherwise, enter one of the following values:
  • 1 - Inventory Forecast
  • 2 - Level Forecast
  • 3 - Group Forecast
  • 4 - Replenish Inventory
  • 5 - Assortment Receipt Plan
  • 6 - Adaptive Allocations
  • 7 - Assortment Prepack
  • 8 - Allocations
  • 9 - Replenish Set
  • 10 - Price Manager
@iUserID 1 Optional. The integer ID of the user that the job task will run as if the job process is submitted without an identified user. This user must have all necessary access rights to complete the work for the @iProcessTypeID value.

Default value is the IOAdmin ID.

@nMarshalID   Required. Unique ID value for this job task. To ensure accurate processing and logging, this value must be unique across all job tasks. Include this ID value in the job name. For example, "Workload Queue Parallel - Nightly - 1".
