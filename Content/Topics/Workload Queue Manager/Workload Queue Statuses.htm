Workload Queue statuses

Active list

  • Unknown: The system includes a default status type that is only used if the system is experiencing an issue. Contact ToolsGroup JustEnough Support in this case.
  • Preparing (a.k.a. Batching): The system is evaluating the items in the submitted batch and estimated workload.
  • Coalescing: The system is marking items in the submitted batch to reduce processing time by removing duplicate items. Last batch submission, with like items, are processed first. Prior batches complete the non-coalesced items and then place them into a status of Purgatory until the last batch, with like items, has been completed.
  • Ready: The system has completed preparation and coalescing the batch to make it ready to be executed by the Workload Queue Marshall.
  • Executing: The system is executing the submitted batch.
  • Purgatory: The batch is waiting for a following batch, with like Items, to be completed.

History list

For all the following statuses, the batch will be removed from the Active Workload Queue and can be referenced in the history logs.

  • Completed: The batch has completed execution without error.
  • Completed With Errors: The batch has completed execution with error.
  • Failed: The batch could not be processed by the Workload Queue.
  • Expired: The batch expired before execution could begin.
  • Removed: The batch was removed.
