Workload Queue grid fields

There are two grids on the Workload Queue home screen. The grids are: Active and History.

The Active list displays the current batches being or waiting to be processed by the system. The list is sorted by Process Type and Status, which cannot be modified.

The History list displays completed batches in the following statuses: complete, partially complete, failed, expired. The list also displays if a batch was removed from processing.

Field descriptions

Field Description
Process Owner The name of the user who submitted the work to the queue.
Description Name used to identify different batches.
  • Inventory Forecast – The system uses the Item filter name and product filter used.
  • Assortment Planning – The system uses the scenario name. If a scenario name is not available, the plan name is displayed with the product filter used.
Process Type Process types represent the supported ToolsGroup JustEnough solutions with the ability to use the Workload Queue. Batches are grouped by process types and are only displayed if there are active batches of that type.
Status Icon In the Active grid, the icon for the current status of the batch. In the History grid, the appropriate icon for the final status of the batch.
Batch Priority The priority calculated by the system or modified by a super user. The batch priority determines the order in which the batches execute in conjunction with batch priority escalation based on the length of time it has been waiting to be processed.
Item Status The number of items completed out of the total number of items to be processed in the batch.
Archive Date (History grid) The date time stamp when the batch was moved from the Active to the History.
Enqueue Date The date time stamp the batch was submitted to the Workload Queue.
Start Date The date time stamp the batch began execution on the Workload Queue. The field is blank until execution begins.
Expire Date The system calculated date time stamp on which the batch is removed from the Workload Queue if its execution has not begun.
Last Changed Date The date time stamp of the last priority change.
Last Changed By The name of the user who last modified the batch’s priority.
