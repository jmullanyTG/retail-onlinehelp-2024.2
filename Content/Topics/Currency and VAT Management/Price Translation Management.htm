Price translation management

Different pricing strategies by currency and price range are set using the Price Translation Management screen. The primary intent of this functionality is to allow users to account for differing costs of doing business in other countries without having to manage pricing for them all individually.

In Assortment Planning, the Multi-Currency Price View side view can be used to make overrides by item/currency to the rules created here. For more information, see Price Exceptions View.

Even if no price translations are needed, at a minimum create at least one rule with a Translation Factor of 1 for all From and To currencies to be used and with a Start Date of the earliest date to be used within any of the assortment plans and assign the rule to the Total Company level of the product hierarchies used in those plans.

This screen can be accessed by navigating to System Utilities > Views > Currency > Price Translation Management.

Fields

Field Description
Currency From The currency to view or create translation rules from in the view.

Generally, Rules only need to be created for the home currency the assortment plans are managed in.
Currency To The currencies to view or create translations to in the view.
Include Unidentified Whether rules that do not yet have From and To currencies defined appear in the Price Translation Rules section. Clicking this parameter automatically refreshes the view.

This should be left checked when creating new rules as From and To currencies can only be defined in the Details section after clicking Save in the Rules section. Clicking Save with this setting unchecked when creating new rules causes new rule lines to disappear from the view.

Toolbar buttons

Button Description
Parameters Toggles between displaying or hiding the Parameters area of the view.
Execute Updates the Rules and Details sections of the view to display rules with the selected Currency From and Currency To assignments.

Rules only have to match one of the Currency From or To assignments to be shown.

Price Translation Rules section

Button Description
New Creates a new price translation rule line.
Delete Deletes the selected price translation rule lines.

There is no pop-up message requesting confirmation of deletion of translation rules.
Fields Opens the Customization dialog where users can drag fields to or from the tab’s header area and this dialog to customize the view.
Save Saves all new price translation rules and changes made to existing rules.
Cancel Removes all new price translation rules and undoes changes.

Saving or restoring a custom layout

Custom layouts can be saved or restored by right-clicking in the Pivot Grid and selecting Save Layout or Restore Layout, respectively.

Pivot Grid fields

  • Price Translation Rule ID: A unique system identifier for the price translation rule.
  • Rule Name: The name for the translation rule.
  • Privacy: The privacy settings for the translation rule:
    • Public: All users with sufficient rights can view and edit the rule
    • Private: Only the user who created the rule and view and edit it
    • Shared: Only specified users with sufficient rights can view and edit the rule
  • Allowed Users: The users allowed to view or edit the rule.
  • From Currencies: The currencies being translated from in the rule.
  • To Currencies: The currencies being translated to in the rule.
  • Price Range Endpoints: The price range end points for the rule (corresponds to the Price Greater Than...? entries in the Price Translation Rule Details section). If the Price Greater Than...? 0 line in the bottom section for the rule does not have a translation factor set, then this field is conditionally formatted:

  • Minimum Date Present: The earliest start date for the rule.
  • Maximum Date Present: The latest start date for the rule.
  • Copy Rule: If checked, creates a copy of the rule with “ – Copy” and the current date appended to the Rule Name. Multiple rules can be copied with a single save.
  • Missing Values: Checked if the Price Greater than 0 line for the selected rule in the Rule Details section does not have Translation Factors set. At a minimum, all rules must have a translation factor for the Price Greater Than 0 line set.

Price Translation Rule Details section

Buttons

  • New: Creates a new translation factor line for the rule.
  • Delete: Deletes the selected price translation rule lines. Users are not asked to confirm deletion of lines.

    The Price Greater Than 0 line populates for new rules by default and cannot be deleted. If deleted, the line regenerates when Save is clicked.

  • Fields: Opens the fields Customization dialog. Users can drag fields to or from the tab’s header area and this dialog to customize the view.
  • Save: Saves all new translation lines created and changes made to existing lines.
  • Cancel: Removes all new translation lines created and undoes deletions.

Saving or restoring custom layouts

Custom layouts are saved or restored by right-clicking in the Pivot Grid and selecting Save Layout or Restore Layout, respectively.

Grid fields

  • Price Greater Than: Sets the starting price for the translation factors defined in the line to be applied and sets the ending price for the previous line to be less than the value entered.
  • Start Date: Sets the date translation factors in the line become active.
  • Price Translation Rule Detail ID: A unique system identifier for the translation rule detail. This is unique by line by currency.
  • Translation Factor: Sets the translation factor for the corresponding currency and price range of the line.

    Translation factors are applied as a multiplier to the home ticket price (Home AUT) of the product in the From Currency and are invoked by Home AUTs in that currency, regardless of what the To Currencies are. Setting a factor of 1 indicates no change.

    Translation factors are applied to Home ticket prices (Home AUTs) that have Home Site VAT values excluded, for the purposes of dependent calculations within Assortment Planning.

    The current ticket price in USD for sites doing business in GBP is $25.00 because the translation factor for products $20 and above is 1.25 ($20 * 1.25 = $25).

    A week later, the product goes on promotion for 20%, setting its new price in USD at $16.00. This means the new USD ticket price for GBP sites is $19.20 ($16 * 1.2 = $19.20).
