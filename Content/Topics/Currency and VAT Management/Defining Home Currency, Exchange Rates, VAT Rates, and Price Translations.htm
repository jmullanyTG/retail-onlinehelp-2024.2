Defining home currency, exchange rates, VAT rates, and price translations

There are four related views that manage how currencies are dealt with by Assortment Planning. The parameters set in these views contribute to the calculation of proper blended AUR (average unit retail) pricing in implementations with multiple currencies and countries. These parameters are also used in single-currency/country implementations being set to maintain flat pricing with no adjustments.

The configuration of these views is a required step in the implementation of Assortment Planning and should be completed prior to the creation of any assortment plans.

Site currency assignments

Home currencies need to be established for all sites in use before configuring currency exchanges and price translations. For more information, see Currency Exchange Rates.

Updating exchange rates

As exchange rates between currencies are volatile, how often these rates are updated needs to be planned effectively. Considerations include the source of the rates and how the rates are updated.  While more frequent updating of rates may yield a greater degree of accuracy, these changes cause recalculations within assortment plans that need to be reviewed.

For more information, see Price Translation Management.

Price Translation rules

Price Translation rules allow users to define rules to adjust pricing by price range within each currency and then assign those rules to levels of the product hierarchy.

For more information, see Price Translation Hierarchy Assignment.

Updating VAT rates

Although VAT rates do change they are much less volatile than currency exchange rates. Users can maintain VAT rates at any level of a given product hierarchy and by Site Groups 1-20.

However, Assortment Planning was not designed to be used with product hierarchies that extend to the style/variant level. That said, VAT rates should be maintained at a level above that.

Any variations occurring beneath (due to fabric type in some countries, etc.) should be handled by entering a blended VAT rate for the level based on the expected % contribution of the products attached to varying VAT rates beneath.

As with currency exchanges, the frequency, source, and method of VAT rate updates needs to be considered. For more information, see Planning VAT Rates.

Target Creation, with Assortment Planning, can be used as a purely analytical tool and could be useful in determining the % contribution of product types that exist beneath established product hierarchy levels. (For more information, see Target Creation.)
