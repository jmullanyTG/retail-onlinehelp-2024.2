Schedule properties

The schedule properties define the layout of the schedule and how users are allowed to interact with it.

  • Resource Name Caption – Name given to the schedule.
  • Resource Tree Visibility – If selected, the resource tree is visible.
  • Auto Node Height – If selected, automatically generates the height for the node.

Use Today as Go to Date – If selected, today’s date (GETDATE) is used as the go to date.

  • Go to Date – The date which presents when the widget is launched.
  • Show Resource Headers – If selected, the resource headers display.
  • Resource Headers Height – Height for the resource headers can be hard coded.
  • Resources Per Page – Limits how many resources display per page. When the widget has been launched, this can be adjusted dynamically using the plus and minus arrow keys.
  • Cells Auto Height – Automatically assigns a height for the cells.
  • Appointment Auto Height – Automatically assigns a height for the appointments.
  • Appointment Height – If not set automatically, appointment height can be specified.
  • Show More Button – In cases where the data does not fit well into the height and width constraints, the More button alerts users to the fact that there is data present which is being hidden.

  • Default Time Scale – The default time scale determines level of granularity which is initially visible within the time dimension. This can be set at:

    • Year-Quarter
    • Quarter-Month
    • Month-Week
    • Week-Days
    • Day-Hours
    • Hour-Time

  • Default View – The view for the schedule.

    • Gantt View – The Gantt view presents the schedule as a series of bars with inter-dependencies.
    • Month View – The month view looks similar to a regular calendar view.
  • Months Per Page – Sets the number of months that display when in month view.
  • Week Count – Shows the number of weeks when in month view.
  • Cell Colors – Lets users change the foreground (text) and background cell colors for certain sections of the schedule.
