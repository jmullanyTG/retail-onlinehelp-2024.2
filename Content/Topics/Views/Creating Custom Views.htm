Creating custom views

Custom view are commonly built for one of two purposes:

  • Custom Reports – Users require a report on data contained within the database but not presented in one of the standard modules. A custom report is built using the Views module. Like other views, custom reports can have user rights assigned where required.
  • Custom Maintenance Screens – There is a requirement to manipulate data outside of the standard modules, either to enhance the performance of the standard modules, or to otherwise adjust the inputs or results of the standard modules. Some common examples include:
    • Site Supersession
    • Sales Forecasting
    • Supplier Lead Time Grading
    • Financial Plan Branching Matrices

To properly build a custom maintenance screen within the Views module, first define a workflow outline and then plan the requirements for the new view. The view should be thought of as a single screen so any information which would need to be accessed at the same time should be considered for inclusion.

Views with multiple widgets should not be cluttered or overly busy (this may be controlled via the view type).

Docking new views in modules

New views can be docked in modules using the Generic View Collection Set Module. For more information, see Generic View Collection Set Module Overview. For more information, refer to the online help.

To build a new custom view:

  1. Within the Configure section on the toolbar, click Views.

    The View Management window displays.

  2. In the View Management window, click New.

    The Add View window displays.

  3. Click Assign Roles or Users to assign roles or users.

  4. Complete the fields as required.

    For more information on each field, see Field Definitions.

  5. Assign a view type in the View Type field.
  6. Use the checkboxes to define how the parameters are treated within the view.

  7. Click the Add button below the widget pane to add widgets and reorder the widgets if required.

    Reordering widgets is especially important in a vertical widget layout.

    After widgets have been added, their respective parameters appear in the Parameters pane.

  8. Select an action as required.
  9. Click Ok.
  10. Optionally, dock the new view in a module.
