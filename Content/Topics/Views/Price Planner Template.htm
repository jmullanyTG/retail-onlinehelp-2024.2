Adding custom views to the Price Manager Planning tab

For Variant by Site plans, custom views can be added as tabs on the Planning view. A View Collection Set, View, and Widget (all three are labeled Price Planner Template) are used to create custom views. A stored procedure can be used for reference to build widgets.

System policy Price Planner View Collection Set must be set to the View Collection Set ID. When the system policy is set to a valid View Collection Set ID, then that associated View is visible as a tab on the Planning view next to the Site Detail tab. When the system policy isn’t set, the tab is not visible.

Accessing the Price Planner Template screen

This screen can be accessed by navigating to System Utilities > Views > Price Manager > Price Manager Template.

Creating custom views

For more information, see Creating custom views.

Parameters

The View Collection Set is passed the following parameters:

Parameter Description
@iUserID The user using price planner.
@iConfigID The currently selected price plan.
@iFilterMasterID The selected filter in price planner, if Show Plan, Show All Sites, or Null is checked, then no filter is passed in.
@ShowMissingPrice1 The value of the Show Missing Price 1 checkbox.
@ShowMissingPrice2 The value of the Show Missing Price 2 checkbox.
@iVariantID The selected variant of in the variant grid.
