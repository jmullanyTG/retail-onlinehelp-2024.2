Schedule tree data selection

The data selection tab will determine how data is brought into the widget. This data specifically pertains to the schedule tree.

  • SQL Connection Type - Choosing an SQL connection type tells the widget from which database to return the data.
    • Operations Database - The normal database containing most of the current and operational data for ToolsGroup JustEnough. This is the selection for most applications.
    • Warehouse Database - If configured, the warehouse database can function as a container for archived data. Select the warehouse database to return data to a widget
  • Method - The method dropdown returns every SQL stored procedure listed in tbl_Common_StoredProcedureLookup, which should be representative of every stored procedure in the database. This is the procedure which will execute when the Execute button is pressed on the widget (when parameters are present) or which returns the data to the widget (i.e. a SELECT statement).
  • Method Parameters - Upon selection of the stored procedure, the Method Parameters window populates with the required parameters for the procedure. These parameters vary greatly depending on the requirements of the stored procedure.
    • Method Parameter Name - The name of the required parameter as specified in the stored procedure.
    • Parameter - The selected parameter entry to be passed. These can be selected from the list of parameters detailed in the Maintain Parameters section.
    • Maintain Parameters - Displays the Select Parameter window, displaying all available parameters. More details can be found in the Maintain Parameters section.
    • Default Value - The default value, if any, passed to the parameter.
