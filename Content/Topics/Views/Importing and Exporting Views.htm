Importing and exporting views

Views can be exported from one instance of ToolsGroup JustEnough and imported into a second instance. This allows for the upgrade of standard versions or transportation of work between development, QA, and production environments.

New views

A new view can be imported using the Import New function.

Importing a view as new creates the view with a new universally unique identifier (UUID) within not just the current instance of ToolsGroup JustEnough but across any version.

Views imported as new do not replace existing views and the feature should not be used to upgrade in place. When customizing or extending an existing standard view, it should first be exported, then re-imported as new to give the view a fresh UUID (and thus not be overwritten when upgrading).

Existing views

An upgraded version of an existing view can be imported using the Import Existing function.

The Import Existing function searches for and replaces an existing view with the same UUID within the system. This should be used to upgrade standard views in place.
