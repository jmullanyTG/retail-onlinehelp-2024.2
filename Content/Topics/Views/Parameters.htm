About parameters

Parameters can be used in multiple methods to create a streamlined user experience throughout the application. There are two types of parameters that are managed through the Select Parameter screen: system parameters and user parameters. Parameters are listed and organized by parameter group on the Select Parameter screen.

System parameters are not visible to users and can have values entered into them (by .NET code) such as user ID and session ID. It is not advised to alter system parameters.

User parameters operate at the user level and can be built to create consistency throughout the application, or at minimum, across widgets.

When importing and exporting parameters (.paramf file), the parameters initially appear in an import group folder. When importing widgets (.widgetf files), their dependent parameters are also imported.

Accessing the Select Parameter screen

Click Parameters on the Views toolbar.

The Select Parameter screen displays the available parameters.

Creating and editing parameters

Several fields must be supplied to create a valid parameter, including some knowledge of the SQL parameter (which will not be available anywhere within the user interface).

  • Parameter Name – The name for the parameter.
  • Parameter Description – An optional description of the parameter.
  • Parameter Caption – The name which presents as part of the user prompt to supply a parameter value.
  • Parameter SQL Name – The linked SQL parameter. Name must match the name in the database.
  • Parameter is Visible – If checked, the parameter is visible to users.
  • Parameter Width – The specified width of the prompt box.
  • Parameter Height – The specified height of the prompt box.
  • Parameter Default Value – The default value supplied without user override.
  • Parameter Type – Defines how values pass to the parameter. Options include single and multi-selects.
  • Parameter Data Type – The associated data type.
  • Allow Null – If selected, will permit a null value to be returned to the parameter.

Testing parameters prior to application

The Test Parameter Control button found on the Create Parameter and Update Parameter dialogs allows users to test parameters before applying them to methods. The parameter displays as defined in the settings.
