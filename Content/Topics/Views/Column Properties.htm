Column properties

The Column Properties section lets users add, modify, and delete columns as needed within the grid.

  • Column – The column name.
  • Column Friendly Name – An optional user readable name.
  • Tool Tip Text – Supplies text for a mouse-over tool tip on the column.
  • Visible – If checked, the column is visible within the widget.
  • Order – The stacked numerical order for the columns in the widget. Any column without the Visible flag checked will have an order of 0.

  • Format String – The format string settings affect how data displays. Format string entries vary depending on the data type. Examples include:

    • P2 – A percentage with two decimal places.
    • F3 – A fixed point decimal with three decimal places.
    • C – A currency.
    • D6 – A decimal with six decimal places.
  • Pivot Area – The default area of the pivot grid for the column (can be moved).
  • Allowed Pivot Areas – If the column is allowed to be moved from its default pivot area, the available areas are listed, or a value of All is present.
  • Show Column Totals – If checked, shows a total for the column.
  • Show Column Grand Totals – If checked, shows a grand total for the column.
  • Fore Color – The text color for the column.
  • Back Color – The background color for the column.
  • Aggregate Type – Determines how the data within the column aggregates.

  • Aggregate Display Type – Determines how the aggregated data displays.

  • Total Aggregate Type – Total aggregate refers to the level above the lowest level. The aggregate type selected determines how that data is aggregated. For example, week level data would be aggregate type and month level data would be the total aggregate type.

  • Show in Customization Form – Display the fields button within the user interface.
  • Show Values – Display the value to users. This is different from the Visible field in that the data is still available for calculations, but not displayed to the end users.
  • Allow Drag – Allows the column to be moved to another pivot area.
  • Allow Drag in Customization Form – Allows the column to be moved to another pivot area within the customization form (fields view).
  • Running Total – If selected, certain aggregation types (SUM, COUNT, etc.) display the running total, which is calculated for values in the data area and corresponds to the current column or row field.
  • Min Width – The minimum number of pixels for width on the column.
  • Width – The standard number of pixels for width on the column.
  • Write Enabled – Allows the field to be edited within the user interface.

  • Writeback Split Type – Determines how the value is split when written back.

    • Direct – Writes the value exactly as defined by the user.
    • Equal Split – Equally divides the value by the number of dependents at the lower level (i.e. a month value splits into four weeks equally).
    • Weighted Split – Requires users to specify how splits are distributed. If selected, requires users to set the writeback weight based on column.
  • Writeback Weight Based on Column – Defines the weighted split value.
  • Writeback to Column Name – If selected, allows a data writeback within the column to affect the value of another column.
  • Row Level Write Enabled Check Field – At row level, determines if the value is write enabled.
  • Pivot Level Write Enabled Check Field – At the pivot level, determines if the value is write enabled. If enabled at pivot level, the row level write back is also enabled.
  • Row Level Locked Check Field – If selected, locks the data at the row level.
  • Aux Write Enabled – If selected, allows a data writeback within the column to affect the value of multiple columns, (which must be specified).

  • Aux Writeback Value Calculation – Allows for an expression to calculate the value which is written back. Any column within the pivot grid can be referenced by using the true column name within square brackets (case sensitive). Available parameters include:

    • @Value – Whatever the user supplied value is becomes the @Value (e.g. @Value/SalesUnits would be used to derive AUR).
    • @OldValue – Use the value before the user changed it to make the calculation (i.e. for holding calculations).

  • Aux Writeback Split Type – Determines how the value splits when written back.

    • Direct – Writes the value exactly as defined by the user.
    • Equal Split – Equally divides the value by the number of dependents at the lower level (i.e. month value splits into four weeks equally).
    • Weighted Split – Requires user to specify how the split is distributed. If selected, requires users to set the writeback weight based on column.
  • Aux Writeback Weight Based on Column – Defines the weighted split value for the aux writeback.
  • Aux Writeback to Column Name – If selected, allows a data writeback within the column to affect the value of another column.
  • Aux Row Level Write Enabled Check Field – At row level, determines if the value is write enabled. 
  • Aux Pivot Level Write Enabled Check Field – At the pivot level, determines if the value is write enabled. If enabled at pivot level, the row level write back is also enabled.
  • Sort Order – Determines the sort order for the data within the column.
  • Sort by Column – Enables the reference column used for sorting.
  • Column Used to Sort – Specifies the reference column used for sorting.
  • Expression Column – If checked, the column is an expression column used to make a calculation.
  • Expression – If the column is an expression column, this will display the value of the expression.

  • Expression Column Type – Will describe the level at which the expression calculation is done.

  • Data Type – Defines the output data type of the expression.
  • Derived Column Type – Derived columns are legacy functionality and all features have been replaced by functionality in expression columns.
  • Derived From Column – Derived columns are legacy functionality and all features have been replaced by functionality in expression columns. 
  • Lookup Column – Checked if column is a lookup column.
