Setting flow period frequency based on Item Cube (ROS & Cube size)

The Assortment Cube Matrix view is used to define and update the Flow Period Frequency based on the intersecting ranges of Placeholder/variant ROS and Cube Size (which references a volumetric attribute). Flow Period Frequency is the default number of periods between receipt dates calculated by the receipt planning engine.

This feature is only applicable in Assortment Planning.

The attribute to reference is defined using system policy Attribute to reference when applying item cube.

Accessing the Assortment Cube Matrix view

Navigate to the Views module to access the Assortment Cube Matrix view.

Working with Assortment Cube Matrixes

All three grids are initially blank when the screen first displays. The Matrix grid populates automatically as entries are added to (or delete entries from) both the ROS Cube and Item Cube grids. Both cube start and end values must be populated on the top grids.

If the cube attribute assignment is removed for a placeholder, then the hierarchy default Flow Period Frequency value applies. Changing the Flow Period Frequency in the Assortment Receipt Plan Constraints grid overrides the values in the Assortment Cube Matrix view.

The decimal formatting for the start and end values of each cube can be modified within the grids.

Flow period frequency defaults to 1 as the entries are created, and cannot be updated to any negative values.

Setting which levels use item cube logic

Specify which levels use item cube logic using the Use Cube for Flow Period Frequency? field in the Assortment Plan Defaults screen within Assortment Planning.

Overlaps in ROS group definitions

Any placeholder with a ROS that exists in 2 ROS cube groups results in the ROS group with the higher Cube Start Value being chosen as the tiebreaker. For example:

ROS GROUP 1 (1.00 - 5.00)

ROS GROUP 2 (3.00 - 5.00)

A placeholder with a ROS of 3.75 is assigned to ROS GROUP 2.
