Custom view field definitions

  • Name – The name for the selected view. View names do not need to be unique within the view group. However, it is recommended that names be a view’s unique identifier for ease of navigation. View groups can be created and maintained in the view groups pane to the left of the view creation/edit window.
  • Description – A description of the view. May include author or date if desired.
  • Group - The view group to which the new custom view is assigned.
  • View Type – Determines the layout possibilities for the widgets contained within the view. Requirements vary depending on the number of widgets.
    • Vertical Widget Layout – Stacks widgets vertically based on the order provided in the Widgets section. If the view contains only one widget, it is common to see this view type selected.
    • Tabbed Widget Layout – Creates a tab style layout similar to those provided by the view collections.
    • Layout Controlled Widget Layout – The most customizable layout allowing for much greater customization and configuration of the view. If selected, the widget order is irrelevant. Users can drag and drop elements within the views to manipulate the widgets, scroll bars and separators to best frame the workflow process.

Assign Roles or Users – Displays the Maintain Users And Roles dialog used to configure included and excluded roles and users.

If users cannot find a view, this is a common troubleshooting spot to check.

checkboxes

  • Hide Parameter(s) – Required parameters for the view to pass to the database to retrieve data display near the top of the view. Check this box to hide the parameter section if more screen space is needed.
  • Auto Refresh on Parameter Change – Select this option if users should be able to toggle between several parameters within the same view. The entire view automatically refreshes without the need to click the Execute button.
  • Auto Refresh on Widget Parameter Change – Refreshes data when a parameter within a widget changes.
  • Allow Update Method Parameter Value – In situations where there is more than one parameter and they are dependent upon one another, selecting this option allows the parameters to update if there is a change to one or the other.
  • Allow Use and Save User View Parameter Values – If selected, saves the parameters when the view is closed and reopened.

Widgets section

The Widget section displays the widgets selected for inclusion in the view.

  • Widget Name – The assigned name as defined in the Maintain Widgets configuration sub-menu.
  • Widget Description – The assigned description.
  • Widget Order – For vertical widget layouts and tabbed widget layouts, this determines the order in which the widgets appear in the view. The lowest number is stacked near the top in a vertical layout and near the left in a tabbed layout.

Parameters section

The Parameters section displays the parameters required for each of the widgets listed in the Widgets section.

  • Parameter Name – The assigned name as defined in the Create Parameters configuration sub-menu.
  • Parameter Description – The assigned description.
  • Parameter Order – The order in which the parameters are display (left to right) across the top of the screen.

Actions section

The Actions section is used to configure View Actions. These Actions are buttons that appear at the top of the view, alongside the built-in actions like New, Delete, Save, and so on.

Some examples of View Actions are in Target Choice Counts. All of the buttons across the top, like Copy, Compare, and Smart Start, are View Actions. To create View Actions, write very specific XML that allows the environment to know which View to open and what parameters from the launching screen to pass through.
