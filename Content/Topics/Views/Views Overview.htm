Views Overview

Views provide a way to view and manage critical data that is key to the decision making process. Views are built and organized in the Views module. View collections and view collection sets can be organized within this module and are an integral part of the Events, Assortment Planning, and Promotions modules as well as several others.

The Views module, in conjunction with the creation of SQL views, provides the required toolkit for building custom reports and maintenance screens to accommodate any necessary custom work flows.

View hierarchy

It is important to understand the hierarchy associated with views to properly organize and display them. Groups may be created to help organize view collection sets, but contain no hierarchical data.

View Description
View Collection Set The highest level of view classification. View collection sets contain view collections, views, and widgets.
View Collection The next level of view classification. View collections are the highest level at which user roles can be assigned. The icon can be changed in the Image field.
Views The next level of view classification. A view is a collection of widgets used to display useful data in an easy to understand way.
Widgets Widgets are the lowest level of view classification being a subset of a particular view. Widgets are a way to display data. There are many varieties of widgets, including data grids, pivot grids, single column pivot grids, charts, forms, schedule, and browser.
