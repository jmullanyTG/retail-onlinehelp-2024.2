Modeling sales data

If sales are reduced significantly due to a large scale event such as a disaster, modeled sales can be put in place using the Disaster Sales Modeling screen. Select a store and have the application system calculate relevant theoretical sales history data for this store during the periods impacted by the event.

The modeled sales history data for selected stores is based on forecasting methods. The data is modeled on the LY periods for the same store at the same levels of the product hierarchies. This data is entered as a special sales type so that it does not conflate or otherwise interfere with actual sales.

Accessing the Disaster Sales Modeling screen

This screen can be accessed by:

  • navigating to System Utilities > Views > Disaster Model > Disaster Sales Modeling
  • adding the view to the Site Manager module. (For more information, see Site Manager Configuration.)

Parameters

Parameter Description
Sales Type Select the sales type (default is COVID-19). Additional sales types can be added.
Time Periods to Replace The time periods to model. The intended periods with missing data that model data needs to be created for.

Site Level Selection and Product Level Selection panes

The Site Level Selection and Product Level Selection panes display the site level hierarchies and product level hierarchies for which model sales and sales history by type can be created. One or more site hierarchies and one or more product hierarchies can be selected at one go.

In fact, all site hierarchies and product hierarchies can be selected and a sales model done at the company wide level. However, this would take time to generate the sales model data and it is recommended that this be scheduled at a convenient time.

Time Model Configuration pane

Use this section to configure how the model sales is designed.

Column Description
Time Period To Replace The intended periods with missing data that model data needs to be created for. The time periods that display in this field are selected in the Time Periods to Replace parameter.
Included? Whether a particular time period will have modeled sales. If checked, the time period will have modeled sales.
Time Period To Copy From The LY time period from which sales data is copied. The LY time period does not have to correspond with the time period with depressed sales. For example, week 16 can be modeled on LY week 12.
Adjustment Factor The desired percentage of sales data to be copied from the previous period.

Not all the sales data from the previous period has to be copied. Leaving the value in this field as 1.00 copies 100% of the previous time period’s sales data. The teal blue portion of each column in the graph that displays in the Modeled Sales Data pane is the actual sales for the time periods to be replaced.

Therefore, subtract the actual sales data from the sales data to be copied over. For example, if the actual sales are 50% of what was expected, then enter .5 in this field to copy over 50% of the previous time periods sales.

Also, a value of 1.2 can be entered for a 20% lift or a value of 0.8 for a 20% depression.

The graph automatically updates when click Save is clicked in this pane.

Formatting

Certain conditions are highlighted by a change in the formatting in the grid.

Formatting Description
Light blue background The time period is selected for inclusion in the sales model.
Orange background The time period is in a mixed state. When multiple sites or product levels have been selected and some sites or products are included and some are not.
Indicates that there is a mix of included and excluded.
  • Time Period To Copy From label is replaced with (Mixed)
  • The value in the Adjustment Factor field is italicized.

Multiple sites levels with existing sales models have been selected and the sites have different time periods as the basis for their sales models. The value in the Adjustment Factor field is now an average of the adjustment factors in the individual sales models.

Example: Selecting multiple site levels with existing sales models

If two site levels are selected each with their own pre-existing sales models and there are differences in the time periods included between the sales models, then the following formatting displays:

The background color is blue for all 3 rows as they are included in both selections. The labels for week 14 and week 16 are spelled out because both site level selections are using week 14 and week 16 in their sales models. The label for week 15 is (Mixed) because one site level sales one includes week 15 and one sales model does not include it. The adjustment factor value for week 15 is italicized because it is the average of the adjustment factors for the 3 individual sales models.

Clicking Save effectively creates a supersession for both site levels as the rule is copied down to the individual site level sales models. This can be seen by clicking on the individual site levels and reviewing their time model configurations. Supersessions can be removed by deselecting an included site level and clicking Save. The supersession sales model data for the site level is deleted.

Modeled Sales Data pane

The Modeled Sales Data pane includes a graph showing the sales counts as they exist today. Teal blue indicates actual sales.

Metric Definition
Real Sales Units The actual sales (both regular and promotional). Used in the modeled sales.
Site Modeled Sales Units Saved site modeling data. Used in the modeled sales.
Disaster Modeled Sales Units The theoretical or modeled sales units for a specified time period.
Build Ratio How sales have decreased over time.

Generating model sales data

To generate model sales data:

  1. Review the graph in the Modeled Sales Data pane to identify a period with a significant decline in sales.

  2. Choose one or more site level hierarchy in the Site Level Selection and one or more product level hierarchy in the Product Level Selection pane.

  3. Select the time periods to be included by checking the checkbox in the Included? field in the Time Model Configuration pane.

    The default time periods that display is last year (LY). The time period is changed by clicking within the field to select a different time period to copy sales data from.

  4. Click the Adjustment Factor field and enter the desired percentage of sales data to be copied from the target time period.
