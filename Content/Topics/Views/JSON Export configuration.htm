Configuring standard export types and order document exports

Standard export types and order document exports can be dynamically configured in the Views module. This functionality creates flexibility in the order export process.

Two Maintenance views support these additional configurations:

  1. Standard Export Type Configuration

    Expand the Payload Type dropdown and select either XML or JSON as the payload type.

    Although the JSON payload type can be seen in the dropdown for other export types, this functionality is only available for Order export types.

  2. Inventory Order Document Export Configuration

    Select which columns to include in the inventory order document and:

    • use the Output Name field to configure the field name, if needed.
    • include the Order Document Section Type ID column to identify which section of the order document the metadata is taken from.

    • filter for specific metadata, whether that is inventory metadata or order header and order line metadata.

      Everything defined in filter metadata is selectable for inventories. This means that the metadata is dynamically read off whatever is configured within the system.

In most if not all cases, users can configure the necessary metadata, fields, and column information needed to export their order document.
