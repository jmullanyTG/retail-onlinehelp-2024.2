Structured report

  • WidgetTypeID = 7

The structured reports widget give power users an opportunity to make WinForms like reports. These can mimic applications such as Microsoft Word and allow for advanced reporting.

General tab

Report Data Selection tab

The Report Data Selection tab determines how data is brought into the widget.

  • SQL Connection Type: Choosing a SQL connection type tells the widget from which database to return the data.
  • Operations Database: The normal database containing most of the current and operational data. This is the selection for most applications.
  • Warehouse Database: If configured, the warehouse database may function as a container for archived data. Selecting the warehouse database to return data to a widget
  • Method: The dropdown returns every SQL stored procedure listed in tbl_Common_StoredProcedureLookup, which should be representative of every stored procedure in the database. This is the procedure which executes when the Execute button is pressed on the widget (when parameters are present) or which returns the data to the widget (i.e. a SELECT statement).
  • Method Parameters: Upon selection of the stored procedure, the Method Parameters window populates with the required parameters for the procedure. These parameters vary greatly depending on the requirements of the stored procedure.
  • Method Parameter Name: The name of the required parameter as specified in the stored procedure.
  • Parameter: The selected parameter entry to be passed. These can be selected from the list of parameters detailed in the Maintain parameters section.
  • Maintain Parameters: Displays the Select Parameter window with all available parameters.
  • Default Value: If any, the default value passed to the parameter.

Report designer

The Report Designer will look familiar to Microsoft Office users as it features many of the same concepts used within that suite to build reports.

The fields available for reporting depend on the method selected to return data.
