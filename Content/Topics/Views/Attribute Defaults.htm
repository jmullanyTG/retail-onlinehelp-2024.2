Setting attribute properties for Variant and VariantPlaceholder entity types

The Attribute Defaults view is used to set and clear default attribute values.

Use Configuration > View Maintenance to add the Attribute Defaults view to the View Selection dropdown menu.

The following can be defined:

  • Lead Time Days (in days). Provides an updated Order By Date for all orders, not only the first. This is for reference only and does not actually move the order date.

    The currently defined lead time displays in the Default Lead Days field in the Assortment Planning main planning view. Once an additional lead time is set, the Assortment Upcoming Commitments Dashboard Report shows an updated Order Date, Due Date, and Is Past Due? date. The Receipt Plan view displays an updated Order By Date.

    Where there is a lead time for a product hierarchy (for example 7, 10, or 14 days) there may be products such as Promotions within that hierarchy that require additional lead times. These exceptions may need to arrive earlier so instead of 7 days they should arrive 10 days earlier. Additional lead times are defined using the Attribute Defaults view.

    The system first looks for an attribute-set default lead time, then a hierarchy set lead time, then the system policy.

  • Periods Before Phase In (by periods). For the first order only, affects the first delivery and moves the order date forward in the receipt plan.

    Periods Before Phase In is editable within the Assortment Receipt Plan Constraints section within the Receipt Plan view, Assortment Plan Defaults By Cluster view, and Assortment Plan Defaults view.

    The last-in value between a user override from the Assortment Receipt Plan Constraints section and a user change in the Attribute Default view takes priority. The new attribute default will only show the updated Periods Before Phase In value after receipt recalculation, and only updates placeholders that have not phased in yet.

    The system first looks for an attribute-set default (Variant), then an attribute-set default (Placeholder), then a hierarchy cluster default, then a hierarchy default, and then the system policy.

Both Lead Time Days and Periods before Phase In can be set at both the Variant and VariantPlaceholder levels. This allows the same attribute to have different times assigned depending if a Placeholder is in Placeholder status or Variant status. If the same lead time is desired for both, then define against both entity types in the dropdown in the Attribute Defaults view.

It is important to be aware which entity type has been selected when using the Attribute Defaults view.

As an additional lead time and Periods Before Phase In can be defined for any attribute, the attribute to be updated needs to be identified. This is done by noting the Attribute ID in the Attribute ID field and setting the system policy AttributeID to determine which default values to apply with that Attribute ID.

The attributes are maintained using the Attributes Management view. For more information, refer to Attribute management and assignment overview. Refer to the Attribute Management User Guide.

Setting a default value for an attribute

  1. Select the plan in the Assortment Plan field.
  2. Choose the entity type in the Entity Type field.
  3. Select an attribute and click New.
  4. Click the dropdown arrow in the Lookup Value field and select a value.
  5. Enter a value in the Lead Time Days or Periods before Phase In field as required.
  6. Click Save.
  7. Note the Attribute ID in the Attribute ID field.
  8. Set the system policy AttributeID to determine which default values to apply with that Attribute ID.
