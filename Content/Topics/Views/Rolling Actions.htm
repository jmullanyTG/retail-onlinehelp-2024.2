Rolling actions

The rolling actions settings determine how changes made to the data affect (or roll out to) other data fields within the widget.

  • Set Fields – Defines the group. Set Fields is a multi-select so users can pick more than one (i.e. product level ID and channel level ID in Merchandise Financial Planning).
  • Order Fields – Determines the direction in which the data roles. Typically, these are date fields.
  • Condition – Determines under which condition users intend to cause the data to roll forward (i.e. in Merchandise Financial Planning, only roll for future periods).
  • Previous Source Row Field – Defines the source of the data to be rolled. Must be a base level field.
  • Next Destination Row Field – Defines the destination. Must be a base level field.
  • Do Accept Changes – Enables the rolling action.
