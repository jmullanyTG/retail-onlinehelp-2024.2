Real-Time Retail (RTR)

Responding real-time to changes to stock on hand (SOH)

If configured, planners can receive a real-time data feed from ToolsGroup Onera that reports any changes to the stock on hand (SOH) value of an inventory. TG Onera receives feeds from client ERP systems, and then feeds real-time data via a Webhook into TG JE.

To take advantage of this functionality, TG Onera is required and an API key from TG Onera configured to access and authenticate with TG JE.

The value provided by TG Onera is the total remaining SOH at that time and not the delta between the original and new values.

The integration of a continuous data feed means users can be notified in real-time that a fast selling item is approaching its defined end of period safety stock level allowing them to react accordingly. This gives planners the ability to avoid unexpected stock outs and lost sales while also avoiding over ordering stock. This process cuts down the reaction time of TG JE’s system and can alert planners a stock out is about to occur in minutes rather than hour or days. The trigger for an updated recommendation calculation is also configurable to cater to different customer needs.

With TG Onera live reporting, planners can generate and approve orders within hours of an approaching stock out event instead of waiting overnight for the nightly batch process to make recommendations. The real-time data feed is in addition to, and faster than, the existing nightly batch process as users don't have to wait overnight to update SOH values. It is important to note that the data feed does not replace the existing overnight batch process and still relies on the nightly batch for the complete import and process.

The Notifications sent to users are silent in that they don't pop up on the screen to avoid being intrusive. In this case, the arrival of Notifications is indicated by an increase in the notification flag count at the top-right of TG JE screens. Additionally, Notifications are aggregated to avoid recipients from being overwhelmed.

Within a Notification, users can click a link that will open the Real-Time Retail Dashboard view to gain insight into the received data.

Getting insight into real-time data received from TG Onera

The real-time data feed can be reviewed using the Real-Time Retail Dashboard view. This dashboard functions as a central control center for today’s recommended actions providing planners insight to if and when they should generate orders for today.

The Real-Time Retail Dashboard view provides a visually interesting and easy way to gain insight into the following data received from TG Onera's Inventory Hub:

  • today’s Stock on Hand events

  • the number of Recalculation events that resulted today.

  • the recommended order quantity and generated order quantity for today

  • the last 25 events that resulted in a threshold process event.

The data that displays is filtered by selecting predefined filters in the Inventory Filter field. The dashboard then displays the data for the defined filtered items only. This prevents planners from being overwhelmed by the vast amount of incoming data allowing them to pay close attention to top priority inventories.

The dashboard is comprised of the following:

Section Description
Today’s Real Time SOH Events The total amount of RTR updates compared to the amount displayed using this particular filter.
Today’s Recalculation Events Out of the total amount of RTR updates, how many triggered a calculation.
Today’s Orders Ready for Generation

  • Generated Order Qty - Out of all the recommended orders in this filter, how many do not have an order generated for the recommendation.

  • Recommended Order Qty - The total amount of recommendations with an order date of today using this filter.
Latest Threshold Processed Events The last 25 events processed by RTR.

The number of events is not configurable.

Today is any entry that has a time stamp from 12 am (midnight) up to but not including midnight of the next day. For example, 1/13/2023 00:00:00 to 1/13/2023 23:59:59 is considered today for 1/13/2023.

Once planners think it is a good time to generate an order form for today’s orders, orders can be created directly in the view by clicking Generate Order. This creates an adhoc order for all the items in the filter. The planner can then review and modify the order in Order Manager just like any other order.

Comparing new SOH values against thresholds

The new real-time SOH values are compared against thresholds defined within TG JE. When the new SOH goes beyond a threshold, the outstanding need for the item is calculated, and an order for today automatically generated ready for review. These new orders are treated by TG JE as if they were manually processed.

The thresholds are recorded at individual inventory. The thresholds are:

  • time based: If the system’s current time exceeds this value, a recalculation is triggered. The value is set to be 1440 minutes (24 hours) from the last known Engine run date or today.

  • value based: If SOH falls below this value, a recalculation is triggered. The value is set to be an Inventory’s End Of Period Safety Stock amount for their current time period (i.e this week or this month).

  • forced flag based: When true, a recalculation is always triggered. This is set to true when there is no recommended projection data for an inventory for their current time period (i.e. this week).

A CUSP hook has been provided to grant customers the ability to set the Forced Flag, Time, Value entries as they see fit. This does not grant the ability to change how the thresholds are evaluated. For Value based checks, an Inventory’s SOH is always used. However, instead of EOP Safety Stock being the value, it may be another value provided by a more complex calculation.

For example, an item has 100 SOH at the start of the day and a defined safety stock level of 20. TG JE receives a continuous feed from TG Onera with updates showing the item is continuing to sell briskly. At a certain point during the day, an update is received with a SOH value of 19. As the SOH has dropped below the defined safety stock level of 20 the Item recalculates and produces new recommended orders.

Allocation and replenishment items

Once a threshold has been crossed, a replenishment runs or an Allocation strategy runs (if there are multiple strategies defined for the item, only the smallest strategy ID runs).

A replenishment item is the item master associated with that replenishment. The item master runs using defined system policies.

These allocations and replenishments are placed into specific Workload Queues (made for this purpose) that only handle intra-day processed items. Items that are queued multiple times are grouped together up to the point when the calculation runs. This helps reduce the noise associated with items with constantly changing stock levels. The 2 Workload Queues are:

  • Real-time Retail Allocations

  • Real-time Retail Replenishment

Batching items to eliminate delays in processing

The RTR Queue Marshall job task eliminates conflicts when items are manually calculated or calculated by RTR. With a stream of current data, and the need to process that data, along with typical user actions, keeping the stream of real-time data separate from the other job tasks is imperative to not delay calculations triggered other user actions. The RTR Queue Marshall batches groups of related items together for much faster batch processing. This greatly accelerates processing times and improves the user experience.

There are two Job Manager jobs for the RTR Queue Marshall job task:

  • Queue Marshal RTR - Allocations

  • Queue Marshal RTR - Replenish
