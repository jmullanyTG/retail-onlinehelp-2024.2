Real-Time Retail (RTR) batch reduction

System policy Generate Orders Starting Tomorrow or Today enables the generation of RTR recommended orders for the following day when the policy is set to Tomorrow (default). By setting the policy to Tomorrow, users also gain the flexibility to reduce their nightly batch run times.

Batch exclusion for recent recommendations

Under the default setting, if an RTR recommended order was generated within the last 18 hours, and Generate Orders Starting Tomorrow or Today is enabled, the system skips generating new recommended orders for related items. This allows customers to avoid wasted time running unnecessary batch calculations within a short time frame.

Orders generated when Generate Orders Starting Tomorrow or Today is set to Today are NOT excluded from the nightly batch process.

Benefits of setting the system policy to the default

When the system policy is set to Tomorrow, users can still take advantage of RTR instant decision making without having to wait for the nightly batch process. This reduces their dependency on the nightly batch and offers the following benefits:

  • Faster Order Processing: Users can process tomorrow's orders today if their supply chain processes allow, facilitating quicker fulfillment.

  • Reduced Nightly Batch Needs: By generating recommended orders for the following day, users can minimize the number of nightly batch runs, optimizing system performance.

Configuring the time elapsed since last calculation for batch exclusion

System policy Time Elapsed Since Last Calculation for Batch Exclusion allows users to reconfigure the default 18-hour time limit. Users can specify the number of hours for batch exclusion, although it is not recommended to exceed 24 hours.

Job tasks

The following job tasks facilitate the recommended order generation and batch exclusion process:

  • (AL, RP) (Daily) Enable Batch Exclusion Process

    Description: Enable the batch exclusion process to skip generating new recommended orders for items with recent recommendations.

  • (AL, RP) (Daily) Disable Batch Exclusion Process

    Description: Disable the batch exclusion process to allow the generation of new recommended orders for all items.

To effectively implement the recommended order generation and batch exclusion process, the Enable Batch Exclusion task needs to run before the Ordering Engine Executions. Disable Batch Exclusion should run immediately after. The reason being if Enable Batch Exclusion is left on, then orders generated by users during the day would be excluded as well.
