Maintaining Rates of Sale (ROS) in Variant Manager

After creating a Variant Manager document, select one or more variants from the filter and edit their details in the Rate of Sale blade located at the bottom of the Variant Manager screen.

ROS blade layout

Within the Rate of Sale blade, there are several columns that can be edited and updated, such as phase dates, Rate of Sale override values, curves and product models, and more. Variant-level phase in and out and ROS values can be applied to and passed down to the inventory level.

Changes in Variant Manager only impact at the Variant level and inventory forecasting. Changes do not impact level forecasting.

Multiple Variants can be selected in the top grid with the lower grid displaying information for the selections.

Updating multiple Variants simultaneously

To update multiple Variants simultaneously, select a Variant in the bottom grid and click the Bulk Copy button. This displays the Rate Of Sale Bulk Copy screen. Select the appropriate checkboxes to apply values to the selected Variants. Unchecking a checkbox clears values from Variants.

For example, the Bulk Copy functionality can be used if everything within the same Variant is to have the same Phase In and Phase Out dates because they are all part of the same campaign. This saves a lot of time if everything is part of the same Variant.

Reviewing inventory for specific items

Selecting an item and clicking the Inventory Manager button displays the selected item’s inventory in Inventory Manager. The Rate of Sale field in the Inventory Manager Document displays the updated ROS for the specific item and site combination.

If the Rate of Sale field is hidden, click Choose and select the field from the Inventory Rate of Sale group.

Within Inventory Manager’s Forecast Viewer blade, click Forecast to display the Forecast Task dialog. The Forecast Task dialog shows which method sets are being used. The method sets being used are highlighted in bold with a check mark.

Method sets can be updated if there are no links or locks. The linked method set can be run with items that have links on them. Filters can be mixed and matched with different method sets on different linked items and the different method sets can be run across all the items.

Field definitions

Most fields are read-only. Field definitions include:

Field Definition
Variant Code A unique variant code provided by the system.
Variant Description A description of the variant. The description often contains the variant’s name.
Min, Max Inventory Phase In Date The Phase In Date pushed down to Inventory level. The Min and Max Inventory Phase In Dates represent the range of phase in dates for the item across all sites (provided in the Site Count column).
Min, Max Inventory Phase Out Date The Phase Out Date pushed down to Inventory level. The Min and Max Inventory Phase Out Dates represent the range of phase out dates for the item across all sites (provided in the Site Count column).
Phase In Date The date in which the variant is first available for selling.
Phase Out Date The date in which the variant is last available for selling.
Apply Phase Dates To Inventory When this field is checked, the Variant Phase dates are pushed down to the Inventory level. For example,

Select the checkbox and click Save.
Rate Of Sale Read-only. The Rate of Sale value as selected from the ROS Calculator module.
Rate Of Sale Override The user override value for the Rate of  Sale.
Actual Rate Of Sale Actualized rate of sale based on sales history, at an inventory level aggregated up to variant.
Store Count The number of stores that variant is planning on being sold at. This gives users valuable information with regards to the unique number of sites active for a particular variant.
Min Forecast Review Date The date when the forecast was first reviewed.
Max Forecast Review Date The date when the forecast was last reviewed.
Method Set Based on configured method sets, users can select specific ones to view their forecasts.

The method sets are configured in System Utilities > Forecast > Method Sets.
Curve Type Either AI Curve or Manual. The chosen type determines whether a Curve can be selected in the Curve field.

If AI Curve is selected, then the Curve field is disabled as a Curve is already assigned by the system.

For more information, see Curves overview. refer to the Curves User Guide.
Curve Based on the Curve Type selected, users can select a specific curve to use for their variant. Click the ellipsis to display the Curve Selection screen.

Product Models A list of Product Models. A blade at the right of the grid allows users to adjust their product model weights if needed.

For more information, see Product models and respective weightings. refer to the Allocations User Guide.
